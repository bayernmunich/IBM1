リソース・アダプター接続ファクトリーの設定

このパネルを使用して、選択したリソース・アダプター接続ファクトリーの構成プロパティーを表示または変更します。

この「アプリケーション・クライアント・リソース構成ツール (ACRCT)」ページを 表示するには、「ファイル」>「オープン」とクリックします。 EAR ファイルのブラウズ後、「オープン」をクリックします。 選択した JAR ファイル >「リソース・アダプター」と展開します。「接続ファクトリー」フォルダーを右マウス・ボタンでクリックして、「新規」をクリックします。 「一般」タブ上に以下のフィールドが表示されます。

名前

WebSphere® Application Server 内で、この接続ファクトリーを管理目的で使用する際の名前。この名前は、製品の管理可能ドメイン全体のリソース・アダプター接続ファクトリー内で固有である必要があります。

データ型 ストリング
説明

IBM® WebSphere Application Server 内で管理目的で使用する、この接続ファクトリーのオプションの説明。

データ型 ストリング
JNDI 名

このリソース・アダプター接続ファクトリー定義をデプロイメント記述子と突き合わせる場合に使用する JNDI 名。 この記入項目は、resource-ref 名である必要があります。

データ型 ストリング
ユーザー名

接続するときに、呼び出し側のアプリケーションがユーザー ID とパスワードを明示的に指定しない場合に、認証のために「パスワード」プロパティーとともに使用されるユーザー名です。このフィールドを使用すると、「プロパティー」フィールドのユーザー名は無視されます。

「ユーザー名」プロパティーの値を指定する場合は、「パスワード」プロパティーの値も指定する必要があります。

接続するときに、呼び出し側のアプリケーションがユーザー ID とパスワードを明示的に指定しない場合は、接続ファクトリーの「ユーザー名」プロパティーと「パスワード」プロパティーが使用されます。

データ型 ストリング
パスワード

暗号化されたパスワードを指定します。 このフィールドに入力すると、「プロパティー」ボックスの「パスワード」フィールドは無視されます。

「ユーザー名」プロパティーの値を指定する場合は、「パスワード」プロパティーの値も指定する必要があり ます。

データ型 ストリング
再入力パスワード

パスワードを確認します。

タイプ

リソース・アダプター・アーカイブ内のファクトリーに対して定義されている すべての connectionFactoryInterfaces のドロップダウン・リスト。

各「タイプ」には、「プロパティー」ボックスで指定されている一連のプロパティーがあります。 この一連のプロパティーは、各接続定義オブジェクトのプロパティーをリトリーブして構成されています。 更新用に表示されている既存の接続ファクトリーの場合、このプロパティーのリストは、オブジェクトに対して指定されているプロパティーでオーバーレイされます。 「タイプ」フィールドを変更すると、そのタイプの適切なプロパティーに合わせてプロパティーも変更されます。
