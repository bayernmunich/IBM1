アプリケーション・クライアントの JMS プロバイダー設定

デフォルト・メッセージング・プロバイダーまたは JMS プロバイダーとしての WebSphere® MQ 以外の JMS プロバイダーを使用する場合は、このページを使用して、Java™ Message Service (JMS) プロバイダーのプロパティーを構成します。

この「アプリケーション・クライアント・リソース構成ツール (ACRCT)」ページを 表示するには、「ファイル」>「オープン」とクリックします。 EAR ファイルのブラウズ後、「オープン」をクリックします。 選択した JAR ファイルを展開します。 「Messaging Providers」を右マウス・ボタンでクリックして、「新規」をクリックします。 「一般」タブ上に以下のフィールドが表示されます。

名前

管理目的で JMS プロバイダーを使用する際の名前。

データ型 ストリング
説明

管理目的で使用する JMS プロバイダーの説明

データ型 ストリング
クラスパス

リソース・プロバイダー・クラスのロケーションを形成する、 パスまたは .jar ファイル名のリスト。

コンテキスト・ファクトリー・クラス

JMS プロバイダーの初期コンテキスト・ファクトリーの Java クラス名。

例えば、LDAP サービス・プロバイダーの場合は、 この値は次のような形式になります。 com.sun.jndi.ldap.LdapCtxFactory

データ型 ストリング
プロバイダー URL

外部 JNDI 検索を行うための JMS プロバイダー URL。

例えば、JMS プロバイダーの LDAP URL は次のような形式になります: ldap://hostname.company.com/contextName

データ型 ストリング
カスタム・プロパティー

このリソースの実行時に作成されるオブジェクトに関する追加のプロパティーを設定するための、 名前と値のペアを指定します。

このオブジェクトのパブリック・プロパティーの名前、およびストリングから このプロパティーのセット・メソッドが必要とするタイプに変換できる値を入力する必要が あります。受け入れ可能なプロパティーと値は作成されるオブジェクトによって 異なります。有効なプロパティーと値のリストについては、 オブジェクトの資料を参照してください。
