デフォルトのプロバイダー・トピック接続ファクトリー設定

WebSphere® Application Server とともにインストールされる内部製品 Java™ Message Service (JMS) プロバイダーで使用するために、このパネルを使用して、選択された JMS トピック接続ファクトリーの構成プロパティーを表示または変更します。この構成プロパティーは、JMS プロバイダーとその JMS プロバイダーが使用するサービス統合バス間で接続が作成される方法を制御します。

この「アプリケーション・クライアント・リソース構成ツール (ACRCT)」ページを 表示するには、「ファイル」>「オープン」とクリックします。 EAR ファイルのブラウズ後、「オープン」をクリックします。 選択した JAR ファイル >「Messaging Providers」>「Default Provider」と展開します。「Topic Connection Factories」を右マウス・ボタン・クリックして、「新規」をクリックします。 「一般」タブ上に以下のフィールドが表示されます。

デフォルト値を持つ設定には、適切な値が表示されます。固定値を持つ設定には、ドロップダウン・メニューがあります。

トピック接続ファクトリーの名前。

データ型 ストリング
説明

IBM® WebSphere Application Server 内で管理目的で使用する、このトピック接続ファクトリーの説明。

データ型 ストリング
JNDI 名

このトピック接続ファクトリー定義をデプロイメント記述子と突き合わせる場合に使用する JNDI 名。この記入項目は、resource-ref 名です。

データ型 ストリング
ユーザー名

呼び出し側のアプリケーションがユーザー ID とパスワードを明示的に指定しない場合に、認証のために「パスワード」プロパティーとともに使用される「ユーザー名」。このフィールドを使用すると、「プロパティー」フィールドのユーザー名は無視されます。

「ユーザー名」プロパティーの値を指定する場合は、「パスワード」プロパティーの値も指定する必要があります。

呼び出し側のアプリケーションが ユーザー ID とパスワードを明示的に提供しない場合は、接続ファクトリーの「ユーザー名」プロパティーと「パスワード」プロパティーが使用されます。 ユーザー名とパスワードが指定されている場合は、パスワードが暗号化されているファクトリーに対して認証別名が作成されます。

データ型 ストリング
パスワード

暗号化を作成する場合に使用されるパスワード。 このフィールドに入力すると、「プロパティー」ボックスの「パスワード」フィールドは無視されます。

「ユーザー名」プロパティーの値を指定する場合は、「パスワード」プロパティーの値も指定する必要があります。

データ型 ストリング
再入力パスワード

データ型 ストリング
非パーシスタント・メッセージの信頼性

この接続ファクトリーを使用して送信される非パーシスタント JMS メッセージに適用される信頼性。

個々の JMS 宛先に異なる信頼性デリバリー・オプションが必要な場合は、 このプロパティーを「バス宛先として」に設定できます。 次に信頼性は、JMS 宛先が割り当てられたバス宛先の Reliability プロパティーによって定義されます。

デフォルト ReliablePersistent
範囲
なし
非パーシスタント・メッセージには、メッセージの信頼性はありません。 非パーシスタント・メッセージが配信できない場合、廃棄されます。
ベストエフォート非パーシスタント
メッセージはディスクに書き込まれず、メモリー・キャッシュがオーバーランした場合に破棄されます。
高速非パーシスタント
メモリー・キャッシュがオーバーランした場合、メッセージは非同期的に永続ストレージへ書き込まれますが、サーバーの再始動の間は保持されません。
高信頼性非パーシスタント
メッセージは、メッセージング・エンジンに障害が発生すると失われることがあり、 通常の作動状態でも失われることがあります。
高信頼性パーシスタント
メッセージは、メッセージング・エンジンに障害が発生すると失われることがあります。 ただし、通常の作動状態では失われません。
保証パーシスタント
保証されたメッセージの配信がサポートされるという最高度の信頼性があります。
バス宛先として
バス宛先用に構成されたデリバリー・オプションを使用します。

永続サブスクリプション・ホーム

永続サブスクリプション・ホームの名前。

データ型 ストリング
永続サブスクリプションの共用

永続サブスクリプションが接続全体でサーバー・クラスターのメンバーと共用されるかどうかを制御します。

通常、特定の永続サブスクリプションに対して TopicSubscriber を持つことができるのは、一度に 1 つのセッションだけです。 このプロパティーにより、この動作をオーバーライドしたり、1 つの永続サブスクリプションで複数の同時コンシューマーを保持したりすることが可能になります。

データ型 選択リスト
デフォルト クラスター
範囲
クラスター
サーバー・クラスター内から接続が行われたときに、永続サブスクリプションの共用が可能になります。
常に共用
接続全体で永続サブスクリプションが共用できます。
共用しない
接続で永続サブスクリプションの共用はできません。

データ型 ストリング
プロバイダー・エンドポイント

ブートストラップ・サーバーへの接続に使用されるコンマ区切りのエンドポイントのリスト。

syntax: host:port:protocol の構文のエンドポイント・トリプレットのコンマ区切りのリストを入力します。

例 
localhost:7777:BootstrapBasicMessaging
各部の意味は、次のとおりです。

BootstrapBasicMessaging は、リモート・プロトコル InboundBasicMessaging (JFAP-TCP/IP) に対応しています。
デフォルト
  • ホスト名が指定されないと、デフォルトの localhost がデフォルト値として使用されます。
  • ポート番号が指定されないと、7276 がデフォルト値として使用されます。
  • チェーン名が指定されないと、BootstrapBasicMessaging などの事前定義チェーンがデフォルト値として使用されます。
