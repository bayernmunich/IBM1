アプリケーション・クライアントの新規 URL プロバイダーの構成

アプリケーション・クライアント・リソース構成ツール (ACRCT) を使用して、 クライアント・アプリケーション用の URL プロバイダーと URL を作成できます。

始める前に

このタスクでは、クライアント・アプリケーションの URL プロバイダーと URL を作成します。 それぞれの管理用タスクで、必要な URL プロバイダーの Java™ コードをクライアント・アプリケーションが常駐するクライアント・マシンにインストールする必要があります。

このタスクについて

手順

  1. アプリケーション・クライアント・リソース構成ツール (ACRCT) を開始します。
  2. 新規 URL プロバイダーを構成する対象である EAR ファイルを開きます。 EAR ファイルの内容がツリー・ビューに表示されます。
  3. ツリーから、新規 URL プロバイダーを構成しようとする JAR ファイルを選択します。
  4. JAR ファイルを展開して、その内容を表示します。
  5. 「URL プロバイダー」という名前のフォルダーをクリックします。 以下のいずれか 1 つのアクションを実行します。
    • フォルダーを右マウス・ボタン・クリックして、「新規」を選択します。
    • メニュー・バーで「編集」>「 新規」をクリックします。
  6. これにより表示されるプロパティー・ダイアログで URL プロバイダーのプロパティーを構成します。
  7. 「OK」をクリックします。
  8. メニュー・バーで、「ファイル」>「保管」をクリックして変更内容を保管します。


マーク付きのリンク (オンライン) では、インターネットにアクセスする必要があります。
