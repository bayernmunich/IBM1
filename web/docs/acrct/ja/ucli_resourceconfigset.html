リソース・アダプター設定

このパネルを使用して、リソース・アダプターの構成プロパティーを表示または変更します。 これらの構成プロパティーによって、リソース・アダプターの 作成方法が制御されます。

この「アプリケーション・クライアント・リソース構成ツール (ACRCT)」ページを 表示するには、「ファイル」>「オープン」とクリックします。 EAR ファイルのブラウズ後、「オープン」をクリックします。 選択した JAR ファイル >「リソース・アダプター」と展開します。「リソース・アダプター」を右マウス・ボタンでクリックして、 「新規」をクリックします。 「一般」タブ上に以下のフィールドが表示されます。

名前

IBM® WebSphere® Application Server 内で、このリソース・アダプターを管理目的で使用する際の名前。この名前は、製品管理可能ドメイン全体のリソース・アダプター内で固有である必要があります。

データ型 ストリング
説明

IBM WebSphere Application Server 内で管理目的で使用する、このリソース・アダプターの説明。

データ型 ストリング
クラスパス

追加のクラスパス。リソース・アダプター・ディレクトリーを参照するパスが 自動的に追加されます。

データ型 ストリング
デフォルト リソース・アダプター・ディレクトリーを参照するパス。
ネイティブ・パス

リソース・アダプターのロケーションのネイティブ・パス。追加のネイティブ・クラスパスがある場合は、 ここに入力します。

データ型 ストリング
リソース・アダプター名

インストール済みリソース・アダプター・サブディレクトリーを指す必須フィールド。 エントリーは、リソース・アダプターの 完全なディレクトリー名を表していません。完全なディレクトリー名は、インストール済みリ ソース・アダプター・パスにリソース・アダプター名を加えたものになります。

データ型 ストリング
インストール済みリソース・アダプター・パス

リソース・アダプターがインストールされているディレクトリー。このフィールドに入力しなかった場合、 デフォルトが有効になります。

値 ${CONNECTOR_INSTALL_ROOT} を指定すると、この値によりクライアント・アプリケーションが稼働しているマシン上の CLIENT_CONNECTOR_INSTALL_ROOT 変数の値が置き換えられます。このアクションにより、クライアントのインストールが別の場所にある場合に、別のマシン上でアプリケーションを容易に実行することができるようになります。

データ型 ストリング
デフォルト ${CONNECTOR_INSTALL_ROOT}
