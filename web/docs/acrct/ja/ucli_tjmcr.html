Java メッセージング・クライアント・リソースの構成

Java™ メッセージング・クライアント・リソースを構成するには、アプリケーション・クライアントの新規 JMS プロバイダー構成を作成します。アプリケーション・クライアントは、Java Message Service API を介して、メッセージング・サービスを使用することができます。JMS プロバイダーは、2 種類の J2EE ファクトリーを提供します。1 つは JMS 接続ファクトリー で、もう 1 つは JMS 宛先ファクトリー です。

始める前に

それぞれの管理用タスクで、アプリケーション・クライアントが常駐するクライアント・マシンに Java Message Service (JMS) クライアントをインストールします。メッセージング製品のベンダーは、JMS クライアントの実装を提供しなければなりません。詳しくは、メッセージング製品の資料を参照してください。

注: このタスクを実行するとき、新規メッセージング・プロバイダーを作成することができます。あるいは、既存のものを使用できます。

手順

  1. アプリケーション・クライアント・リソース構成ツール (ACRCT) を開始します。
  2. 新規 JMS プロバイダーを構成する対象である EAR ファイルを開きます。 EAR ファイルの内容が、現行のツリー・ビューに表示されます。
  3. ツリーから、新規 JMS プロバイダーを構成しようとする JAR ファイルを選択します。
  4. JAR ファイルを展開して、その内容を表示します。
  5. オプションで、新規メッセージング・プロバイダーを作成して使用する場合、「メッセージング・プロバイダー」を右クリックして、「新規」を選択します。
  6. これにより表示されるプロパティー・ダイアログで JMS プロバイダーのプロパティーを構成します。
  7. 「OK」をクリックします。
  8. 「ファイル」>「保管」をクリックします。


マーク付きのリンク (オンライン) では、インターネットにアクセスする必要があります。
