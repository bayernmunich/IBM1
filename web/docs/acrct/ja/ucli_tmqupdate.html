アプリケーション・クライアントの Java Message Service プロバイダーとしての WebSphere MQ およびその JMS リソース構成の更新

Java™ Message Service (JMS) プロバイダーとしての WebSphere® MQ の既存の構成を更新し、WebSphere MQ 接続ファクトリーまたは WebSphere MQ 宛先の構成を更新できます。

このタスクについて

このタスクを使用して、Java Message Service (JMS) プロバイダーとしての WebSphere MQ の既存の構成を更新し、WebSphere MQ 接続ファクトリーまたは WebSphere MQ 宛先の構成を更新します。

手順

  1. アプリケーション・クライアント・リソース構成ツール (ACRCT) を開始します。
  2. WebSphere MQ JMS プロバイダー、WebSphere MQ 接続ファクトリー、または WebSphere MQ 宛先が入ったエンタープライズ・アーカイブ (EAR) ファイルを開きます。 EAR ファイルの内容が、ナビゲーション・ツリー・ビューに表示されます。
  3. 更新対象の JMS プロバイダー、接続ファクトリー、または宛先が入った Java アーカイブ (JAR) ファイルを選択します。
  4. 更新する特定の JMS プロバイダー、接続ファクトリー、 または宛先が見付かるまで JAR ファイルを展開し、その内容を表示します。 以下のいずれか 1 つのアクションを実行します。
    • 該当するオブジェクトを右マウス・ボタンでクリックして、「プロパティー」をクリックします。
    • メニュー・バーで「編集」>「プロパティー」をクリックします。
  5. 表示されるフィールドで、プロパティーを更新します。 フィールド・ヘルプの詳細については、以下を参照してください。
    • JMS プロバイダーのプロパティー
    • MQ キュー接続ファクトリーのプロパティー
    • MQ トピック接続ファクトリーのプロパティー
    • MQ キュー宛先のプロパティー
    • MQ トピック宛先のプロパティー
  6. 「OK」をクリックします。
  7. 「ファイル」>「保管」をクリックして、変更内容を保管します。


マーク付きのリンク (オンライン) では、インターネットにアクセスする必要があります。
