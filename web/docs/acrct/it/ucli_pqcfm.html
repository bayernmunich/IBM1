Impostazioni di produzione di connessioni code del provider MQ di WebSphere per i client delle applicazioni

Utilizzare questo pannello per visualizzare o modificare le proprietà di configurazione della factory di connessione code selezionata da utilizzare con il provider JMS (Java™ Message Service) del prodotto MQSeries®. Queste proprietà di configurazione controllano in che modo vengono create le connessioni tra il provider JMS e WebSphere® MQ.

Per visualizzare questa pagina dello Strumento di configurazione delle risorse del client delle applicazioni (ACRCT), fare clic su File > Apri. Dopo aver individuato un file EAR, fare clic su Apri. Espandere il file JAR selezionato > Provider di messaggistica > Provider WebSphere MQ. Fare clic con il pulsante destro del mouse su Produzioni connessioni code e su Nuovo. Vengono visualizzati i seguenti campi sulla scheda Generale.

Nota:
  • I valori delle proprietà specificati devono corrispondere ai valori specificati durante la configurazione delle risorse JMS per WebSphere MQ. Per ulteriori informazioni sulla configurazione delle risorse JMS per WebSphere MQ, consultare la sezione Utilizzo di Java del centro informazioni di WebSphere MQ.
  • In WebSphere MQ, i nomi possono avere un massimo di 48 caratteri, ad eccezione dei canali, che possono contenere un massimo di 20 caratteri.

Un factory di connessione code per il provider di JMS ha le seguenti proprietà.

Nome

Il nome attribuito a questa factory di connessione code per scopi di gestione all'interno di WebSphere Application Server. Il nome deve essere univoco all'interno delle produzioni connessioni JMS nel dominio di gestione WebSphere.

Tipo dati Stringa
Descrizione

Una descrizione di questo factory di connessione per scopi di gestione all'interno di IBM WebSphere Application Server.

Tipo dati Stringa
Valore predefinito Nullo
Nome JNDI

Il runtime del client delle applicazioni utilizza questo campo per richiamare le informazioni relative alla configurazione.

ID utente

L'ID utente utilizzato, con la proprietà password, per l'autenticazione se l'applicazione chiamante non fornisce esplicitamente un ID utente e una password.

Se viene specificato un valore per la proprietà ID utente, è necessario specificare anche un valore per la proprietà password.

Le proprietà ID utente e password della factory di connessione vengono utilizzate se l'applicazione chiamante non fornisce esplicitamente un ID utente e una password; ad esempio se l'applicazione chiamante utilizza il metodo createQueueConnection(). Il client JMS inoltra l'ID utente e la password al server JMS.

Tipo dati Stringa
Password

La password, con la proprietà ID utente, utilizzata per l'autenticazione, se l'applicazione chiamante non fornisce esplicitamente un id utente ed una password.

Se viene specificato un valore per la proprietà ID utente, è necessario specificare anche un valore per la proprietà password.

Tipo dati Stringa
Valore predefinito Nullo
Immettere di nuovo la password

Conferma la password.

Gestore code

Il nome del gestore code di WebSphere MQ della factory di connessione prescelta.

Le connessioni create da questa produzione si collegano a questo gestore code.

Tipo dati Stringa
Immettere le informazioni relative al nome host ed alla porta

Questo pulsante di opzione è selezionato per impostazione predefinita e, se selezionato, abilita le proprietà host e porta e disabilita la proprietà dell'elenco di nomi connessione.

Tipo dati Pulsante di opzione
Valore predefinito Selezionato
Host

Il nome dell'host su cui è in esecuzione il Gestore code WebSphere MQ, solo per connessioni client.

Tipo dati Stringa
Valore predefinito Nullo
Intervallo Un nome host TCP/IP valido
Porta

Il numero della porta TCP/IP utilizzata per la connessione al gestore code WebSphere MQ, solo per connessioni client.

È necessario configurare questa porta sul gestore code WebSphere MQ.

Tipo dati Numero intero
Valore predefinito Nullo
Intervallo Un numero porta TCP/IP valido, configurato sul gestore code WebSphere MQ.
Immettere le informazioni dell'Elenco nomi connessione

Se selezionato, questo pulsante opzione abilita la proprietà dell'elenco di nomi connessione e disabilita le proprietà nome host e porta. Selezionare questo pulsante opzione se si intende connettersi ad un gestore code di un'istanza multipla.

Tipo dati Pulsante di opzione
Valore predefinito Deselezionato
Elenco nomi connessione

Un elenco separato da virgole di informazioni relative ad host ed a porte che può essere utilizzato per connettersi ad un gestore code ad istanza multipla.

Il formato dell'elenco è:

host[(porta)],[host[(porta)]]
dove la porta è facoltativa ed ha un valore predefinito pari a 1414 se non viene impostata. Ad esempio:
hostname1,hostname2(1415)

Per ulteriori informazioni sui gestori code ad istanza multipla, consultare il centro informazioni di WebSphere MQ.

Questa proprietà deve essere utilizzata solo per effettuare la connessione ad un gestore code ad istanze multiple. Non deve essere utilizzata per la connessioni ad un elenco di gestori code distinti poiché ciò potrebbe comportare problemi di integrità della transazione.

Canale

Il nome del canale utilizzato per la connessione al gestore code WebSphere MQ, solo per la connessione client.

Tipo dati Stringa
Valore predefinito Nullo
Intervallo Da 1 a 20 caratteri ASCII
Tipo di trasporto

Specifica se la connessione client WebSphere MQ o i collegamenti JNDI vengono utilizzati per la connessione al gestore code WebSphere MQ. Il provider JMS esterno controlla i protocolli di comunicazione tra i client JMS e i server JMS. Ottimizzare il tipo di trasporto quando si utilizza la messaggistica non transazionale, temporanea, non permanente, non ASF o quando si intende rispettare questioni di protezione ed il client è locale sul nodo del gestore code.

Tipo dati Enum
Unità di misura Non applicabile
Valore predefinito COLLEGAMENTI
Intervallo
COLLEGAMENTI
I collegamenti JNDI vengono utilizzati per collegarsi al gestore code. BINDINGS è un protocollo di memoria condivisa e può essere utilizzato solo quando il Gestore code si trova sullo stesso nodo del client JMS e si verificano dei rischi di sicurezza che dovrebbero essere risolti utilizzando i ruoli EJB.
CLIENT
La connessione client WebSphere MQ viene utilizzata per connettersi al gestore code. CLIENT è un protocollo basato su TCP tipico.
DIRECT
Per il broker eventi WebSphere MQ che utilizza la modalità DIRECT. DIRECT è un protocollo dei socket leggero utilizzato nella messaggistica Pubblicazione/sottoscrizione non persistente, non duratura e non transazionale. DIRECT funziona solo per i client e i bean basati sui messaggi che utilizzano il protocollo non-ASF.
QUEUED
QUEUED è un protocollo TCP standard.
Consigliato
Tipo di trasporto della factory di connessione di una coda
BINDINGS è più rapido di almeno il 30% ma è meno sicuro. Quando si hanno problemi di protezione, CLIENT è più consigliato di BINDINGS.
Tipo di trasporto della factory di connessione topic
DIRECT rappresenta il tipo più veloce e dovrebbe essere utilizzato dove possibile. Utilizzare BINDINGS quando si desidera soddisfare ulteriori attività di protezione ed il gestore code è locale per il client JMS. QUEUED è il fallback in tutti gli altri casi. WebSphere MQ 5.3 prima di CSD2 con l'impostazione DIRECT poteva perdere i messaggi quando veniva utilizzato con i bean basati sui messaggi e in condizioni di sovraccarico. Questa perdita si verifica anche con le applicazioni lato client a meno che maxClientQueueSize del broker non sia impostato a 0. È possibile impostare questo valore su 0 con il comando: 
#wempschangeproperties WAS_nodeName_server1
-e default -o DynamicSubscriptionEngine -n maxClientQueueSize -v 0 -x executionGroupUUID
dove è possibile individuare executionGroupUUID avviando il broker ed eseguendo una ricerca nel Registro eventi/Applicazioni per l'evento 2201. Tale valore è generalmente ffffffff-0000-0000-000000000000.
Nota: WebSphere MQ 5.3 JMS non può essere utilizzato con WAS 6.1 in quanto WAS 6.1 ha un runtime Java 5. Pertanto, le connessioni tra memorie non possono essere stabilite con i gestori code WebSphere MQ 5.3. Ciò provoca una riduzione delle prestazioni se in precedenza, per le connessioni, si utilizzava WebSphere MQ 5.3 e BINDINGS ed in seguito si è passati alle connessioni di rete CLIENT durante la migrazione a WebSphere Application Server versione 6.1. Se si utilizza WebSphere MQ 5.3 per z/OS®, sarà necessario acquistare anche un ulteriore feature pack.

Quando si esegue su z/OS a 64 bit, il tipo di trasporto deve essere impostato su CLIENT in quanto WebSphere MQ z/OS a 64 bit non è disponibile e la modalità BINDINGS non può essere utilizzata per connettersi a WebSphere MQ z/OS a 31 bit. sarà inoltre necessario acquistare un ulteriore feature pack di WebSphere MQ per questo supporto.
ID client

L'identificatore del client JMS utilizzato per le connessioni al gestore code MQSeries.

Tipo dati Stringa
CCSID

L'identificatore di un insieme di caratteri codificati da utilizzare con il gestore code WebSphere MQ.

Questo CCSID (Coded Character Set IDentifier) deve essere tra quelli supportati da WebSphere MQ.

Tipo dati Stringa

Per ulteriori informazioni sui CCSID supportati e sulla conversione dei dati dei messaggi da un insieme di caratteri codificati ad un altro, consultare le sezioni System Administration e Application Programming Reference del centro informazioni WebSphere MQ.

Tipo dati Stringa

Tipo dati Stringa

Specifica l'intervallo, espresso in millisecondi, in cui un argomento viene sottoposto a scansione per ricercare i messaggi che sono stati aggiunti a un argomento fuori servizio.

Questo intervallo controlla la scansione dei messaggi che sono stati aggiunti all'argomento non in sequenza in relazione a un cursore browse di WebSphere MQ.

Tipo dati Numero intero
Unità di misura Millisecondi
Valore predefinito 5000
Suite di cifratura SSL

Specifica il pacchetto di crittografia da utilizzare per la connessione SSL a WebSphere MQ.

Impostare questa proprietà su un pacchetto di crittografia valido fornito dal provider JSSE. Il valore deve corrispondere al CipherSpec specificato sul canale SVRCONN come proprietà Canale.

È necessario impostare questa proprietà se si imposta la proprietà Nome peer SSL.

Archivio certificati SSL

Specifica un elenco di nessuno o più server CRL (Certificate Revocation List) utilizzati per ricercare la revoca del certificato SSL. Se si specifica un valore per questa proprietà, è necessario utilizzare WebSphere MQ JVM su Java 2 versione 1.4.

Il valore è un elenco di voci delimitato da spazi, nel seguente formato:
ldap://nomehost:[porta]

Questo valore è seguito da una barra singola (/). Se il valore porta viene omesso, si utilizza il valore della porta LDAP predefinita, 389. Durante la connessione, il certificato SSL presentato dal server viene verificato sui server CRL specificati. Per ulteriori informazioni sulla sicurezza CRL, consultare le informazioni relative all'"Utilizzo degli Elenchi di revoca certificato" nella sezione Sicurezza del centro informazioni di WebSphere MQ.

Nome peer SSL

Per SSL, uno skeleton DN (Distinguished Name) che deve corrispondere al nome fornito dal gestore code WebSphere MQ. Il DN viene utilizzato per controllare il certificato di identificazione presentato dal server al momento della connessione.

Se questa proprietà non è impostata, viene eseguito il controllo del certificato.

La proprietà nome peer SSL viene ignorata se non viene specificata la proprietà Suite di cifratura SSL.

Questa proprietà è rappresentata da un elenco di coppie nome e valore attributo separate da virgole o punti e virgola. Ad esempio: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

L'esempio fornito controlla il certificato di identificazione presentato dal server al momento della connessione. Per la riuscita della connessione, il nome comune (CN, Common Name) del certificato deve iniziare con QMGR. e deve disporre di almeno due nomi unità organizzative (OU, Organizational Unit), ossia la prima IBM e la seconda WEBSPHERE. Il controllo non distingue tra maiuscolo e minuscolo.

Per ulteriori informazioni dettagliate su DN (distinguished name) e sul relativo utilizzo con WebSphere MQ, consultare la sezione "Distinguished Names" nel centro informazioni WebSphere MQ.

Pool di connessioni

Specifica un insieme facoltativo di impostazioni del lotto di connessioni.

Le proprietà del pool di connessioni sono comuni a tutti i connettori J2C.

Il server delle applicazioni raggruppa in pool le connessioni e le sessioni insieme al provider JMS, per migliorare le prestazioni. Ciò non dipende da alcun lotto di connessioni di WebSphere MQ. È necessario configurare le proprietà pool connessioni e sessioni in modo adeguato alle applicazioni, altrimenti non è possibile richiamare il funzionamento connessioni e sessioni desiderato.

Modificare le dimensioni del pool di connessioni se l'accesso simultaneo lato server alla risorsa JMS supera il valore predefinito. Le dimensioni del pool di connessioni sono impostate su base dell'argomento o della coda.

Tipo dati Casella di spunta
Valore predefinito Selezionato
Opzioni di riconnessione del client

Specifica se una connessione in modalità client effettua automaticamente la riconnessione nel caso in cui si verifichi un errore di comunicazione o del gestore code. Questa proprietà viene ignorata a meno che la factory di connessione viene utilizzata in un ambiente thin client o a client gestito.

Tipo dati Elenco a discesa
Valore predefinito DISABLED
Intervallo
DISABLED
La riconnessione del client non avviene in modo automatico.
ASDEF
Viene utilizzato il valore dell'attributo DefRecon della stanza canali del file di configurazione del client. Se non si specifica alcun valore DefRecon, questa impostazione avrà lo stesso comportamento di quando si imposta il valore DISABLED.
RECONNECT
La riconnessione viene effettuata per qualsiasi gestore code che abbia un valore congruente a quello dell'attributo gestore code, che potrebbe essere un gestore code diverso da quello verso cui si era originariamente effettuata la connessione.
QMGR
La riconnessione viene effettuata solo per il gestore code verso cui si era originariamente effettuata la connessione.

Per ulteriori informazioni sulla riconnessione automatica del client, consultare il centro informazioni di WebSphere MQ.

Timeout di riconnessione del client

La quantità massima di tempo, in secondi, che una connessione in modalità client spenderà per cercare di effettuare la riconnessione automatica ad un gestore code dopo un errore di comunicazione o del gestore code. Questo parametro viene ignorato a meno che la factory di connessione non venga utilizzata in un ambiente thin client o a client gestito. L'utilizzo di questo parametro dipende dal valore dei parametri delle opzioni di riconnessione del client.

Tipo dati Numero intero
Unità di misura Secondi
Valore predefinito 1800
Intervallo Un valore compreso tra zero e 2147483647

Per ulteriori informazioni sulla riconnessione automatica del client, consultare il centro informazioni di WebSphere MQ.
