Impostazioni della destinazione argomento del Provider MQ di WebSphere per i client delle applicazioni

Utilizzare questo pannello per visualizzare o modificare le proprietà di configurazione della destinazione dell'argomento selezionato da utilizzare con il provider JMS (Java™ Message Service) del prodotto WebSphere® MQ.

Per visualizzare questa pagina dello Strumento di configurazione delle risorse del client delle applicazioni (ACRCT), fare clic su File > Apri. Dopo aver individuato un file EAR, fare clic su Apri. Espandere il file JAR selezionato > Provider dei messaggi > Provider WebSphere MQ. Fare clic con il pulsante destro del mouse su Destinazioni argomenti e su Nuovo. Vengono visualizzati i seguenti campi sulla scheda Generale.

Nota:
  • I valori delle proprietà specificati devono corrispondere ai valori specificati durante la configurazione delle risorse JMS per WebSphere MQ. Per ulteriori informazioni sulla configurazione delle risorse JMS per WebSphere MQ, consultare Utilizzo di Java nel centro informazioni di WebSphere MQ.
  • In WebSphere MQ, i nomi non possono contenere più di 48 caratteri.

Una destinazione argomento viene utilizzata per configurare le proprietà di un argomento JMS per il provider JMS associato. Un argomento da utilizzare con il provider JMS del prodotto WebSphere MQ ha le seguenti proprietà.

Nome

Il nome attribuito all'argomento per scopi di gestione all'interno di WebSphere Application Server.

Tipo dati Stringa
Descrizione

Una descrizione dell'argomento per scopi di gestione all'interno di WebSphere Application Server.

Tipo dati Stringa
Nome JNDI

L'ambiente di runtime del client delle applicazioni utilizza questo campo per richiamare le informazioni relative alla configurazione.

Persistenza

Specifica se tutti i messaggi inviati alla destinazione sono persistenti, non persistenti o se tale proprietà viene definita dall'applicazione.

Tipo dati Enum
Valore predefinito APPLICATION_DEFINED
Intervallo
Definita dall'applicazione
I messaggi sulla destinazione hanno la persistenza definita dall'applicazione che li ha inseriti nella coda.
Definita dalla coda
[Solo una destinazione WebSphere MQ] I messaggi sulla destinazione hanno la persistenza definita dalle proprietà di definizione delle code di WebSphere MQ.
Persistente
I messaggi sulla destinazione sono permanenti.
Non persistente
I messaggi sulla destinazione non sono permanenti.
Priorità

Se la priorità messaggio di questa destinazione viene definita dall'applicazione o dalla proprietà Priorità specificata.

Tipo dati Enum
Valore predefinito APPLICATION_DEFINED
Intervallo
Definita dall'applicazione
La priorità dei messaggi su questa destinazione viene definita dall'applicazione che li ha inseriti nella destinazione.
Definita dalla coda
[Solo una destinazione WebSphere MQ] I messaggi sulla destinazione hanno la persistenza definita dalle proprietà di definizione delle code di WebSphere MQ.
Specificata
La priorità dei messaggi in questa destinazione è definita dalla proprietà Priorità specificata. Se viene selezionata questa opzione, è necessario definire una priorità per la proprietà Priorità specificata.
Priorità specificata

Se la proprietà Priorità è impostata su Specificato, indicare la priorità da applicare ai messaggi di questa coda nell'intervallo compreso tra 0 (il valore più basso) a 9 (il valore più alto).

Se la proprietà Priorità è impostata su Specificata, questa proprietà specifica il valore della priorità dei messaggi inviati a tale coda.

Tipo dati Numero intero
Unità di misura Livello di priorità messaggio
Intervallo 0 (priorità minima) - 9 (priorità massima)
Scadenza

Se il timeout di scadenza della coda prescelta è definito dall'applicazione o dalla proprietà Scadenza specificata, oppure se i messaggi nella coda sono permanenti (cioè ha un timeout di scadenza illimitato).

Tipo dati Enum
Valore predefinito APPLICATION_DEFINED
Intervallo
Definita dall'applicazione
Il timeout di scadenza dei messaggi presenti in questa coda viene definito dall'applicazione che li ha inseriti nella coda.
Specificata
Il timeout di scadenza dei messaggi presenti in questa coda viene definito dalla proprietà Scadenza specificata. Se viene selezionata questa opzione, è necessario definire un valore di timeout per la proprietà Scadenza specificata.
Illimitato
I messaggi presenti in questa coda non hanno un timeout di scadenza, per cui non scadono mai.
Scadenza specificata

Se la proprietà Timeout di scadenza viene impostata su Specificato, digitare il numero di millisecondi (maggiore di 0) trascorsi i quali i messaggi presenti in questa coda scadono.

Tipo dati Numero intero
Unità di misura Millisecondi
Intervallo Maggiore o uguale a 0
  • 0 indica che i messaggi non prevedono timeout.
  • Altri valori sono espressi con un numero intero di millisecondi.
Nome argomento di base

Il nome dell'argomento a cui vengono inviati i messaggi.

Tipo dati Stringa
CCSID

L'identificatore di un insieme di caratteri codificati da utilizzare con il Gestore code WebSphere MQ.

Questo CCSID (Coded Character Set IDentifier) deve essere tra quelli supportati da WebSphere MQ. Per ulteriori informazioni consultare Impostazioni delle proprietà avanzate argomenti e code del provider dei messaggi WebSphere MQ

Tipo dati Stringa
Unità di misura Numero intero
Intervallo Da 1 a 65535

Codifica nativa

Indica che la destinazione coda utilizza la codifica nativa (valori di codifica appropriati alla piattaforma Java), quando si seleziona questa casella di controllo.

BrokerDurSubQueue

Il nome della coda broker da cui vengono richiamati i messaggi di sottoscrizione duraturi.

L'autore specifica il nome della coda quando registra una sottoscrizione.

Tipo dati Stringa
Unità di misura Caratteri En_US ASCII
Intervallo Da 1 a 48 caratteri ASCII
BrokerCCDurSubQueue

Il nome della coda broker da cui vengono richiamati i messaggi di sottoscrizione duraturi per un ConnectionConsumer. Questa proprietà viene applicata solo per utilizzare il contenitore web.

Tipo dati Stringa
Unità di misura Caratteri En_US ASCII
Intervallo Da 1 a 48 caratteri ASCII
Client di destinazione

Se l'applicazione ricevente è compatibile con JMS oppure se si tratta di un'applicazione WebSphere MQ tradizionale.

Tipo dati Enum
Valore predefinito WebSphere MQ
Intervallo
WebSphere MQ
La destinazione è un'applicazione WebSphere MQ tradizionale che non supporta JMS.
JMS
L'applicazione di destinazione supporta JMS.
Corpo del messaggio

Specifica se un'applicazione elabora l'intestazione RFH versione 2 di un messaggio WebSphere MQ come parte del corpo del messaggio JMS.

Tipo dati Elenco a discesa
Valore predefinito UNSPECIFIED
Intervallo
UNSPECIFIED
Durante l'invio di un messaggio, il provider dei messaggi WebSphere MQ genera o meno, ed include un'intestazione RFH versione 2, a seconda del valore della proprietà Aggiungi intestazioni RFH versione 2 ai messaggi inviati a questa destinazione. Quando riceve dei messaggi, il provider dei messaggi di WebSphere MQ si comporta come se il valore fosse impostato su JMS.
JMS
Quando invia messaggi, il provider dei messaggi WebSphere MQ genera automaticamente un'intestazione RFH versione 2 e la include nel messaggio WebSphere MQ. Quando riceve messaggi, il provider dei messaggi di WebSphere MQ imposta le proprietà del messaggio JMS in base ai valori presenti in Intestazione RFH versione 2 (se ve ne sono); non presenta l'intestazione RFH versione 2 come parte del corpo del messaggio JMS.
MQ
Quando invia messaggi, il provider dei messaggi WebSphere MQ non genera un'intestazione RFH versione 2. Quando riceve messaggi, il provider d ei messaggi WebSphere MQ presenta l'intestazione RFH versione 2 come parte del corpo del messaggio JMS.
Stile di destinazione ReplyTo

Specifica il formato del campo JMSReplyTo.

Tipo dati Elenco a discesa
Valore predefinito DEFAULT
Intervallo
DEFAULT
Il valore predefinito è equivalente alle informazioni contenute nell'intestazione RFH versione 2.
MQMD
Utilizza il valore fornito nel MQMD. Ciò riempirà il campo Rispondi a gestore code con il valore contenuto in MQMD, equivalente al comportamento predefinito di WebSphere MQ Versione 6.0.2.4 e 6.0.2.5.
RFH2
Utilizza il valore fornito nell'intestazione RFH versione 2. Se nell'applicazione mittente è impostato un valore JMSReplyTo, si utilizzerà tale valore.
Multicast

Tipo dati Enum

Intervallo

DISABLED
Questa produzione connessioni non utilizza il trasporto multicast.
NOT_RELIABLE
Questa produzione connessione utilizza sempre il trasporto multicast.
RELIABLE
Questa produzione connessione utilizza il trasporto multicast quando la destinazione argomento non è affidabile.
ENABLED
Questa produzione connessione utilizza il trasporto multicast affidabile.

Contesto del messaggio MQMD

Definisce le opzioni di contesto del messaggio specificate quando si invia un messaggio ad una destinazione.

Tipo dati Elenco a discesa
Valore predefinito DEFAULT
Intervallo
DEFAULT
La chiamata MQOPEN API e la struttura MQPMO non specificano alcuna opzione di contesto del messaggio esplicita.
SET_IDENTITY_CONTEXT
La chiamata MQOPEN API specifica l'opzione di contesto del messaggio MQOO_SET_IDENTITY_CONTEXT e la struttura MQPMO specifica MQPMO_SET_IDENTITY_CONTEXT.
SET_ALL_CONTEXT
La chiamata MQOPEN API specifica l'opzione di contesto del messaggio MQOO_SET_ALL_CONTEXT e la struttura MQPMO specifica MQPMO_SET_ALL_CONTEXT.
