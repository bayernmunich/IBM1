Configurazione di nuovi provider JMS con lo Strumento di configurazione risorse del client delle applicazioni

È possibile creare nuove configurazioni JMS (Java Message Service) per il client delle applicazioni. Il client delle applicazioni utilizza il servizio di messaggistica tramite le interfacce JMS.

Informazioni su questa attività

Durante questa attività, creare nuove configurazioni del provider JMS (Java Message Service) per il client delle applicazioni. Il client delle applicazioni utilizza il servizio di messaggistica tramite le interfacce JMS. Un provider JMS fornisce due tipi di risorse J2EE. La prima è una factory di connessione JMS, la seconda è una destinazione JMS.

In un'attività di gestione separata, installare il client JMS sulla macchina client dove risiede il client delle applicazioni specifico. Il fornitore del prodotto di messaggistica deve fornire un'implementazione del client JMS. Per ulteriori informazioni, consultare la documentazione del prodotto di messagistica.

Procedura

  1. Avviare lo strumento ACRCT e aprire il file EAR per cui si desidera configurare il nuovo provider JMS (Java Message Service). I contenuti del file EAR sono visualizzati in una vista ad albero.
  2. Dalla vista ad albero, selezionare il file JAR in cui si desidera configurare il nuovo provider di JMS.
  3. Espandere il file JAR per visualizzarne i contenuti.
  4. Fare clic con il tasto destro del mouse su Provider dei messaggi. Completare una delle seguenti operazioni:
    • Fare clic con il tasto destro del mouse sulla cartella e selezionare Nuovo.
    • Sulla barra dei menu, fare clic su Modifica > Nuovo.
  5. Nella finestra di dialogo risultante, configurare le proprietà del provider di JMS.
  6. Fare clic su OK al termine dell'operazione.
  7. Fare clic su File > Salva sulla barra dei menu, per salvare le modifiche.

Esempio

Il seguente codice di esempio illustra come effettuare la configurazione delle impostazioni del provider JMS, della factory di connessione JMS e della destinazione JMS per i client dell'applicazione. 

<resources.jms:JMSProvider xmi:id="JMSProvider_3" name="genericJMSProvider:name"
description="genericJMSProvider:description"
externalInitialContextFactory="genericJMSProvider:contextFactoryClass"
externalProviderURL="genericJMSProvider:providerUrl">
<classpath>genericJMSProvider:classpath</classpath>
<factories xmi:type="resources.jms:GenericJMSDestination"
xmi:id="GenericJMSDestination_1" name="jmsDestination:name"
jndiName="jmsDestination:jndiName" description="jmsDestination:description"
externalJNDIName="jmsDestination:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_15">
<resourceProperties xmi:id="J2EEResourceProperty_17" name="jmsDestination:customName"
value="jmsDestination:customValue"/>
</propertySet>
</factories>
<factories xmi:type="resources.jms:GenericJMSConnectionFactory"
xmi:id="GenericJMSConnectionFactory_1" name="jmsCF:name" jndiName="jmsCF:jndiName"
description="jmsCF:description" userID="jmsCF:user" password="{xor}NTIsHBllMT4yOg=="
externalJNDIName="jmsCF:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_16">
<resourceProperties xmi:id="J2EEResourceProperty_18" name="jmsCF:customName"
value="jmsCF:customValue"/>
</propertySet>
</factories>
<propertySet xmi:id="J2EEResourcePropertySet_17">
<resourceProperties xmi:id="J2EEResourceProperty_19"
name="genericJMSProvider:customName" value="genericJMSProvider:customValue"/>
</propertySet>
</resources.jms:JMSProvider>

Tra i campi obbligatori sono inclusi:

  • Pagina Proprietà di provider di JMS: nome e almeno un provider di protocollo
  • Pagina Proprietà di factory di connessione JMS: nome, jndiName, tipo di destinazione
  • Pagina Proprietà di destinazione JMS: nome, jndiName, tipo di destinazione

Casi speciali:

  • Il tipo di destinazione deve essere QUEUE o TOPIC.
