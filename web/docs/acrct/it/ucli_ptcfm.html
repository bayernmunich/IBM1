Impostazioni di factory di connessione argomenti del provider MQ di WebSphere per i client delle applicazioni

Utilizzare questo pannello per visualizzare o modificare le proprietà di configurazione della factory di connessione argomento selezionata per essere utilizzata con il provider JMS (Java™ Message Service) del prodotto WebSphere®. Queste proprietà di configurazione controllano in che modo vengono create le connessioni tra il provider JMS e WebSphere MQ.

Per visualizzare questa pagina dello Strumento di configurazione delle risorse del client delle applicazioni (ACRCT), fare clic su File > Apri. Dopo aver individuato un file EAR, fare clic su Apri. Espandere il file JAR selezionato > Provider di messaggistica > Provider WebSphere MQ. Fare clic con il pulsante destro del mouse su Produzioni connessioni argomenti e su Nuovo.

Nota:
  • I valori delle proprietà specificati devono corrispondere ai valori specificati durante la configurazione delle risorse JMS per WebSphere MQ. Per ulteriori informazioni sulla configurazione delle risorse JMS per WebSphere MQ, consultare la sezione Utilizzo di Java del centro informazioni di WebSphere MQ.
  • In WebSphere MQ, i nomi possono avere un massimo di 48 caratteri, ad eccezione dei canali, che possono contenere un massimo di 20 caratteri.
Broker MA0C: quando si crea una produzione di connessione per la sezione WebSphere Application Server v6 per il broker MA0C, è necessario considerare i seguenti valori degli attributi:
BrokerControlQueue
Questo valore è fisso per SYSTEM.BROKER.CONTROL.QUEUE per il broker MA0C e rappresenta la coda da cui il broker legge.
BrokerVersion
Impostare questo valore su BASIC per il broker MA0C.
ClientID
Impostare questo valore su qualsiasi valore desiderato per il broker MA0C (il valore è string ed è semplicemente un identificativo per l'applicazione client).
Abilitato per XA
Impostare questo valore su TRUE o FALSE per il broker MA0C (l'impostazione che viene utilizzata è un indicatore di miglioramento delle prestazioni, ma nella maggior parte dei casi verrà impostata su 'true').
Selezione BrokerMessage
Questo valore è fisso su CLIENT per il broker MA0C in quanto questo si basa sulla selezione di messaggi lato client.
Tipo di autorizzazione broker diretta
Questo valore non è richiesto per il broker MA0C.

Una factory di connessione argomenti del provider JMS del prodotto WebSphere MQ ha le seguenti proprietà.

Nome

Il nome attribuito a questo factory di connessione argomenti per scopi di gestione all'interno di WebSphere Application Server. Il nome deve essere univoco all'interno del provider JMS.

Tipo dati Stringa
Descrizione

Una descrizione di questa factory di connessione argomenti per scopi di gestione all'interno di WebSphere Application Server.

Tipo dati Stringa
Nome JNDI

Il nome JNDI (Java Naming and Directory Interface) utilizzato per collegare la factory di connessione argomenti allo spazio nome del server delle applicazioni.

Per convenzione, utilizzare il nome completo JNDI; ad esempio, nel modulo jms/Nome, dove Nome è il nome logico della risorsa.

Tale nome viene utilizzato per il binding alle informazioni specifiche della piattaforma. Il binding associa le risorse definite dal descrittore di distribuzione del modulo alle risorse effettive (fisiche) collegate a JNDI dalla piattaforma.

Tipo dati Stringa
Unità di misura Caratteri En_US ASCII
Intervallo Da 1 a 45 caratteri ASCII
ID utente

L'ID utente utilizzato, con la proprietà password, per l'autenticazione se l'applicazione chiamante non fornisce esplicitamente un ID utente e una password.

Se viene specificato un valore per la proprietà ID utente, è necessario specificare anche un valore per la proprietà password.

Le proprietà ID utente e password della factory di connessione vengono utilizzate se l'applicazione chiamante non fornisce esplicitamente un ID utente e una password, ad esempio se l'applicazione chiamante utilizza il metodo createTopicConnection(). Il client JMS inoltra l'ID utente e la password al server JMS.

Tipo dati Stringa
Password

La password, con la proprietà ID utente, utilizzata per l'autenticazione, se l'applicazione chiamante non fornisce esplicitamente un id utente ed una password.

Se viene specificato un valore per la proprietà ID utente, è necessario specificare anche un valore per la proprietà password.

Tipo dati Stringa
Immettere di nuovo la password

Confermare la password.

Gestore code

Il nome del gestore code di WebSphere MQ della factory di connessione prescelta. Le connessioni create da questa factory di connessione si collegano a questo gestore code.

Tipo dati Stringa
Immettere le informazioni relative al nome host ed alla porta

Questo pulsante di opzione è selezionato per impostazione predefinita e, se selezionato, abilita le proprietà host e porta e disabilita la proprietà dell'elenco di nomi connessione.

Tipo dati Pulsante di opzione
Valore predefinito Selezionato
Host

Il nome dell'host su cui è in esecuzione il Gestore code WebSphere MQ, solo per connessioni client.

Tipo dati Stringa
Intervallo Un nome host TCP/IP valido
Porta

Il numero della porta TCP/IP utilizzata per la connessione al gestore code WebSphere MQ, solo per connessioni client.

È necessario configurare questa porta sul gestore code WebSphere MQ.

Tipo dati Numero intero
Intervallo Un numero porta TCP/IP valido, configurato sul gestore code WebSphere MQ.
Immettere le informazioni dell'Elenco nomi connessione

Se selezionato, questo pulsante opzione abilita la proprietà dell'elenco di nomi connessione e disabilita le proprietà nome host e porta. Selezionare questo pulsante opzione se si intende connettersi ad un gestore code di un'istanza multipla.

Tipo dati Pulsante di opzione
Valore predefinito Deselezionato
Elenco nomi connessione

Un elenco separato da virgole di informazioni relative ad host ed a porte che può essere utilizzato per connettersi ad un gestore code ad istanza multipla.

Il formato dell'elenco è:

host[(porta)],[host[(porta)]]
dove la porta è facoltativa ed ha un valore predefinito pari a 1414 se non viene impostata. Ad esempio:
hostname1,hostname2(1415)

Per ulteriori informazioni sui gestori code ad istanza multipla, consultare il centro informazioni di WebSphere MQ.

Questa proprietà deve essere utilizzata solo per effettuare la connessione ad un gestore code ad istanze multiple. Non deve essere utilizzata per la connessioni ad un elenco di gestori code distinti poiché ciò potrebbe comportare problemi di integrità della transazione.

Canale

Il nome del canale utilizzato per le connessioni client al Gestore code WebSphere MQ, solo per la connessione client.

Tipo dati Stringa
Intervallo Da 1 a 20 caratteri ASCII
Tipo di trasporto

Suite di cifratura SSL

Il pacchetto di crittografia da utilizzare per la connessione SSL a WebSphere MQ.

Impostare questa proprietà su un pacchetto di crittografia valido fornito dal provider JSSE. Il valore deve corrispondere al CipherSpec specificato sul canale SVRCONN come proprietà Canale.

È necessario impostare questa proprietà se si imposta la proprietà Nome peer SSL.

Archivio certificati SSL

Un elenco di nessuno o più server CRL (Certificate Revocation List) utilizzati per ricercare la revoca del certificato SSL. Se si specifica un valore per questa proprietà, è necessario utilizzare WebSphere MQ JVM su Java 2 versione 1.4.

Il valore è un elenco di voci delimitato da spazi, nel seguente formato:
ldap://nomehost:[porta]

Questo valore è seguito da una barra singola (/). Se il valore porta viene omesso, si utilizza il valore della porta LDAP predefinita, 389. Durante la connessione, il certificato SSL presentato dal server viene verificato sui server CRL specificati. Per ulteriori informazioni sulla sicurezza CRL, consultare le informazioni relative all'"Utilizzo degli Elenchi di revoca certificato" nella sezione Sicurezza del centro informazioni di WebSphere MQ.

Nome peer SSL

Per SSL, uno skeleton DN (Distinguished Name) che deve corrispondere al nome fornito dal gestore code WebSphere MQ. Il DN viene utilizzato per controllare il certificato di identificazione presentato dal server al momento della connessione.

Se questa proprietà non è impostata, viene eseguito il controllo del certificato.

La proprietà nome peer SSL viene ignorata se non viene specificata la proprietà Suite di cifratura SSL.

Questa proprietà è rappresentata da un elenco di coppie nome e valore attributo separate da virgole o punti e virgola. Ad esempio: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

L'esempio fornito controlla il certificato di identificazione presentato dal server al momento della connessione. Per la riuscita della connessione, il nome comune (CN, Common Name) del certificato deve iniziare con QMGR. e deve disporre di almeno due nomi unità organizzative (OU, Organizational Unit), ossia la prima IBM e la seconda WEBSPHERE. Il controllo non distingue tra maiuscolo e minuscolo.

Per ulteriori informazioni dettagliate su DN (distinguished name) e sul relativo utilizzo con WebSphere MQ, consultare le informazioni relative a "Distinguished Names" nella sezione Sicurezza del centro informazioni WebSphere MQ.

Pool di connessioni

Un insieme facoltativo di impostazioni lotto connessioni.

Le proprietà del pool di connessioni sono comuni a tutti i connettori J2C.

Il server delle applicazioni raggruppa in pool le connessioni e le sessioni insieme al provider JMS, per migliorare le prestazioni. Ciò non dipende da alcun lotto di connessioni di WebSphere MQ. È necessario configurare le proprietà pool connessioni e sessioni in modo adeguato alle applicazioni, altrimenti non è possibile richiamare il funzionamento connessioni e sessioni desiderato.

Modificare le dimensioni del pool di connessioni se l'accesso simultaneo lato server alla risorsa JMS supera il valore predefinito. Le dimensioni del pool di connessioni sono impostate su base dell'argomento o della coda.

Tipo dati Casella di spunta
Valore predefinito Selezionato
Opzioni di riconnessione del client

Specifica se una connessione in modalità client effettua automaticamente la riconnessione nel caso in cui si verifichi un errore di comunicazione o del gestore code. Questa proprietà viene ignorata a meno che la factory di connessione non venga utilizzata in un ambiente thin client o a client gestito.

Tipo dati Elenco a discesa
Valore predefinito DISABLED
Intervallo
DISABLED
La riconnessione del client non avviene in modo automatico.
ASDEF
Viene utilizzato il valore dell'attributo DefRecon della stanza canali del file di configurazione del client. Se non si specifica alcun valore DefRecon, questa impostazione avrà lo stesso comportamento di quando si imposta il valore DISABLED.
RECONNECT
La riconnessione viene effettuata per qualsiasi gestore code che abbia un valore congruente a quello dell'attributo gestore code, che potrebbe essere un gestore code diverso da quello verso cui si era originariamente effettuata la connessione.
QMGR
La riconnessione viene effettuata solo per il gestore code verso cui si era originariamente effettuata la connessione.

Per ulteriori informazioni sulla riconnessione automatica del client, consultare il centro informazioni di WebSphere MQ.

Timeout di riconnessione del client

La quantità massima di tempo, in secondi, che una connessione in modalità client spenderà per cercare di effettuare la riconnessione automatica ad un gestore code dopo un errore di comunicazione o del gestore code. Questo parametro viene ignorato a meno che la factory di connessione non venga utilizzata in un ambiente thin client o a client gestito. L'utilizzo di questo parametro dipende dal valore dei parametri delle opzioni di riconnessione del client.

Tipo dati Numero intero
Unità di misura Secondi
Valore predefinito 1800
Intervallo Un valore compreso tra zero e 2147483647

Per ulteriori informazioni sulla riconnessione automatica del client, consultare il centro informazioni di WebSphere MQ.
