URL de clients d'application

Une adresse URL est un identificateur qui pointe vers une ressource accessible par des moyens informatiques, comme un fichier de répertoire sur une machine réseau ou un document dans une base de données.

Le format des URL est le suivant : schéma:informations_schéma.

Un schéma peut être représenté par http, ftp, file ou tout autre terme identifiant le type de ressource et le mécanisme à l'aide duquel vous pouvez accéder à la ressource.

Dans la zone d'adresse d'un navigateur Web, l'adresse URL d'un fichier accessible à l'aide du protocole HTTP (HyperText Transfer Protocol) commence par http. (par exemple, http://www.ibm.com), L'adresse des fichiers accessibles par le biais du protocole FTP (File Transfer Protocol) commence par ftp:. Celle des fichiers accessibles localement commence par file:.

La partie informations_schéma identifie en général la machine qui publie une ressource sur Internet, le chemin d'accès à cette ressource et le nom de la ressource. Pour le schéma HTTP, FTP et File, ces informations commencent par deux barres obliques (//), suivies de l'adresse Internet séparée du nom de chemin de la ressource par une barre oblique (/). Exemple,

http://www.ibm.com/software/webservers/appserv/library.html.

Pour HTTP et FTP, le chemin d'accès se termine par une barre oblique lorsque l'adresse URL pointe vers un répertoire. Dans ce cas, le serveur renvoie en général l'index par défaut de ce répertoire.



Les liens marqués (en ligne) requièrent un accès à Internet.
