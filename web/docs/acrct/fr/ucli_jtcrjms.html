Paramètres de session JavaMail des clients d'application

Cette page permet de configurer les propriétés de la session de messagerie.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > instance de fournisseur de messagerie. Cliquez avec le bouton droit de la souris sur Sessions Javamail, puis cliquez sur Nouveau. Les zones ci-dessous apparaissent dans l'onglet Général.

Nom

Indique le nom administratif de l'objet de session JavaMail.

Description

(Facultatif) Décrit vos enregistrements administratifs.

Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

Hôte de transport des messages

Indique le serveur auquel se connecter lors de l'envoi de messages.

Protocole de transport des messages

Indique le protocole de transport à utiliser lors de l'envoi de messages.

Utilisateur pour le transport du courrier

Indique l'ID utilisateur à employer lorsque l'hôte de transport des messages doit être authentifié.

Mot de passe de transport des messages

Indique le mot de passe à employer lorsque l'hôte de transport des messages doit être authentifié.

Activer l'analyse d'adresses Internet stricte

Indique si les adresses des destinataires doivent être analysées strictement en fonction de RFC 822, qui est un document de spécification émis par le conseil IAB (Internet Architecture Board).

Ce paramètre n'est généralement pas utilisé pour la plupart des applications de messagerie. La syntaxe RFC 822 permettant d'analyser les adresses applique une définition stricte d'une adresse électronique valide. Si vous sélectionnez ce paramètre, JavaMail se conforme à la syntaxe RFC 822 et refuse les adresses des destinataires qui ne sont pas des adresses électroniques valides (les adresses électroniques valides sont définies par la spécification). Si vous ne sélectionnez pas ce paramètre, JavaMail ne se conforme pas à la syntaxe RFC 822 et accepte les adresses des destinataires qui ne sont pas conformes à la spécification. Par défaut, ce paramètre n'est pas sélectionné. Vous pouvez consulter la spécification RFC 822 à l'URL suivante pour W3C (World Wide Web Consortium) : http://www.w3.org/Protocols/rfc822/.

Message de

Hôte de stockage des messages

Utilisateur de stockage des messages

Spécifie l'ID utilisateur du compte de messagerie.

Mot de passe de stockage des messages

Spécifie le mot de passe du compte de messagerie.

Confirmation du mot de passe

Confirme le mot de passe.

Protocole de stockage des messages

Indique le protocole à utiliser lors de la réception de messages.

Débogage du courrier

Lorsque la valeur est true, les données d'interaction JavaMail avec les serveurs de courrier, ainsi que les propriétés des sessions JavaMail, sont dirigées vers le fichier stdout.

Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.
