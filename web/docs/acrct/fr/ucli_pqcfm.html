Paramètre de la fabrique de connexions de file d'attente du fournisseur WebSphere MQ pour tous les clients d'application

Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions de file d'attente sélectionnée à utiliser avec le fournisseur JMS (Java™ Messaging Service) du produit MQSeries®. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et WebSphere® MQ.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > Fournisseur WebSphere MQ. Cliquez avec le bouton droit de la souris sur Fabriques de connexions de file d'attente puis cliquez sur Nouveau. Les zones ci-dessous sont affichées dans la page Général.

Remarque :
  • Les valeurs de propriété que vous indiquez doivent correspondre aux valeurs que vous avez indiquées lors de la configuration des ressources JMS pour WebSphere MQ. Pour plus d'informations sur la configuration des ressources JMS pour WebSphere MQ, reportez-vous à la section Using Java du centre de documentation WebSphere MQ.
  • Dans WebSphere MQ, les noms peuvent inclure jusqu'à 48 caractères, à l'exception des canaux, qui sont limités à 20.

La fabrique de connexions de file d'attente du fournisseur JMS possède les propriétés suivantes :

Nom

Nom sous lequel la fabrique de connexions de file d'attente est connue dans WebSphere Application Server à des fins d'administration. Le nom doit être unique parmi les fabriques de connexions JMS du domaine administratif WebSphere.

Type de données Chaîne
Description

Description de cette fabrique de connexions en vue de son administration dans WebSphere Application Server.

Type de données Chaîne
Valeur par défaut Null
Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

ID utilisateur

ID utilisateur, utilisé avec la propriété de mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement d'ID utilisateur ou de mot de passe.

Si vous indiquez une valeur pour la propriété d'ID utilisateur vous devez également indiquer une valeur pour la propriété de mot de passe.

Les propriétés d'ID utilisateur et de mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas expressément d'ID utilisateur et de mot de passe ; par exemple, si l'application appelante utilise la méthode createQueueConnection(). Le client JMS transmet l'ID utilisateur et le mot de passe au serveur JMS.

Type de données Chaîne
Mot de passe

Mot de passe utilisé avec la propriété d'ID utilisateur pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété d'ID utilisateur vous devez également indiquer une valeur pour la propriété de mot de passe.

Type de données Chaîne
Valeur par défaut Null

Confirme le mot de passe.

Liste de noms de connexion

Liste séparée par des virgules d'informations d'hôte et de port pouvant être utilisée pour se connecter à un gestionnaire de files d'attente multi-instance.

Le format de la liste est : 

hôte[(port)],[hôte[(port)]]
où port est facultatif et prend la valeur par défaut 1414 s'il n'est pas défini. Exemple :

Pour plus d'informations sur les gestionnaires de files d'attente multi-instance, voir le centre de documentation WebSphere MQ.

Cette propriété ne doit être utilisée que pour la connexion à un gestionnaire de files d'attente multi-instance. Elle ne doit pas être employée pour la connexion à une liste de gestionnaires de files d'attente distincts car cela peut entraîner des problèmes d'intégrité des transactions.

Type de transport

Indique si la connexion client WebSphere MQ ou les liaisons JNDI sont utilisées pour établir la connexion au gestionnaire de files d'attente WebSphere MQ. Le fournisseur JMS externe contrôle les protocoles de communication entre les clients JMS et les serveurs JMS. Ajustez le type de transport lorsque vous utilisez des messageries non ASF, non persistantes, non durables et non transactionnelles ou lorsque vous souhaitez répondre aux exigences de sécurité et que le client et le gestionnaire de files d'attente sont sur le même noeud.

Type de données Enum
Unités Non applicable
Valeur par défaut BINDINGS
Options
BINDINGS
Les liaisons JNDI permettent d'établir des connexions au gestionnaire de files d'attente. BINDINGS est un protocole de mémoire partagée utilisable uniquement lorsque le gestionnaire de files d'attente est sur le même noeud que le client JMS et présente quelques risques au niveau de la sécurité, qu'il est possible de gérer par le biais des rôles d'EJB.
CLIENT
La connexion client WebSphere MQ est utilisée pour se connecter au gestionnaire de files d'attente. CLIENT est un protocole TCP standard.
DIRECT
Pour WebSphere MQ Event Broker utilisant le mode DIRECT. DIRECT est un protocole sockets simple utilisé dans les messageries de publication/d'abonnement non-transactionnelles, non-durables et non-persistantes. DIRECT fonctionne uniquement pour les clients et les beans gérés par message utilisant le protocole non ASF.
QUEUED
QUEUED est un protocole TCP standard.
Recommandations
Type de transport de fabrique de connexions de file d'attente
BINDINGS est plus rapide d'au moins 30 % mais sa sécurité est faible. Si la sécurité fait partie de vos préoccupations, BINDINGS est préférable à CLIENT.
Type de transport de fabrique de connexions de sujet
DIRECT est le protocole le plus rapide et doit être utilisé dans la mesure du possible. Utilisez BINDINGS pour effectuer des tâches de sécurité supplémentaires lorsque le gestionnaire de files d'attente et le client JMS sont sur le même noeud. Dans les autres cas, utilisez QUEUED. WebSphere MQ 5.3, avant CSD2 et avec le paramètre DIRECT, peut perdre des messages lorsqu'il est utilisé avec des beans gérés par messages (MDB) et soumis à une forte charge de travail. Cette perte se produit aussi avec les applications côté client, sauf si la propriété maxClientQueueSize du courtier a pour valeur 0. Pour associer la valeur 0 à cette propriété, vous pouvez utiliser la commande : 
#wempschangeproperties WAS_nodeName_server1 -e default -o DynamicSubscriptionEngine -n maxClientQueueSize -v 0 -x executionGroupUUID
Vous pouvez trouver la valeur de executionGroupUUID en démarrant le courtier et en recherchant l'événement 2201 dans le journal des événements. Cette valeur est généralement ffffffff-0000-0000-000000000000.
Remarque : Le JMS WebSphere MQ 5.3 JMS n'est pas utilisable dans WAS 6.1 car ce dernier dépend d'une exécution Java 5. Par conséquent, les connexions intermémoire ne peuvent pas être établies avec des gestionnaires de file d'attente WebSphere MQ 5.3. Ceci peut entraîner une dégradation des performances si vous utilisiez préalablement WebSphere MQ 5.3 et BINDINGS pour vos connexions et que vous passez à des connexions réseau CLIENT en migrant vers WebSphere Application Server Version 6.1. Si vous utilisez WebSphere MQ 5.3 pour z/OS®, vous devrez sans doute acheter également un module de fonctions supplémentaire.

Sous 64 bits z/OS, le type de transport doit être défini sur CLIENT car WebSphere MQ z/OS 64 bits n'est actuellement pas disponible et le mode BINDINGS ne peut pas servir à se connecter à WebSphere MQ z/OS 31 bits. Il peut également s'avérer nécessaire d'acheter un module de fonctions WebSphere MQ supplémentaire pour ce support.

Algorithme de chiffrement SSL

Indique l'algorithme de chiffrement à utiliser pour la connexion SSL à WebSphere MQ.

Attribuez un algorithme de chiffrement valide proposé par votre fournisseur JSSE à cette propriété. La valeur doit correspondre à l'élément CipherSpec indiqué sur le canal SVRCONN en tant que propriété Canal.

Vous devez définir cette propriété si vous définissez la propriété Nom de l'homologue SSL.

Magasin de certificats SSL

Indique une liste contenant de zéro à plusieurs serveurs CRL (Certificate Revocation List) utilisés pour vérifier le retrait de certificats SSL. Si vous indiquez une valeur pour cette propriété, vous devez utiliser la machine JVM WebSphere MQ Java 2 version 1.4.

La valeur est une liste d'entrées séparées par un espace au format suivant :
ldap://nom_hôte:[port]

Une barre oblique (/) apparaît après cette valeur. Si port n'est pas spécifié, le port LDAP par défaut (le port 389) est utilisé. Au moment de la connexion, le certificat SSL présenté par le serveur est comparé à ceux stockés sur les serveurs CRL spécifiés. Pour plus d'informations sur la sécurité CRL, voir les informations sur l'utilisation de liste de révocation de certificat dans la section Security du centre de documentation WebSphere MQ.

Nom de l'homologue SSL

Pour SSL, squelette d'un nom distinctif qui doit correspondre au nom fourni par le gestionnaire de files d'attente WebSphere MQ. Le nom distinctif permet de vérifier le certificat d'identification présenté par le serveur lors de la connexion.

Lorsque cette propriété n'est pas définie, une telle vérification de certificat est effectuée.

La propriété de nom d'homologue SSL est ignorée si la propriété Algorithme de chiffrement SSL n'est pas spécifiée.

Cette propriété est une liste de paires nom d'attribut et valeur. Chaque élément de la liste est séparée par une virgule ou un point-virgule. par exemple : 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

L'exemple indiqué vérifie le certificat d'identification présenté par le serveur lors de la connexion. Pour que la connexion aboutisse, le Nom commun du certificat doit commencer par QMGR. et être associé à au moins deux noms d'unité d'organisation (le premier étant IBM et le second WEBSPHERE). La vérification ne fait pas de distinction entre les majuscules et les minuscules.

Pour obtenir plus d'informations sur les noms distinctifs et leur utilisation avec WebSphere MQ, reportez-vous à la section "Distinguished Names" du centre de documentation WebSphere MQ.

Pool de connexions

Indique un ensemble facultatif de paramètres de pool de connexions.

Les propriétés du pool de connexions sont communes à tous les connecteurs J2C.

Le serveur d'applications regroupe les connexions et les sessions dans un pool avec le fournisseur JMS pour optimiser les performances. Cette opération ne dépend pas de la mise en pool des connexions WebSphere MQ. Vous devez configurer correctement les propriétés du pool de connexions et de sessions pour les applications. Dans le cas contraire, le comportement des connexions et des sessions obtenu risque de ne pas répondre à vos besoins.

Modifiez la taille du pool de connexions si l'accès concurrent côté serveur à la ressource JMS dépasse la valeur par défaut. La taille du pool de connexions est définie sur la base d'une file d'attente ou d'u sujet individuel.

Type de données Case à cocher
Valeur par défaut Sélectionnée
Options de reconnexion du client

Indique si une connexion en mode client se reconnecte automatiquement ou non en cas d'échec d'un gestionnaire de files d'attente ou de communications. Cette propriété est ignorée, sauf si la fabrique de connexions est utilisée dans un environnement client léger ou géré.

Type de données Liste déroulante
Valeur par défaut DISABLED
Options
DISABLED
La reconnexion du client n'a pas lieu automatiquement.
ASDEF
La valeur de l'attribut DefRecon des sections channels du fichier de configuration client est utilisée. Si aucune valeur n'est spécifiée pour DefRecon, ce paramètre a le même effet que la valeur DISABLED.
RECONNECT
La reconnexion a lieu pour tout gestionnaire de files d'attente cohérent avec la valeur de l'attribut de gestionnaire de files d'attente, qui peut être un gestionnaire de files d'attente autre que celui de la connexion d'origine.
QMGR
La reconnexion a lieu uniquement au gestionnaire de files d'attente de la la connexion d'origine.

Pour plus d'informations sur la reconnexion automatique du client, voir le centre de documentation WebSphere MQ.

Délai de reconnexion du client

Délai maximal, en secondes, pendant lequel une connexion en mode client tente de se reconnecter automatiquement à un gestionnaire de files d'attente après l'échec d'un gestionnaire de files d'attente ou de communications. Ce paramètre est ignoré, sauf si la fabrique de connexions est utilisée dans un environnement client léger ou géré. L'utilisation ou non de ce paramètre dépend de la valeur du paramètre Options de reconnexion du client.

Type de données Entier
Unités Valeur par défaut
Valeur par défaut 1800
Options Valeur supérieure à zéro égale à 2147483647 au maximum

Pour plus d'informations sur la reconnexion automatique du client, voir le centre de documentation WebSphere MQ.
