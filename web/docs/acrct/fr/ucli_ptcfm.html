Paramètre de la fabrique de connexions de sujet du fournisseur WebSphere MQ pour les clients d'application

Ce panneau permet d'afficher ou de modifier les propriétés de configuration de la fabrique de connexions de sujet sélectionnée à utiliser avec le fournisseur JMS (Java™ Messaging Service) du produit WebSphere® MQ. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et WebSphere MQ.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > Fournisseur WebSphere MQ. Cliquez avec le bouton droit de la souris sur Fabriques de connexions de sujet, puis cliquez sur Nouveau.

Remarque :
  • Les valeurs de propriété que vous indiquez doivent correspondre aux valeurs que vous avez indiquées lors de la configuration des ressources JMS pour WebSphere MQ. Pour plus d'informations sur la configuration des ressources JMS pour WebSphere MQ, reportez-vous à la section Using Java du centre de documentation WebSphere MQ.
  • Dans WebSphere MQ, les noms peuvent inclure jusqu'à 48 caractères, à l'exception des canaux, qui sont limités à 20.
Courtier MA0C : Lors de la création d'une fabrique de connexions de sujet pour WebSphere Application Server v6 relative au courtier MA0C, tenez compte des valeurs d'attribut suivantes :
File d'attente de contrôle du courtier
Cette valeur est réglée sur SYSTEM.BROKER.CONTROL.QUEUE pour le courtier MA0C et correspond à la file d'attente que consulte le courtier.
Version de courtier
Réglez cette valeur sur BASIC pour le courtier MA0C.
ClientID
Réglez cette valeur comme vous le souhaitez pour le courtier MA0C (il s'agit d'une chaîne tenant lieu d'identificateur pour l'application client).
Compatible XA
Réglez cette valeur sur TRUE ou FALSE pour le courtier MA0C (le paramètre utilisé est un indicateur d'extension des performances que vous allez certainement le plus souvent paramétrer sur 'true').
Sélection de messages de courtier
Cette valeur est réglée sur CLIENT pour le courtier MA0C dont la sélection de messages s'effectue côté client.
Type d'autorisation du courtier direct
Cette valeur n'est pas obligatoire pour le courtier MA0C.

La fabrique de connexions de sujet du fournisseur JMS du produit WebSphere MQ possède les propriétés suivantes :

Nom

Nom sous lequel cette fabrique de connexions de sujet est connue dans WebSphere Application Server, à des fins d'administration. Il doit être unique dans le fournisseur JMS.

Type de données Chaîne
Description

Description de cette fabrique de connexions de sujet dans WebSphere Application Server, à des fins d'administration.

Type de données Chaîne
Nom JNDI

Nom JNDI (Java Naming and Directory Interface) utilisé pour lier la fabrique de connexions de sujet à l'espace de nom du serveur d'applications.

Par convention, utilisez le nom JNDI complet, par exemple sous la forme jms/Nom, Nom correspondant au nom logique de la ressource.

Ce nom est utilisé pour lier les informations relatives à la liaison des plateformes. La liaison associe les ressources définies par le descripteur de déploiement du module aux ressources réelles (physiques) liées à JNDI par la plateforme.

Type de données Chaîne
Unités Caractères ASCII En_US
Options 1 à 45 caractères ASCII
ID utilisateur

ID utilisateur, utilisé avec la propriété de mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement d'ID utilisateur ou de mot de passe.

Si vous indiquez une valeur pour la propriété d'ID utilisateur vous devez également indiquer une valeur pour la propriété de mot de passe.

Les propriétés d'ID utilisateur et de mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas expressément d'ID utilisateur et de mot de passe, par exemple, si l'application appelante utilise la méthode createTopicConnection(). Le client JMS transmet l'ID utilisateur et le mot de passe au serveur JMS.

Type de données Chaîne
Mot de passe

Mot de passe utilisé avec la propriété d'ID utilisateur pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété d'ID utilisateur vous devez également indiquer une valeur pour la propriété de mot de passe.

Type de données Chaîne
Confirmation du mot de passe

Confirme le mot de passe.

Liste de noms de connexion

Liste séparée par des virgules d'informations d'hôte et de port pouvant être utilisée pour se connecter à un gestionnaire de files d'attente multi-instance.

Le format de la liste est : 

hôte[(port)],[hôte[(port)]]
où port est facultatif et prend la valeur par défaut 1414 s'il n'est pas défini. Exemple :
nomhôte1,nomhôte2(1415)

Pour plus d'informations sur les gestionnaires de files d'attente multi-instance, voir le centre de documentation WebSphere MQ.

Cette propriété ne doit être utilisée que pour la connexion à un gestionnaire de files d'attente multi-instance. Elle ne doit pas être employée pour la connexion à une liste de gestionnaires de files d'attente distincts car cela peut entraîner des problèmes d'intégrité des transactions.

Algorithme de chiffrement SSL

Algorithme de chiffrement à utiliser pour la connexion SSL à WebSphere MQ.

Attribuez un algorithme de chiffrement valide proposé par votre fournisseur JSSE à cette propriété. La valeur doit correspondre à l'élément CipherSpec indiqué sur le canal SVRCONN en tant que propriété Canal.

Vous devez définir cette propriété si vous définissez la propriété Nom de l'homologue SSL.

Magasin de certificats SSL

Indique une liste contenant de zéro à plusieurs serveurs CRL (Certificate Revocation List) utilisés pour vérifier le retrait de certificats SSL. Si vous indiquez une valeur pour cette propriété, vous devez utiliser la machine JVM WebSphere MQ Java 2 version 1.4.

La valeur est une liste d'entrées séparées par un espace au format suivant :
ldap://nom_hôte:[port]

Une barre oblique (/) apparaît après cette valeur. Si port n'est pas spécifié, le port LDAP par défaut (le port 389) est utilisé. Au moment de la connexion, le certificat SSL présenté par le serveur est comparé à ceux stockés sur les serveurs CRL spécifiés. Pour plus d'informations sur la sécurité CRL, voir les informations sur l'utilisation de liste de révocation de certificat dans la section Security du centre de documentation WebSphere MQ.

Nom de l'homologue SSL

Pour SSL, squelette d'un nom distinctif qui doit correspondre au nom fourni par le gestionnaire de files d'attente WebSphere MQ. Le nom distinctif permet de vérifier le certificat d'identification présenté par le serveur lors de la connexion.

Lorsque cette propriété n'est pas définie, une telle vérification de certificat est effectuée.

La propriété de nom d'homologue SSL est ignorée si la propriété Algorithme de chiffrement SSL n'est pas spécifiée.

Cette propriété est une liste de paires nom d'attribut et valeur. Chaque élément de la liste est séparée par une virgule ou un point-virgule. par exemple : 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

L'exemple indiqué vérifie le certificat d'identification présenté par le serveur lors de la connexion. Pour que la connexion aboutisse, le Nom commun du certificat doit commencer par QMGR. et être associé à au moins deux noms d'unité d'organisation (le premier étant IBM et le second WEBSPHERE). La vérification ne fait pas de distinction entre les majuscules et les minuscules.

Pour obtenir plus d'informations sur les noms distinctifs et leur utilisation avec WebSphere MQ, reportez-vous aux informations de "Distinguished Names" dans la section Security du centre de documentation WebSphere MQ.

Pool de connexions

Ensemble facultatif de paramètres de pool de connexions.

Les propriétés du pool de connexions sont communes à tous les connecteurs J2C.

Le serveur d'applications regroupe les connexions et les sessions dans un pool avec le fournisseur JMS pour optimiser les performances. Cette opération ne dépend pas de la mise en pool des connexions WebSphere MQ. Vous devez configurer correctement les propriétés du pool de connexions et de sessions pour les applications. Dans le cas contraire, le comportement des connexions et des sessions obtenu risque de ne pas répondre à vos besoins.

Modifiez la taille du pool de connexions si l'accès concurrent côté serveur à la ressource JMS dépasse la valeur par défaut. La taille du pool de connexions est définie sur la base d'une file d'attente ou d'u sujet individuel.

Type de données Case à cocher
Valeur par défaut Sélectionnée
Options de reconnexion du client

Indique si une connexion en mode client se reconnecte automatiquement ou non en cas d'échec d'un gestionnaire de files d'attente ou de communications. Cette propriété est ignorée, sauf si la fabrique de connexions est utilisée dans un environnement client léger ou géré.

Type de données Liste déroulante
Valeur par défaut DESACTIVE
Options
DISABLED
La reconnexion du client n'a pas lieu automatiquement.
ASDEF
La valeur de l'attribut DefRecon des sections channels du fichier de configuration client est utilisée. Si aucune valeur n'est spécifiée pour DefRecon, ce paramètre a le même effet que la valeur DISABLED.
RECONNECT
La reconnexion a lieu pour tout gestionnaire de files d'attente cohérent avec la valeur de l'attribut de gestionnaire de files d'attente, qui peut être un gestionnaire de files d'attente autre que celui de la connexion d'origine.
QMGR
La reconnexion a lieu uniquement au gestionnaire de files d'attente de la la connexion d'origine.

Pour plus d'informations sur la reconnexion automatique du client, voir le centre de documentation WebSphere MQ.

Délai de reconnexion du client

Ce paramètre est ignoré, sauf si la fabrique de connexions est utilisée dans un environnement client léger ou géré. L'utilisation ou non de ce paramètre dépend de la valeur du paramètre Options de reconnexion du client.

Type de données Entier
Unités Valeur par défaut
Valeur par défaut 1800
