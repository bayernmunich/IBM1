Outil de configuration des ressources du client d'application

Naviguez jusqu'aux rubriques d'aide de cette section.

  • Adaptateur de ressources pour le client
    Un adaptateur de ressources est un pilote de niveau système qu'une application Java utilise pour se connecter à un système d'information d'entreprise (EIS). Il se "branche" dans un client d'application et assure la connectivité entre le système EIS et l'application d'entreprise.
  • Fournisseurs d'environnement de ressources et entrées d'environnement de ressources
    Une référence d'environnement de ressource mappe un nom logique utilisé par l'application client vers le nom physique d'un objet.
  • Fournisseurs JMS (Java Message Service) pour les clients
    Les applications client peuvent utiliser les ressources de messagerie de trois principaux types de fournisseurs JMS (Java Message Service) dans WebSphere Application Server : le fournisseur de messagerie par défaut WebSphere Application Server (utilise l'intégration de service comme fournisseur), le fournisseur de messagerie WebSphere MQ (utilise votre système WebSphere MQ comme fournisseur) et les fournisseurs de messagerie tiers (utilisent le produit d'une autre entreprise comme fournisseur).
  • Messagerie asynchrone dans WebSphere Application Server avec JMS
    WebSphere Application Server prend en charge la messagerie asynchrone en tant que méthode de communication basée sur l'interface de programmation JMS (Java Message Service). L'interface JMS offre une méthode commune aux programmes Java (clients et Java Platform Enterprise Edition (applications J2EE)) pour créer, envoyer, recevoir et lire les demandes asynchrones comme des messages JMS.
  • Configuration de nouvelles fabriques de connexions pour les adaptateurs de ressource pour le client
    Utilisez l'outil de configuration des ressources du client d'application (ACRCT) afin de configurer les nouvelles fabriques de connexions pour les adaptateurs de ressources pour le client.
  • Fournisseurs d'URL pour l'outil de configuration des ressources du client d'application
    Un fournisseur d'URL implémente la fonction d'un protocole d'URL particulier, tel que HTTP (HyperText Transfer Protocol). Ce fournisseur, composé d'une paire de classes, étend les classes java.net.URLStreamHandler et java.net.URLConnection.
  • Paramètre de la fabrique de connexions de file d'attente du fournisseur WebSphere MQ pour tous les clients d'application
    Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions de file d'attente sélectionnée à utiliser avec le fournisseur JMS (Java Messaging Service) du produit MQSeries. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et WebSphere MQ.
  • Paramètre de la fabrique de connexions par rubrique du fournisseur WebSphere MQ pour les clients d'application
    Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions par rubrique sélectionnée à utiliser avec le fournisseur JMS (Java Message Service) du produit WebSphere MQ. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et WebSphere MQ.
  • Paramètres d'adaptateur de ressources
    Utilisez ce panneau pour visualiser ou modifier les propriétés de configuration d'un adaptateur de ressources. Ces propriétés de configuration déterminent comment les adaptateurs de ressources sont créés.
  • Paramètres d'entrée d'environnement de ressources des clients d'application

Un fournisseur JMS offre deux types de fabrique J2EE : une fabrique de connexions JMS et une fabrique de destinations JMS.
