Paramètres de fabrique de connexions de sujet du fournisseur de messagerie par défaut V5 pour les clients d'application

Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions de sujet sélectionnée à utiliser avec le fournisseur JMS (Java Message Service) du produit interne. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et le système de messagerie qu'il utilise.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > Fournisseur par défaut version 5. Cliquez avec le bouton droit de la souris sur Fabriques de connexions de sujet puis cliquez sur Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Une fabrique de connexions de sujet du fournisseur de messagerie par défaut de la version 5 possède les propriétés suivantes :

Nom

Nom sous lequel la fabrique de connexions de file d'attente est connue dans WebSphere Application Server en vue de son administration. Il doit être unique parmi les fabriques de connexions JMS du domaine administratif WebSphere Application Server.

Type de données String (chaîne)
Description

Description de cette fabrique de connexions de rubriques dans WebSphere Application Server, à des fins d'administration.

Type de données String (chaîne)
Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

ID utilisateur

ID utilisateur utilisé avec la propriété Mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Les propriétés ID utilisateur et Mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas explicitement d'ID utilisateur et de mot de passe ; par exemple, si l'application appelante utilise la méthode createTopicConnection(). Le client JMS transmet l'ID utilisateur et le mot de passe au serveur JMS.

Type de données String (chaîne)
Password

Mot de passe utilisé avec la propriété ID utilisateur pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Type de données String (chaîne)
Confirmation du mot de passe

Confirme le mot de passe.

Noeud

Nom WebSphere Application Server du noeud administratif sur lequel s'exécute le serveur JMS pour cette fabrique de connexions. Les connexions créées par cette fabrique se connectent à ce serveur JMS.

Type de données Enum
Plage de valeurs Liste déroulante des noeuds du domaine administratif WebSphere Application Server.
Application Server

Entrez le nom du serveur d'applications. Ce nom n'est pas le nom d'hôte de la machine, mais le nom du serveur d'applications configuré.

Port

Port utilisé pour la connexion au serveur JMS. Le port QUEUED est destiné au support complet des publications/abonnements JMS et le port DIRECT à celui des abonnements non persistants, non transactionnels et non durables uniquement.

Remarque : Les beans gérés par message ne peuvent pas utiliser le port d'écoute direct pour la prise en charge des publications ou abonnements. Par conséquent, toutes les fabriques de connexions de sujet dont le port a pour valeur Direct ne peuvent pas être utilisées avec des beans gérés par message (MDB).
Type de données Enum
Portée par défaut QUEUED
Plage de valeurs
QUEUED
Port d'écoute utilisé pour la prise en charge complète des publications ou des abonnements compatibles avec JMS.
DIRECT
Port d'écoute utilisé pour les connexions TCP/IP directes (abonnements non transactionnels, non persistants et non durables uniquement) pour la prise en charge des publications ou des abonnements.

Ces numéros de port TCP/IP sont définis dans le serveur JMS interne du produit.
ID client

Identificateur de client JMS utilisé pour établir les connexions avec le gestionnaire de file d'attente MQSeries.

Type de données String (chaîne)
Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.
