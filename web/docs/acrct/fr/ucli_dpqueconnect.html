Paramètres de fabrique de connexions de file d'attente du fournisseur par défaut

Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions de file d'attente JMS sélectionnée à utiliser avec le fournisseur JMS (Java Message Service) du produit interne installé avec WebSphere Application Server. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et le bus d'intégration de services qu'il utilise.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > Fournisseur par défaut. Cliquez avec le bouton droit de la souris sur fabriques de connexions de file d'attente, puis sélectionnez Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Pour les paramètres disposant d'une valeur par défaut, cette dernière est affichée. Pour les paramètres dont les valeurs ne peuvent être choisies que parmi un jeu de valeurs fixes, une liste déroulante est proposée.

Nom

Nom de la fabrique de connexions de file d'attente.

Type de données String (chaîne)
Description

Description de cette fabrique de connexions de file d'attente dans IBM WebSphere Application Server en vue de son administration.

Type de données String (chaîne)
Nom JNDI

Nom JNDI utilisé pour associer cette fabrique de connexions de file d'attente au descripteur de déploiement. Cette entrée est un nom resource-ref.

Type de données String (chaîne)
Nom de l'utilisateur

Nom d'utilisateur utilisé avec la propriété Mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement d'ID utilisateur et de mot de passe. Si cette zone est utilisée, la zone de propriété Nom d'utilisateur est ignorée.

Si vous indiquez une valeur pour la propriété Nom d'utilisateur, vous devez également indiquer une valeur pour la propriété Mot de passe.

Les propriétés Nom d'utilisateur et Mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas explicitement d'ID utilisateur et de mot de passe. Si un nom d'utilisateur et un mot de passe sont spécifiés, un alias d'authentification est créé pour la fabrique et le mot de passe est chiffré.

Type de données String (chaîne)
Password

Mot de passe utilisé pour le chiffrement. Si vous complétez cette zone, la zone Mot de passe de la boîte Propriétés est ignorée.

Si vous indiquez une valeur pour la propriété Nom d'utilisateur, vous devez également indiquer une valeur pour la propriété Mot de passe.

Type de données String (chaîne)
Confirmation du mot de passe

Confirme le mot de passe.

Nom du bus

Nom du bus auquel la fabrique de connexions de file d'attente se connecte.

Type de données String (chaîne)
Identificateur du client

Identificateur du client. Requis pour les abonnements aux sujets durables.

Type de données String (chaîne)
Fiabilité des messages non persistants

Fiabilité appliquée aux messages JMS non persistants envoyés à l'aide de cette fabrique de connexions.

Si vous souhaitez des options de distribution à fiabilité différente pour chaque destination JMS, vous pouvez associer cette propriété à Comme destination du bus. La fiabilité est alors définie par la propriété Fiabilité de la destination de bus à laquelle la destination JMS est affectée.

Portée par défaut ReliablePersistent
Plage de valeurs
Aucun
Aucune fiabilité des messages pour les messages non persistants. Si un message non persistant ne peut pas être distribué, il est éliminé.
Meilleur effort non persistant
Les messages ne sont jamais enregistrés sur disque et sont supprimés en cas de saturation de la mémoire cache.
Rapide non persistant
Les messages sont placés de manière asynchrone dans la mémoire persistante en cas de saturation de la mémoire cache mais ne sont pas conservés lors du redémarrage du serveur.
Fiable non persistant
Les messages peuvent être perdus en cas de défaillance d'un moteur de messagerie et dans des conditions normales de fonctionnement.
Persistant - Fiable
Les messages peuvent être perdus en cas de défaillance d'un moteur de messagerie, mais pas dans des conditions normales de fonctionnement.
Persistant - Assuré
Degré le plus élevé de fiabilité lorsque le système prend en charge la distribution assurée des messages.
Comme destination du bus
Utilisez l'option de distribution configurée pour la destination du bus.

Chaîne de transport cible des communications entrantes

Nom du protocole qui se convertit en un groupe de moteurs de messagerie.

Type de données String (chaîne)
Noeuds finals du fournisseur

Liste des noeuds finals séparés par des virgules, utilisés pour établir une connexion avec un serveur d'amorçage.

Entrez une liste de triplets de noeuds finals séparés par des virgules au format suivant : hôte:port:protocole.

Exemple 
localhost:7777:BootstrapBasicMessaging
où

BootstrapBasicMessaging correspond au protocole éloigné InboundBasicMessaging (JFAP-TCP/IP).
Portée par défaut
  • Si le nom d'hôte n'est pas spécifié, localhost est utilisé comme valeur par défaut.
  • Si le numéro de port n'est pas spécifié, 7276 est utilisé comme valeur par défaut.
  • Si le nom de chaîne n'est pas spécifié, une chaîne prédéfinie, telle que BootstrapBasicMessaging, est utilisée comme valeur par défaut.
Proximité des connexions

Proximité du moteur de messagerie par rapport au demandeur.

Portée par défaut Bus, Cluster, Serveur
Plage de valeurs Bus, Hôte
Préfixe du nom de file d'attente temporaire

Préfixe à ajouter aux noms des files d'attente temporaires. Ce nom comporte au maximum douze caractères.
