Paramètres d'une destination de sujet du fournisseur par défaut

Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la destination par rubrique JMS sélectionnée à utiliser avec le fournisseur JMS (Java Message Service) du produit interne installé avec WebSphere Application Server.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > Fournisseur par défaut. Cliquez avec le bouton droit de la souris sur Destinations de sujet, puis cliquez sur Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Nom

Nom de l'entrée de destination de sujet.

Type de données String (chaîne)
Description

Description de l'entrée.

Type de données String (chaîne)
Nom JNDI

Nom JNDI permettant d'associer cette définition à un nom resource-env-ref de descripteur de déploiement.

Type de données String (chaîne)
Espace de sujet

Nom de l'espace de sujet. Cette zone est obligatoire.

Type de données String (chaîne)
Portée par défaut DEFAULT_TOPIC_SPACE
Nom du sujet

Nom du sujet. Cette zone est obligatoire.

Type de données String (chaîne)
Mode de livraison

Mode par défaut pour les messages envoyés à cette destination.

Type de données String (chaîne)
Plage de valeurs Application, Persistant ou Non persistant
Portée par défaut Application
Durée de vie

Durée par défaut pendant laquelle un message doit être conservé par le système à compter du moment où il est envoyé à cette destination. La valeur 0 indique une durée illimitée (absence d'expiration). Si cette zone n'est pas renseignée, la valeur de l'expéditeur du message est utilisée.

Type de données Long
Unités Millisecondes
Priority

Priorité des messages envoyés à cette destination. Si cette zone n'est pas renseignée, la valeur de l'expéditeur du message est utilisée.

Type de données Entier
Plage de valeurs 0 à 9, 0 étant la priorité la plus basse et 9 la priorité la plus élevée
Lecture anticipée

Contrôle l'optimisation par lecture anticipée lors de la livraison des messages.

Type de données String (chaîne)
Plage de valeurs AsConnection, AlwaysOn et AlwaysOff
Portée par défaut AsConnection
