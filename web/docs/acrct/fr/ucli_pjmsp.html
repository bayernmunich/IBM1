Paramètres du fournisseur JMS des clients d'application

Cette page permet de configurer les propriétés d'un fournisseur JMS (Java Message Service) s'il s'agit d'un fournisseur autre que le fournisseur de messagerie par défaut ou WebSphere MQ comme fournisseur JMS.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Agrandissez la fenêtre du fichier JAR. Cliquez avec le bouton droit de la souris sur Fournisseurs de messagerie et sélectionnez Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Nom

Nom sous lequel le fournisseur JMS est connu à des fins d'administration.

Type de données String (chaîne)
Description

Description du fournisseur JMS, indiquée en vue de son administration.

Type de données String (chaîne)
Chemin d'accès aux classes

Liste des chemins d'accès ou des noms de fichier .jar correspondant à l'emplacement des classes du fournisseur de ressources.

Classe de fabrique de contexte

Nom de classe Java de la fabrique de contexte initial du fournisseur JMS.

Par exemple, pour un fournisseur de services LDAP, cette valeur est au format : com.sun.jndi.ldap.LdapCtxFactory.

Type de données String (chaîne)
URL du fournisseur

URL du fournisseur JMS pour les recherches JNDI externes.

Par exemple, le format d'une URL LDAP de fournisseur JMS est ldap://hostname.company.com/contextName.

Type de données String (chaîne)
Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.
