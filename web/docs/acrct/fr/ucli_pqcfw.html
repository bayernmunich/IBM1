Paramètres de la fabrique de connexions de file d'attente du fournisseur de messagerie par défaut V5 pour les clients d'application

Ce panneau permet de parcourir ou modifier les propriétés de configuration de la fabrique de connexions de file d'attente JMS sélectionnée pour la messagerie point-à-point à utiliser par les applications WebSphere Application Server, version 5. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et le système de messagerie par défaut qu'il utilise.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseur de messagerie > Fournisseur par défaut version 5. Cliquez avec le bouton droit de la souris sur Fabriques de connexions de file d'attente, puis sélectionnez Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Une fabrique de connexions de file d'attente permet de créer des connexions JMS vers des destinations de file d'attente. La fabrique de connexions de file d'attente est créée par le fournisseur JMS interne de WebSphere Application Server. Une fabrique de connexions de file d'attente du fournisseur de messagerie par défaut de la version 5 possède les propriétés suivantes :

Nom

Nom sous lequel la fabrique de connexions de file d'attente est connue dans IBM WebSphere Application Server en vue de son administration. Ce nom doit être unique au sein des fabriques de connexions JMS dans le domaine d'administration WebSphere.

Type de données String (chaîne)
Description

Description de cette fabrique de connexions dans IBM WebSphere Application Server à des fins d'administration.

Type de données String (chaîne)
Portée par défaut Null
Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

ID utilisateur

ID utilisateur, utilisé avec la propriété Mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Les propriétés ID utilisateur et Mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas expressément d'ID utilisateur et de mot de passe ; par exemple, si l'application appelante utilise la méthode createQueueConnection(). Le client JMS transmet l'ID utilisateur et le mot de passe au serveur JMS.

Type de données String (chaîne)
Password

Mot de passe utilisé avec la propriété ID utilisateur pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Confirmation du mot de passe

Confirme le mot de passe.

Noeud

Nom du noeud d'administration WebSphere où le serveur JMS s'exécute pour cette fabrique de connexions. Les connexions créées par cette fabrique se connectent à ce serveur JMS.

Type de données String (chaîne)
Application Server

Entrez le nom du serveur d'applications. Ce nom n'est pas le nom d'hôte de la machine, mais le nom du serveur d'applications configuré.

Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.
