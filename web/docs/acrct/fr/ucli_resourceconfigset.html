Paramètres d'adaptateur de ressources

Utilisez ce panneau pour visualiser ou modifier les propriétés de configuration d'un adaptateur de ressources. Ces propriétés de configuration déterminent comment les adaptateurs de ressources sont créés.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Adaptateur de ressources. Cliquez avec le bouton droit de la souris sur Adaptateur de ressources, puis sélectionnez Nouveau. Les zones ci-après apparaissent dans l'onglet Général.

Nom

Nom sous lequel cet adaptateur de ressources est connu dans IBM WebSphere Application Server en vue de son administration. Il doit être unique parmi les adaptateurs de ressources du domaine d'administration du produit.

Type de données String (chaîne)
Description

Description de cet adaptateur de ressources dans IBM WebSphere Application Server en vue de son administration.

Type de données String (chaîne)
Chemin d'accès aux classes

Eventuel chemin de classes supplémentaire. Le chemin du répertoire des adaptateurs de ressources est automatiquement ajouté.

Type de données String (chaîne)
Portée par défaut Chemin de votre répertoire d'adaptateurs de ressources.
Chemin d'accès natif

Chemin natif où est situé l'adaptateur de ressources. Si des chemins d'accès aux classes supplémentaires doivent être spécifiés, entrez-les ici.

Type de données String (chaîne)
Nom de l'adaptateur de ressources

Zone obligatoire pointant sur le sous-répertoire d'un adaptateur de ressources installé. L'entrée ne représente pas le nom de répertoire complet de l'adaptateur de ressources. Le nom de répertoire complet est le chemin des adaptateurs de ressources installés plus le nom de l'adaptateur de ressources.

Type de données String (chaîne)
Chemin des adaptateurs de ressources installés

Répertoire où les adaptateurs de ressources sont installés. Si vous ne renseignez pas cette zone, la valeur par défaut est utilisée.

Si vous indiquez la valeur, ${RACINE_INSTALL_CONNECTEUR}, alors cette valeur remplace la valeur de la variable RACINE_INSTALL_CONNECTEUR_CLIENT sur la machine sur laquelle l'application client est en cours d'exécution. Cette action permet à l'application de s'exécuter facilement sur différentes machines alors que l'installation client peut se trouver à divers emplacements.

Type de données String (chaîne)
Portée par défaut ${RACINE_INSTALL_CONNECTEUR}

Nom du fichier : ucli_resourceconfigset.html
