Mise à jour de WebSphere MQ comme fournisseur JMS (Java Message Service) et de ses configurations de ressource JMS pour les clients d'application

Vous pouvez mettre à jour une configuration de WebSphere MQ comme fournisseur JMS (Java Message Service) ainsi que la configuration des fabriques de connexions et des destinations WebSphere MQ.

A propos de cette tâche

Cette tâche vous permet de mettre à jour une configuration existante de WebSphere MQ comme fournisseur JMS (Java Message Service) ainsi que la configuration des fabriques de connexions et des destinations WebSphere MQ.

Procédure

  1. Démarrez l'outil de configuration des ressources du client d'application (ACRCT).
  2. Ouvrez le fichier EAR contenant le fournisseur JMS WebSphere MQ, la fabrique de connexions WebSphere MQ ou la destination WebSphere MQ. Le contenu du fichier EAR apparaît dans une vue arborescente.
  3. Sélectionnez le fichier JAR (fichier d'archive Java) contenant le fournisseur JMS, la fabrique de connexions ou la destination à mettre à jour.
  4. Développez le contenu du fichier JAR pour trouver le fournisseur JMS, la fabrique de connexions ou la destination à mettre à jour. Effectuez l'une des actions suivantes :
    • Cliquez avec le bouton droit de la souris sur l'objet approprié et sélectionnez Propriétés.
    • Cliquez sur Edition > Propriétés dans la barre de menus.
  5. Mettez à jour les propriétés dans les zones de la boîte de dialogue affichée. Pour obtenir une aide détaillée sur les zones, reportez-vous aux rubriques suivantes :
    • Propriétés du fournisseur JMS
    • Propriétés de la fabrique de connexions de file d'attente MQ
    • Propriétés de la fabrique de connexions de sujet MQ
    • Propriétés de la destination de file d'attente MQ
    • Propriétés de la destination de sujet MQ
  6. Cliquez sur OK.
  7. Cliquez sur Fichier > Sauvegarder pour sauvegarder vos modifications.


Les liens marqués (en ligne) requièrent un accès à Internet.
