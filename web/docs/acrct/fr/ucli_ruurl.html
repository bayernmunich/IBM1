Paramètres d'URL des clients d'application

Cette page permet d'implémenter la fonction d'un protocole d'URL particulier tel que HTTP (Hyper Text Transfer Protocol).

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs URL > instance de fournisseur URL. Cliquez avec le bouton droit de la souris sur URL, puis cliquez sur Nouveau. Les zones suivantes apparaissent dans l'onglet Général :

Ce fournisseur, composé de classes étend les classes java.net.URLStreamHandler et java.net.URLConnection.

Nom

Nom administratif de l'URL.

Description

Description facultative de l'URL pour vos enregistrements administratifs.

Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

URL

Nom désignant une ressource Internet ou intranet. Par exemple http://www.ibm.com.

Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.



Les liens marqués (en ligne) requièrent un accès à Internet.
