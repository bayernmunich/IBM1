Paramètres des fabriques de connexions JMS génériques des clients d'application

Ce panneau permet de visualiser ou de modifier les propriétés de configuration de la fabrique de connexions JMS (Java Message Service) sélectionnée à utiliser avec le fournisseur JMS associé. Ces propriétés de configuration contrôlent le mode de création des connexions entre le fournisseur JMS et le système de messagerie qu'il utilise.

Pour afficher la page de l'outil de configuration des ressources du client d'application (ACRCT), cliquez sur Fichier > Ouvrir. Recherchez un fichier EAR et cliquez sur Ouvrir. Développez le fichier JAR sélectionné > Fournisseurs de messagerie > nouvelle_instance_fournisseur_JMS. Cliquez avec le bouton droit de la souris sur Fabriques de connexions, puis sélectionnez Nouveau. Les zones ci-dessous sont affichées dans la page Général.

Une fabrique de connexions JMS (Java Message Service) est utilisée pour créer des connexions vers des destinations JMS. Elle est créée par le fournisseur JMS associé. La fabrique de connexions JMS d'un fournisseur JMS générique (autre que le fournisseur de messagerie par défaut interne ou le fournisseur JMS WebSphere MQ) possède les propriétés suivantes :

Nom

Nom sous lequel la fabrique de connexions JMS est connue dans IBM WebSphere Application Server à des fins d'administration. Il doit être unique dans le fournisseur JMS associé.

Type de données String (chaîne)
Description

Description de cette fabrique de connexions dans IBM WebSphere Application Server à des fins d'administration.

Type de données String (chaîne)
Portée par défaut Null
Nom JNDI

L'environnement d'exécution du client d'application utilise cette zone pour extraire les informations de configuration.

ID utilisateur

ID utilisateur utilisé avec la propriété Mot de passe pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Les propriétés ID utilisateur et Mot de passe de la fabrique de connexions sont utilisées si l'application appelante ne fournit pas expressément d'ID utilisateur et de mot de passe ; par exemple, si l'application appelante utilise la méthode createQueueConnection(). Le client JMS transmet l'ID utilisateur et le mot de passe au serveur JMS.

Type de données String (chaîne)
Password

Mot de passe utilisé avec la propriété ID utilisateur pour l'authentification si l'application appelante ne fournit pas explicitement un ID utilisateur et un mot de passe.

Si vous indiquez une valeur pour la propriété ID utilisateur vous devez également indiquer une valeur pour la propriété Mot de passe.

Type de données String (chaîne)
Portée par défaut Null
Confirmation du mot de passe

Confirme le mot de passe indiqué dans la zone Mot de passe.

Nom JNDI externe

Nom JNDI utilisé pour lier la file d'attente dans l'espace de nom du serveur d'applications.

Par convention, utilisez le nom JNDI complet (par exemple, jms/Nom, Nom correspondant au nom logique de la ressource).

Ce nom est utilisé pour lier les informations relatives à la liaison des plateformes. La liaison associe les ressources définies par le descripteur de déploiement du module aux ressources réelles (physiques) liées dans l'API JNDI par la plateforme.

Type de données String (chaîne)
Type de connexion

Indique si cette destination JMS est une file d'attente (pour système point à point) ou un sujet (pour la publication ou l'abonnement).

Sélectionnez l'une des options suivantes :
File d'attente
Destination JMS de file d'attente pour système de messagerie point-à-point
Sujet
Destination JMS de type sujet pour la messagerie publication/abonnement.
Propriétés personnalisées

Spécifie les paires nom-valeur permettant de définir des propriétés supplémentaires pour l'objet créé lors de l'exécution pour cette ressource.

Vous devez entrer un nom correspondant à une propriété publique dans l'objet et une valeur pouvant être convertie d'une chaîne vers le type requis par la méthode set de la propriété. Les valeurs et propriétés admises dépendent de l'objet créé. Pour obtenir la liste des valeurs et des propriétés valides, reportez-vous à la documentation de l'objet.
