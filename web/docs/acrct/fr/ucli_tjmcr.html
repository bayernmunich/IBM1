Configuration des ressources du client de messagerie Java

Pour configurer les ressources du client de messagerie Java, vous créez de nouvelles configurations de fournisseur JMS pour le client d'application. Ce dernier peut accéder à un service de messagerie par l'intermédiaire des interfaces API du service JMS (Java Message Service). Un fournisseur JMS offre deux types de fabrique J2EE : une fabrique de connexions JMS et une fabrique de destinations JMS.

Avant de commencer

Il est nécessaire, au cours d'une tâche d'administration distincte, d'installer le client JMS (Java Messaging Service) sur la machine sur laquelle réside le client d'application. Le fournisseur du produit de messagerie doit assurer une implémentation du client JMS. Pour plus d'informations, consultez la documentation du produit de messagerie.

Remarque : Pour effectuer cette tâche, vous pouvez créer un nouveau fournisseur de messagerie ou en utiliser un existant.

Procédure

  1. Démarrez l'outil de configuration des ressources du client d'application (ACRCT).
  2. Ouvrez le fichier EAR pour lequel le nouveau fournisseur JMS doit être configuré. Le contenu du fichier EAR se trouve dans la vue d'arborescence affichée.
  3. Dans l'arborescence, sélectionnez le fichier JAR dans lequel configurer le nouveau fournisseur JMS.
  4. Développez le fichier JAR pour en afficher le contenu.
  5. (Facultatif) Cliquez avec le bouton droit de la souris sur Fournisseurs de messagerie, puis sélectionnez Nouveau si vous voulez créer et utiliser un nouveau fournisseur de messagerie.
  6. Configurez les propriétés du fournisseur JMS dans la boîte de dialogue des propriétés qui s'affiche.
  7. Cliquez sur OK.
  8. Cliquez sur Fichier > Sauvegarder.
