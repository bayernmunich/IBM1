Fournisseurs d'environnement de ressources et entrées d'environnement de ressources

Une référence d'environnement de ressource mappe un nom logique utilisé par l'application client vers le nom physique d'un objet.

Les objets liés dans l'espace de noms JNDI du serveur ne sont pas tous utilisés par un client d'application. Par exemple, l'environnement d'exécution du client WebSphere Application Server ne prend pas en charge l'utilisation des objets Java 2 Connector (J2C) sur le client. L'objet doit pouvoir être supprimé et les implémentations côté client doivent être rendues disponibles dans le chemin d'accès aux classes de l'environnement d'exécution du client d'application.

Les références d'environnement de ressource diffèrent des références de ressource. Les premières permettent au client d'application d'utiliser un nom logique pour rechercher une liaison de ressource dans l'espace de noms JNDI du serveur. Les secondes permettent à votre application d'utiliser un nom logique pour rechercher une ressource J2EE locale. La spécification J2EE n'indique pas d'implémentation de ressource particulière.



Les liens marqués (en ligne) requièrent un accès à Internet.
