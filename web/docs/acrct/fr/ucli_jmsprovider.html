Fournisseurs JMS (Java Message Service) pour les clients

Les applications client peuvent utiliser les ressources de messagerie de trois principaux types de fournisseurs JMS (Java Message Service) dans WebSphere Application Server : le fournisseur de messagerie par défaut WebSphere Application Server (utilise l'intégration de service comme fournisseur), le fournisseur de messagerie WebSphere MQ (utilise votre système WebSphere MQ comme fournisseur) et les fournisseurs de messagerie tiers (utilisent le produit d'une autre entreprise comme fournisseur).

IBM WebSphere Application Server prend en charge la messagerie asynchrone par le biais d'un fournisseur JMS et son système de messagerie associé. Les fournisseurs JMS doivent être conformes à la spécification JMS version 1.1. Pour pouvoir utiliser des beans gérés par message, le fournisseur JMS doit prendre en charge la fonction ASF (Application Server Facility) facultative définie dans cette spécification ou un adaptateur de ressources de communications entrantes, comme défini dans la version 1.5 de la spécification JCA.
Fournisseur de messagerie par défaut
Si vous souhaitez utiliser essentiellement la messagerie entre des applications dans WebSphere Application Server, avec éventuellement des interactions avec un système WebSphere MQ, le choix du fournisseur de messagerie par défaut s'impose. Ce fournisseur de messagerie exploite les technologies d'intégration de services et est entièrement intégré à l'environnement d'exécution de WebSphere Application Server.
Fournisseur de messagerie WebSphere MQ
Si votre entreprise utilise également WebSphere MQ et que vous souhaitez intégrer des applications de messagerie WebSphere Application Server à un réseau essentiellement WebSphere MQ, choisissez le fournisseur de messagerie WebSphere MQ, qui vous permet de définir des ressources à connecter à tout gestionnaire de files d'attente sur le réseau WebSphere MQ.
Fournisseur de messagerie tiers
Vous pouvez configurer n'importe quel fournisseur de messagerie tiers prenant en charge les fabriques de connexion unifiées JMS version 1.1. Cela peut être utile, par exemple, en raison des investissements existants.
Fonction obsolète : Dans WebSphere Application Server 7.0, le fournisseur de messagerie de la version 5 est obsolète.depfeat
Remarque : Pour une compatibilité amont avec les versions antérieures, WebSphere Application Server version 7 prend en charge le fournisseur de messagerie par défaut (obsolète) version 5 et le fournisseur de messagerie WebSphere MQ version 6. Cette prise en charge permet à vos applications utilisant toujours ces ressources de communiquer avec les noeuds de version 5 et 6 dans les cellules mixtes de la version 7.

Les applications WebSphere peuvent utiliser les ressources de messagerie mises à disposition par l'un de ces fournisseurs JMS. Toutefois, le choix du fournisseur est le plus souvent dicté par la nécessité d'utiliser un système de messagerie existant ou de s'y intégrer. Par exemple, il est possible qu'une infrastructure de messagerie fondée sur WebSphere MQ existe déjà. Dans ce cas, vous pouvez soit vous connecter directement à l'aide de la prise en charge intégrée à WebSphere MQ, tel un fournisseur JMS, soit configurer un bus d'intégration des services avec des liaisons à un réseau WebSphere MQ, puis accéder au bus par l'intermédiaire du fournisseur de messagerie par défaut.
