URL szolgáltató beállítások alkalmazásügyfelekhez

Ezen oldal használatával új URL szolgáltatókat hozhat létre.

Az alábbi Alkalmazásügyfél erőforrás beállítási eszköz (ACRCT) oldal megtekintéséhez kattintson a Fájl > Megnyitás elemre. Miután keresett egy EAR fájlt, kattintson a Megnyitás gombra. Bontsa ki a kijelölt JAR fájlt. Kattintson a jobb egérgombbal a URL szolgáltatók lehetőségre, majd válassza az előugró menü Új menüpontját. A következő mezők jelennek meg az Általános lapon.

Egy URL szolgáltató egy bizonyos URL protokoll funkcióját valósítja meg, mint a Hiperszöveg átviteli protokoll (HTTP). A szolgáltató osztályokból áll és a java.net.URLStreamHandler és java.net.URLConnection osztályokat terjeszti ki.

Név

Adminisztrációs név az URL címhez.

Leírás

Az URL elhagyható leírása az adminisztrációs rekordok számára.

Osztályútvonal

Útvonalak listája vagy JAR fájlnevek, amelyek együtt formálják az erőforrás szolgáltató osztályok helyét.

Protokoll

A folyamkezelő által támogatott protokoll. Például, nntp, smtp, ftp és így tovább.

Az alapértelmezett protokoll használatához hagyja ezt a mezőt üresen.

Folyamkezelő osztály

A felhasználó által megadott, a java.net.URLStreamHandler osztályt kiterjesztő Java osztály teljes képzésű neve egy teljes képzésű URL protokollhoz, mint az FTP.

Az alapértelmezett folyamkezelő használatához hagyja ezt a mezőt üresen.

Egyéni tulajdonságok

Név-érték párokat határoz meg további tulajdonságok beállításához azon az objektumon, ami futás közben jön létre ehhez az erőforráshoz.

Meg kell adnia egy nevet, ami nyilvános tulajdonsága az objektumnak és egy értéket, ami átváltható karaktersorozatból a tulajdonság beállító metódusához szükséges típusba. Az elfogadható tulajdonságok és értékek a létrehozott objektumtól függnek. Forduljon az objektum dokumentációjához az érvényes tulajdonságok és értékek listájához.
