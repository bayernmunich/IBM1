Aszinkron üzenetkezelés a WebSphere Application Server termékben JMS használatával

A WebSphere Application Server támogatja az aszinkron üzenetkezelést, mint a Java Üzenetszolgáltatás (JMS) programozási felületen alapuló kommunikációs módszert. A JMS felület egy közös módszert biztosít Java programoknak (ügyfeleknek és Java Enterprise Edition (Java EE) alkalmazásoknak) aszinkron kérések JMS üzenetként létrehozásához, küldéséhez, fogadásához és olvasásához.

Ez a témakör egy általános áttekintést biztosít az aszinkron üzenetkezelésről a WebSphere Application Server által biztosított JMS támogatás használatával.

A JMS alkalmazás programozási felületet használó aszinkron üzenetkezelés alap támogatása biztosítja a JMS felületek és kapcsolódó szemantikák (amelyek meghatározzák, hogy egy JMS ügyfél hogyan férhet hozzá egy JMS szolgáltató lehetőségeihez). Ez a támogatás lehetővé teszi, hogy a WebSphere termék Java EE alkalmazások, JMS ügyfelekként, aszinkron módon cseréljenek üzeneteket más JMS ügyfelekkel JMS címzettek használatával (sorok és témakörök). Egy Java EE alkalmazás használhat JMS sor címzetteket pont-pont üzenetkezeléshez és JMS témakör címzetteket közzététel/előfizetés üzenetkezeléshez. Egy Java EE alkalmazás kifejezetten lekérdezheti a címzetteket üzenetekért, majd lekérheti az üzeneteket feldolgozáshoz üzleti komponensekkel (EJB-komponens).

Az alap JMS és XA támogatással a Java EE alkalmazás szabványos JMS hívásokat használ üzenetek feldolgozásához, beleértve bármilyen választ és kimenő üzenetkezelést. Egy EJB-komponens kezelhet válaszokat küldő komponensként viselkedve, vagy a bejövő üzeneteket fogadó EJB-komponensen belül. Választhatóan ez a folyamat használhat kétfázisú végrehajtást egy tranzakció hatáskörén belül. Az aszinkron üzenetkezelés ezen szintű működését komponens-kezelt üzenetkezelésnek hívják, és az üzenetkezelés infrastruktúra felett (például a kapcsolat és szekció tár kezelés) teljes irányítást ad egy EJB-komponensnek. A közös tárolónak nincs szerepe a komponens-kezelt üzenetkezelésben.

A WebSphere Application Server szintén támogatja az automatikus aszinkron üzenetkezelést üzenet-vezérelt komponensek (az Enterprise JavaBeans (EJB) 2.0 specifikációban meghatározott EJB-komponens típus) és JMS figyelők (a JMS alkalmazáskiszolgáló lehetőségek része) használatával. Az üzenetek automatikusan lekérdezésre kerülnek a JMS címzettektől, választhatóan egy tranzakcióban, majd elküldésre kerülnek az üzenetvezérelt komponenshez a Java EE alkalmazásban, anélkül hogy az alkalmazásnak kifejezetten le kellene kérnie a JMS címzettet.



A jelzett hivatkozások (online) Internet-hozzáférést igényelnek.
