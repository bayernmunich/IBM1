JMS szolgáltató beállítások alkalmazásügyfelekhez

Ezen oldal használatával a Java üzenetszolgáltatás (JMS) szolgáltató tulajdonságait állíthatja be, ha szeretne az alapértelmezett üzenetkezelési szolgáltatótól, vagy a WebSphere MQ programtól, mint JMS szolgáltatótól eltérő JMS szolgáltatót használni.

Az alábbi Alkalmazásügyfél erőforrás beállítási eszköz (ACRCT) oldal megtekintéséhez kattintson a Fájl > Megnyitás elemre. Miután keresett egy EAR fájlt, kattintson a Megnyitás gombra. Bontsa ki a kijelölt JAR fájlt. Kattintson a jobb egérgombbal a Üzenetkezelési szolgáltatók lehetőségre, majd válassza az előugró menü Új menüpontját. A következő mezők jelennek meg az Általános lapon.

Név

A név, amellyel a JMS szolgáltató megkülönböztethető adminisztrációs célokhoz.

Adattípus Karaktersorozat
Leírás

A JMS szolgáltató leírása adminisztrációs célokhoz.

Adattípus Karaktersorozat
Osztályútvonal

Útvonalak listája vagy .jar fájlnevek, amelyek együtt formálják az erőforrás szolgáltató osztályok helyét.

Kontextus gyár osztály

A kezdeti kontextus gyár Java osztályneve a JMS szolgáltatóhoz.

Például, egy LDAP szolgáltatás szolgáltató esetében az érték a következő formátummal rendelkezik: com.sun.jndi.ldap.LdapCtxFactory.

Adattípus Karaktersorozat
Szolgáltató URL

A JMS szolgáltató URL a külső JNDI kikeresésekhez.

Például, egy LDAP URL cím a következő formátummal rendelkezik egy JMS szolgáltató esetében: ldap://hostname.company.com/contextName.

Adattípus Karaktersorozat
Egyéni tulajdonságok

Név-érték párokat határoz meg további tulajdonságok beállításához azon az objektumon, ami futás közben jön létre ehhez az erőforráshoz.

Meg kell adnia egy nevet, ami nyilvános tulajdonsága az objektumnak és egy értéket, ami átváltható karaktersorozatból a tulajdonság beállító metódusához szükséges típusba. Az elfogadható tulajdonságok és értékek a létrehozott objektumtól függnek. Forduljon az objektum dokumentációjához az érvényes tulajdonságok és értékek listájához.
