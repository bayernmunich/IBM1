Configurând furnizori noi JMS cu Unealta de configurare resurse client de aplicaţii

Puteţi crea configuraţii furnizor Serviciu mesaj noi Java (JMS) pentru Clientul de aplicaţii. Clientul de aplicaţii face utilizare de serviciul de mesagerie prin interfeţe JMS.

Despre acest task

În timpul acestui task, creaţi configuraţii furnizor Serviciu mesaj noi Java (JMS) pentru Clientul de aplicaţii. Clientul de aplicaţii face utilizare de serviciul de mesagerie prin interfeţe JMS. Un furnizor JMS asigură două tipuri de resurse J2EE. Una este o fabrică de conexiuni JMS, şi cealaltă este o destinaţie JMS.

Într-un task administrativ separat, trebuie să instalaţi clientul JMS pe maşina client unde clientul de aplicaţii particular se află. Vânzătorul produs de mesagerie trebuie să furnizeze o implementare a clientului JMS. Pentru mai multe informaţii suplimentare, vedeţi documentaţia produs de mesagerie.

Procedură

  1. Porniţi Application Client Resource Configuration Tool şi deschideţi fişierul EAR pentru care doriţi să configuraţi noul furnizor JMS. Conţinuturile fişierelor EAR sunt afişate într-o vizualizare arbore.
  2. Din arbore, selectaţi fişierul JAR în care doriţi să configuraţi noul furnizor JMS.
  3. expandaţi fişierul JAR pentru a vizualiza conţinutul.
  4. Clic dreapta Furnizori de mesagerie. Finalizaţi una din următoarele acţiuni:
    • Faceţi clic dreapta pe folder şi selectaţi Nou.
    • Pe bara de meniu, faceţi clic Editare > Nou.
  5. În dialogul proprietate rezultat, configuraţi proprietăţile furnizor JMS.
  6. Faceţi clic OK când terminaţi.
  7. Faceţi clic pe Fişier > Salvaţi pe bara de meniu pentru a salva modificările.

Exemplu

Exemplul următor de cod ilustrează cum se configurează setările JMS Provider, JMS Connection Factory şi JMS Destination pentru clienţii aplicaţiilor.

<resources.jms:JMSProvider xmi:id="JMSProvider_3" name="genericJMSProvider:name"
description="genericJMSProvider:description"
externalInitialContextFactory="genericJMSProvider:contextFactoryClass"
externalProviderURL="genericJMSProvider:providerUrl">
<classpath>genericJMSProvider:classpath</classpath>
<factories xmi:type="resources.jms:GenericJMSDestination"
xmi:id="GenericJMSDestination_1" name="jmsDestination:name"
jndiName="jmsDestination:jndiName" description="jmsDestination:description"
externalJNDIName="jmsDestination:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_15">
<resourceProperties xmi:id="J2EEResourceProperty_17" name="jmsDestination:customName"
value="jmsDestination:customValue"/>
</propertySet>
</factories>
<factories xmi:type="resources.jms:GenericJMSConnectionFactory"
xmi:id="GenericJMSConnectionFactory_1" name="jmsCF:name" jndiName="jmsCF:jndiName"
description="jmsCF:description" userID="jmsCF:user" password="{xor}NTIsHBllMT4yOg=="
externalJNDIName="jmsCF:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_16">
<resourceProperties xmi:id="J2EEResourceProperty_18" name="jmsCF:customName"
value="jmsCF:customValue"/>
</propertySet>
</factories>
<propertySet xmi:id="J2EEResourcePropertySet_17">
<resourceProperties xmi:id="J2EEResourceProperty_19"
name="genericJMSProvider:customName" value="genericJMSProvider:customValue"/>
</propertySet>
</resources.jms:JMSProvider>

Câmpurile necesare includ:

  • Pagina Proprietăţi furnizor JMS: nume, şi cel puţin un furnizor de protocol
  • Pagina Proprietăţi fabrică de conexiuni JMS: nume, jndiName, tip destinaţie
  • Pagina Proprietăţi destinaţie JMS: nume, jndiName, tip destinaţie

Cazuri speciale:

  • Tipul destinaţiei trebuie să fie COADĂ, sau SUBIECT.


Legăturile marcate (online) necesită acces la Internet.
