Setările fabrică de conexiuni coadă Furnizor WebSphere MQ pentru clienţii de aplicaţii

Utilizaţi acest panou pentru a vizualiza sau modifica proprietăţile configuraţiei fabricii de conexiuni pentru coada selectată pentru utilizare cu furnizorul de MQSeries® product Java™ Message Service (JMS. Aceste proprietăţi de configurare controlează cum sunt create conexiunile între furnizorul de JMS şi WebSphere® MQ.

Pentru a vedea această pagină Unealtă configurare resursă client aplicaţie (ACRCT), faceţi clic pe Fişier > Deschidere. După ce răsfoiţi după un fişier EAR, faceţi clic pe Deschidere. Expandaţi fişierul JAR selectat > Furnizori de mesagerie > Furnizor MQ WebSphere. Faceţi clic dreapta pe Fabrici de conexiuni coadă şi faceţi clic pe Nou. Următoarele câmpuri sunt afişate în fişa General.

Notă:
  • Valorile proprietăţilor pe care le specificaţi trebuie să se potrivească cu valorile pe care le-aţi specificat la configurarea resurselor JMS pentru WebSphere MQ. Pentru informaţii suplimentare despre configurarea resurselor JMS pentru WebSphere MQ, citiţi secţiunea Utilizare Java din centrul de informare WebSphere MQ.
  • În MQ WebSphere, numele pot avea un maxim de 48 de caractere, cu excepţia canalelor care au maxim 20 de caractere.

O fabrică de conexiuni coadă pentru furnizorul JMS are următoarele proprietăţi.

Tip de transport

Specifică dacă conexiunea client MQ WebSpheresau legături JNDI sunt utilizate pentru conexiune la managerul coadă MQ WebSphere. Furnizorul JMS extern controlează protocoalele de comunicaţie dintre clienţi JMS şi servere JMS. Reglaţi tipul de transport când utilizaţi mesagerie non-ASF nepersistentă, nedurabilă, netranzacţională sau când doriţi să satisfaceţi probleme de securitate şi clientul este local la nodul manager coadă.

Tip de date Enum
Unităţi Nu se aplică
Valoare implicită LEGĂTURI
Interval
BINDINGS
Legăturile JNDI sunt utilizate pentru conectarea la managerul coadă. LEGĂTURI este un protocol memorie partajată şi poate fi utilizat doar când managerul coadă este pe acelaşi nod ca şi clientul JMS şi posedă riscuri de securitate care ar trebui să fie adresate prin utilizarea de roluri EJB.
CLIENT
Conexiunea client MQ WebSphere este utilizat pentru conectare la managerul coadă. CLIENT este un protocol TCP-bazat tipic.
DIRECT
Pentru Broker-ul eveniment MQ WebSphere utilizând modul DIRECT. DIRECT este un protocol socket de categorie uşoară utilizat în mesageria netranzacţională, nedurabilă şi nepersistentă Publicare/Abonament. DIRECT funcţionează doar pentru clienţi şi bean-uri de mesagerie conduse utilizând protocolul non-ASF.
QUEUED
QUEUED este un protocol TCP standard.
Recomandat
Tip de transport al fabricii de conexiuni de cozi
LEGĂTURI este mai rapid cu 30% sau mai mult, dar duce lipsă de securitate. Când aveţi preocupări de securitate, LEGĂTURI este mai dorit decât CLIENT.
Tip de transport al fabricii de conexiuni de subiect
DIRECT este cel mai rapid tip şi ar trebui utilizat unde este posibil. Utilizaţi LEGĂTURI când vreţi să satisfaceţi task-uri de securitate suplimentare şi managerul coadă este local la clientul JMS. PUS ÎN COADĂ este regresia pentru toate celelalte cazuri. MQ 5.3 WebSphere înainte de CSD2 cu setarea DIRECT poate pierde mesaje când este utilizat cu bean-uri de mesagerie conduse şi subîncărcate. Această pierdere de asemenea se întâmplă cu aplicaţii parte-client doar dacă broker-ul maxClientQueueSize este setat la 0. Puteţi seta aceasta la 0 cu comanda: 
#wempschangeproperties WAS_nodeName_server1
 -e default -o DynamicSubscriptionEngine -n
 maxClientQueueSize -v 0 -x executionGroupUUID
unde executionGroupUUID poate fi găsit prin pornirea broker-ului şi prin căutarea în Istoric evenimente/Aplicaţii pentru evenimentul 2201. Această valoare este de obicei ffffffff-0000-0000-000000000000.
Notă: WebSphere MQ 5.3 JMS nu poate fi utilizat în WAS 6.1 întrucât WAS 6.1 are un runtime Java 5. Prin urmare, conexiunile memorie încrucişată nu pot fi realizate cu managerii coadă MQ 5.3 WebSphere. Aceasta poate determina degradarea performanţelor dacă aţi utilizat anterior WebSphere MQ 5.3 şi BINDINGS pentru co nexiunile dvs. şi muta la conexiunile de reţea CLIENT la migrarea la WebSphere Application Server Versiunea 6.1. Dacă utilizaţi WebSphere MQ 5.3 pentru z/OS®, puteţi achiziţiona de asemenea un pachet de caracteristici suplimentare.

Când se rulează pe un 64-bit z/OS, Tipul de Transport trebuie set at la CLIENT întrucât 64-bit WebSphere MQ z/OS nu este disponibil în mod curent şi modul BINDINGS nu poate fi utilizat pentru conectarea la 31-bit WebSphere MQ z/OS. Aţi putea de asemenea necesita să cumpăraţi un pachet caracteristică MQ suplimentar WebSphere pentru acest suport.

Suită de cifrare SSL

Specifică suita de cifrare pentru utilizare pentru conexiunea SSL la MQ WebSphere.

Setaţi această proprietate la o suită de cifrare validă furnizată de furnizorul dumneavoastră JSSE. Valoarea trebuie să se potrivească cu CipherSpec specificat la canalul SVRCONN ca Canal proprietate.

Trebuie să setaţi această proprietate dacă setaţi proprietatea SSL Peer Name .

Container de certificate SSL

Specifică o listă de niciun sau mai multe servere Listă revocare certificat (CRL) care sunt utilizate pentru verificarea revocărilor certificat SSL. Dacă specificaţi o valoare pentru această proprietate, trebuie să utilizaţi JVM MQ WebSpherela 2 versiune 1.4 Java.

Valoarea este o listă delimitată de spaţii de intrări de forma:
ldap://nume gazdă:[port]

Un singur slash (/) urmează această valoare. Dacă portul este omis, portul LDAP implicit de 389 este presupus. La timpul de conectare, certificatul SSL prezentat de server este verificat împotriva serverelor CRL specificate. Pentru informaţii suplimentare despre securitatea CRL, recurgeţi la informaţiile despre "Operarea cu liste de revocare certificate" în secţiunea Securitate a centrului de informare WebSphere MQ.

Nume pentru peer SSL

Pentru SSL, structura nume distinctiv care trebuie să se potrivească cu managerul de cozi WebSphere MQ. Numele distinctiv este utilizat pentru verificarea certificatului de identificare prezentat de server la timpul de conexiune.

Dacă această proprietate nu este setată, asemenea verificare de certificat este realizată.

Proprietatea nume peer SSL este ignorată dacă nu este specificată SSL Cipher Suite proprietatea .

Această proprietate este o listă de nume atribute şi perechi de valori separate de virgule sau punct şi virgulă. De exemplu: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

Exemplul dat verifică certificatul de identificare prezentat de server la momentul conexiunii. Pentru ca conexiunea să reuşească, certificatul trebuie să aibă un QMGR. care începe Numele comun, şi trebuie să aibă cel puţin două nume Unitate organizaţională, primul fiind IBM şi al doilea WEBSPHERE. Verificarea nu este sensibilă la majuscule.

Pentru mai multe detalii despre numele distinctive şi utilizarea lor cu WebSphere MQ, urmăriţi secţiunea "Nume distinctive" din centrul de informare WebSphere MQ.

Pool de conexiune

Specifică un set opţional de setări de pool de conexiuni.

Proprietăţile pool-ului de conexiuni sunt comune tuturor conectorilor J2C.

Conexiunile şi sesiunile pool-uri server de aplicaţii cu furnizorul JMS pentru a îmbunătăţi performanţa. Aceasta este independentă de orice pool de conexiuni MQ WebSphere. Trebuie să configuraţi proprietăţile pool conexiune şi sesiune în mod corespunzător pentru aplicaţiile dumneavoastră, altfel aţi putea să nu primiţi comportamentul conexiune şi sesiune pe care îl vreţi.

Modificaţi dimensiunea pool-ului de conexiuni dacă accesul concurent de parte de server la resursa JMS depăşeşte valoarea implicită. Dimensiunea pool-ului de conexiuni este setat pe bază coadă sau subiect.

Dacă acest parametru este utilizat sau nu depinde de valoarea parametrului de opţiuni reconectare client.
