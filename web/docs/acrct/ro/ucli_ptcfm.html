Setările fabrică de conexiuni Furnizor WebSphere MQ pentru clienţii de aplicaţii

Utilizaţi acest panou pentru a vizualiza sau modifica proprietăţile configuraţiei fabricii de conexiuni pentru subiectul selectat pentru utilizare cu furnizorul WebSphere® MQ produs Java™ Message Service (JMS). Aceste proprietăţi de configurare controlează cum conexiunile sunt create între furnizorul JMS şi MQ WebSphere.

Pentru a vedea această pagină Unealtă configurare resursă client aplicaţie (ACRCT), faceţi clic pe Fişier > Deschidere. După ce răsfoiţi după un fişier EAR, faceţi clic pe Deschidere. Expandaţi fişierul JAR selectat > Furnizori de mesagerie > Furnizor MQ WebSphere. Faceţi clic dreapta pe Fabrici de conexiuni subiect şi faceţi clic pe Nou.

Note:
  • Valorile proprietăţilor pe care le specificaţi trebuie să se potrivească cu valorile pe care le-aţi specificat la configurarea resurselor JMS pentru WebSphere MQ. Pentru informaţii suplimentare despre configurarea resurselor JMS pentru WebSphere MQ, citiţi secţiunea Utilizare Java din centrul de informare WebSphere MQ.
  • În MQ WebSphere, numele pot avea un maxim de 48 de caractere, cu excepţia canalelor care au maxim 20 de caractere.
Agent MA0C: La crearea unei fabrici de conexiuni pe subiectul WebSphere Application Server v6 pentru agentul MA0C, trebuie să consideraţi urmatoarele valori ale atributelor:
BrokerControlQueue
Această valoare este fixată la SYSTEM.BROKER.CONTROL.QUEUE pentru broker-ul MA0C şi este coada de la care broker-ul citeşte.
BrokerVersion
Setaţi această valoare la BASIC pentru broker-ul MA0C.
ClientID
Setaţi această valoare la orice doriţi pentru broker-ul MA0C (valoarea este şir şi este doar un identificator pentru aplicaţia client a dumneavoastră).
Activat XA
Setaţi această valoare la ADEVĂRAT sau FALS pentru broker-ul MA0C (setarea utilizată este un steguleţ intensificare performanţă - veţi vrea probabil să setaţi aceasta la 'adevărat' în majoritatea timpului).
Selecţie BrokerMessage
Această valoare este fixată la CLIENT pentru broker-ul MA0C deoarece broker-ul se bazează pe selecţia mesaj parte client.
Tip autorizaţie broker direct
Această valoare nu este necesară de broker-ul MA0C.

Nume JNDI

Numele Interfaţă director şi nume Java(JNDI) care este utilizat pentru a lega fabrica de conexiuni subiect de spaţiul de nume server de aplicaţii.

Suită de cifrare SSL

Suita de cifrare de utilizat pentru conexiunea SSL la WebSphere MQ.

Setaţi această proprietate la o suită de cifrare validă furnizată de furnizorul dumneavoastră JSSE. Valoarea trebuie să se potrivească cu CipherSpec specificat la canalul SVRCONN ca Canal proprietate.

Trebuie să setaţi această proprietate dacă setaţi proprietatea Nume peer SSL .

Container de certificate SSL

O listă de niciun sau mai multe servere Listă revocare certificat (CRL) care sunt utilizate pentru verificarea revocărilor certificat SSL. Dacă specificaţi o valoare pentru această proprietate, trebuie să utilizaţi JVM MQ WebSpherela 2 versiune 1.4 Java.

Valoarea este o listă delimitată de spaţii de intrări de forma:
ldap://nume gazdă:[port]

Un singur slash (/) urmează această valoare. Dacă portul este omis, portul LDAP implicit de 389 este presupus. La timpul de conectare, certificatul SSL prezentat de server este verificat împotriva serverelor CRL specificate. Pentru informaţii suplimentare despre securitatea CRL, recurgeţi la informaţiile despre "Operarea cu liste de revocare certificate" în secţiunea Securitate a centrului de informare WebSphere MQ.

Nume pentru peer SSL

Pentru SSL, o structură nume distinctiv care trebuie să se potrivească cu managerul de cozi WebSphere MQ. Numele distinctiv este utilizat pentru verificarea certificatului de identificare prezentat de server la timpul de conexiune.

Dacă această proprietate nu este setată, asemenea verificare de certificat este realizată.

Proprietatea nume peer SSL este ignorată dacă nu este specificată proprietatea Suită de cifrare SSL .

Această proprietate este o listă de nume atribute şi perechi de valori separate de virgule sau punct şi virgulă. De exemplu: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

Exemplul dat verifică certificatul de identificare prezentat de server la momentul conexiunii. Pentru ca conexiunea să reuşească, certificatul trebuie să aibă un QMGR. care începe Numele comun, şi trebuie să aibă cel puţin două nume Unitate organizaţională, primul fiind IBM şi al doilea WEBSPHERE. Verificarea nu este sensibilă la majuscule.

Pentru mai multe detalii despre numele distinctive şi utilizarea lor cu WebSphere MQ, urmăriţi informaţiile despre "Nume distinctive" în secţiunea Securitate din centrul de informare WebSphere MQ.

Pool de conexiune

Un set opţional de setări pool de conexiuni.

Proprietăţile pool-ului de conexiuni sunt comune tuturor conectorilor J2C.

Conexiunile şi sesiunile pool-uri server de aplicaţii cu furnizorul JMS pentru a îmbunătăţi performanţa. Aceasta este independentă de orice pool de conexiuni MQ WebSphere. Trebuie să configuraţi proprietăţile pool conexiune şi sesiune în mod corespunzător pentru aplicaţiile dumneavoastră, altfel aţi putea să nu primiţi comportamentul conexiune şi sesiune pe care îl vreţi.

Modificaţi dimensiunea pool-ului de conexiuni dacă accesul concurent de parte de server la resursa JMS depăşeşte valoarea implicită. Dimensiunea pool-ului de conexiuni este setat pe bază coadă sau subiect.
