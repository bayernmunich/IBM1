应用程序客户机的 URL 提供程序设置

使用此页面新建 URL 提供程序。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件。右键单击 URL 提供程序，并单击新建。 以下字段出现在常规选项卡上。

URL 提供程序实现特定 URL 协议的功能，例如，超文本传输协议（HTTP）。此由类组成的提供程序扩展 java.net.URLStreamHandler 和 java.net.URLConnection 类。

名称

URL 的管理名称。

描述

对 URL 的可选描述，用于您的管理记录。

类路径

共同组成资源提供程序类的位置的路径或 JAR 文件名的列表。

协议

此流处理程序支持的协议。例如，nntp、smtp、ftp 等等。

要使用缺省协议，保持该字段为空白。

流处理程序类

为特定 URL 协议（如 FTP）扩展 java.net.URLStreamHandler 的用户定义的 Java™ 类的标准名称。

要使用缺省流处理程序，保持该字段为空白。

定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。



标有（在线）的链接要求访问因特网。
