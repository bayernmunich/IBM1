应用程序客户机的一般 JMS 连接工厂设置

使用此面板查看或更改所选 Java™ 消息服务（JMS）连接工厂的配置属性，以与关联的 JMS 提供程序配合使用。这些配置属性控制如何创建 JMS 提供程序和它使用的消息传递系统之间的连接。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 消息传递提供程序 > new_JMS_Provider_instance。右键单击连接工厂，并单击新建。以下字段出现在常规选项卡上。

Java 消息服务（JMS）连接工厂创建到 JMS 目标的连接。JMS 连接工厂由关联的 JMS 提供程序创建。一般 JMS 提供程序（除了内部缺省消息传递提供程序或者作为 JMS 提供程序的 WebSphere® MQ 之外）的 JMS 连接工厂具有以下属性：

名称

在 IBM® WebSphere Application Server 中，为了进行管理而提供的用于识别此 JMS 连接工厂的名称。此名称在相关联的 JMS 提供程序中必须唯一。

数据类型 字符串
描述

在 IBM WebSphere Application Server 中，为了进行管理而提供的对此连接工厂的描述。

数据类型 字符串
缺省值 Null
JNDI 名称

应用程序客户机运行时使用此字段检索配置信息。

用户标识

表明如果调用应用程序未显式提供用户标识和密码，那么将带有密码属性的用户标识用于认证。

如果您指定用户标识属性的值，那么您还必须指定密码属性的值。

如果调用应用程序未显式提供用户标识和密码，那么使用连接工厂用户标识和密码属性；例如，如果调用应用程序使用方法 createQueueConnection()。JMS 客户机将用户标识和密码传递到 JMS 服务器。

数据类型 字符串
密码

如果调用应用程序未显式提供用户标识和密码，那么将带有用户标识属性的密码用于认证。

如果您指定用户标识属性的值，那么您还必须指定密码属性的值。

数据类型 字符串
缺省值 Null
重新输入密码

确认输入到密码字段中的密码。

外部 JNDI 名称

用于将队列绑定到应用程序服务器名称空间的 JNDI 名称。

按约定，使用标准 JNDI 名称，例如，jms/Name，其中 Name 是资源的逻辑名称。

该名称用于链接平台绑定信息。绑定将模块的部署描述符定义的资源与由平台绑定到 JNDI API 的实际（物理）资源相关联。

数据类型 字符串
连接类型

此 JMS 目标是一个队列（用于点到点）还是主题（用于发布或预订）。

选择下列其中一个选项：
队列
点到点消息传递的 JMS 队列目标。
主题
用于发布预订消息传递的 JMS 主题目标。
定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
