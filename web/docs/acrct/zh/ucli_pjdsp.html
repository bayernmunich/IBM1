应用程序客户机的通用 JMS 目标设置

使用此面板查看或更改所选 JMS 目标的配置属性，以由关联的 JMS 提供程序使用。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 消息传递提供程序 > 新的 JMS 提供程序实例。右键单击目标，然后单击新建。以下字段出现在常规选项卡上。

JMS 目标用于配置关联通用 JMS 提供程序的 JMS 目标的属性。与 JMS 目标的连接由相关联的 JMS 连接工厂创建。与一般 JMS 提供程序（不是缺省消息传递提供程序或者作为 JMS 提供程序的 WebSphere® MQ）配合使用的 JMS 目标具有下列属性。

名称

在 WebSphere Application Server 中，为了进行管理而提供的用于识别该队列的名称。

数据类型 字符串
描述

用于管理的队列描述。

JNDI 名称

绑定到 JNDI 中的 JMS 目标实际（物理）名的 JNDI 名称。

外部 JNDI 名称

用于将队列绑定到应用程序服务器名称空间的 JNDI 名称。

按约定，使用标准 JNDI 名称；例如，格式为 jms/Name，其中 Name 是资源的逻辑名称。

该名称用于链接平台绑定信息。绑定使模块部署描述符定义的资源与由平台绑定到 JNDI 的实际（物理）资源相关联。

数据类型 字符串
目标类型

此 JMS 目标是一个队列（用于点到点）还是主题（用于发布或预订）。

选择下列其中一个选项：
队列
点到点消息传递的 JMS 队列目标。
主题
发布/预订消息传递的 JMS 主题目标。
定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
