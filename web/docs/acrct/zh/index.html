应用程序客户机资源配置工具

浏览至此部分中的帮助主题。

  • 为应用程序客户机更新作为 Java 消息服务提供程序的 WebSphere MQ，以及它的 JMS 资源配置
    可以更新作为 Java 消息服务（JMS）提供程序的 WebSphere MQ 的现有配置，并且更新 WebSphere MQ 连接工厂或 WebSphere MQ 目标的配置。
  • 为应用程序客户机配置新数据源提供程序（JDBC 提供程序）
    可以使用应用程序客户机资源配置工具（ACRCT）为应用程序客户机创建新的数据源提供程序（也称为 JDBC 提供程序）。
  • 为应用程序客户机配置新的 URL 提供程序
    可以使用应用程序客户机资源配置工具（ACRCT）为客户机应用程序创建 URL 提供程序和 URL。
  • 为应用程序客户机配置新的资源环境条目
    可以使用应用程序客户机资源配置工具（ACRCT）为客户机应用程序创建新的资源环境条目。
  • 为应用程序客户机配置新的邮件会话
    可以使用应用程序客户机资源配置工具（ACRCT）为应用程序客户机配置新的邮件会话。
  • 为应用程序客户机配置新资源环境提供程序
    可以使用应用程序客户机资源配置工具（ACRCT）为应用程序客户机创建新的资源环境提供程序配置。
  • 为应用程序客户机配置邮件提供者和会话
    可以使用应用程序客户机资源配置工具（ACRCT）来编辑应用程序客户机的邮件会话配置和提供程序配置。
  • 配置 Java 消息传递客户机资源
    要配置 Java 消息传递客户机资源，需要为应用程序客户机创建新的 JMS 提供程序配置。应用程序客户机可以通过 Java 消息服务 API 使用消息传递服务。JMS 提供程序提供两种 J2EE 工厂。 一个是 JMS 连接工厂，另一个是 JMS 目标工厂。
  • 配置受管对象
    本节帮助您配置新的受管对象。
  • 在 WebSphere Application Server 中使用 JMS 进行异步消息传递
    WebSphere Application Server 支持将异步消息传递作为基于 Java 消息服务（JMS）编程接口的通信方法。JMS 接口为 Java 程序（客户机和 Java Platform Enterprise Edition（Java EE）应用程序）提供了创建、发送、接收和读取异步请求（作为 JMS 消息）的公共方法。
  • 客户机的 Java 消息服务提供程序
    本主题描述客户机应用程序可以将 Java 消息服务（JMS）提供程序和 WebSphere Application Server 配合使用的不同方法。
  • 客户机的资源适配器
    资源适配器是系统级的软件驱动程序，Java 应用程序使用它来连接到企业信息系统（EIS）。资源适配器插进应用程序客户机，并提供 EIS 和企业应用程序之间的连接。
  • 资源环境提供程序和资源环境条目
    一个资源环境引用将客户机应用程序使用的逻辑名映射至一个对象的物理名称。
  • Application Client Resource Configuration Tool 的 URL 提供程序
    URL 提供程序实现特定 URL 协议的功能，例如，超文本传输协议（HTTP）。该提供程序由一对类组成，扩展 java.net.URLStreamHandler 和 java.net.URLConnection 类。
  • URLs for application clients
    A Uniform Resource Locator (URL) is an identifier that points to an electronically accessible resource, such as a directory file on a machine in a network, or a document stored in a database.
  • 为资源适配器配置新的连接工厂

  • 应用程序客户机的 JMS 提供程序设置
