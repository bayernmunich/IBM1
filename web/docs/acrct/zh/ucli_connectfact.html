资源适配器连接工厂设置

使用此面板查看或更改所选资源适配器连接工厂的配置属性。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 资源适配器。右键单击连接工厂文件夹，并单击新建。以下字段出现在常规选项卡上。

名称

在 WebSphere® Application Server 中，为了进行管理而提供的用于识别此连接工厂的名称。此名称在跨产品管理域的资源适配器连接工厂中必须唯一。

数据类型 字符串
描述

在 IBM® WebSphere Application Server 中，为了进行管理而提供的对此连接工厂的可选描述。

数据类型 字符串
JNDI 名称

用于将此资源适配器连接工厂定义与部署描述符匹配起来的 JNDI 名称。此条目应该是 resource-ref 名称。

数据类型 字符串
用户名

如果在获取连接时调用应用程序未显式提供用户标识和密码，那么将带有密码属性的用户名用于认证。如果使用此字段，那么用户名“属性”字段被忽略。

如果您指定用户名属性的值，那么还必须指定密码属性的值。

如果在获取连接时调用应用程序未显式提供用户标识和密码，那么使用连接工厂用户名和密码属性。

数据类型 字符串
密码

指定加密的密码。如果您完成此字段，那么“属性”框中的密码字段被忽略。

如果您指定用户名属性的值，那么还必须指定密码属性的值。

数据类型 字符串
重新输入密码

确认密码。

类型

为资源适配器归档中的工厂定义的所有 connectionFactoryInterfaces 的下拉列表。

对于每种类型，都有在“属性”框中指定的一组属性。通过从每个连接定义对象获取属性来构造该组属性。对于任何为更新而显示的现有连接工厂，用为对象指定的属性覆盖此属性列表。当类型字段已更改时，属性也更改，以反映该类型的正确属性。

数据类型 字符串

文件名： ucli_connectfact.html
