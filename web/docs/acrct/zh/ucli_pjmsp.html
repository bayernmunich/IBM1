应用程序客户机的 JMS 提供程序设置

如果您要使用除缺省消息传递提供程序之外的 JMS 提供程序或者作为 JMS 提供程序的 WebSphere® MQ，使用此页面来配置 Java™ 消息服务（JMS）提供程序的属性。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件。右键单击消息传递提供程序，然后单击新建。 以下字段出现在常规选项卡上。

名称

为了进行管理而提供的用于识别 JMS 提供程序的名称。

数据类型 字符串
描述

为了进行管理而提供的 JMS 提供程序的描述。

数据类型 字符串
类路径

路径或 .jar 文件名的列表，它们共同组成资源提供程序类的位置。

上下文工厂类

用于 JMS 提供程序的初始上下文工厂的 Java 类名。

例如，对于 LDAP 服务提供者，该值的格式为：com.sun.jndi.ldap.LdapCtxFactory。

数据类型 字符串
提供程序 URL

用于外部 JNDI 查找的 JMS 提供程序 URL。

例如，JMS 提供程序的 LDAP URL 的格式为：ldap://hostname.company.com/contextName。

数据类型 字符串
定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
