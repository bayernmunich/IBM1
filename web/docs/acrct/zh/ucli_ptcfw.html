应用程序客户机的 V5 缺省提供程序主题连接工厂设置

使用此面板查看或更改所选主题连接工厂的配置属性，以供内部产品 Java™ 消息服务（JMS）提供程序使用。这些配置属性控制如何创建 JMS 提供程序和它使用的消息传递系统之间的连接。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 消息传递提供程序 > V5 缺省提供程序。右键单击 主题连接工厂，并单击新建。以下字段出现在常规选项卡上。

V5 缺省提供程序主题连接工厂具有下列属性：

名称

在 WebSphere® Application Server 中，为了进行管理而提供的用于识别此队列连接工厂的名称。此名称在跨越 WebSphere Application Server 管理域的 JMS 连接工厂中必须唯一。

数据类型 字符串
描述

在 WebSphere Application Server 中，为了进行管理而提供的对此主题连接工厂的描述。

数据类型 字符串
JNDI 名称

应用程序客户机运行时使用此字段检索配置信息。

用户标识

如果调用应用程序未显式提供用户标识和密码，那么将带有密码属性的用户标识用于认证。

如果您指定用户标识属性的值，那么您还必须指定密码属性的值。

如果调用应用程序未显式提供用户标识和密码，那么使用连接工厂用户标识和密码属性，例如，如果调用应用程序使用方法 createTopicConnection()。JMS 客户机将用户标识和密码传递到 JMS 服务器。

数据类型 字符串
密码

如果调用应用程序未显式提供用户标识和密码，那么将带有用户标识属性的密码用于认证。

如果您指定用户标识属性的值，那么您还必须指定密码属性的值。

数据类型 字符串
重新输入密码

确认密码。

节点

管理节点的 WebSphere Application Server 节点名，在该节点上为此连接工厂运行 JMS 服务器。此工厂创建的连接与该 JMS 服务器连接。

数据类型 枚举
范围 WebSphere Application Server 管理域中的节点的下拉列表。
Application Server

输入应用程序服务器名。该名称不是机器的主机名，而是配置的应用程序服务器的名称。

端口

连接使用两个端口中的哪一个来连接到 JMS 服务器。QUEUED 端口用于全功能 JMS 发布/预订支持，DIRECT 端口仅用于非持续性、非事务性、非持久性预订。

注： 消息驱动的 bean 不能为发布或预订支持使用直接侦听器端口。因此，端口设置为 Direct 的任何主题连接工厂都不能由消息驱动的 bean 使用。
数据类型 枚举
缺省值 QUEUED
范围
QUEUED
用于符合 JMS 的全功能、发布或预订支持的侦听器端口。
DIRECT
用于发布或预订支持的直接 TCP/IP 连接（仅非事务、非持久和非持久预订）的侦听器端口。

在产品内部 JMS 服务器上定义这些端口的 TCP/IP 端口号。
客户机标识

用于连接到 MQSeries® 队列管理器的 JMS 客户机标识。

数据类型 字符串
定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
