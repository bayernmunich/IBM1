客户机的 Java 消息服务提供程序

本主题描述客户机应用程序可以将 Java™ 消息服务（JMS）提供程序和 WebSphere® Application Server 配合使用的不同方法。

IBM® WebSphere Application Server 支持通过使用 JMS 提供程序及其相关消息传递系统来进行异步消息传递。JMS 提供程序必须符合 JMS 规范 V1.1。要使用消息驱动的 Bean，JMS 提供程序必须支持此规范中定义的可选应用程序服务器工具（ASF）功能，或支持 JCA 规范 V1.5 中定义的入站资源适配器。

如果已配置通过缺省消息传递提供程序访问的服务集成总线，那么可以将 IBM WebSphere Application Server 的服务集成技术用作消息传递系统。此支持作为 WebSphere Application Server 的一部分安装，通过管理控制台进行管理，并且与 WebSphere Application Server 运行时完全集成在一起。

WebSphere Application Server 还包含对下列 JMS 提供程序的支持：
WebSphere MQ
提供以与 WebSphere MQ 的支持版本配合使用。
通用
提供以与支持 ASF 的任何第三方消息传递系统配合使用。

由于 WebSphere Application Server 能够向后兼容先前发行版，因此它还包含对 V5 缺省消息传递提供程序的支持，此支持使您可以配置与 WebSphere Application Server V5 嵌入式消息传递系统配合使用的资源。还可以将此 V5 缺省消息传递提供程序与服务集成总线配合使用。

WebSphere 应用程序可以使用由这些 JMS 提供程序中的任何一个提供的消息传递资源。但是，使用现有的消息传递系统或与其集成的需求通常决定了对提供程序的选择。例如，可能已经存在基于 WebSphere MQ 的消息传递基础结构。在这种情况下，您可以将所包括对 WebSphere MQ 的支持用作 JMS 提供程序以进行直接连接，也可以配置与 WebSphere MQ 网络相链接的服务集成总线并通过缺省消息传递提供程序来访问该总线。

服务集成总线还提供了对缺省消息传递提供程序的访问。这是符合 J2EE 1.4 的 JMS 消息传递提供程序，它完全集成在 WebSphere Application Server 中。可以在多个服务器配置中使用它，以便与 WebSphere MQ 网络进行消息传递交互。
