配置 Java 消息传递客户机资源

要配置 Java™ 消息传递客户机资源，需要为应用程序客户机创建新的 JMS 提供程序配置。应用程序客户机可以通过 Java 消息服务 API 使用消息传递服务。JMS 提供程序提供两种 J2EE 工厂。 一个是 JMS 连接工厂，另一个是 JMS 目标工厂。

在您开始之前

在独立的管理任务中，在应用程序客户机驻留的客户机上安装 Java 消息服务（JMS）客户机。消息传递产品供应商必须提供 JMS 客户机的实现。有关更多信息，请参阅您的消息传递产品文档。

注： 完成本任务时，可以创建新的消息传递提供程序，也可以使用现有的消息传递提供程序。

过程

  1. 启动应用程序客户机资源配置工具（ACRCT）。
  2. 打开要为其配置新的 JMS 提供程序的 EAR 文件。 EAR 文件内容在显示的树形视图中。
  3. 从树中选择您要在其中配置新 JMS 提供程序的 JAR 文件。
  4. 展开 JAR 文件以查看其内容。
  5. （可选）如果要创建并使用新的消息传递提供程序，那么右键单击消息传递提供程序并选择新建。
  6. 在结果属性对话框中，配置 JMS 提供程序属性。
  7. 单击确定。
  8. 单击文件 > 保存。


标有（在线）的链接要求访问因特网。
