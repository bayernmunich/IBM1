应用程序客户机的数据源属性

使用此页面创建或修改数据源。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 数据源提供程序 > 数据源提供程序实例。右键单击数据源并单击新建。以下字段出现在常规选项卡上：

名称

指定此数据源的显示名称。

数据类型 字符串
描述

指定描述数据源的文本。

数据类型 字符串
JNDI 名称

应用程序客户机运行时使用此字段检索配置信息。

数据库名称

需要连接的数据库的名称。

用户

如果调用应用程序未显式提供用户标识和密码，那么使用具有密码属性的用户标识来认证。

如果您指定用户标识属性的值，那么还必须指定密码属性的值。如果调用应用程序未显式提供用户标识和密码，那么使用连接工厂用户标识和密码属性。

密码

如果调用应用程序未显式提供用户标识和密码，那么使用具有用户标识属性的密码来认证。

如果您指定密码属性的值，那么还必须指定用户标识属性的值。

重新输入密码

确认密码。

定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。



标有（在线）的链接要求访问因特网。
