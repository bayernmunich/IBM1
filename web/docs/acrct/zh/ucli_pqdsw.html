应用程序客户机的 V5 缺省提供程序队列目标设置

使用此面板查看或更改所选队列目标的配置属性，以供产品 Java™ 消息服务（JMS）提供程序使用。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 消息传递提供程序 > V5 缺省提供程序。右键单击队列目标，并单击新建。以下字段出现在常规选项卡上。

队列目标用于配置 JMS 队列的属性。 V5 缺省提供程序队列目标具有下列属性。

名称

在 WebSphere® Application Server 中，为了进行管理而提供的用于识别该队列的名称。

数据类型 字符串
描述

用于管理的队列描述。

数据类型 字符串
JNDI 名称

应用程序客户机运行时使用此字段检索配置信息。

持久性

发送到目标的所有消息是持久的、非持久的，还是由应用程序定义其持久性。

数据类型 枚举
缺省值 APPLICATION_DEFINED
范围
应用程序定义
目标上的消息由将它们放到队列的应用程序定义其持久性。
队列定义
［仅用于 WebSphere MQ 目标］目标上的消息由 WebSphere MQ 队列定义属性定义其持久性。
持久
目标上的消息是持久的。
非持久
目标上的消息是不持久的。
优先级

是由应用程序还是由指定的优先级属性定义此目标的消息优先级。

数据类型 枚举
缺省值 APPLICATION_DEFINED
范围
应用程序定义
此目标上消息的优先级由将它们放到目标上的应用程序定义。
队列定义
［仅用于 WebSphere MQ 目标］目标上的消息由 WebSphere MQ 队列定义属性定义其持久性。
指定的
此目标上的消息的优先级由指定的优先级属性来定义。如果您选择此选项，那么必须在指定的优先级属性上定义优先级。
指定的优先级

如果优先级属性设置为 Specified，那么在此处输入此队列的消息优先级，范围是从 0（最低）到 9（最高）。

如果优先级属性设置为 Specified，那么发送到此队列的消息具有此属性指定的优先级值。

数据类型 整型
单元 消息优先级级别
范围 0（最低优先级）到 9（最高优先级）
到期

此队列的到期超时是由应用程序定义还是由指定的到期属性定义，或者队列上的消息是否到期（具有无限到期超时）。

数据类型 枚举
缺省值 APPLICATION_DEFINED
范围
应用程序定义
此队列中的消息的到期超时由将它们放到队列上的应用程序定义。
指定的
此队列中的消息的到期超时由指定的到期属性定义。如果您选择此选项，那么必须在指定的到期属性上定义超时。
无限
此队列中的消息无到期超时，所以那些消息从不到期。
指定的到期

如果到期超时属性设置为 Specified，那么指定此队列上的消息到期前的毫秒数（大于 0）。

数据类型 整型
单元 毫秒
范围 大于或等于 0
  • 0 表明消息从不超时。
  • 其他值是整数毫秒。
定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
