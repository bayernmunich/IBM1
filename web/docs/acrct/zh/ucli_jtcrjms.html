应用程序客户机的邮件会话设置

使用此页面配置邮件会话属性。

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 邮件提供者 > 邮件提供者实例。右键单击邮件会话并单击 新建。以下字段出现在常规选项卡上：

名称

表示 JavaMail 会话对象的管理名。

描述

为管理记录提供可选的描述。

JNDI 名称

应用程序客户机运行时使用此字段检索配置信息。

邮件传输主机

指定发送邮件时要连接到的服务器。

邮件传输协议

指定发送邮件时使用的传输协议。

邮件传输用户

指定当邮件传输主机要求进行认证时要使用的用户标识。

邮件传输密码

指定当邮件传输主机要求进行认证时要使用的密码。

启用严格的因特网地址解析

指定接收方地址是否必须严格遵从 RFC 822 来进行解析，RFC 822 是因特网体系结构委员会发出的规范文档。

此设置通常不用于大多数邮件应用程序。有效解析地址的 RFC 822 语法执行有效电子邮件地址的严格定义。如果您选择此设置，那么 JavaMail 将遵从 RFC 822 语法，并拒绝未解析为有效电子邮件地址（如规范所定义的那样）的接收方地址。如果您未选择此设置，那么 JavaMail 不会遵从 RFC 822 语法，并接受不遵守规范的接收方地址。缺省情况下，取消选择此设置。您可以查看 RFC 822 规范，它位于以下万维网联盟（W3C）URL：http://www.w3.org/Protocols/rfc822/。

重新输入密码

确认密码。

邮件来源

指定邮件发件人。

邮件存储主机

指定邮件帐户主机（或“域”）名。

邮件存储用户

指定邮件帐户的用户标识。

邮件存储密码

指定邮件帐户的密码。

重新输入密码

确认密码。

邮件存储协议

指定接收邮件时使用的协议。

邮件调试

为 true 时，JavaMail 与邮件服务器的交互作用以及邮件会话属性被打印到 stdout 文件。

定制属性

指定用于设置运行时为此资源创建的对象上的其他属性的“名称/值”对。

您必须输入是对象上的公用属性的名称，以及可以从字符串转换到属性的 set 方法必需的类型的值。可接受的属性和值取决于创建的对象。请参阅对象文档，以获取有效属性和值的列表。
