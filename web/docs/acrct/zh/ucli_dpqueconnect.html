缺省提供程序队列连接工厂设置

使用此面板查看或更改所选 JMS 队列连接工厂的配置属性，以供随 WebSphere® Application Server 安装的内部产品 Java™ 消息服务（JMS）提供程序使用。这些配置属性控制如何创建 JMS 提供程序和它使用的服务集成总线之间的连接

要查看该 Application Client Resource Configuration Tool（ACRCT）页面，单击文件 > 打开。在您浏览 EAR 文件后，单击打开。展开所选 JAR 文件 > 消息传递提供程序 > 缺省提供程序。右键单击队列连接工厂，并单击新建。以下字段出现在常规选项卡上。

具有缺省值的设置显示相应的值。任何具有修正值的设置都有下拉菜单。

名称

队列连接工厂的名称。

数据类型 字符串
描述

在 IBM® WebSphere Application Server 中，为了进行管理而提供的对此队列连接工厂的描述。

数据类型 字符串
JNDI 名称

用于将此队列连接工厂定义与部署描述符匹配的 JNDI 名称。此条目是 resource-ref 名称。

数据类型 字符串
用户名

如果调用应用程序未显式提供用户标识和密码，那么将带有密码属性的用户名用于认证。如果使用此字段，那么用户名“属性”字段被忽略。

如果您指定用户名属性的值，那么还必须指定密码属性的值。

如果调用应用程序未显式提供用户标识和密码，那么使用连接工厂用户名和密码属性。如果指定了用户名和密码，那么为加密密码的工厂创建认证别名。

数据类型 字符串
密码

用于创建加密的密码。如果您完成此字段，那么“属性”框中的“密码”字段被忽略。

如果您指定用户名属性的值，那么还必须指定密码属性的值。

数据类型 字符串
重新输入密码

确认密码。

总线名称

队列连接工厂连接到的总线的名称。

数据类型 字符串
客户机标识

客户机标识。它是持久主题预订所必需的。

数据类型 字符串
非持久消息传递可靠性

可靠性适用于使用此连接工厂发送的非持久 JMS 消息。

如果您要为各个 JMS 目标使用不同的可靠性传递选项，那么可以将此属性设置为作为总线目标。然后，可靠性由将 JMS 目标指定给的总线目标的 Reliability 属性定义。

缺省值 ReliablePersistent
范围
None
非持久消息没有消息可靠性。如果无法传递非持久消息，那么将其废弃。
最大努力非持久
如果内存高速缓存溢出，那么从不将消息写入磁盘并且丢弃它们。
快速非持久
如果内存高速缓存溢出，那么消息异步写入持久存储器，但在服务器重新启动后不保存它们。
可靠非持久
如果消息传递引擎失败则可能会丢失消息，并且在正常操作情况下也可能丢失消息。
可靠持久
如果消息传递引擎失败则可能会丢失消息，但在正常操作情况下不会丢失消息。
有保证的持久
支持有保证的消息传递的最高等级可靠性。
作为总线目标
使用为总线目标配置的传递选项。

目标

包含消息传递引擎的工作负载管理器目标组的名称。

数据类型 字符串
目标类型

包含消息传递引擎的工作负载管理器目标组的类型。

缺省值 BusMember
范围 BusMember、Custom、Destination、ME
目标重要性

指定目标的重要性优先级。

缺省值 首选
范围 首选、必需
目标入站传输链

解析到一组消息传递引擎的协议的名称。

数据类型 字符串
提供程序端点

用于连接到引导服务器的以逗号分隔的端点列表。

使用语法（主机:端口:协议）输入端点三元组的以逗号分隔的列表。

示例 
localhost:7777:BootstrapBasicMessaging
其中

BootstrapBasicMessaging 相应于远程协议 InboundBasicMessaging（JFAP-TCP/IP）。
缺省值
  • 如果未指定主机名，那么缺省 localhost 用作缺省值。
  • 如果未指定端口号，那么 7276 用作缺省值。
  • 如果未指定链名，那么预定义的链（例如 BootstrapBasicMessaging）用作缺省值。
连接近似性

消息传递引擎应该与请求者有近似性。

缺省值 总线、集群、服务器
范围 总线、主机
临时队列名称前缀

要应用到临时队列的名称的前缀。此名称最多为 12 个字符。

数据类型 字符串
