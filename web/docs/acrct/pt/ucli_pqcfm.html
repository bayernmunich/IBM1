Configurações do Connection Factory da Fila do Provedor WebSphere MQ para Aplicativos Clientes

Use este painel para visualizar ou alterar as propriedades de configuração do factory de conexão da fila selecionado para uso com o provedor MQSeries® do produto Java™ Message Service (JMS). Essas propriedades de conexão controlam o modo como as conexões são criadas entre o provedor JMS e o WebSphere® MQ.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor WebSphere MQ. Clique com o botão direito em Connection Factories de Fila e clique em Novo. Os campos a seguir são exibidos na guia Geral.

Observação:
  • Os valores de propriedade especificados devem corresponder aos valores especificados ao configurar os recursos JMS para o WebSphere MQ. Para obter informações adicionais sobre a configuração dos recursos JMS para o WebSphere MQ, consulte a seção Uso do Java do centro de informações do WebSphere MQ.
  • No WebSphere MQ, os nomes podem ter no máximo 48 caracteres, com exceção de canais os quais têm no máximo 20 caracteres.

Um connection factory e filas para o provedor JMS tem as seguintes propriedades.

Nome

O nome pelo qual o connection factory da fila é conhecido para propósitos administrativos dentro do WebSphere Application Server. O nome deve ser exclusivo dentro dos connection factories JMS pelo domínio administrativo do WebSphere.

Tipo de Dados Seqüência de Caracteres
Descrição

Uma descrição desse connection factory para propósitos administrativos no WebSphere Application Server.

Tipo de Dados Seqüência de Caracteres
Padrão Nulo
Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

ID do Usuário

O ID de usuário usado, com a propriedade Senha, para autenticação se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, deverá também especificar um valor para a propriedade Senha.

As propriedades ID de Usuário e Senha do Connection Factory serão usadas se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente; por exemplo, se o aplicativo de chamada usar o método createQueueConnection(). O cliente JMS envia o ID de usuário e a senha para o servidor JMS.

Tipo de Dados Seqüência de Caracteres
Senha

A senha usada, com a propriedade ID de Usuário, para autenticação se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, deverá também especificar um valor para a propriedade Senha.

Tipo de Dados Seqüência de Caracteres
Padrão Nulo
Reinserir Senha

Confirma a senha.

Tipo de Dados Botão de Opções
Padrão Não selecionado
Lista de Nomes de Conexão

Uma lista separada por vírgulas das informações de host e de porta que podem ser usadas para se conectar a um gerenciador de filas com várias instâncias.

O formato da lista é:

host[(port)],[host[(port)]]
em que port é opcional e padronizado como 1414, se não for configurado. Por exemplo:
hostname1,hostname2(1415)

Para obter informações adicionais sobre os gerenciadores de filas de várias instâncias, consulte o centro de informações do WebSphere MQ.

Essa propriedade só deve ser usada para a conexão com um gerenciador de filas com várias instâncias. Ela não deve ser usada para conexão com uma lista de gerenciadores de fila diferentes, pois isso pode resultar em problemas de integridade da transação.

Canal

O nome do canal utilizado para conexão com o gerenciador de filas do WebSphere MQ, apenas para conexão de cliente.

Tipo de Dados Seqüência de Caracteres

Tipo de Transporte

Especifica se a conexão do cliente do WebSphere MQ ou as ligações JNDI são usadas para conexão com o gerenciador de filas do WebSphere MQ. O provedor JMS externo controla os protocolos de comunicação entre os clientes e os servidores JMS. Ajuste o tipo de transporte quando estiver utilizando sistema de mensagens não-ASF, não persistente, não durável, não transacional ou quando quiser atender questões de segurança e o cliente for local para o nó do gerenciador de filas.

Tipo de Dados Enum
Unidades Não aplicável
Padrão BINDINGS
Range
BINDINGS
As ligações JNDI são utilizadas para conexão com o gerenciador de filas. BINDINGS é um protocolo de memória compartilhada e pode ser utilizado somente quando o gerenciador de filas estiver no mesmo nó que o cliente JMS e impuser riscos de segurança que devam ser abordados através da utilização de funções EJB.
CLIENT
A conexão do cliente do WebSphere MQ é utilizada para conexão com o gerenciador de filas. CLIENT é um protocolo típico baseado em TCP.
DIRETA
Para WebSphere MQ Event Broker utilizando o modo DIRECT. DIRECT é um protocolo de soquete leve utilizado em sistema de mensagens de Publicação/Assinatura não transacional, não durável e não persistente. DIRECT funciona apenas para clientes e beans orientados a mensagens que utilizam o protocolo não-ASF.
QUEUED
QUEUED é um protocolo TCP padrão.
Recomendado
Tipo de transporte do factory de conexão da filas
BINDINGS é mais rápido 30% ou mais, mas perde na segurança. Quando houver preocupação com a segurança, BINDINGS é mais desejável do que CLIENT.
Tipo de transporte do connection factory de tópico
DIRECT é o tipo mais rápido e deve ser utilizado quando possível. Utilize BINDINGS quando quiser atender tarefas de segurança adicionais e o gerenciador de filas for local em relação ao cliente JMS. QUEUED é o retrocesso a todos os outros casos. WebSphere MQ 5.3 antes do CSD2 com a definição DIRECT pode perder mensagens quando utilizado com beans acionados por mensagens e sob carga. Essa perda também ocorre com aplicativos no lado do cliente, a não ser que o maxClientQueueSize do intermediário seja definido para 0. Ele pode ser definido para 0 com o comando: 
#wempschangeproperties WAS_nodeName_server1
 -e default -o DynamicSubscriptionEngine -n
 maxClientQueueSize -v 0 -x executionGroupUUID
em que executionGroupUUID pode ser localizado iniciando o intermediário e procurando o Registro de Eventos/Aplicativos no evento 2201. Esse valor geralmente é ffffffff-0000-0000-000000000000.
Observação: O JMS do WebSphere MQ 5.3 não pode ser usado no WAS 6.1, pois o WAS 6.1 tem um tempo de execução Java 5. Portanto, as conexões entre memórias não podem ser estabelecidas com os gerenciadores de filas do WebSphere MQ 5.3. Isso pode resultar em uma redução de desempenho se você já estivesse usando o WebSphere MQ 5.3 e as LIGAÇÕES para as conexões e mudar para as conexões de rede do CLIENTE ao migrar para o WebSphere Application Server Versão 6.1. Se estiver usando o WebSphere MQ 5.3 para z/OS®, talvez seja necessário também adquirir um fix pack adicional.

Ao executar no z/OS de 64 bits, o Tipo de Transporte deve ser configurado como CLIENTE, porque o z/OS do WebSphere MQ de 64 bits não está mais disponível e o modo de LIGAÇÕES não pode ser usado para conexão com o z/OS do WebSphere MQ de 31 bits. Talvez seja necessário adquirir também um pacote de recursos adicional do WebSphere MQ para esse suporte.

Conjunto de Criptografia SSL

Especifica o conjunto de criptografia a ser utilizado para conexão SSL com o WebSphere MQ.

Defina essa propriedade para um conjunto de criptografia válido fornecido pelo provedor JSSE. O valor deve corresponder ao CipherSpec especificado no canal SVRCONN como a propriedade Canal.

Você deve configurar essa propriedade se a propriedade Nome do Peer SSL for configurada.

Armazenamento de Certificados SSL

Especifica uma lista com zero ou mais servidores CRL (Certificate Revocation List) utilizados para verificar revogações de certificado SSL. Se você especificar um valor para essa propriedade, deverá utilizar o WebSphere MQ JVM em Java 2 versão 1.4.

O valor é uma lista delimitada por espaços com entradas na forma:
ldap://hostname:[port]

Uma única barra (/) segue esse valor. Se port for omitido, a porta LDAP padrão 389 será utilizada. Durante o tempo de conexão, o certificado SSL apresentado pelo servidor é verificado em comparação com os servidores CRL especificados. Para obter informações adicionais sobre a segurança CRL, consulte as informações de "Trabalhando com Listas de Revogação de Certificado" na seção Segurança do centro de informações do WebSphere MQ.

Nome do Peer SSL

Para SSL, um esqueleto de nome distinto que deve corresponder ao nome fornecido pelo gerenciador de filas do WebSphere MQ. O nome distinto é utilizado para verificar o certificado de identificação apresentado pelo servidor na conexão.

Se essa propriedade não estiver definida, a verificação de certificado será executada.

A propriedade Nome do Peer SSL será ignorada se a propriedade Conjunto de Criptografia SSL não for especificada.

Esta propriedade é uma lista de pares nome e valor do atributo separados por vírgulas ou pontos e vírgulas. Por exemplo: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

O exemplo verifica o certificado de identificação apresentado pelo servidor no tempo de conexão. Para que a conexão seja bem-sucedida, o certificado deve ter um Nome Comum começando com QMGR. e, pelo menos, dois nomes de Unidade Organizacional, sendo o primeiro IBM e o segundo, WEBSPHERE. A verificação não faz distinção entre maiúsculas e minúsculas.

Para obter detalhes adicionais sobre os nomes distintos e seu uso com o WebSphere MQ, consulte a seção "Nomes Distintos" do centro de informações do WebSphere MQ.

Conjunto de Conexões

Especifica um conjunto opcional de definições do pool de conexão.

As propriedades do pool de conexão são comuns a todos os conectores J2C.

O servidor de aplicativos une as conexões e as sessões em conjuntos com o provedor JMS para melhorar o desempenho. Isso não depende de qualquer conjunto de conexões do WebSphere MQ. É necessário configurar as propriedades do conjunto de conexões e sessões corretamente para os aplicativos, caso contrário, é possível não obter o comportamento desejado da conexão e da sessão.

Altere o tamanho do pool de conexão se o acesso simultâneo do lado do servidor ao recurso JMS exceder o valor padrão. O tamanho do pool de conexão é definido por fila ou por tópico.

Tipo de Dados Caixa de Seleção
Padrão Selecionadas
Opções de Reconexão do Cliente

Especifica se uma conexão do modo do cliente se reconectará automaticamente, ou não, na ocorrência de uma falha de comunicação ou do gerenciador de filas. Essa propriedade será ignorada, a menos que o connection factory esteja sendo usado em um ambiente do cliente thin ou gerenciado.

Tipo de Dados Lista drop down
Padrão DISABLED
Range
DISABLED
A reconexão do cliente não ocorrerá automaticamente.
ASDEF
O valor do atributo DefRecon da sub-rotina dos canais do arquivo de configuração do cliente será usado. Se não houver um valor de DefRecon especificado, essa configuração terá o mesmo efeito que um valor igual a DISABLED.
RECONNECT
A reconexão ocorrerá com qualquer gerenciador de filas consistente com o valor do atributo do gerenciador de filas, que pode ser um gerenciador de filas diferente daquele com o qual a conexão estava sendo feita originalmente.
QMGR
A reconexão ocorrerá apenas com o gerenciador de filas com o qual a conexão estava sendo feita originalmente.

Para obter informações adicionais sobre a reconexão automática do cliente, consulte o centro de informações do WebSphere MQ.

Tempo Limite de Reconexão do Cliente

O tempo máximo, em segundos, que uma conexão do modo do cliente passará tentando se reconectar automaticamente com um gerenciador de filas após uma falha de comunicação ou do gerenciador de filas. Esse parâmetro será ignorado, a menos que o connection factory esteja sendo usado em um ambiente do cliente thin ou gerenciado. O uso ou não desse parâmetro depende do valor do parâmetro de opções de reconexão do cliente.

Tipo de Dados Integer
Unidades Segundos
Padrão 1800
Range Um valor maior que zero e até 2147483647

Para obter informações adicionais sobre a reconexão automática do cliente, consulte o centro de informações do WebSphere MQ.
