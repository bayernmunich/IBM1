Configurações do Destino de Tópico do Provedor do Sistema de Mensagens Padrão V5 para Aplicativos Clientes

Use este painel para visualizar ou alterar as propriedades de configuração do destino de tópico selecionado para uso com o provedor Java Message Service (JMS) do produto interno.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores de Sistema de Mensagens > Provedor Padrão da Versão 5. Clique com o botão direito em Destinos de Tópico e clique em Novo. Os seguintes campos aparecem na guia Geral.

Um destino de tópico é utilizado para configurar as propriedades de um tópico JMS do provedor JMS associado. Um tópico do provedor do sistema de mensagens padrão V5 possui as seguintes propriedades.

Nome

O nome através do qual o tópico é conhecido para propósitos administrativos.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição do tópico, para propósitos administrativos, dentro do WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Nome JNDI

O ambiente do tempo de execução do cliente aplicativo utiliza esse campo para recuperar as informações sobre configuração.

Nome do Tópico

O nome do tópico conforme definido pelo provedor JMS.

Tipo de Dado Seqüência de Caracteres
Persistˆncia

Se todas as mensagens enviadas ao destino são persistentes, não-persistentes ou têm a persistência definida pelo aplicativo.

Tipo de Dado Enum
Padrão APPLICATION_DEFINED
Intervalo
Definida pelo aplicativo
As mensagens no destino possuem sua persistência definida pelo aplicativo que as coloca na fila.
Definida pela fila
[Apenas destino do WebSphere MQ] As mensagens no destino têm sua persistência definida pelas propriedades de definição de fila do WebSphere MQ.
Persistente
As mensagens no destino são persistentes.
Não Persistente
As mensagens no destino não são persistentes.
Prioridade

Se a prioridade da mensagem para esse destino for definida pelo aplicativo ou pela propriedade Prioridade Especificada

Tipo de Dado Enum
Padrão APPLICATION_DEFINED
Intervalo
Definida pelo aplicativo
A prioridade de mensagens neste destino é definida pelo aplicativo que as coloca no destino.
Definida pela fila
[Apenas destino do WebSphere MQ] As mensagens no destino têm sua persistência definida pelas propriedades de definição de fila do WebSphere MQ.
Especificado
A prioridade das mensagens nesse destino é definida pela propriedade Prioridade Especificada.Se essa opção for selecionada, você deverá definir uma prioridade na propriedade Prioridade Especificada.
Prioridade Especificada

Se a propriedade Prioridade for definida para Especificada, especifique a prioridade para essa fila no intervalo de 0 (mais baixo) a 9 (mais alto).

Se a propriedade Prioridade for definido como Especificado, as mensagens enviadas para esta fila terão o valor de prioridade especificado por esta propriedade.

Tipo de Dado Inteiro
Unidades Nível de prioridade da mensagem
Intervalo 0 (prioridade mais baixa) até 9 (prioridade mais alta)
Expira‡Æo

Se o tempo limite de expiração desta fila estiver definido pelo aplicativo ou pela propriedade Expiração Especificada, ou mensagens na fila nunca expiram (têm um tempo limite de expiração ilimitado)

Tipo de Dado Enum
Padrão APPLICATION_DEFINED
Intervalo
Definida pelo aplicativo
O tempo limite para expirar as mensagens nesta fila é definido pelo aplicativo que as coloca na fila.
Especificado
O tempo limite de expiração para as mensagens dessa fila é definido pela propriedade Expiração Especificada . Se você selecionar essa opção, você deve definir um tempo limite na propriedade Expiração Especificada.
Ilimitado
As mensagens nesta fila não possuem tempo limite de expiração, portanto elas nunca expiram.
Expiração Especificada

Se a propriedade Tempo Limite de Expiração for definida para Especificado, digite aqui o número de milissegundos (maior do que 0) depois do qual as mensagens da fila expiram.

Tipo de Dado Inteiro
Unidades Milissegundos
Intervalo Maior que ou igual a 0
  • 0 indica que as mensagens nunca têm o tempo limite excedido.
  • Outros valores são um número inteiro de milissegundos.
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
