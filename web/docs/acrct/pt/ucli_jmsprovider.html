Provedores Java Message Service para Clientes

Os aplicativos clientes podem usar recursos de sistema de mensagens de três tipos principais de provedores Java Message Service (JMS) no WebSphere Application Server: o provedor de sistema de mensagens padrão WebSphere Application Server (que usar a integração de serviços como provedor), o provedor de sistema de mensagens WebSphere MQ (que usa o sistema do WebSphere MQ como provedor) e os provedores de sistema de mensagens de terceiros (que usam outro produto da empresa como provedor).

O IBM WebSphere Application Server suporta o sistema de mensagens assíncronas através do uso de um provedor JMS e seu sistema de mensagens relacionado. Os provedores JMS devem estar de acordo com a especificação JMS versão 1.1. Para utilizar beans orientados a mensagens, o provedor JMS deve suportar a função opcional ASF (Application Server Facility) definida nessa especificação ou suportar um adaptador de recursos de entrada, conforme definido na especificação JCA versão 1.5.
Provedor de sistemas de mensagens padrão
Se você desejar usar principalmente o sistema de mensagens entre aplicativos no WebSphere Application Server, talvez com alguma interação com um sistema WebSphere MQ, o provedor de sistema de mensagens padrão será a opção natural. Esse provedor é baseado em tecnologias de integração de serviços e está totalmente integrado ao ambiente de tempo de execução do WebSphere Application Server.
Fornecedor de sistema de mensagens WebSphere MQ
Se os seus negócios também usarem o WebSphere MQ e você desejar integrar os aplicativos de sistema de mensagens do WebSphere Application Server em uma rede do WebSphere MQ, predominantemente, escolha o provedor de sistema de mensagens do WebSphere MQ, que permite definir recursos para conexão com qualquer gerenciador de filas na rede do WebSphere MQ.
Provedor de Sistemas de Mensagens de Terceiros
É possível configurar qualquer provedor de sistemas de mensagens de terceiros que suporte a connection factory unificada do JMS Versão 1.1. É possível desejar fazer isso, por exemplo, devido aos investimentos existentes.
Recurso Reprovado: No WebSphere Application Server 7.0, o provedor de sistema de mensagens padrão na Versão 5 é reprovado. depfeat
Nota: Para compatibilidade com versões anteriores, o WebSphere Application Server Versão 7 suporta a Versão 5 do provedor de sistema de mensagens padrão (reprovada) e a Versão 6 do provedor de sistema de mensagens do WebSphere MQ. Esse suporte permite que os aplicativos que ainda utilizam esses recursos se comuniquem com os nós das Versões 5 e 6 nas células mistas da Versão 7.

Os aplicativos do WebSphere podem usar recursos do sistema de mensagens fornecidos por qualquer um desses provedores JMS. No entanto, a opção de provedor geralmente é mais imposta pelos requisitos para utilização ou integração com um sistema de mensagens existente. Por exemplo, você já pode ter uma infraestrutura do sistema de mensagens baseada no WebSphere MQ. Neste caso você pode se conectar diretamente usando o suporte incluído para o WebSphere MQ como um provedor JMS ou configurar um barramento de integração de serviços com links com uma rede do WebSphere MQ e, em seguida, acessar o barramento através do provedor de sistema de mensagens padrão.
