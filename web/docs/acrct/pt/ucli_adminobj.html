Definições de Objetos Administrados

Utilize esse painel para visualizar ou alterar as propriedades de configuração dos objetos administrados selecionados.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Adaptadores de Recursos > resource_adapter_instance. Clique com o botão direito do mouse em Objetos Administrados e clique em Novo. Os seguintes campos aparecem na guia Geral.

As definições para os objetos administrados são manipuladas de forma semelhante às connection factories. Ao atualizar os objetos administrados, utilize os mesmos painéis utilizados para criar objetos administrados.

Nome

O nome pelo qual este objeto administrado é conhecido para propósitos administrativos dentro do IBM WebSphere Application Server. O nome deve ser exclusivo nos objetos administrados do adaptador de recursos em todo o domínio administrativo do produto.

Descrição

Uma descrição opcional deste connection factory para propósitos administrativos dentro do IBM WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Nome JNDI

Essa entrada é um nome resource-env-ref, um nome message-destination-ref (se message-destination-ref não tiver nenhum link) ou um link message-destination.

Tipo de Dado Seqüência de Caracteres
Tipo

Uma lista drop-down de todos os pares classe-interface de objetos administrados conforme definido para os objetos admin no arquivo RAR (Resource Adapter Archive).

Para cada Tipo, há um conjunto de propriedades especificado na caixa Propriedades. Esse conjunto de propriedades é construído pela recuperação das propriedades de cada definição de objeto administrado. Para os objetos administrados existentes exibidos para atualização, essa lista de propriedades é sobreposta com as propriedades especificadas para os objetos. Quando o campo Tipo for alterado, as propriedades também são alteradas para refletir as propriedades corretas para o tipo.
