Configurações do Destino do JMS Genérico para Clientes Aplicativos

Utilize esse painel para exibir ou alterar as propriedades de configuração do destino JMS selecionado para uso com o provedor JMS associado.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores de Sistema de Mensagens > new JMS Provider instance. Clique com o botão direito do mouse em Destinos e clique em Novo. Os campos a seguir são exibidos na guia Geral.

Um destino JMS é utilizado para configurar as propriedades de um destino JMS do provedor JMS genérico associado. As conexões do destino JMS são criadas pela connection factory JMS associada. Um destino JMS para uso com um provedor JMS genérico (não o provedor de sistema de mensagens padrão ou WebSphere MQ como provedor JMS) tem as seguintes propriedades.

Nome

O nome pelo qual a fila é conhecida para propósitos administrativos dentro do WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição da fila, para propósitos administrativos.

Nome JNDI

O nome de JNDI do nome real (físico) do destino JMS ligado ao JNDI.

Nome de JNDI Externo

O nome de JNDI utilizado para ligar a fila no espaço de nomes do servidor de aplicativos.

Como uma convenção, utilize o nome do JNDI completo; por exemplo, no formato jms/Nome, onde Nome é o nome lógico do recurso.

Este nome é utilizado para vincular as informações de ligação da plataforma. A ligação associa os recursos definidos pelo descritor de implementação do módulo aos recursos reais (físicos) vinculados ao JNDI pela plataforma.

Tipo de Dado Seqüência de Caracteres
Tipo de Destino

Se esse destino JMS é uma fila (para ponto-a-ponto) ou um tópico (para publicação ou assinatura).

Selecione uma das seguintes opções:
Fila
Um destino de fila JMS para mensagens ponto a ponto.
Um destino de tópico JMS para mensagem pub/sub.
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
