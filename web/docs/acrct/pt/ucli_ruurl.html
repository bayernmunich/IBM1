Definições de URL para Aplicativos Clientes

Utilize esta página para implementar a função de um protocolo de URL específico, como um HTTP (Hyper Text Transfer Protocol).

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Provedores de URL > URL provider instance. Clique com o botão direito em URLs e clique em Novo. Os seguintes campos aparecem na guia Geral.

Este provedor, composto de classes, amplia as classes java.net.URLStreamHandler e java.net.URLConnection.

Nome

O nome administrativo da URL.

Descrição

Esta é uma descrição opcional da URL para seus registros administrativos.

Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

URL

Um nome de URL (Uniform Resource Locator) que aponta para um recurso de Internet ou intranet. Por exemplo: http://www.ibm.com.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.



Links marcados (on-line) requerem acesso à Internet.
