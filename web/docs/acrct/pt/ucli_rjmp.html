Definições de Provedores de Correio para Aplicativos Clientes

Utilize esta página para implementar a API JavaMail e criar sessões de correio.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado. Clique com o botão direito do mouse em Provedores de Correio > e clique em Novo. Os campos a seguir aparecem na guia Geral:

Nome

O nome do provedor de recursos JavaMail.

Descrição

Uma descrição opcional para o provedor de recursos.

Caminho da Classe

Especifica uma lista de caminhos ou nomes de arquivos JAR que, juntos, formam a localização das classes do provedor de recursos.

Protocolo

Especifica o nome do protocolo.

Nome da Classe

Especifica o nome da classe que implementa o protocolo. Deixe este campo em branco se desejar utilizar a implementação padrão.

Tipo

Este menu contém os dois valores a seguir: TRANSPORT ou STORE.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
