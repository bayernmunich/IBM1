Configurações do Factory de Conexão da Fila do Provedor do Sistema de Mensagens Padrão V5 para Aplicativos Clientes

Use este painel para procurar ou alterar as propriedades de configuração do factory de conexão da fila JMS selecionado para sistema de mensagens ponto a ponto para uso por aplicativos do WebSphere Application Server Versão 5. Estas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o sistema de mensagens padrão que ele utiliza.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedor de Sistema de Mensagens > Provedor Padrão da Versão 5. Clique com o botão direito do mouse em Connection Factories de Filas e clique em Novo. Os seguintes campos aparecem na guia Geral.

Uma connection factory de fila é utilizada para criar conexões JMS para destinos de fila. O factory de conexão da fila é criado pelo provedor JMS do produto WebSphere Application Server interno. Um factory de conexão da fila do provedor do sistema de mensagens padrão V5 possui as seguintes propriedades:

Nome

O nome pelo qual este factory de conexão da fila é conhecido para propósitos administrativos dentro do IBM WebSphere Application Server. O nome deve ser exclusivo dentro dos connection factories JMS pelo domínio administrativo do WebSphere.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição dessa connection factory para fins administrativos no IBM WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Padrão Nulo
Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

ID do usuário

O ID do Usuário utilizado, com a propriedade Senha, para autenticação se o aplicativo de chamada não fornecer um ID do Usuário e Senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

As propriedades ID do Usuário e Senha da connection factory são utilizadas se o aplicativo de chamada não fornecer um ID de Usuário e uma senha explicitamente, por exemplo, se o aplicativo de chamada utilizar o método createQueueConnection(). O cliente JMS encaminha o ID do Usuário e a Senha ao servidor JMS.

Tipo de Dado Seqüência de Caracteres
Senha

A senha utilizada com a propriedade ID do Usuário para autenticação, se o aplicativo de chamado não fornecer um ID do Usuário e uma Senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

Digitar Senha Novamente

Confirma a senha.

Nó

O nome do nó administrativo do WebSphere onde o servidor JMS é executado para este connection factory. As conexões criadas por esse depósito de informações conectam-se a esse servidor JMS.

Tipo de Dado Seqüência de Caracteres
Application Server

Digite o nome do servidor de aplicativos. Este nome não é o nome do host da máquina, mas o nome do servidor de aplicativos configurado.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
