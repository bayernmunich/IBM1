Configurações de Sessão de Correio para Clientes Aplicativos

Utilize esta página para configurar as propriedades da sessão de correio.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Provedores de Correio > mail provider instance. Clique com o botão direito em Sessões de Correio e clique em Novo. Os campos a seguir aparecem na guia Geral:

Nome

Representa o nome administrativo do objeto de sessão JavaMail.

Descrição

Fornece uma descrição opcional para os registros administrativos.

Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

Host de Transporte de Correio

Especifica o servidor para conexão ao enviar correio.

Protocolo de Transporte de Correio

Especifica o protocolo de transporte a ser utilizado no envio de correio.

Usuário de Transporte de Correio

Especifica o ID do usuário a ser utilizado quanto o host de transporte de correio requer autenticação.

Senha do Transporte do Correio

Especifica a senha a ser utilizada quando o host de transporte de correio exigir autenticação.

Ativar Análise Rígida do Endereço da Internet

Especifica se os endereços de destinatário devem ser analisados de forma rígida em conformidade com RFC 822, que é um documento de especificações emitido pelo Internet Architecture Board.

Essa definição não é geralmente utilizada para a maioria dos aplicativos de correio. A sintaxe de RFC 822 para análise efetiva dos endereços utiliza uma definição rígida de um endereço de e-mail válido. Se você selecionar essa definição, JavaMail irá aderir à sintaxe de RFC 822 e rejeitará endereços de destinatários que não sejam analisados em endereços de e-mail válidos (conforme definido pela especificação). Se você não selecionar essa definição, o JavaMail não irá aderir à sintaxe de RFC 822 e aceitará os endereços de destinatários que não estejam de acordo com a especificação. Por padrão, essa definição não está selecionada. É possível visualizar a especificação de RFC 822 no seguinte URL do W3C (World Wide Web Consortium): http://www.w3.org/Protocols/rfc822/.

Correio De

Especifica o originador do correio.

Host de Depósito de Correio

Especifica o nome do host da conta de correio (ou "domínio").

Usuário de Depósito de Correio

Especifica o ID do usuário da conta de correio.

Senha de Armazenamento do Correio:

Especifica a senha da conta de correio.

Digitar Senha Novamente

Confirma a senha.

Protocolo de Depósito de Correio

Especifica o protocolo a ser utilizado ao receber correio.

Depuração de Correio

Quando true, a interação do JavaMail com os servidores de correio juntamente com essas propriedades da sessão de correio são impressas no arquivo stdout.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
