Definições de Connection Factory de Provedor Padrão

Use este painel para visualizar ou alterar as propriedades de configuração do connection factory JMS selecionado para uso com o provedor Java Message Service (JMS) do produto interno que está instalado com o WebSphere Application Server. Estas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o barramento de integração de serviço que ele utiliza

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor Padrão. Clique com o botão direito do mouse em Connection Factories e clique em Novo. Os seguintes campos aparecem na guia Geral.

As definições que possuem um valor padrão exibem o valor apropriado. As definições com valores fixos têm um menu drop-down.

ID do Usuário

O Nome do Usuário utilizado com a propriedade Senha para conexão com um aplicativo.

Se você especificar um valor para a propriedade Nome do Usuário, é necessário também especificar um valor para a propriedade Senha.

As propriedades ID do Usuário e Senha da connection factory são utilizadas se o aplicativo de chamada não fornecer ID do usuário e senha explicitamente. Se um nome de usuário e uma senha forem especificados, então, um alias de autenticação é criado para a fábrica onde a senha é criptografada.

Confiabilidade do Sistema de Mensagens Não-persistentes

A confiança aplicada a mensagens JMS não-persistentes enviadas utilizando esta connection factory.

Se desejar diferentes opções de entrega de confiança para destinos JMS individuais, será possível definir esta propriedade como Conforme Destino de Barramento. A confiança é então definida pela propriedade Confiança do destino do barramento ao qual o destino JMS está designado.

Compartilhar Assinaturas Duráveis

Controla se assinaturas duráveis serão ou não compartilhadas em conexões com membros de um cluster de servidores.

Normalmente, apenas uma sessão por vez pode ter TopicSubscriber para uma determinada assinatura durável. Esta propriedade permite substituir este comportamento, para permitir que uma assinatura durável tenha vários consumidores simultâneos.

Nós de Extremidade do Fornecedor

A lista separada por vírgulas de nós de extremidade utilizados para conexão com um servidor de auto-inicialização.

Digite uma lista separada por vírgulas de grupos de três nós de extremidade com a sintaxe: host:port:protocol.

Exemplo: 
merlin:7276:BootstrapBasicMessaging,Gandalf:
5557:BootstrapSecureMessaging
em que

BootstrapBasicMessaging corresponde ao protocolo remoto InboundBasicMessaging (JFAP-TCP/IP).
Padrão
  • Se o nome do host não for especificado, o padrão localhost será utilizado como um valor padrão.
  • Se o número da porta não for especificado, 7276 será utilizado como um valor padrão.
  • Se o nome da cadeia não for especificado, uma cadeia predefinida, como BootstrapBasicMessaging, será utilizada como um valor padrão.
Proximidade de Conexão
