Ferramenta de Configuração de Recursos do Aplicativo Cliente

Navegue para os tópicos de ajuda dentro desta seção.

  • Adaptadores de Recursos para o Cliente
    Um adaptador de recursos é um driver de software no nível do sistema que um aplicativo Java utiliza para conectar a um EIS (sistema de informações corporativas). Um adaptador de recursos é conectado a um cliente aplicativo e fornece conectividade entre o EIS e o aplicativo corporativo.
  • Provedores do Ambiente de Recursos e Entradas do Ambiente de Recursos
    Uma referência de ambiente de recurso mapeia um nome lógico, utilizado pelo aplicativo cliente, para o nome físico de um objeto.
  • Provedores Java Message Service para Clientes
    Os aplicativos clientes podem usar recursos de sistema de mensagens de três tipos principais de provedores Java Message Service (JMS) no WebSphere Application Server: o provedor de sistema de mensagens padrão WebSphere Application Server (que usar a integração de serviços como provedor), o provedor de sistema de mensagens WebSphere MQ (que usa o sistema do WebSphere MQ como provedor) e os provedores de sistema de mensagens de terceiros (que usam outro produto da empresa como provedor).
  • Sistema de Mensagens Assíncronas no WebSphere Application Server Usando JMS
    O WebSphere Application Server suporta sistema de mensagens assíncronas como método de comunicação baseado na interface de programação JMS (Java Message Service). A interface JMS fornece uma maneira comum para que os programas Java (clientes e Java Platform, Enterprise Edition (aplicativos Java EE)) criem, enviem, recebam e leiam pedidos assíncronos como mensagens JMS.

  • Provedores de URL para a Ferramenta de Configuração de Recursos do Cliente Aplicativo
    Um provedor de URL implementa a função para um determinado protocolo de URL, como HTTP (HyperText Transfer Protocol). Este provedor, composto de um par de classes, estende as classes java.net.URLStreamHandler e java.net.URLConnection.
  • URLs para Aplicativos Clientes
    Uma URL (Uniform Resource Locator) é um identificador que aponta para um recurso acessível eletronicamente, como um arquivo de diretório em uma máquina na rede, ou um documento armazenado em um banco de dados.

  • Configurando Recursos do Cliente do Sistema de Mensagens Java
    Para configurar os recursos do cliente do sistema de mensagens Java, crie novas configurações do provedor JMS para o aplicativo cliente. O aplicativo cliente pode usar um serviço de sistema de mensagens através das APIs do Java Message Service. Um provedor de JMS oferece dois tipos de fábricas de J2EE. Um é fábrica de conexão de JMS e o outro é fábrica de destino de JMS.
