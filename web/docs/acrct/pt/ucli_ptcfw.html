Configurações do Connection Factory de Tópico do Provedor do Sistema de Mensagens Padrão V5 para Aplicativos Clientes

Use este painel para visualizar ou alterar as propriedades de configuração do connection factory de tópico selecionado para uso com o provedor Java Message Service (JMS) do produto interno. Estas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o sistema de mensagens que ele utiliza.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores de Sistema de Mensagens > Provedor Padrão da Versão 5. Clique com o botão direito do mouse em Connection Factories de Tópicos e clique em Novo. Os seguintes campos aparecem na guia Geral.

Um connection factory de tópico do provedor do sistema de mensagens padrão V5 possui as seguintes propriedades.

Nome

O nome pelo qual o factory de conexão da fila é conhecido para propósitos administrativos dentro do WebSphere Application Server. O nome deve ser exclusivo dentro dos connection factories JMS pelo domínio administrativo do WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição deste connection factory de tópico para propósitos administrativos dentro do WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

ID do usuário

O ID do usuário utilizado, com a propriedade Senha, para autenticação se o aplicativo de chamada não fornecer um ID de usuário e senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

As propriedades ID do Usuário e Senha da connection factory são utilizadas se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente, por exemplo, se o aplicativo de chamada utilizar o método createTopicConnection(). O cliente JMS encaminha o ID do Usuário e a Senha ao servidor JMS.

Tipo de Dado Seqüência de Caracteres
Senha

A senha utilizada com a propriedade ID do Usuário para autenticação, se o aplicativo de chamado não fornecer um ID do Usuário e uma Senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

Tipo de Dado Seqüência de Caracteres
Digitar Senha Novamente

Confirma a senha.

Nó

O nome do nó administrativo do WebSphere Application Server onde o servidor JMS é executado para este connection factory. As conexões criadas por essa fábrica conectam-se a esse servidor JMS.

Tipo de Dado Enum
Intervalo A lista suspensa de nós no domínio administrativo do WebSphere Application Server.
Application Server

Digite o nome do servidor de aplicativos. Este nome não é o nome do host da máquina, mas o nome do servidor de aplicativos configurado.

Port

Qual das duas portas as conexões utilizam para conexão ao Servidor JMS. A porta QUEUED serve para suporte à publicação/assinatura JMS de função completa, a porta DIRECT serve apenas para assinaturas não-persistentes, não-transacionais e não-duráveis.

Nota: Os beans orientados a mensagens não podem utilizar a porta listener direta para suporte à publicação ou assinatura. Portanto, qualquer connection factory de tópicos configurada com a Porta definida para Direta não pode ser utilizada com os beans orientados por mensagens.
Tipo de Dado Enum
Padrão QUEUED
Intervalo
QUEUED
A porta listener utilizada para suporte à publicação ou assinatura compatível com JMS de função completa.
DIRECT
A porta listener utilizada para conexão TCP/IP direta (apenas assinaturas não-transacionais, não-persistentes e não-duráveis) para suporte à publicação ou assinatura.

Os números da porta TCP/IP dessas portas são definidos no servidor JMS interno do produto.
ID do Cliente

O identificador do cliente JMS utilizado para conexões com o gerenciador de filas do MQSeries.

Tipo de Dado Seqüência de Caracteres
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
