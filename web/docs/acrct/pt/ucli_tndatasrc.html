Configurando Novos Provedores de Origem de Dados (provedores JDBC) para Aplicativos Clientes

É possível criar novos fornecedores de origem de dados, também conhecidos como provedores JDBC, para seu cliente aplicativo utilizando a ACRCT (Application Client Resource Configuration Tool).

Antes de Começar

Durante esta tarefa, você criará novos provedores de origens de dados, também conhecidos como provedores JDBC, para seu aplicativo cliente. Em uma tarefa administrativa separada, instale o código Java para o provedor de origem de dados necessário no computador cliente no qual o aplicativo cliente reside.

Sobre Esta Tarefa

Utilize esta tarefa para conectar clientes aplicativos a bancos de dados relacionais.

Procedimento

  1. Inicie a ACRCT (Application Client Resource Configuration Tool) e abra o arquivo EAR para o qual deseja configurar o novo fornecedor de origem de dados. O conteúdo do arquivo EAR é exibido em uma visualização em árvore.
  2. Selecione o arquivo JAR no qual deseja configurar o novo provedor de origem de dados a partir da árvore.
  3. Expanda o arquivo JAR para exibir seu conteúdo.
  4. Clique na pasta Provedores de Origem de Dados. Execute um dos seguintes procedimentos:
    • Clique com o botão direito na pasta e clique em Novo Provedor.
    • Clique em Editar > Novo na barra de menus.
  5. Configure as propriedades do provedor de origem de dados no diálogo de propriedades resultante.
  6. Clique em OK quando você terminar.
  7. Clique em Arquivo > Salvar na barra de menus para salvar suas alterações.


Links marcados (on-line) requerem acesso à Internet.
