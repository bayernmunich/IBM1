Configurações do Provedor de URL para Clientes Aplicativos

Utilize esta página para criar novos provedores de URLs.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado. Clique com o botão direito do mouse em Provedores de URL e clique em Novo. Os seguintes campos aparecem na guia Geral.

Um provedor URL implementa a função para um determinado protocolo de URL, como HTTP (Hyper Text Transfer Protocol). Este provedor, composto de classes, amplia as classes java.net.URLStreamHandler e java.net.URLConnection.

Nome

Nome administrativo para a URL.

Descrição

Descrição opcional da URL, para os registros administrativos.

Caminho da Classe

Uma lista de caminhos de nomes de arquivos JAR que, juntos, formam a localização das classes de provedores de recursos.

Protocolo

Protocolo suportado por esta rotina de tratamento de fluxos. Por exemplo, nntp, smtp, ftp e outros.

Para utilizar o protocolo padrão, deixe esse campo em branco.

Classe da Rotina de Tratamento de Fluxos

Nome completo de uma classe Java definida pelo usuário que estende o java.net.URLStreamHandler para um protocolo URL específico, como FTP.

Para utilizar a rotina de tratamento de fluxo padrão, deixe esse campo em branco.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
