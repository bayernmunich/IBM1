Propriedades da Origem de Dados para Clientes Aplicativos

Utilize esta página para criar ou modificar as origens de dados.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores de Origens de Dados > Instância do fornecedor de origem de dados. Clique com o botão direito em Origens de Dados e clique em Novo. Os campos a seguir são exibidos na guia Geral:

Nome

Especifica o nome de exibição dessa origem de dados.

Tipo de D]ados Seqüência de Caracteres
Descrição

Especifica uma descrição de texto da origem de dados.

Tipo de D]ados Seqüência de Caracteres
Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

Nome do Banco de Dados

O nome do banco de dados ao qual você deseja se conectar.

Usuário

Utilize o ID do usuário com a propriedade Senha para autenticação se o aplicativo de chamada não fornecer um ID do usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, você também deverá especificar um valor para a propriedade Senha. As propriedades ID do Usuário e Senha da connection factory são utilizadas se o aplicativo que está sendo chamado não fornecer um ID de usuário e uma senha explicitamente.

Senha

Utilize a senha com a propriedade ID do Usuário para autenticação, se o aplicativo de chamada não fornecer um ID do usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade Senha, também deverá especificar um valor para a propriedade ID do Usuário.

Digitar Senha Novamente

Confirma a senha.

Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.



Links marcados (on-line) requerem acesso à Internet.
