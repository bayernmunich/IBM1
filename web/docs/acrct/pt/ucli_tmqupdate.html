Atualizando o WebSphere MQ Como um Provedor Java Message Service, e suas Configurações de Recursos JMS para Aplicativos Clientes

É possível atualizar uma configuração existente do WebSphere MQ como um provedor Java Message Service (JMS), e atualizar a configuração dos connection factories do WebSphere MQ ou dos destinos do WebSphere MQ.

Sobre Esta Tarefa

Use essa tarefa para atualizar uma configuração existente do WebSphere MQ como um provedor Java Message Service (JMS), e atualizar a configuração dos connection factories do WebSphere MQ ou dos destinos do WebSphere MQ.

Procedimento

  1. Inicie a ACRTCT (Application Client Resource Configuration Tool).
  2. Abra o arquivo Enterprise Archive (EAR) que contém o provedor JMS do WebSphere MQ, o connection factory do WebSphere MQ ou o destino do WebSphere MQ. O conteúdo do arquivo EAR é exibido na visualização da árvore de navegação.
  3. Selecione o arquivo Java Archive (JAR) que contém o provedor JMS, o connection factory ou destino a ser atualizado.
  4. Expanda o arquivo JAR para visualizar seu conteúdo até localizar o fornecedor JMS específico, a connection factory ou o destino que deseja atualizar. Conclua uma das ações a seguir:
    • Clique com o botão direito no objeto apropriado e clique em Propriedades.
    • Clique em Editar > Propriedades na barra de menus.
  5. Atualize as propriedades nos campos exibidos. Para obter ajuda de campo detalhada, consulte:
    • Propriedades do provedor JMS
    • Propriedades da fábrica de conexões de Fila MQ
    • Propriedades da fábrica de conexões de Tópico MQ
    • Propriedades do destino de Fila MQ
    • Propriedades do destino de Tópico MQ
  6. Clique em OK.
  7. Clique em Arquivo > Salvar para salvar suas alterações.


Links marcados (on-line) requerem acesso à Internet.
