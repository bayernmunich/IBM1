Configurações de Destino de Tópico do Provedor Padrão

Use este painel para visualizar ou alterar as propriedades de configuração do destino de tópico do JMS selecionado para uso com o provedor Java Message Service (JMS) do produto interno que está instalado com o WebSphere Application Server.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor Padrão. Clique com o botão direito do mouse em Destinos de Tópico e clique em Novo. Os seguintes campos aparecem na guia Geral.

Nome

O nome da entrada do destino de tópico.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição da entrada.

Tipo de Dado Seqüência de Caracteres
Nome JNDI

O nome de JNDI utilizado para corresponder essa definição a um nome resource-env-ref do descritor de implementação.

Tipo de Dado Seqüência de Caracteres
Espaço do Tópico

O nome do espaço do tópico. Este campo é obrigatório.

Tipo de Dado Seqüência de Caracteres
Padrão DEFAULT_TOPIC_SPACE
Nome do Tópico

O nome do tópico. Este campo é obrigatório.

Tipo de Dado Seqüência de Caracteres
Modo de Entrega

O modo padrão para mensagens enviadas para o destino.

Tipo de Dado Seqüência de Caracteres
Intervalo Aplicativo, Persistente ou Não-persistente
Padrão Aplicativo
Tempo de Vida

A duração de tempo padrão a partir de sua hora de dispatch pela qual uma mensagem enviada a esse destino deve ser retida pelo sistema, sendo que 0 indica que o valor do tempo de vida não expira. O valor do produtor é utilizado se não for preenchido.

Tipo de Dado Número Longo
Unidades Milissegundos
Prioridade

A prioridade para as mensagens enviadas a esse destino. O valor do produtor é utilizado se não for preenchido.

Tipo de Dado Inteiro
Intervalo 0 a 9, sendo 0 a prioridade mais baixa e 9 a prioridade mais alta
Ler Adiante

Utilizada para controlar a otimização de leitura adiante durante a entrega das mensagens.

Tipo de Dado Seqüência de Caracteres
Intervalo Como Conexão, Sempre Ativo e Sempre Inativo
Padrão Como Conexão
