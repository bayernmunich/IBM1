Definições do Adaptador de Recursos

Utilize esse painel para visualizar ou alterar as propriedades de configuração do adaptador de recursos. Essas propriedades de configuração controlam como os adaptadores de recursos são criados.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Adaptador de Recursos. Clique com o botão direito do mouse em Adaptador de Recursos e clique em Novo. Os seguintes campos aparecem na guia Geral.

Nome

Nome pelo que esse Adaptador de Recursos é conhecido para propósitos administrativos no IBM WebSphere Application Server. O nome deve ser exclusivo nos Adaptadores de Recursos no domínio administrativo do produto.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição desse adaptador de recursos para propósitos administrativos dentro do IBM WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Caminho da Classe

Qualquer caminho de classe adicional. O caminho para o diretório do adaptador de recursos é incluído automaticamente.

Tipo de Dado Seqüência de Caracteres
Padrão O caminho para o diretório do Adaptador de Recursos.
Caminho Nativo

O caminho nativo onde o Adaptador de Recursos está localizado. Digite qualquer caminho de classe nativo adicional aqui.

Tipo de Dado Seqüência de Caracteres
Nome do Adaptador de Recursos

O campo obrigatório aponta para um subdiretório do adaptador de recursos. A entrada não representa o nome completo do diretório para o adaptador de recursos. O nome completo do diretório é o caminho do adaptador de recursos instalado, mais o nome do adaptador de recursos.

Tipo de Dado Seqüência de Caracteres
Caminho do Adaptador de Recursos Instalado

O diretório onde os adaptadores de recursos são instalados. Se você não preencher esse campo, então, o padrão será utilizado.

Se você especificar o valor, ${CONNECTOR_INSTALL_ROOT}, esse valor substituirá o valor da variável CLIENT_CONNECTOR_INSTALL_ROOT na máquina na qual o aplicativo cliente é executado. Essa ação permite que o aplicativo seja executado facilmente em diferentes máquinas, sendo que a instalação do cliente poderá ser em diferentes locais.

Tipo de Dado Seqüência de Caracteres
Padrão ${CONNECTOR_INSTALL_ROOT}
