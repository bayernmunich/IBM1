URLs para Aplicativos Clientes

Uma URL (Uniform Resource Locator) é um identificador que aponta para um recurso acessível eletronicamente, como um arquivo de diretório em uma máquina na rede, ou um documento armazenado em um banco de dados.

As URLs aparecem no formato esquema:scheme_information.

Você pode representar um esquema como http, ftp, arquivo, ou outro termo que identifica o tipo de recurso e o mecanismo pelo qual o recurso pode ser acessado.

Em uma caixa de endereço ou localização do navegador da World Wide Web, um URL de um arquivo disponível utilizando HTTP (HyperText Transfer Protocol) começa com http:. Um exemplo é http://www.ibm.com. Os arquivos disponíveis utilizando o FTP (File Transfer Protocol) começam com ftp:. Os arquivos disponíveis localmente começam com file:.

scheme_information geralmente identifica a máquina da Internet que está disponibilizando um recurso, o caminho para este recurso e o nome do recurso. As scheme_information para HTTP, FTP e Arquivo são iniciadas geralmente com duas barras (//), em seguida, fornece o endereço de Internet separado do nome do caminho do recurso com uma barra (/). Por exemplo,

http://www.ibm.com/software/webservers/appserv/library.html.

Para HTTP e FTP, o nome do caminho termina com uma barra inclinada quando o URL aponta para um diretório. Nestes casos, o servidor geralmente retorna o índice padrão para o diretório.



Links marcados (on-line) requerem acesso à Internet.
