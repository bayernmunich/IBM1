Definições de Connection Factory de Fila de Provedor Padrão

Use este painel para visualizar ou alterar as propriedades de configuração do factory de conexão da fila JMS selecionado para uso com o provedor Java Message Service (JMS) do produto interno que está instalado com o WebSphere Application Server. Estas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o barramento de integração de serviço que ele utiliza

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor Padrão. Clique com o botão direito do mouse em Connection Factories de Filas e clique em Novo. Os seguintes campos aparecem na guia Geral.

As definições que possuem um valor padrão exibem o valor apropriado. As definições com valores fixos têm um menu drop-down.

ID do Usuário

O Nome do Usuário utilizado com a propriedade Senha para autenticação, se o aplicativo de chamada não fornecer um ID do usuário e senha explicitamente. Se esse campo for utilizado, então, o UserName do campo Propriedades é ignorado.

Se você especificar um valor para a propriedade Nome do Usuário, é necessário também especificar um valor para a propriedade Senha.

As propriedades Nome do Usuário e Senha da connection factory serão utilizadas se o aplicativo de chamada não fornecer um ID do usuário e senha explicitamente. Se um nome de usuário e uma senha forem especificados, então, um alias de autenticação é criado para a fábrica onde a senha é criptografada.

Senha

A senha utilizada para criar uma criptografia. Se esse campo for preenchido, então, o campo Senha da caixa Propriedades será ignorado.

Se você especificar um valor para a propriedade Nome do Usuário, é necessário também especificar um valor para a propriedade Senha.

Confiabilidade do Sistema de Mensagens Não-persistentes

A confiança aplicada a mensagens JMS não-persistentes enviadas utilizando esta connection factory.

Se desejar diferentes opções de entrega de confiança para destinos JMS individuais, será possível definir esta propriedade como Conforme Destino de Barramento. A confiança é então definida pela propriedade Confiança do destino do barramento ao qual o destino JMS está designado.

Nós de Extremidade do Fornecedor

A lista separada por vírgulas de nós de extremidade utilizados para conexão com um servidor de auto-inicialização.

Digite uma lista separada por vírgulas de grupos de três nós de extremidade com a sintaxe: host:port:protocol.

Exemplo: 
localhost:7777:BootstrapBasicMessaging
em que

BootstrapBasicMessaging corresponde ao protocolo remoto InboundBasicMessaging (JFAP-TCP/IP).
Padrão
  • Se o nome do host não for especificado, o padrão localhost será utilizado como um valor padrão.
  • Se o número da porta não for especificado, 7276 será utilizado como um valor padrão.
  • Se o nome da cadeia não for especificado, uma cadeia predefinida, como BootstrapBasicMessaging, será utilizada como um valor padrão.
Proximidade de Conexão

A proximidade que o mecanismo do sistema de mensagens deve ter com o solicitante.
