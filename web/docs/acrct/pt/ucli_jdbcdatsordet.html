Definições do Provedor de Origem de Dados para Aplicativos Clientes

Utilize esta página para criar uma origem de dados sob um provedor JDBC que fornece a classe de implementação específica do driver JDBC.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado. Clique com o botão direito em Fornecedores de origens de dados > e clique em Novo. Os campos a seguir aparecem na guia Geral:

Nome

Especifica o nome de exibição para a origem de dados.

Por exemplo, você pode definir este campo para Testar Origem de Dados.

Tipo de D]ados Seqüência de Caracteres
Descrição

Especifica uma descrição de texto do recurso.

Tipo de D]ados Seqüência de Caracteres
Caminho da Classe

Uma lista de caminhos ou nomes de arquivos .jar que juntos formam o local para as classes do provedor de recursos.

Classe da Implementação

Utilize esta definição para executar funções específicas do banco de dados.

Tipo de D]ados Seqüência de Caracteres
Padrão Dependente da classe de implementação do driver JDBC
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
