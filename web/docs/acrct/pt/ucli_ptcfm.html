Configurações do Connection Factory de de Tópico do Provedor WebSphere MQ para Aplicativos Clientes

Use este painel para visualizar ou alterar as propriedades de configuração do factory de conexão de tópico selecionado para uso com o provedor WebSphere® MQ do produto Java™ Message Service (JMS). Essas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o WebSphere MQ.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor WebSphere MQ. Clique com o botão direito do mouse em Connection Factories de Tópicos e clique em Novo.

Observação:
  • Os valores de propriedade especificados devem corresponder aos valores especificados ao configurar os recursos JMS para o WebSphere MQ. Para obter informações adicionais sobre a configuração dos recursos JMS para o WebSphere MQ, consulte a seção Uso do Java do centro de informações do WebSphere MQ.
  • No WebSphere MQ, os nomes podem ter no máximo 48 caracteres, com exceção de canais os quais têm no máximo 20 caracteres.
Broker MA0C: Ao criar um connection factory de tópico v6 do WebSphere Application Server para o broker MA0C, devem ser considerados os seguintes valores de atributo:
BrokerControlQueue
Esse valor é fixado no SYSTEM.BROKER.CONTROL.QUEUE para o intermediário MA0C e é a fila a partir da qual o intermediário lê.
BrokerVersion
Configure esse valor para BASIC para o intermediário MA0C.
ClientID
Configure esse valor para o que preferir para o intermediário MA0C (o valor é string e é meramente um identificador para seu aplicativo cliente).
Ativado para XA
Configure esse valor para TRUE ou FALSE para o intermediário MA0C (a configuração utilizada é um sinalizador de aprimoramento de desempenho - você provavelmente irá querer configurá-lo para 'true' na maioria das vezes).
Seleção de BrokerMessage
Esse valor é fixado em CLIENT para o intermediário MA0C, pois o intermediário depende da seleção da mensagem do lado do cliente.
Tipo de Autorização Direta do Intermediário
Esse valor não é requerido pelo intermediário MA0C.

Um connection factory de tópico para o provedor JMS do produto do WebSphere MQ possui as seguintes propriedades.

Nome

O nome pelo qual esse connection factory de tópico é conhecido para propósitos administrativos no WebSphere Application Server. O nome deve ser exclusivo dentro do provedor JMS

Tipo de Dados Seqüência de Caracteres
Descrição

Uma descrição desse connection factory de tópico para propósitos administrativos no WebSphere Application Server.

Tipo de Dados Seqüência de Caracteres
Nome JNDI

O nome Java Naming and Directory Interface (JNDI) usando para ligar o connection factory de tópico ao namespace do servidor de aplicativos.

Como uma convenção, utilize o nome JNDI completo; por exemplo, no formato jms/Nome, onde Nome é o nome lógico do recurso.

Este nome é utilizado para vincular as informações de ligação da plataforma. A ligação associa os recursos definidos pelo descritor de implementação do módulo aos recursos reais (físicos) vinculados ao JNDI pela plataforma.

Tipo de Dados Seqüência de Caracteres
Unidades Caracteres En_US ASCII
Range 1 a 45 caracteres ASCII
ID do Usuário

O ID de usuário usado, com a propriedade Senha, para autenticação se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, deverá também especificar um valor para a propriedade Senha.

As propriedades ID de Usuário e Senha do connection factory serão usadas se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente, por exemplo, se o aplicativo de chamada usar o método createTopicConnection(). O cliente JMS envia o ID de usuário e a senha para o servidor JMS.

Tipo de Dados Seqüência de Caracteres
Senha

A senha usada, com a propriedade ID de Usuário, para autenticação se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, deverá também especificar um valor para a propriedade Senha.

Tipo de Dados Seqüência de Caracteres
Reinserir Senha

Confirma a senha.

Gerenciador de Filas

O nome do gerenciador de filas do WebSphere MQ para esta fábrica de conexões. As conexões criadas por esse connection factory são conectadas a esse gerenciador de filas.

Tipo de Dados Seqüência de Caracteres
Inserir Informações do Nome do Host e da Porta

Esse botão de opções é selecionado por padrão e, se estiver selecionado, ativará as propriedades de host e de porta e desativará a propriedade da lista de nomes de conexões.

Tipo de Dados Botão de Opções

O número da porta TCP/IP utilizada para conexão com o gerenciador de filas do WebSphere MQ, apenas para conexão de cliente.

Essa porta deve ser configurada no gerenciador de filas do WebSphere MQ.

Tipo de Dados Integer
Range Um número de porta TCP/IP válido, configurado no gerenciador de filas do WebSphere MQ.
Inserir Informações da Lista de Nomes de Conexões

Se selecionado, esse botão de opções ativará a propriedade da lista de nomes de conexões e desativará as propriedades de nome de host e porta. Selecione esse botão de opções se desejar se conectar com um gerenciador de filas com várias instâncias.

Tipo de Dados Botão de Opções
Padrão Não selecionado
Lista de Nomes de Conexão

Uma lista separada por vírgulas das informações de host e de porta que podem ser usadas para se conectar a um gerenciador de filas com várias instâncias.

O formato da lista é:

host[(port)],[host[(port)]]
em que port é opcional e padronizado como 1414, se não for configurado. Por exemplo:
hostname1,hostname2(1415)

Para obter informações adicionais sobre os gerenciadores de filas de várias instâncias, consulte o centro de informações do WebSphere MQ.

Essa propriedade só deve ser usada para a conexão com um gerenciador de filas com várias instâncias. Ela não deve ser usada para conexão com uma lista de gerenciadores de fila diferentes, pois isso pode resultar em problemas de integridade da transação.

No nome do canal usado para conexões do cliente com o gerenciador de filas do WebSphere MQ, apenas para conexão do cliente.

Conjunto de Criptografia SSL

O conjunto de criptografia a ser utilizado para a conexão SSL ao WebSphere MQ.

Defina essa propriedade para um conjunto de criptografia válido fornecido pelo provedor JSSE. O valor deve corresponder ao CipherSpec especificado no canal SVRCONN como a propriedade Canal.

Você deve configurar essa propriedade se a propriedade Nome do Peer SSL for configurada.

Armazenamento de Certificados SSL

Uma lista de zero ou mais servidores Certificate Revocation List (CRL) usados para verificar a revogação de certificado SSL. Se você especificar um valor para essa propriedade, deverá utilizar o WebSphere MQ JVM em Java 2 versão 1.4.

O valor é uma lista delimitada por espaços com entradas na forma:
ldap://hostname:[port]

Uma única barra (/) segue esse valor. Se port for omitido, a porta LDAP padrão 389 será utilizada. Durante o tempo de conexão, o certificado SSL apresentado pelo servidor é verificado em comparação com os servidores CRL especificados. Para obter informações adicionais sobre a segurança CRL, consulte as informações de "Trabalhando com Listas de Revogação de Certificado" na seção Segurança do centro de informações do WebSphere MQ.

Nome do Peer SSL

Para SSL, um esqueleto de nome distinto que deve corresponder ao nome fornecido pelo gerenciador de filas do WebSphere MQ. O nome distinto é utilizado para verificar o certificado de identificação apresentado pelo servidor na conexão.

Se essa propriedade não estiver definida, a verificação de certificado será executada.

A propriedade Nome do Peer SSL será ignorada se a propriedade Conjunto de Criptografia SSL não for especificada.

Esta propriedade é uma lista de pares nome e valor do atributo separados por vírgulas ou pontos e vírgulas. Por exemplo: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

O exemplo verifica o certificado de identificação apresentado pelo servidor no tempo de conexão. Para que a conexão seja bem-sucedida, o certificado deve ter um Nome Comum começando com QMGR. e, pelo menos, dois nomes de Unidade Organizacional, sendo o primeiro IBM e o segundo, WEBSPHERE. A verificação não faz distinção entre maiúsculas e minúsculas.

Para obter detalhes adicionais sobre os nomes distintos e seu uso com o WebSphere MQ, consulte as informações sobre "Nomes Distintos" na seção Segurança do centro de informações do WebSphere MQ.

Conjunto de Conexões

Um conjunto opcional de definições do pool de conexões

As propriedades do pool de conexão são comuns a todos os conectores J2C.

O servidor de aplicativos une as conexões e as sessões em conjuntos com o provedor JMS para melhorar o desempenho. Isso não depende de qualquer conjunto de conexões do WebSphere MQ. É necessário configurar as propriedades do conjunto de conexões e sessões corretamente para os aplicativos, caso contrário, é possível não obter o comportamento desejado da conexão e da sessão.

Altere o tamanho do pool de conexão se o acesso simultâneo do lado do servidor ao recurso JMS exceder o valor padrão. O tamanho do pool de conexão é definido por fila ou por tópico.

Tipo de Dados Caixa de Seleção
Padrão Selecionadas
Opções de Reconexão do Cliente

Especifica se uma conexão do modo do cliente se reconectará automaticamente, ou não, na ocorrência de uma falha de comunicação ou do gerenciador de filas. Essa propriedade será ignorada, a menos que o connection factory esteja sendo usado em um ambiente do cliente thin ou gerenciado.

Tipo de Dados Lista drop down
Padrão DISABLED
Range
DISABLED
A reconexão do cliente não ocorrerá automaticamente.
ASDEF
O valor do atributo DefRecon da sub-rotina dos canais do arquivo de configuração do cliente será usado. Se não houver um valor de DefRecon especificado, essa configuração terá o mesmo efeito que um valor igual a DISABLED.
RECONNECT
A reconexão ocorrerá com qualquer gerenciador de filas consistente com o valor do atributo do gerenciador de filas, que pode ser um gerenciador de filas diferente daquele com o qual a conexão estava sendo feita originalmente.
QMGR
A reconexão ocorrerá apenas com o gerenciador de filas com o qual a conexão estava sendo feita originalmente.

Para obter informações adicionais sobre a reconexão automática do cliente, consulte o centro de informações do WebSphere MQ.

Tempo Limite de Reconexão do Cliente

O tempo máximo, em segundos, que uma conexão do modo do cliente passará tentando se reconectar automaticamente com um gerenciador de filas após uma falha de comunicação ou do gerenciador de filas. Esse parâmetro será ignorado, a menos que o connection factory esteja sendo usado em um ambiente do cliente thin ou gerenciado. O uso ou não desse parâmetro depende do valor do parâmetro de opções de reconexão do cliente.

Tipo de Dados Integer
Unidades Segundos
Padrão 1800
Range Um valor maior que zero e até 2147483647

Para obter informações adicionais sobre a reconexão automática do cliente, consulte o centro de informações do WebSphere MQ.
