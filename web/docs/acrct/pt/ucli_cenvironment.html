Provedores do Ambiente de Recursos e Entradas do Ambiente de Recursos

Uma referência de ambiente de recurso mapeia um nome lógico, utilizado pelo aplicativo cliente, para o nome físico de um objeto.

Nem todos os objetos ligados ao espaço de nomes JNDI do servidor podem ser utilizados por um aplicativo cliente. Por exemplo, o tempo de execução do cliente do WebSphere Application Server não suporta o uso de objetos do Java 2 Connector (J2C) no cliente. O objeto precisa ser remoto e as implementações do lado do cliente devem estar disponíveis no classpath do tempo de execução do aplicativo cliente.

As referências do ambiente de recursos são diferentes das referências de recursos. As referências do ambiente de recursos permite que o aplicativo cliente utilize um nome lógico para procurar um recurso ligado ao espaço de nome de JNDI do servidor. Uma referência de recurso permite que seu aplicativo utilize um nome lógico para procurar um recurso J2EE local. A especificação do J2EE não especifica uma implementação de um recurso em particular.
