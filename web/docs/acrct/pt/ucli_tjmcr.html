Configurando Recursos do Cliente do Sistema de Mensagens Java

Para configurar os recursos do cliente do sistema de mensagens Java, crie novas configurações do provedor JMS para o aplicativo cliente. O aplicativo cliente pode usar um serviço de sistema de mensagens através das APIs do Java Message Service. Um provedor de JMS oferece dois tipos de fábricas de J2EE. Um é fábrica de conexão de JMS e o outro é fábrica de destino de JMS.

Antes de Começar

Em uma tarefa administrativa separada, instale o cliente Java Message Service (JMS) no computador cliente onde o aplicativo cliente reside. O fornecedor do produto de mensagens deve oferecer uma implementação do cliente JMS. Para obter informações adicionais, consulte a documentação do produto do sistema de mensagens.

Nota: Ao concluir essa tarefa, você pode criar um novo fornecedor de sistema de mensagens ou pode utilizar um existente.

Procedimento

  1. Inicie a ACRTCT (Application Client Resource Configuration Tool).
  2. Abra o arquivo EAR para o qual deseja configurar o novo provedor JMS. O conteúdo do arquivo EAR é exibido na exibição em árvore.
  3. Selecione o arquivo JAR no qual deseja configurar o novo provedor JMS a partir da árvore.
  4. Expanda o arquivo JAR para exibir seu conteúdo.
  5. Como opção, clique com o botão direito do mouse em Fornecedores de Sistema de Mensagens e selecione Novo, se quiser criar e utilizar um novo fornecedor de sistema de mensagens.
  6. Configure as propriedades do provedor JMS no diálogo de propriedades resultante.
  7. Clique em OK.
  8. Clique em Arquivo > Salvar.
