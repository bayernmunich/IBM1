Configurações do Provedor JMS para Clientes Aplicativos

Use esta página para configurar as propriedades do provedor Java Message Service (JMS), se desejar usar um provedor JMS diferente do provedor de sistema de mensagens padrão ou o WebSphere MQ como um provedor JMS.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado. Clique com o botão direito do mouse em Provedores de Sistema de Mensagens e clique em Novo. Os seguintes campos aparecem na guia Geral.

Nome

O nome através do qual o provedor JMS é conhecido para finalidades administrativas.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição do provedor JMS, para propósitos administrativos.

Tipo de Dado Seqüência de Caracteres
Caminho da Classe

Uma lista de caminhos ou nomes de arquivos .jar que juntos formam o local para as classes do provedor de recursos.

Classe do Depósito de Informações do Provedor de Contexto

O nome de classe Java do factory de contexto inicial para o provedor JMS.

Por exemplo, para um fornecedor de serviços LDAP o valor possui o formato: com.sun.jndi.ldap.LdapCtxFactory.

Tipo de Dado Seqüência de Caracteres
URL do Provedor

A URL do provedor JMS para procurar JNDI externas.

Por exemplo, uma URL do LDAP para um provedor JMS possui o formato: ldap://hostname.company.com/contextName.

Tipo de Dado Seqüência de Caracteres
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
