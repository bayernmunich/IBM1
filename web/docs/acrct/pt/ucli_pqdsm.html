Configurações do Destino de Fila do Provedor WebSphere MQ para Aplicativos Clientes

Use este painel para visualizar ou alterar as propriedades de configuração do destino de fila selecionado para uso com o provedor WebSphere® MQ do produto Java™ Message Service (JMS).

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores do Sistema de Mensagens > Provedor WebSphere MQ. Clique com o botão direito em Destinos de Fila e clique em Novo. Os campos a seguir são exibidos na guia Geral.

Observação:
  • Os valores de propriedade especificados devem corresponder aos valores especificados ao configurar os recursos JMS para o WebSphere MQ. Para obter informações adicionais sobre a configuração dos recursos JMS para o WebSphere MQ, consulte Uso do Java no centro de informações do WebSphere MQ.
  • No WebSphere MQ, os nomes podem ter no máximo 48 caracteres.

Uma fila para uso com o provedor JMS do produto WebSphere MQ possui as seguintes propriedades.

Nome

O nome pelo qual a fila é conhecida para propósitos administrativos dentro do WebSphere Application Server.

Tipo de Dados Seqüência de Caracteres
Descrição

Uma descrição da fila, para propósitos administrativos, no WebSphere Application Server.

Tipo de Dados Seqüência de Caracteres
Nome JNDI

O ambiente do tempo de execução do cliente aplicativo utiliza esse campo para recuperar as informações sobre configuração.

Tipo de Dados Enum
Padrão APPLICATION_DEFINED
Range

Definida pela fila

Persistente
As mensagens no destino são persistentes.
Não persistente
As mensagens no destino não são persistentes.

Prioridade Especificada

Se a propriedade Prioridade for definida para Especificada, especifique a prioridade para essa fila no intervalo de 0 (mais baixo) a 9 (mais alto).

Tipo de Dados Integer

Expiração

Se o valor de tempo de limite de expiração para essa fila for definido pelo aplicativo ou pela propriedade Expiração Especificada ou se as mensagens da fila nunca expiram (têm um tempo limite de expiração ilimitado).

Tipo de Dados Enum
Unidades Não aplicável
Padrão APPLICATION_DEFINED
Range
Definida pelo aplicativo
O tempo limite para expirar as mensagens nesta fila é definido pelo aplicativo que as coloca na fila.
Especificado
O tempo limite de expiração para as mensagens dessa fila é definido pela propriedade Expiração Especificada . Se você selecionar essa opção, você deve definir um tempo limite na propriedade Expiração Especificada.
Ilimitado
As mensagens dessa fila não têm tempo limite de expiração e essas mensagens nunca expiram.
Expiração Especificada

Se a propriedade Tempo Limite de Expiração for definida para Especificado, digite aqui o número de milissegundos (maior do que 0) depois do qual as mensagens da fila expiram.

Tipo de Dados Integer

Range Maior que ou igual a 0

  • Outros valores são um número inteiro de milissegundos

Corpo da Mensagem

Especifica se um aplicativo processa o cabeçalho do RFH versão 2 de uma mensagem do WebSphere MQ como parte do corpo da mensagem JMS.

Tipo de Dados Lista drop down
Padrão UNSPECIFIED
Intervalo
UNSPECIFIED
Ao enviar mensagens, o provedor de sistemas de mensagens do WebSphere MQ gera ou não gera um cabeçalho do RFH versão 2 e o inclui, dependendo do valor da versão dos cabeçalhos do Append RFH versão 2 para as mensagens enviadas para essa propriedade de destino. Ao receber mensagens, o provedor de sistemas de mensagens do WebSphere MQ age como se o valor estivesse configurado como JMS.
JMS
Ao enviar mensagens, o provedor de sistemas de mensagens do WebSphere MQ gera automaticamente um cabeçalho do RFH versão 2 e o inclui na mensagem do WebSphere MQ. Ao receber mensagens, o provedor de sistemas de mensagens do WebSphere MQ configura as propriedades de mensagem do JMS de acordo com os valores do cabeçalho do RFH versão 2 (se esse valor for apresentado); e não apresenta o cabeçalho do RFH versão 2 como parte do corpo da mensagem do JMS.
MQ
Ao enviar mensagens, o provedor de sistemas de mensagens do WebSphere MQ não gera um cabeçalho do RFH versão 2. Ao receber mensagens, o provedor de sistemas de mensagens do WebSphere MQ apresenta o cabeçalho do RFH versão 2 como parte do corpo da mensagem do JMS.

Tipo de Dados Lista drop down
Padrão DEFAULT
Range
DEFAULT
O valor padrão é equivalente às informações contidas no cabeçalho do RFH versão 2.
MQMD
Use o valor fornecido no MQMD. Preenche a resposta para o campo gerenciador de filas com o valor do MQMD, equivalente ao comportamento padrão do WebSphere MQ Versão 6.0.2.4 e 6.0.2.5.

Use o valor fornecido no cabeçalho do RFH versão 2. Se o aplicativo de envio configurar um valor de JMSReplyTo, esse valor será usado.

Contexto de Mensagens do MQMD

Define as opções de contexto da mensagem especificadas ao enviar mensagens para um destino.

Tipo de Dados Lista drop down
Padrão DEFAULT
Range
DEFAULT
A chamada da API do MQOPEN e a estrutura do MQPMO não especificam opções de contexto de mensagens explícitas.
SET_IDENTITY_CONTEXT
A chamada da API do MQOPEN especifica a opção de contexto da mensagem MQOO_SET_IDENTITY_CONTEXT e a estrutura do MQPMO especifica MQPMO_SET_IDENTITY_CONTEXT.
SET_ALL_CONTEXT
A chamada da API do MQOPEN especifica a opção de contexto da mensagem MQOO_SET_ALL_CONTEXT e a estrutura do MQPMO especifica MQPMO_SET_ALL_CONTEXT.
Propriedades Customizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
