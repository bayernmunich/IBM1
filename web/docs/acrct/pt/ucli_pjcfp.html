Configurações do Connection Factory JMS Genérico para Clientes Aplicativos

Use este painel para visualizar ou alterar as propriedades de configuração do connection factory Java Message Service (JMS) selecionado para uso com o provedor JMS associado. Estas propriedades de configuração controlam como as conexões são criadas entre o provedor JMS e o sistema de mensagens que ele utiliza.

Para visualizar essa página da ACRCT (Application Client Resource Configuration Tool), clique em Arquivo > Abrir. Após procurar por um arquivo EAR, clique em Abrir. Expanda o arquivo JAR selecionado > Fornecedores de Sistema de Mensagens > new_JMS_Provider_instance. Clique com o botão direito do mouse em Connection Factories e clique em Novo. Os campos a seguir são exibidos na guia Geral.

Um connection factory Java Message Service (JMS) cria conexões com destinos JMS. A connection factory JMS é criada pelo provedor JMS associado. Um connection factory JMS para um provedor JMS genérico (diferente do provedor interno de sistema de mensagens padrão ou WebSphere MQ como provedor JMS) tem as seguintes propriedades:

Nome

Nome pelo qual essa connection factory do JMS é conhecida para fins administrativos no IBM WebSphere Application Server.O nome deve ser exclusivo dentro do provedor JMS associado.

Tipo de Dado Seqüência de Caracteres
Descrição

Uma descrição dessa connection factory para fins administrativos no IBM WebSphere Application Server.

Tipo de Dado Seqüência de Caracteres
Padrão Nulo
Nome JNDI

O tempo de execução do aplicativo cliente utiliza esse campo para recuperar as informações sobre configuração.

ID do usuário

Indica o ID do usuário utilizado com a propriedade Senha, para autenticação, se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

As propriedades do fator de conexão ID do Usuário e Senha são utilizados se o aplicativo chamado não fornecer um ID do usuário e senha explicitamente; por exemplo, se o aplicativo chamado utilizar o método createQueueConnection(). O cliente JMS encaminha o ID do Usuário e a Senha ao servidor JMS.

Tipo de Dado Seqüência de Caracteres
Senha

A senha utilizada com a propriedade ID do Usuário para autenticação se o aplicativo de chamada não fornecer um ID de usuário e uma senha explicitamente.

Se você especificar um valor para a propriedade ID do Usuário, também é necessário especificar um valor para a propriedade Senha.

Tipo de Dado Seqüência de Caracteres
Padrão Nulo
Digitar Senha Novamente

Confirma a senha digitada no campo Senha.

Nome de JNDI Externo

O nome de JNDI utilizado para ligar a fila no espaço de nomes do servidor de aplicativos.

Como uma convenção, utilize o nome de JNDI completo, por exemplo, jms/Name, em que Name é o nome lógico do recurso.

Este nome é utilizado para vincular as informações de ligação da plataforma. A ligação associa os recursos definidos pelo descritor de implementação do módulo aos recursos reais (físicos) ligados na API de JNDI pela plataforma.

Tipo de Dado Seqüência de Caracteres
Tipo de Conexão

Se esse destino JMS é uma fila (para ponto-a-ponto) ou um tópico (para publicação ou assinatura).

Selecione uma das seguintes opções:
Fila
Um destino de fila JMS para mensagens ponto a ponto.
Um destino de tópico JMS para o sistema de mensagens de publicação/assinatura.
Propriedades Personalizadas

Especifica pares de nome-valor para definição de propriedades adicionais no objeto criado no tempo de execução deste recurso.

Você deve inserir um nome que é uma propriedade pública no objeto e um valor que pode ser convertido de cadeia para o tipo requerido pelo método de definição da propriedade. As propriedades e valores aceitáveis dependem do objeto que é criado. Consulte a documentação do objeto para obter uma lista de propriedades e valores válidos.
