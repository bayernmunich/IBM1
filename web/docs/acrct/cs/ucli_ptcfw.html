Nastavení faktorie připojení tématu výchozího poskytovatele verze 5 pro aplikační klienty

Tento panel slouží k zobrazení nebo změně konfiguračních vlastností vybrané faktorie připojení tématu, která má být použita v kombinaci s interním poskytovatelem rozhraní JMS (Java Message Service). Tyto konfigurační vlastnosti řídí způsob vytváření připojení mezi poskytovatelem rozhraní JMS a systémem zpráv, který využívá.

Chcete-li zobrazit tuto stránku Konfiguračního nástroje prostředků aplikačního klienta (ACRCT), klepněte na volbu Soubor > Otevřít. Po vyhledání souboru EAR klepněte na tlačítko Otevřít. Rozbalte vybraný soubor JAR > Poskytovatelé systémů zpráv > Výchozí poskytovatel verze 5. Klepněte pravým tlačítkem myši na volbu Faktorie připojení tématu a poté klepněte na volbu Nový. Na kartě Obecné se zobrazí následující pole.

Faktorie připojení tématu výchozího poskytovatele verze 5 má následující vlastnosti.

Název

Název označující tuto faktorii připojení fronty pro účely administrace v rámci produktu WebSphere Application Server. Název musí být v rámci administrativní domény produktu WebSphere Application Server jedinečný.

Datový typ Řetězec
Popis

Popis této faktorie připojení tématu pro účely administrace v rámci produktu WebSphere Application Server.

Datový typ Řetězec
Název rozhraní JNDI

Běhová komponenta aplikačního klienta používá toto pole k načítání konfiguračních informací.

Jméno uživatele

Jméno uživatele použité spolu s vlastností Heslo pro ověřování v případě, že volající aplikace nepředá hodnoty userid (jméno uživatele) a password (heslo) explicitně.

Zadáte-li hodnotu pro vlastnost Jméno uživatele, musíte rovněž zadat hodnotu pro vlastnost Heslo.

Vlastnosti Jméno uživatele a Heslo faktorie připojení se používají v případě, že volající aplikace nepředá hodnoty userid (jméno uživatele) a password (heslo) explicitně, například v případě, že volající aplikace používá metodu createTopicConnection(). Klient JMS předá hodnoty userid a password serveru JMS.

Datový typ Řetězec
Heslo

Heslo použité spolu s vlastností Jméno uživatele pro ověřování v případě, že volající aplikace nepředá hodnoty userid (jméno uživatele) a password (heslo) explicitně.

Zadáte-li hodnotu pro vlastnost Jméno uživatele, musíte rovněž zadat hodnotu pro vlastnost Heslo.

Datový typ Řetězec
Opětné zadání hesla

Potvrzuje heslo.

Uzel

Určuje název administračního uzlu WebSphere Application Server, ve kterém pracuje server JMS pro tuto faktorii připojení. Připojení vytvořená v této faktorii využívají přiřazený server JMS.

Datový typ Výčet
Rozsah Rozevírací seznam uzlů v administrativní doméně produktu WebSphere Application Server.
Aplikační server

Zadejte název aplikačního serveru. Tento název není názvem hostitele počítače, ale název konfigurovaného aplikačního serveru.

Port

Jeden ze dvou portů, který bude používán pro připojení k serveru JMS. Port S FRONTOU je určen pro podporu všech funkcí publikování a odběru JMS. PŘÍMÝ port je určen pouze pro neperzistentní přechodné odběry mimo transakce.

Poznámka: Objekty bean řízené zprávami nemohou používat přímý port modulu listener pro podporu publikování nebo odběrů. Z toho důvodu nelze žádnou faktorii připojení tématu konfigurovanou s vlastností Port nastaveno na hodnotu Přímý používat spolu s objekty bean řízenými zprávami.
Datový typ Výčet
Výchozí hodnota QUEUED
Rozsah
QUEUED
Port modulu listener je používán pro podporu publikování či odběru, která je plně kompatibilní s funkcí JMS.
DIRECT
Port modulu listener je používán pro podporu přímého připojení TCP/IP (pouze netransakční a přechodné odběry) pro publikování nebo odběr.

Čísla portů TCP/IP pro tyto porty jsou definována na interním serveru JMS produktu.
ID klienta

Identifikátor klienta JMS použitý pro připojení ke správci front MQSeries.

Datový typ Řetězec
Přizpůsobené vlastnosti

Určuje dvojice název-hodnota pro nastavení dalších vlastností objektu vytvářeného pro tento prostředek během zpracování.

Musíte zadat název, který je veřejnou vlastností objektu, a hodnotu, kterou lze převést z řetězce na typ vyžadovaný metodou set příslušné vlastnosti. Použitelné vlastnosti a hodnoty jsou závislé na vytvářeném objektu. Seznam platných vlastností a hodnot naleznete v dokumentaci k objektu.
