Zpracování asynchronních zpráv v produktu WebSphere Application Server prostřednictvím rozhraní JMS

Server WebSphere Application Server podporuje asynchronní systém zpráv jako metodu komunikace založenou na programovacím rozhraní JMS (Java Message Service). Rozhraní JMS poskytuje běžný způsob vytváření, odesílání, přijímání a čtení asynchronních žádostí jako zpráv rozhraní JMS pro programy prostředí Java (klienti i aplikace Java Platform, Enterprise Edition - Java EE).

Toto téma poskytuje obecný přehled asynchronního systému zpráv používajícího podporu rozhraní JMS, kterou poskytuje server WebSphere Application Server.

Základní podpora asynchronního systému zpráv používajícího rozhraní API JMS poskytuje běžnou sadu rozhraní JMS a přiřazených sémantik, které určují, jak může klient rozhraní JMS přistupovat k prostředkům poskytovatele rozhraní JMS. Tato podpora povoluje aplikacím produktu WebSphere založeným na platformě Java EE, jako jsou klienti rozhraní JMS, vyměňovat si zprávy asynchronně s jinými klienty rozhraní JMS pomocí míst určení rozhraní JMS (fronty nebo témata). Aplikace Java EE může používat místa určení v podobě front rozhraní JMS pro zasílání zpráv mezi dvěma body a místa určení v podobě témat rozhraní JMS pro systém zpráv publikování a odběru. Aplikace Java EE si může explicitně vyžádat zprávy z místa určení a pak načíst zprávy pro zpracování objekty typu bean obchodní logiky (objekty enterprise bean).

Se základní podporou rozhraní JMS a XA používá aplikace Java EE standardní volání rozhraní JMS pro zpracování zpráv včetně veškerých odpovědí nebo odesílání odchozích zpráv. Objekt enterprise bean může zpracovávat odpovědi jako objekt typu bean odesílatele nebo v rámci objektu enterprise bean, který přijímá příchozí zprávy. Tento proces může volitelně používat dvoufázové potvrzování v rámci oboru transakce. Tato úroveň funkcí pro zpracován asynchronních zpráv je označována výrazem zpracování zpráv spravované objektem typu bean a poskytuje objektu enterprise bean úplnou kontrolu nad infrastrukturou systému zpráv, například pro připojení a správu fondu relací. Společný kontejner se zpracování zpráv spravovaného objektem typu bean neúčastní.

Server WebSphere Application Server také podporuje automatický asynchronní systém zasílání zpráv pomocí objektů bean řízených zprávami (typ objektu enterprise bean definovaný ve specifikaci EJB - Enterprise JavaBeans 2.0) a modulů listener rozhraní JMS (součást prostředků aplikačního serveru rozhraní JMS). Zprávy jsou automaticky načítány z míst určení rozhraní JMS, volitelně v rámci transakce, a poté odeslány na objekty bean řízené transakcemi v aplikaci Java EE, aniž by aplikace musela provádět explicitní výzvu pro místa určení rozhraní JMS.
