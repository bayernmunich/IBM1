Настройка новых провайдеров источников данных (провайдеров JDBC) для приложений-клиентов

Вы можете создать новый провайдеры источников данных, также называемых провайдерами JDBC, для ваших приложений-клиентов с помощью инструмента настройки ресурса приложения-клиента (ACRCT).

Перед тем, как начать

При выполнении этой задачи вы создадите новые провайдеры источников данных для приложения-клиента. В отдельной административной задаче установите код Java для обязательного провайдера источника данных в клиентской системе, в которой находится приложение-клиент.

О задаче

Используйте эту задачу для подключения приложений-клиентов к реляционным базам данных.

Процедура

  1. Запустите инструмент настройки ресурсов приложения-клиента (ACRCT) и откройте файл EAR для которого требуется настроить новый провайдер источника данных. Содержимое файла EAR будет показано в иерархическом списке.
  2. Выберите файл JAR в котором требуется настроить новый провайдер источника данных из иерархического списка.
  3. Разверните файл JAR для просмотра его содержимого.
  4. Щелкните на папке провайдеры источников данных. Выполните одно из следующих действий:
    • Щелкните правой кнопкой мышки на папке и выберите Создать провайдер.
    • Выберите в меню пункт Изменить > Создать.
  5. Настройте свойства источника данных в открывшемся диалоге свойств.
  6. Нажмите OK для завершения.
  7. Выберите в меню пункт Файл > Сохранить для сохранения изменений.
