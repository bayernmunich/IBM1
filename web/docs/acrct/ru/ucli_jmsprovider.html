Провайдеры Java Message Service (JMS) для клиентов

Этот раздел содержит описание различных способов применения приложением-клиентом провайдеров Java Message Service (JMS) с WebSphere Application Server.

IBM WebSphere Application Server поддерживает асинхронный обмен сообщениями посредством использования провайдера JMS и связанной с ним системы обмена сообщениями. Провайдеры JMS должны соответствовать спецификации JMS версии 1.1. Для использования управляемых сообщениями объектов EJB провайдер JMS должен поддерживать необязательную функцию Application Server Facility (ASF), определенную в этом приложении, либо поддерживать встроенный адаптер ресурсов, определенный в спецификации JCA версии 1.5.

Технологии интеграции служб сервера IBM WebSphere Application Server могут выступать в роли системы обмена сообщениями, если настроить шину интеграции служб, доступ к которой осуществляется посредством провайдера обмена сообщениями по умолчанию. Эта поддержка установлена как часть WebSphere Application Server, управляется с помощью административной консоли и полностью интегрирована в среду выполнения WebSphere Application Server.

В WebSphere Application Server также включена поддержка следующих провайдеров JMS:
WebSphere MQ
Позволяет работать с поддерживаемыми версиями WebSphere MQ.
Базовый
Позволяет работать с любыми системами обмена сообщениями других фирм, поддерживающими ASF.

Для целей совместимости с предыдущими выпусками в WebSphere Application Server также включена поддержка провайдера обмена сообщениями V5, что позволяет настраивать ресурсы для работы с системой Embedded Messaging сервера WebSphere Application Server версии 5. Провайдер обмена сообщениями V5 можно также использовать с шиной интеграции служб.

Приложения WebSphere могут использовать ресурсы обмена сообщениями, предоставленные одним из этих провайдеров JMS. Однако выбор провайдера чаще всего определяется требованием работы с существующей системой обмена сообщениями или интеграции с этой системой. Например, уже может существовать инфраструктура обмена сообщениями, основанная на WebSphere MQ. В этом случае можно либо подключиться напрямую с помощью включенной поддержки WebSphere MQ в качестве провайдера JMS, либо настроить шину интеграции служб со связями с сетью WebSphere MQ, а затем получить доступ к шине с помощью провайдера обмена сообщений по умолчанию.

Шина интеграции служб обеспечивает доступ к провайдеру обмена сообщениями по умолчанию. Это совместимый с J2EE 1.4 провайдер обмена сообщениями JMS, полностью интегрированный с WebSphere Application Server. Использовать его можно в конфигурациях с несколькими серверами для обмена сообщениями с сетью WebSphere MQ.
