Адаптеры ресурсов для клиента

Адаптер ресурсов является драйвером программного обеспечения уровня системы, который используется приложением на Java для подключения к информационной системе предприятия (EIS). Адаптер ресурсов встраивается в приложение-клиент и обеспечивает возможность соединения между EIS и приложением J2EE.

Поддержка адаптера ресурсов для приложений-клиентов J2EE является подмножеством поддержки сервера. Для всех адаптеров ресурсов, установленных с помощью инструмента clientRAR, адаптер ресурсов клиента используется в неуправляемой среде и должен соответствовать спецификации архитектуры коннектора J2EE версии 1.5 и выше. В интерфейсах ManagedConnectionFactory поддерживаются только выходящие соединения к EIS. Поддержка входящих сообщений (от EIS), управление жизненным циклом и управление потоком операций не поддерживаются в клиенте.

Чтобы приложения-клиенты могли использовать адаптер ресурсов следует установить его в каталоге, указанном переменной среды CLIENT_CONNECTOR_INSTALL_ROOT, задаваемой при выполнении сценария setupCmdLine. Утилита launchClient, инструмент настройки ресурсов приложения-клиента (ACRCT) и утилита clientRAR используют эту переменную для поиска расположения по умолчанию всех установленных адаптеров ресурсов. Для установки адаптера ресурсов в клиенте используйте инструмент clientRAR. После установки адаптера ресурсов следует настроить его с помощью ACRCT. Инструмент настройки клиента добавляет конфигурацию адаптера ресурсов в файл EAR. Затем задаются фабрики соединений и администрируемые объекты.

При выполнении приложений-клиентов J2EE, сценарий launchClient указывает системное свойство com.ibm.ws.client.installedConnector, для которого задается значение, равное переменной CLIENT_CONNECTOR_INSTALL_ROOT. Это расположение по умолчанию установленных адаптеров ресурсов и оно может быть переопределено для каждого вызова launchClient при указании параметра -CCD. После активации контейнера клиента все подкаталоги адаптера ресурсов в указанном расположении по умолчанию для каталога адаптеров ресурсов добавляются в путь к классам. Это позволяет приложению-клиенту использовать адаптеры ресурсов без применения ACRCT для указания ресурсов клиента.

Применение адаптеров ресурсов является новым механизмом простого расширения приложений-клиентов.
