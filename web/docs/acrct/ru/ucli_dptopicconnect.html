Параметры фабрики соединений темы провайдера по умолчанию

Используйте эту панель для просмотра или изменения свойств выбранной фабрики соединений темы JMS для применения с внутренним провайдером Java Message Service (JMS) продукта, устанавливаемым вместе с WebSphere Application Server. Эти свойства конфигурации контролируют создание соединений между провайдером JMS и использующей его шиной интеграции служб.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер по умолчанию. Щелкните правой кнопкой на опции Фабрики соединений темы и выберите Создать. На вкладке Общие появятся следующие поля.

Будут показаны свойства со значениями по умолчанию. Свойства с фиксированными значениями будут снабжены выпадающим списком.

Имя

Имя фабрики соединений темы.

Тип данных Строка
Описание

Описание данной фабрики соединений для административных целей на сервере IBM WebSphere Application Server.

Тип данных Строка
Имя JNDI

Имя JNDI, используемое для сопоставления этого определения фабрики соединений темы с файлом описания. Эта запись является именем resource-ref.

Тип данных Строка
Имя пользователя

Имя пользователя с Паролем, используемое для идентификации если вызывающее приложение не предоставляет явно указанные ИД пользователя и пароль. Если это поле используется, поле свойств Имя пользователя игнорируется.

Если указать значение свойства Имя пользователя, следует также задать значение свойства Пароль.

Свойства Имя пользователя и Пароль фабрики соединений используются если вызывающее приложение не указывает явно ИД пользователя и пароль. Если имя пользователя и пароль указаны, создается псевдоним идентификации для фабрики с зашифрованным паролем.

Тип данных Строка
Общие гарантированные подписки

Позволяет выбрать, являются ли гарантированные подписки общими для соединений с элементами кластера сервера.

Обычно TopicSubscriber для определенной гарантированной подписки может принадлежать только одному сеансу одновременно. Это свойство позволяет переопределить данное поведение чтобы разрешить гарантированной подписке одновременно иметь несколько получателей.

Тип данных Список выбора
Значение по умолчанию В кластере
Диапазон
В кластере
Разрешает общее использование гарантированных подписок при установке подключений из кластера сервера.
Всегда общие
Гарантированные подписки могут совместно использоваться соединениями.
Не общие
Гарантированные подписки не могут совместно использоваться соединениями.

Входящая целевая транспортная цепочка

Имя протокола, определяющего группу служб обмена сообщениями.

Тип данных Строка
Конечные точки провайдера

Список разделенных запятой конечных точек, используемых для подключения к серверу начальной загрузки.

Укажите через запятую список триплетов конечных точек, используя следующий синтаксис: хост:порт:протокол.

Пример 
localhost:7777:BootstrapBasicMessaging
где

BootstrapBasicMessaging является InboundBasicMessaging удаленного протокола (JFAP-TCP/IP).

Значение по умолчанию
  • Если имя хоста не указано, будет использовано значение по умолчанию localhost.
  • Если не указан номер порта, будет использовано значение по умолчанию 7276.
  • Если не указано имя цепочки, будет использована в качестве значения по умолчанию предустановленная цепочка BootstrapBasicMessaging.
