Параметры адресата очереди провайдера WebSphere MQ для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранного адресата очереди для применения с провайдером JMS для WebSphere MQ®.

Для просмотра страницы инструмента настройки ресурсов приложения-клиента (ACRCT) выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер WebSphere MQ. Щелкните правой кнопкой на пункте Адресаты очереди и нажмите кнопку Создать. На вкладке Общие появятся следующие поля.

Примечание:
  • Указываемые вами значения свойств должны совпадать со значениями, указанными при настройке ресурсов JMS продукта WebSphere MQ. Дополнительная информация о настройке ресурсов JMS для WebSphere MQ приведена в разделе Работа с Java справочной системы WebSphere MQ Information Center.
  • В WebSphere MQ максимальная длина имен не может превышать 48 символов.

Очередь, используемая с провайдером JMS продукта WebSphere MQ, имеет следующие свойства.

Имя

Имя, идентифицирующее очередь для административных задач в WebSphere Application Server.

Тип данных Строка
Описание

Описание очереди для администрирования в WebSphere Application Server.

Тип данных Строка
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

Сохранение состояния

Указывает, являются ли все сообщения, отправляемые в адресаты, постоянными, временными или их постоянство задается приложением.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Задает приложением
Постоянство сообщений в местоположении определяется приложением, поместившим их в очередь.
Задается очередью
[Только для местоположений WebSphere MQ] Постоянство сообщений в местоположении определяется свойствами определения очереди WebSphere MQ.
Устойчивый
Сообщения по назначению сохраняются.
Неустойчивый
Сообщения по назначению не сохраняются.
Приоритет

Указывает, задается ли приоритет сообщения для данного адресата приложением или свойством Указанный приоритет.

Тип данных Перечисление
Единицы Не применимо
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Задает приложением
Приоритет сообщений в данном местоположении определяется приложением, поместившим их в это местоположение.
Задается очередью
[Только для местоположений WebSphere MQ] Постоянство сообщений в местоположении определяется свойствами определения очереди WebSphere MQ.
Указан
Приоритет сообщений этого адресата задается свойством Указанный приоритет. При выборе этого варианта следует задать свойство Указанный приоритет.
Указанный приоритет

Если для свойства Приоритет задано значение Указан, введите приоритет сообщений для этой очереди в диапазоне от 0 (низший) до 9 (высший).

Тип данных Целое число
Единицы Уровень приоритета сообщения
Диапазон От 0 (низший приоритет) до 9 (высший приоритет)
Срок хранения

Указывает, задается ли значение тайм-аута для этой очереди приложением или свойством Указанный срок хранения, либо сообщения очереди имеют неограниченный срок хранения (неограниченный тайм-аут хранения).

Тип данных Перечисление
Единицы Не применимо
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Задает приложением
Тайм-аут срока хранения для сообщений в этой очереди задается приложением, поместившим их в нее.
Указан
Тайм-аут срока хранения для сообщений в этой очереди задается свойством Указанный срок хранения. При выборе этой опции следует указать значение тайм-аута для свойства Указанный срок хранения.
Не ограничено
Сообщения в этой очереди не имеют тайм-аута срока хранения.
Указанный срок хранения

Если для свойства Тайм-аут срока хранения задано значение Указан, введите число миллисекунд (больше 0), по истечении которых сообщение в этой очереди будет удалено.

Тип данных Целое число
Единицы Миллисекунды
Диапазон Больше либо равно 0
  • 0 указывает на бесконечный срок хранения сообщений
  • Другие значения представляют время в миллисекундах
Имя основной очереди

Имя очереди, в которую отправляется сообщение, в диспетчере очередей, указанным в свойстве Имя базового диспетчера очередей.

Тип данных Строка
Имя базового администратора очередей

Имя администратора очередей WebSphere MQ, которому отправляются сообщения.

Этот администратор очередей предоставляет очередь, указанную в свойство Имя базовой очереди.

Тип данных Строка
Единицы ru_RU ASCII-символы
Диапазон Допустимое имя администратора очередей WebSphere MQ, от 1 до 48 ASCII-символов
CCSID

Идентификатор кодированного набора символов для использования с администратором очередей WebSphere MQ.

Этот идентификатор набора символов (CCSID) должен относиться к поддерживаемым администратором очередей WebSphere MQ идентификаторам CCSID. Дополнительная информация приведена в разделе Параметры дополнительных свойств очередей и тем провайдера сообщений WebSphere MQ.

Тело сообщения

Задает, нужно ли приложению обрабатывать заголовок RFH версии 2 сообщения WebSphere MQ как часть текста сообщения JMS.

Тип данных Выпадающий список
Значение по умолчанию UNSPECIFIED
Диапазон
UNSPECIFIED
При отправке сообщений, созданием заголовка RFH версии 2 провайдером сообщений WebSphere MQ управляет свойство Добавлять заголовки RFH версии 2 в сообщения, отправляемые этому получателю. При получении сообщений, провайдер сообщений WebSphere MQ работает таким образом, как если бы было указано значение JMS.
JMS
При отправке сообщений, провайдер сообщений WebSphere MQ автоматически создает заголовок RFH версии 2 и добавляет его в сообщение WebSphere MQ. При получении сообщений, провайдер сообщений WebSphere MQ задает свойства сообщения JMS с учетом содержимого заголовка RFH версии 2 (если он задан); заголовок RFH версии 2 не рассматривается как часть тела сообщения JMS.
MQ
При отправке сообщений, провайдер сообщений WebSphere MQ не создает заголовок RFH версии 2. В ходе приема сообщений провайдер сообщений WebSphere MQ добавляет заголовок RFH версии 2 в тело сообщения JMS.

Применяется значение из заголовка RFH версии 2. Применяется значение JMSReplyTo, указанное отправляющим приложением.
