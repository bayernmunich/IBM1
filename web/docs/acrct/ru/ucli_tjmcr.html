Настройка ресурсов клиента обмена сообщениями Java

Для настройки ресурсов клиента обмена сообщениями Java вы создадите новые конфигурации провайдера JMS для приложения-клиента. Приложение-клиент может использовать службу обмена сообщениями с помощью API Java Message Service. Провайдер JMS предоставляет два типа фабрик J2EE. Одним из них является Фабрика соединений JMS, а вторым - фабрика адресатов JMS.

Перед тем, как начать

В отдельной административной задаче установите клиент Java Message Service (JMS) в клиентской системе, в которой расположено приложение-клиент. Реализация клиента JMS предоставляется производителем продукта обмена сообщениями. Дополнительные сведения приведены в документации продукта обмена сообщениями.

Прим.: При выполнении этой задачи вы можете либо создать новый провайдер обмена сообщениями, либо использовать уже существующий.

Процедура

  1. Запустите инструмент настройки ресурсов приложения-клиента (ACRCT).
  2. Откройте файл EAR для которого требуется настроить новый провайдер JMS. Содержимое файла EAR будет показано в иерархическом списке навигации.
  3. Выберите файл JAR в котором требуется настроить новый провайдер JMS из иерархического списка.
  4. Разверните файл JAR для просмотра его содержимого.
  5. При необходимости щелкните на опции Провайдеры обмена сообщениями и выберите Создать для создания и применения нового провайдера обмена сообщениями.
  6. Настройте свойства провайдера JMS в открывшемся диалоге свойств.
  7. Нажмите OK.
  8. Выберите Файл > Сохранить.
