Параметры фабрики соединений темы провайдера по умолчанию версии 5 для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранной фабрики соединений темы для применения с внутренним провайдером JMS (Java Message Service) продукта. Эти свойства конфигурации контролируют способ создания соединений между провайдером JMS и используемой им системой обмена сообщениями.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер по умолчанию версии 5. Щелкните правой кнопкой на опции Фабрики соединений темы и выберите Создать. На вкладке Общие появятся следующие поля.

Фабрика соединений темы провайдера по умолчанию версии 5 имеет следующие свойства.

Имя

Имя, идентифицирующее эту фабрику соединений очереди для администрирования в WebSphere Application Server. Имя должно быть уникальным среди фабри соединений JMS в административном домене WebSphere Application Server.

Тип данных Строка
Описание

Описание данной фабрики соединений темы для администрирования в WebSphere Application Server.

Тип данных Строка
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

ИД пользователя

ИД пользователя, используемый вместе со свойством Пароль, для идентификации если вызывающее приложений не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Свойства ИД пользователя и Пароль фабрики соединений используются если вызывающее приложение не указывает явно ИД пользователя и пароль, например, если вызывающее приложение использует метод createQueueConnection(). Клиент JMS передает ИД пользователя и пароль серверу JMS.

Тип данных Строка
Пароль

Пароль, используемый вместе со свойством ИД пользователя, для идентификации если вызывающее приложение не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Тип данных Строка
Повторно укажите пароль

Подтверждение пароля.

Узел

Имя узла WebSphere Application Server для административного узла, в котором выполняется сервер JMS для этой фабрики соединений. Соединения, созданные этой фабрикой соединений, подключаются к указанному серверу JMS.

Тип данных Перечисление
Диапазон Выпадающий список узлов административного домена WebSphere Application Server.
Application Server

Укажите имя сервера приложений. Это не имя хоста системы, а имя настроенного сервера приложений.

Порт

Какой из двух портов соединений используется для подключения к серверу JMS. Порт QUEUED используется для полнофункциональной поддержки публикации/подписки JMS, тогда как порт DIRECT используется только для непостоянной, нетранзакционной, негарантированной подписки.

Прим.: Управляемые сообщениями объекты не могут использовать порт прямого прослушивания для поддержки публикации или подписки. Таким образом, все фабрики соединений темы, настроенные со значением Напрямую свойства Порт, не могут быть использованы с управляемыми сообщениями объектам.
Тип данных Перечисление
Значение по умолчанию QUEUED
Диапазон
QUEUED
Порт получателя запросов, используемый с полнофункциональной совместимой с JMS поддержкой публикации и подписки.
DIRECT
Порт получателей запросов, используемый для установки прямых соединений TCP/IP (только для нетранзакционных, непостоянных и негарантированных подписок) для поддержки публикации или подписки.

Номера портов TCP/IP для этих портов задаются на внутреннем сервере JMS продукта.

ИД клиента

Идентификатор клиента JMS, используемый для установки подключений к администратору очередей MQSeries.

Тип данных Строка
Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
