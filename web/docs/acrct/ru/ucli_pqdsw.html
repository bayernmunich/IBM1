Параметры адресата очереди провайдера по умолчанию версии 5 для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранного адресата очереди для применения с провайдером JMS (Java Message Service) для продукта.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер по умолчанию версии 5. Щелкните правой кнопкой на Адресатах очереди и выберите Создать. На вкладке Общие появятся следующие поля.

Адресат очереди используется для настройки свойство очереди JMS. адресат очереди провайдера по умолчанию версии 5 имеет следующие свойства.

Имя

Имя, идентифицирующее очередь для административных задач в WebSphere Application Server.

Тип данных Строка
Описание

Описание очереди для административных задач.

Тип данных Строка
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

Сохранение состояния

Указывает, являются ли все сообщения, отправляемые в адресаты, постоянными, временными или их постоянство задается приложением.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Определяется приложением
Постоянство сообщений в местоположении определяется приложением, поместившим их в очередь.
Определяется очередью
[Только для местоположений WebSphere MQ] Постоянство сообщений в местоположении определяется свойствами определения очереди WebSphere MQ.
Хранимое
Сообщения в местоположении являются хранимыми.
Временное
Сообщения в местоположении не являются хранимыми.
Приоритет

Указывает, задается ли приоритет сообщения для данного адресата приложением или свойством Указанный приоритет.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Определяется приложением
Приоритет сообщений в данном местоположении определяется приложением, поместившим их в это местоположение.
Определяется очередью
[Только для местоположений WebSphere MQ] Постоянство сообщений в местоположении определяется свойствами определения очереди WebSphere MQ.
Заданный
Приоритет сообщений этого адресата задается свойством Указанный приоритет. При выборе этого варианта следует задать свойство Указанный приоритет.
Указанный приоритет

Если для свойства Приоритет задано значение Указан, введите приоритет сообщений для этой очереди в диапазоне от 0 (низший) до 9 (высший).

Если для свойства Приоритет задано значение Указан, отправляемые в эту очередь сообщения используют значение приоритета, указанное в этом свойстве.

Тип данных Целое число
Единицы измерения Уровень приоритета сообщения
Диапазон От 0 (низший приоритет) до 9 (высший приоритет)
Срок действия

Указывает, задается ли тайм-аут для этой очереди приложением, свойством Указанный срок хранения, или сообщения в очереди имеют неограниченный срок хранения.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Определяется приложением
Тайм-аут срока хранения для сообщений в этой очереди задается приложением, поместившим их в нее.
Заданный
Тайм-аут срока хранения сообщений в этой очереди задается свойством Указанный срок хранения. При выборе этой опции следует указать тайм-аут в свойстве Указанный срок хранения.
Не ограничено
Сообщения в этой очереди не имеют тайм-аута срока хранения.
Указанный срок хранения

Если для свойства Тайм-аут срока хранения задано значение Указан, введите число миллисекунд (больше 0), по истечении которых сообщение в этой очереди будет удалено.

Тип данных Целое число
Единицы измерения Миллисекунды
Диапазон Больше либо равно 0
  • 0 указывает на бесконечный срок хранения сообщений.
  • Другие значения должны быть целочисленными и задают число миллисекунд.
Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
