Параметры фабрики соединений темы провайдера WebSphere MQ для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранной фабрики соединений для применения с провайдером JMS для WebSphere MQ®. Эти свойства конфигурации контролируют способ создания соединений между провайдером JMS и WebSphere MQ.

Для просмотра страницы инструмента настройки ресурсов приложения-клиента (ACRCT) выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер WebSphere MQ. Щелкните правой кнопкой на опции Фабрики соединений темы и выберите Создать.

Примечание:
  • Указываемые вами значения свойств должны совпадать со значениями, указанными при настройке ресурсов JMS продукта WebSphere MQ. Дополнительная информация о настройке ресурсов JMS для WebSphere MQ приведена в разделе Работа с Java справочной системы WebSphere MQ Information Center.
  • В WebSphere MQ имена могут иметь не более 48 символов, за исключением каналов, имеющих не более 20 символов.
Посредник MA0C: При создании фабрики соединений темы WebSphere Application Server версии 6 для посредника MA0C нужно учитывать следующие значения атрибутов:
BrokerControlQueue
Это значение фиксируется в SYSTEM.BROKER.CONTROL.QUEUE для посредника MA0C и указывает очередь, их которой посредник выполняет чтение.
BrokerVersion
Задайте значение этого атрибута равным BASIC для применения посредника MA0C.
ClientID
Задайте нужное значение для посредника MA0C (это строковое значение является простым идентификатором для приложения-клиента).
Включен XA
Задайте для этого атрибута значение TRUE или FALSE для посредника MA0C (это флаг расширения производительности, скорее всего, в большинстве случаев вы будете использовать значение 'true').
BrokerMessage Selection
Это значение фиксируется в CLIENT для посредника MA0C так как посредник зависит от выбора сообщений на стороне клиента.
Тип прямой проверки прав доступа посредника
Это значение посреднику MA0C не требуется.

Хранилище сертификатов SSL

Указывает список из нуля или нескольких серверов списка аннулированных сертификатов (CRL), используемых для проверки аннулирования сертификатов SSL. Если указать значение для этого свойства, необходимо использовать WebSphere MQ JVM в Java 2 версии 1.4.

Значением является список разделенных пробелом записей в следующем виде:
ldap://имя-хоста:[порт]

После этого значения нужно добавить одну косую черту (/). Если порт не указан, используется порт LDAP по умолчанию 389. При установке соединения сертификат SSL, представленный сервером, проверяется относительно указанных серверов CRL. Дополнительная информация о защите CRL приведена в разделе "Работа со списками аннулированных сертификатов" в главе Защита справочной системы WebSphere MQ Information Center.

Равноправное имя SSL

Для SSL каркас отличительного имени, который должен соответствовать имени, предоставленному администратором очередей WebSphere MQ. Отличительное имя используется для проверки сертификата идентификации, представленного сервером при установке соединения.

Если это свойство не задано, выполняется подобная проверка сертификата.

Свойство Равноправное имя SSL игнорируется, если не задано свойство Комплект шифров SSL.

Это свойство представляет собой список пар имя атрибута/значение, разделенных запятой или точкой с запятой. Пример: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

В данном примере проверяется идентификационный сертификат, представленный сервером во время соединения. Для успешного соединения сертификат должен иметь имя, которое начинается с QMGR., и по крайней мере два имени подразделений организации, первое из которых - IBM, а второе - WEBSPHERE. Проверка выполняется без учета регистра.

Дополнительная информация об отличительных именах и особенностях их применения в WebSphere MQ приведена в разделе "Отличительные имена" в главе Защита справочной системы WebSphere MQ Information Center.

Пул соединений

Необязательный набор параметров пула соединений.

Параметры пула соединений являются общими для всех коннекторов J2C.

Для увеличения производительности сервер приложений объединяет соединения и сеансы в пулы с помощью провайдера JMS. Этот процесс является независимым от объединения в пулы соединений WebSphere MQ. Необходимо настроить свойства пулов соединений и сеансов в соответствии с имеющимися приложениями, иначе работа соединений и сеансов может не соответствовать ожидаемому алгоритму.

Измените размер пула соединений, если параллельный доступ на сервере к ресурсу JMS превышает значение по умолчанию. Размер пула соединений задается для каждой темы или очереди.

Тайм-аут повторного соединения клиента

Максимальное количество времени, в секундах, которое соединение в клиентском режиме будет тратить на повторное соединение с администратором очереди после сбоя администратора связи или очереди. Этот параметр игнорируется, пока фабрика соединений используется в среде тонкого или управляемого клиента. Используется этот параметр или нет зависит от значения параметра опций повторного соединения клиента.

Тип данных Целое число
Единицы сек
Значение по умолчанию 1800
Диапазон Значение в диапазоне от 1 до 2147483647.
