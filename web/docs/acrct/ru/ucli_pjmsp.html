Параметры провайдера JMS для приложений-клиентов

С помощью этой страницы можно настроить свойства провайдера Java Message Service (JMS) при необходимости использовать провайдер JMS, отличный от провайдера обмена сообщениями по умолчанию или WebSphere MQ, выполняющего роль провайдера JMS.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR. Щелкните правой кнопкой мыши на опции Провайдеры обмена сообщениями и выберите Создать. На вкладке Общие появятся следующие поля.

Имя

Имя провайдера JMS для административных целей.

Тип данных Строка
Описание

Описание провайдера JMS для административных задач.

Тип данных Строка
Путь к классам

Список путей или имен файлов .jar, вместе указывающих на расположение классов провайдера ресурсов.

Класс фабрики контекста

Имя класса Java исходной фабрики контекста для провайдера JMS.

Например, для провайдера служб LDAP значение имеет следующую форму: com.sun.jndi.ldap.LdapCtxFactory.

Тип данных Строка
URL провайдера

URL провайдера JMS для внутреннего поиска JNDI.

Например, URL LDAP для провайдера JMS имеет форму ldap://хост.компания.com/имя_контекста.

Тип данных Строка
Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.


Имя файла: ucli_pjmsp.html
