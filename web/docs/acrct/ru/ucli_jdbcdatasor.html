Свойства источника данных для приложений-клиентов

На этой странице можно создать или изменить источники данных.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры источника данных > Экземпляр провайдера источника данных. Щелкните правой кнопкой на пункте Источники данных и нажмите кнопку Создать. На вкладке Общие появятся следующие поля.

Имя

Указывает отображаемое имя этого источника данных.

Тип данных Строка
Описание

Указывает текстовое описание источника данных.

Тип данных Строка
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

Имя базы данных

Имя базы данных, к которой выполняется подключение.

Пользователь

Если вызывающее приложение не указывает явно ИД пользователя и пароль, используйте свойство ИД пользователя и Пароль для идентификации.

Если вы указали значение свойства ИД пользователя, укажите также и значение свойства Пароль. Свойства ИД пользователя и Пароль фабрики соединений используются если вызывающее приложение явно не предоставило ИД пользователя и пароль.

Пароль

Используйте пароль вместе со свойством ИД пользователя для идентификации если вызывающее приложение не указало явно ИД пользователя и пароль.

Если вы указали значение для свойства Пароль, укажите также и значение свойства ИД пользователя.

Повторно укажите пароль

Подтверждение пароля.

Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
