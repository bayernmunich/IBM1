Параметры фабрики соединений очереди провайдера WebSphere MQ для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранной фабрики соединений очереди для применения с провайдером JMS продукта MQSeries®. Эти свойства конфигурации контролируют способ создания соединений между провайдером JMS и WebSpheree® MQ.

Для просмотра страницы инструмента настройки ресурсов приложения-клиента (ACRCT) выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер WebSphere MQ. Щелкните правой кнопкой на опции Фабрики соединений очереди и выберите Создать. На вкладке Общие появятся следующие поля.

Примечание:
  • Указываемые вами значения свойств должны совпадать со значениями, указанными при настройке ресурсов JMS продукта WebSphere MQ. Дополнительная информация о настройке ресурсов JMS для WebSphere MQ приведена в разделе Работа с Java справочной системы WebSphere MQ Information Center.
  • В WebSphere MQ имена могут иметь не более 48 символов, за исключением каналов, имеющих не более 20 символов.

Тип транспортного протокола

Указывает, используется ли для подключения к администратору очередей WebSphere MQ клиентское соединение WebSphere MQ иди привязка JNDI. Внешний провайдер JMS контролирует протокол связи между клиентами JMS и серверами JMS. Настройте тип транспортного протокола при использовании временных не ASF, негарантированный, нетранзакционный обмен сообщениями или необходимо выполнить требования защиты, а клиент является локальным для узла администратора очередей.

Тип данных Перечисление
Единицы Не применимо
Значение по умолчанию BINDINGS
Диапазон
BINDINGS
Привязки JNDI, используемые для соединения с администратором очередей. BINDINGS является протоколом общего использования памяти и может применяться только когда администратор очередей находится в том же узле, что и клиент JMS и представляет риски безопасности, которые должны быть устранены с помощью применения ролей EJB.
CLIENT
Соединение с клиентом WebSphere MQ применяется для соединения с администратором очередей. CLIENT является типичным основанным на TCP протоколом.
DIRECT
Для посредника сообщений WebSphere MQ в режиме DIRECT. DIRECT - это легковесный протокол сокетов, используемый при нетранзакционном, негарантированном и непостоянном обмене сообщениями Публиковать/Подписаться. DIRECT подходит только для клиентов и управляемых сообщениями объектов, использующих протокол не-ASF.
QUEUED
QUEUED - это стандартный протокол.
Рекомендуется
Тип транспортного протокола фабрики соединений очереди
BINDINGS быстрее на 30% и более, однако менее защищен. Если уровень защиты для вас важен, использовать BINDINGS предпочтительней, чем CLIENT.
Тип транспортного протокола фабрики соединений тем
DIRECT является самым быстрым типом транспортного протокола и его следует использовать при любой возможности. Используйте BINDINGS при необходимости выполнить дополнительные задачи защиты, а администратор очередей является локальным для клиента JMS. QUEUED можно применять во всех остальных случаях. WebSphere MQ 5.3 до CSD2 с настройкой DIRECT может терять сообщения при использовании с управляемыми сообщениями объектами и при повышенной нагрузке. Также это может произойти с клиентскими приложениями, если только свойство maxClientQueueSize посредника на задано равным 0. Чтобы задать для него значение 0 можно использовать команду: 
#wempschangeproperties WAS_nodeName_server1
 -e default -o DynamicSubscriptionEngine -n
 maxClientQueueSize -v 0 -x executionGroupUUID
где executionGroupUUID можно определить, запустив посредника и выполнив поиск события 2201 в Журнале событий/Приложениях. Обычно это значение равно ffffffff-0000-0000-000000000000.
Примечание: Использовать WebSphere MQ 5.3 JMS с WAS 6.1 нельзя, так как WAS 6.1 содержит среду выполнения Java 5. Таким образом, соединения между разной памятью установить с помощью администратора очередей WebSphere MQ 5.3 нельзя. Это может привести к снижению производительности если ранее использовался WebSphere MQ 5.3 и BINDINGS для установки соединений и переход к сетевым соединениям CLIENT при переходе к WebSphere Application Server 6.1. При использовании WebSphere MQ 5.3 для z/OS® может потребоваться приобрести дополнительный пакет компонентов.

При выполнении в 64-разрядной z/OS тип транспортного протокола должен быть указан как CLIENT, так как 64-разрядный WebSphere MQ z/OS пока недоступен, а режим BINDINGS нельзя использовать для подключения к 31-разрядному WebSphere MQ z/OS. Вам также может потребоваться приобрести дополнительный пакет компонентов WebSphere MQ для этой поддержки.

Хранилище сертификатов SSL

Указывает список из нуля или нескольких серверов списка аннулированных сертификатов (CRL), используемых для проверки аннулирования сертификатов SSL. Если указать значение для этого свойства, необходимо использовать WebSphere MQ JVM в Java 2 версии 1.4.

Значением является список разделенных пробелом записей в следующем виде:
ldap://имя-хоста:[порт]

После этого значения нужно добавить одну косую черту (/). Если порт не указан, используется порт LDAP по умолчанию 389. При установке соединения сертификат SSL, представленный сервером, проверяется относительно указанных серверов CRL. Дополнительная информация о защите CRL приведена в разделе "Работа со списками аннулированных сертификатов" в главе Защита справочной системы WebSphere MQ Information Center.

Равноправное имя SSL

Для SSL каркас отличительного имени, который должен соответствовать имени, предоставленному администратором очередей WebSphere MQ. Отличительное имя используется для проверки сертификата идентификации, представленного сервером при установке соединения.

Если это свойство не задано, выполняется подобная проверка сертификата.

Свойство Равноправное имя SSL игнорируется, если не задано свойство Комплект шифров SSL.

Это свойство представляет собой список пар имя атрибута/значение, разделенных запятой или точкой с запятой. Пример: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

В данном примере проверяется идентификационный сертификат, представленный сервером во время соединения. Для успешного соединения сертификат должен иметь имя, которое начинается с QMGR., и по крайней мере два имени подразделений организации, первое из которых - IBM, а второе - WEBSPHERE. Проверка выполняется без учета регистра.

Дополнительная информация об отличительных именах и особенностях их применения в WebSphere MQ приведена в разделе "Отличительные имена" справочной системы WebSphere MQ Information Center.

Пул соединений

Задает необязательный набор параметров пула соединений.

Параметры пула соединений являются общими для всех коннекторов J2C.

Для увеличения производительности сервер приложений объединяет соединения и сеансы в пулы с помощью провайдера JMS. Этот процесс является независимым от объединения в пулы соединений WebSphere MQ. Необходимо настроить свойства пулов соединений и сеансов в соответствии с имеющимися приложениями, иначе работа соединений и сеансов может не соответствовать ожидаемому алгоритму.

Измените размер пула соединений, если параллельный доступ на сервере к ресурсу JMS превышает значение по умолчанию. Размер пула соединений задается для каждой темы или очереди.
