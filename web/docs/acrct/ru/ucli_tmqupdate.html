Обновление WebSphere MQ, выполняющего роль провайдера Java Message Service, и его конфигураций ресурсов JMS для приложений-клиентов

Вы можете обновить имеющуюся конфигурацию WebSphere MQ, выполняющего роль провайдера Java Message Service (JMS) и обновить конфигурацию фабрик соединений WebSphere MQ или адресатов WebSphere MQ.

О задаче

С помощью этой задачи можно обновить имеющуюся конфигурацию WebSphere MQ, выполняющего роль провайдера Java Message Service (JMS) и обновить конфигурацию фабрик соединений WebSphere MQ или адресатов WebSphere MQ.

Процедура

  1. Запустите инструмент настройки ресурсов приложения-клиента (ACRCT).
  2. Откройте файл EAR, содержащий провайдер JMS WebSphere MQ, фабрику соединений WebSphere MQ или адресат WebSphere MQ. Содержимое файла EAR отображается в иерархическом списке навигации.
  3. Выберите файл архива Java (JAR), содержащий провайдер JMS, фабрику соединений или адресат для обновления.
  4. Разверните файл JAR для просмотра его содержимого и поиска определенного провайдера JMS, фабрики соединений или адресата, который требуется обновить. Выполните одно из следующих действий:
    • Щелкните правой кнопкой мыши на соответствующем объекте и выберите Свойства.
    • Выберите в меню пункт Правка > Свойства.
  5. Обновите свойства в отображаемых полях. Более подробная справка о полях приведена в разделах:
    • Свойства провайдера JMS
    • Свойства фабрики соединений очереди MQ
    • Свойства фабрики соединений темы MQ
    • Свойства адресата MQ
    • Свойства адресата темы MQ
  6. Нажмите OK.
  7. Выберите Файл > Сохранить для сохранения внесенных изменений.
