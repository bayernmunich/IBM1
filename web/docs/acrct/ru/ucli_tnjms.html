Настройка новых провайдеров JMS с помощью инструмента настройки ресурсов приложения-клиента

Вы можете создавать новые конфигурации Java Message Service (JMS) для приложения-клиента. Приложение-клиент использует службу обмена сообщениями через интерфейсы JMS.

О задаче

При выполнении этой задачи вы создадите конфигурации провайдера Java Message Service (JMS) для приложения клиента. Приложение-клиент использует службу обмена сообщениями через интерфейсы JMS. Провайдер JMS предоставляет два типа ресурсов J2EE. Одним из них является фабрика соединений JMS, а вторым - адресат JMS.

В отдельной административной задаче следует установить клиент JMS в клиентской системе, в которой располагается определенное приложение-клиент. Реализация клиента JMS предоставляется производителем продукта обмена сообщениями. Дополнительные сведения приведены в документации продукта обмена сообщениями.

Процедура

  1. Запустите инструмент настройки ресурсов приложения-клиента и откройте файл EAR для которого требуется настроить новый провайдер JMS. Содержимое файла EAR будет показано в иерархическом списке.
  2. Выберите из иерархического списка файл JAR в котором требуется настроить новый провайдер JMS.
  3. Разверните файл JAR для просмотра его содержимого.
  4. Щелкните правой кнопкой на опции Провайдеры обмена сообщениями. Выполните одно из следующих действий:
    • Щелкните правой кнопкой на папке и выберите Создать.
    • В строке меню выберите Правка > Создать.
  5. В открывшемся окне диалога настройте свойства провайдера JMS.
  6. Для завершения нажмите кнопку OK.
  7. Выберите в строке меню пункт Файл > Сохранить для сохранения изменений.

Пример

В следующем примере рассмотрены особенности настройки параметров провайдера JMS, фабрики соединений JMS и получателя JMS для приложений-клиентов.

<resources.jms:JMSProvider xmi:id="JMSProvider_3" name="genericJMSProvider:name"
description="genericJMSProvider:description"
externalInitialContextFactory="genericJMSProvider:contextFactoryClass"
externalProviderURL="genericJMSProvider:providerUrl">
<classpath>genericJMSProvider:classpath</classpath>
<factories xmi:type="resources.jms:GenericJMSDestination"
xmi:id="GenericJMSDestination_1" name="jmsDestination:name"
jndiName="jmsDestination:jndiName" description="jmsDestination:description"
externalJNDIName="jmsDestination:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_15">
<resourceProperties xmi:id="J2EEResourceProperty_17" name="jmsDestination:customName"
value="jmsDestination:customValue"/>
</propertySet>
</factories>
<factories xmi:type="resources.jms:GenericJMSConnectionFactory"
xmi:id="GenericJMSConnectionFactory_1" name="jmsCF:name" jndiName="jmsCF:jndiName"
description="jmsCF:description" userID="jmsCF:user" password="{xor}NTIsHBllMT4yOg=="
externalJNDIName="jmsCF:externalJndiName" type="QUEUE">
<propertySet xmi:id="J2EEResourcePropertySet_16">
<resourceProperties xmi:id="J2EEResourceProperty_18" name="jmsCF:customName"
value="jmsCF:customValue"/>
</propertySet>
</factories>
<propertySet xmi:id="J2EEResourcePropertySet_17">
<resourceProperties xmi:id="J2EEResourceProperty_19"
name="genericJMSProvider:customName" value="genericJMSProvider:customValue"/>
</propertySet>
</resources.jms:JMSProvider>

К обязательным полям относятся:

  • Страница свойств провайдера JMS: имя и хотя бы один провайдер протокола
  • Страница свойств фабрики соединений JMS: имя, имя JNDI, тип адресата
  • Страница свойств адресата JMS: имя, имя JNDI, тип адресата

Особые случаи:

  • Типом адресата должен быть QUEUE или TOPIC.
