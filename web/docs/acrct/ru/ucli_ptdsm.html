Параметры адресата темы провайдера WebSphere MQ для приложений-клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранного адресата темы для применения с провайдером JMS для WebSphere MQ®.

Для просмотра страницы инструмента настройки ресурсов приложения-клиента (ACRCT) выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > Провайдер WebSphere MQ. Щелкните правой кнопкой на Адресатах тем и выберите Создать. На вкладке Общие появятся следующие поля.

Примечание:
  • Указываемые вами значения свойств должны совпадать со значениями, указанными при настройке ресурсов JMS продукта WebSphere MQ. Дополнительная информация о настройке ресурсов JMS для WebSphere MQ приведена в разделе Работа с Java справочной системы WebSphere MQ Information Center.
  • В WebSphere MQ максимальная длина имен не может превышать 48 символов.

Адресат темы используется для настройки свойство темы JMS для связанного провайдера JMS. Тема, используемая с провайдером JMS продукта WebSphere MQ, имеет следующие свойства.

Имя, идентифицирующее тему для административных задач в WebSphere Application Server.

Тип данных Строка
Описание

Описание темы для администрирования в WebSphere Application Server.

Указывает, являются ли все сообщения, отправляемые в адресаты, постоянными, временными или их постоянство задается приложением.

Тип данных Перечисление

Диапазон
Задает приложением
Постоянство сообщений, отправляемых в адресаты, определяется приложением, поместившим их в очередь.
Задается очередью
[Только для местоположений WebSphere MQ] Постоянство сообщений в местоположении определяется свойствами определения очереди WebSphere MQ.

Неустойчивый

Приоритет

Указывает, задается ли приоритет сообщения для данного адресата приложением или свойством Указанный приоритет.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон

Указанный приоритет

Если для свойства Приоритет задано значение Указан, введите приоритет сообщений для этой очереди в диапазоне от 0 (низший) до 9 (высший).

Если для свойства Приоритет задано значение Указан, отправляемые в эту очередь сообщения используют значение приоритета, указанное в этом свойстве.

Диапазон От 0 (низший приоритет) до 9 (высший приоритет)
Срок хранения

Указывает, задается ли тайм-аут для этой очереди приложением, свойством Указанный срок хранения, или сообщения в очереди имеют неограниченный срок хранения.

Тип данных Перечисление
Значение по умолчанию APPLICATION_DEFINED
Диапазон
Задает приложением
Тайм-аут срока хранения для сообщений в этой очереди задается приложением, поместившим их в нее.
Указан
Тайм-аут срока хранения для сообщений в этой очереди задается свойством Указанный срок хранения. При выборе этой опции следует указать значение тайм-аута для свойства Указанный срок хранения.
Не ограничено
Сообщения в этой очереди не имеют тайм-аута срока хранения.
Указанный срок хранения

Если для свойства Тайм-аут срока хранения задано значение Указан, введите число миллисекунд (больше 0), по истечении которых сообщение в этой очереди будет удалено.

Тип данных Целое число
Единицы Миллисекунды
Диапазон Больше либо равно 0
  • 0 указывает на бесконечный срок хранения сообщений.
  • Другие значения должны быть целочисленными и задают число миллисекунд.

Тело сообщения

Задает, нужно ли приложению обрабатывать заголовок RFH версии 2 сообщения WebSphere MQ как часть текста сообщения JMS.

Тип данных Выпадающий список
Значение по умолчанию UNSPECIFIED
Диапазон
UNSPECIFIED
При отправке сообщений, созданием заголовка RFH версии 2 провайдером сообщений WebSphere MQ управляет свойство Добавлять заголовки RFH версии 2 в сообщения, отправляемые этому получателю. При получении сообщений, провайдер сообщений WebSphere MQ работает таким образом, как если бы было указано значение JMS.
JMS
При отправке сообщений, провайдер сообщений WebSphere MQ автоматически создает заголовок RFH версии 2 и добавляет его в сообщение WebSphere MQ. При получении сообщений, провайдер сообщений WebSphere MQ задает свойства сообщения JMS с учетом содержимого заголовка RFH версии 2 (если он задан); заголовок RFH версии 2 не рассматривается как часть тела сообщения JMS.
MQ
При отправке сообщений, провайдер сообщений WebSphere MQ не создает заголовок RFH версии 2. В ходе приема сообщений провайдер сообщений WebSphere MQ добавляет заголовок RFH версии 2 в тело сообщения JMS.
