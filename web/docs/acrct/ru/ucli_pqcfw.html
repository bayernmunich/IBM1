Параметры фабрики соединений очереди провайдера по умолчанию версии 5 для приложений-клиентов

С помощью этой панели можно просмотреть или изменить параметры конфигурации выбранной фабрики соединений очередей JMS для двухточечного обмена сообщениями для работы с приложениями WebSphere Application Server версии 5. Эти свойства конфигурации управляют способом создания соединений между модулем JMS и системой обмена сообщениями по умолчанию, которая его использует.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдер обмена сообщениями > Провайдер по умолчанию версии 5. Щелкните правой кнопкой на опции Фабрики соединений очереди и выберите Создать. На вкладке Общие появятся следующие поля.

Фабрика соединений очереди используется для создания соединений JMS с адресатами очереди. Фабрика соединений очереди создается внутренним провайдером JMS продукта WebSphere Application Server. Фабрика соединений очереди провайдера по умолчанию версии 5 имеет следующие свойства:

Имя

Имя фабрики очередей соединений для административных целей на сервере IBM WebSphere Application Server. Имя должно быть уникальным среди фабрик соединений JMS в административном домене WebSphere.

Тип данных Строка
Описание

Описание данной фабрики соединений для административных целей на сервере IBM WebSphere Application Server.

Тип данных Строка
Значение по умолчанию Null
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

ИД пользователя

ИД пользователя, используемый вместе со свойством Пароль, для идентификации если вызывающее приложений не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Свойства ИД пользователя и Пароль используются если вызывающее приложение не указывает явно ИД пользователя и пароль, например, если вызывающее приложение использует метод createQueueConnection(). Клиент JMS передает ИД пользователя и пароль серверу JMS.

Тип данных Строка
Пароль

Пароль, используемый вместе со свойством ИД пользователя, для идентификации если вызывающее приложение не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Повторно укажите пароль

Подтверждение пароля.

Узел

Имя административного узла WebSphere, на котором запущен сервер JMS для данной фабрики соединений. Соединения, созданные этой фабрикой соединений, подключаются к указанному серверу JMS.

Тип данных Строка
Application Server

Укажите имя сервера приложений. Это не имя хоста системы, а имя настроенного сервера приложений.

Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
