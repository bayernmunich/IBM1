Параметры общей фабрики соединений JMS для приложений клиентов

С помощью этой панели можно просмотреть или изменить свойства конфигурации выбранной фабрики соединений Java Message Service (JMS) для применения со связанным провайдером JMS. Эти свойства конфигурации контролируют способ создания соединений между провайдером JMS и используемой им системой обмена сообщениями.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > создать_экземпляр_провайдера_JMS. Щелкните правой кнопкой мыши на пункте Фабрики соединений и выберите Создать. На вкладке Общие появятся следующие поля.

Фабрика соединений Java Message Service (JMS) создаст соединения с адресатами JMS. Фабрика соединений JMS создается связанным провайдером JMS. Фабрика соединений JMS для общего провайдера JMS (отличного от внутреннего провайдера обмена сообщениями по умолчанию или WebSphere MQ, выступающего в роли провайдера JMS) имеет следующие свойства:

Имя

Имя фабрики соединений JMS для административных целей на сервере IBM WebSphere Application Server. Имя должно быть уникальным в связанном провайдере.

Тип данных Строка
Описание

Описание данной фабрики соединений для административных целей на сервере IBM WebSphere Application Server.

Тип данных Строка
Значение по умолчанию Null
Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

ИД пользователя

Указывает ИД пользователя, используемый со свойством Пароль для идентификации если вызывающее приложение не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Свойства ИД пользователя и Пароль используются если вызывающее приложение не указывает явно ИД пользователя и пароль, например, если вызывающее приложение использует метод createQueueConnection(). Клиент JMS передает ИД пользователя и пароль серверу JMS.

Тип данных Строка
Пароль

Пароль, используемый вместе со свойством ИД пользователя, для идентификации если вызывающее приложение не указывает явно ИД пользователя и пароль.

Если указать значение свойства ИД пользователя, следует также задать значение свойства Пароль.

Тип данных Строка
Значение по умолчанию Null
Повторно укажите пароль

Подтверждение указанного в поле Пароль пароля.

Внешнее имя JNDI

Имя JNDI, используемое для привязки очереди к пространству имен сервера приложений.

Используйте полное имя JNDI, например, jms/Имя, где Имя - это логическое имя ресурса.

Это имя используется для ссылки на сведения привязки платформы. Привязка связывает ресурс, заданный файлом описания модуля, с фактическим (физическим) ресурсом, связанным с JNDI API платформой.

Тип данных Строка
Тип соединения

Является ли этот адресат очередью (для двухточечного соединения) или темой (для публикации или подписки).

Выберите одну из следующих опций:
Очередь
Местоположение очереди JMS для двухточечного обмена сообщениями.
Раздел
Адресат темы JMS для обмена сообщениями публикации и подписки.
Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
