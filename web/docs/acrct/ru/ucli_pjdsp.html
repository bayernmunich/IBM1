Параметры общего адресата JMS для приложений клиентов

Используйте эту панель для просмотра или изменения свойств конфигурации выбранного адресата JMS для применения со связанным провайдером JMS.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Провайдеры обмена сообщениями > новый_экземпляр_провайдера_JMS. Щелкните правой кнопкой на пункте Адресаты и нажмите кнопку Создать. На вкладке Общие появятся следующие поля.

Адресат JMS используется для настройки свойств адресата JMS для связанного общего адресата JMS. Соединения с адресатом JMS создаются связанной фабрикой соединений JMS. Адресат JMS для применения с общим провайдером JMS (отличным от провайдера обмена сообщениями по умолчанию или WebSphere MQ, выполняющим роль провайдера JMS) обладает следующими свойствами.

Имя

Имя, идентифицирующее очередь для административных задач в WebSphere Application Server.

Тип данных Строка
Описание

Описание очереди для административных задач.

Имя JNDI

Имя JNDI фактического (физического) имени адресата JMS, привязанного к JNDI.

Внешнее имя JNDI

Имя JNDI, используемое для привязки очереди к пространству имен сервера приложений.

По соглашению, следует использовать полные имена JNDI; например, в форме jms/имя, где имя - это логическое имя ресурса.

Это имя используется для ссылки на сведения привязки платформы. Привязка связывает ресурсы, определенные в файле описания модуля, с фактическими (физическими) ресурсами, соединенные платформой в JNDI.

Тип данных Строка
Тип адресата

Является ли этот адресат очередью (для двухточечного соединения) или темой (для публикации или подписки).

Выберите одну из следующих опций:
Очередь
Местоположение очереди JMS для двухточечного обмена сообщениями.
Раздел
Адресат темы JMS для обмена сообщениями публикации и подписки.
Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
