Параметры провайдера URL для приложений-клиентов

Используйте эту страницу для создания новых провайдеров URL.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR. Щелкните правой кнопкой мыши на опции Провайдеры URL и выберите Создать. На вкладке Общие появятся следующие поля.

Провайдер URL реализует функцию для определенного протокола URL, такого как HTTP. Этот провайдер, состоящий из классов, расширяет классы java.net.URLStreamHandler и java.net.URLConnection.

Имя

Административное имя для URL.

Описание

Необязательное описание URL для административных записей.

Путь к классам

Указывает список путей или имен файлов JAR, вместе образующих расположение классов провайдера ресурса.

Протокол

Поддерживаемый обработчиком потока протокол. Например, nntp, smtp, ftp и так далее.

Для использования протокола по умолчанию оставьте это поле пустым.

Класс обработчика потока

Полное имя пользовательского класса Java, расширяющего java.net.URLStreamHandler для определенного протокола URL, такого как FTP.

Для применения обработчика потока по умолчанию оставьте это поле пустым.

Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
