Параметры почтового сеанса для приложений-клиентов

С помощью этой страницы можно настроить свойства почтового сеанса.

Для того чтобы открыть эту страницу Application Client Resource Configuration Tool (ACRCT), выберите Файл > Открыть. Выберите файл EAR и нажмите кнопку Открыть. Разверните выбранный файл JAR > Почтовые провайдеры > экземпляр почтового провайдера. Щелкните правой кнопкой на опции Почтовые сеансы и выберите Создать. На вкладке Общие появятся следующие поля:

Имя

Представляет административное имя объекта сеанса JavaMail.

Описание

Предоставляет необязательное описание для административных записей.

Имя JNDI

Это поле используется при выполнении приложения-клиента для извлечения сведений о настройке.

Хост почтового транспортного протокола

Указывает сервер для подключения при отправке почты.

Почтовый транспортный протокол

Указывает транспортный протокол для применения при отправке почты.

Пользователь почтового протокола почты

Указывает ИД пользователя, применяемый когда хост почтового транспортного протокола требует выполнения идентификации.

Пароль почтового транспортного протокола

Указывает пароль, применяемый когда хост почтового транспортного протокола требует выполнения идентификации.

Включить строгий анализ IP-адресов

Задает, следует ли анализировать адреса получателя строгим образом, согласно RFC 822, спецификации, созданной организацией Internet Architecture Board.

Для большинства приложений почты этот параметр обычно не используется. Синтаксис RFC 822 строго определяет понятие допустимого электронного адреса. При выборе этого параметра JavaMail будет придерживаться синтаксиса RFC 822 и отклонит адреса получателей, не соответствующие определению допустимого адреса электронной почты (согласно спецификации). Если этот параметр не выбран, JavaMail не будет придерживаться синтаксиса RFC 822 и будет принимать адреса получателей, не соответствующие спецификации. По умолчанию этот параметр не выбран. Вы можете просмотреть спецификацию RFC 822 по следующему адресу на сайте World Wide Web Consortium (W3C): http://www.w3.org/Protocols/rfc822/.

Повторно укажите пароль

Подтверждение пароля.

Отправитель почты

Задает отправитель почты.

Хост хранилища почты

Указывает имя хоста (или домена) почтовой учетной записи.

Пользователь хранилища почты

Указывает ИД пользователя почтовой учетной записи.

Пароль хранилища почты

Указывает пароль почтовой учетной записи.

Повторно укажите пароль

Подтверждение пароля.

Протокол хранилища почты

Указывает протокол, используемый для получения почты.

Отладка почты

Если задано значение true, взаимодействие JavaMail с почтовыми серверами, а также свойства этих почтовых сеансов, сохраняются в файле stdout.

Настраиваемые свойства

Указывает пары имя-значение для настройки дополнительных свойств объекта, создаваемого при выполнении этого ресурса.

Следует ввести имя, являющееся внешним свойством объекта, и значение, которое может быть преобразовано из строки в тип, необходимый методу set свойства. Допустимые свойства и значения зависят от создаваемого объекта. Список допустимых свойств и значений приведен в документации по объекту.
