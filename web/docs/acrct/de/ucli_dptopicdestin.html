Einstellungen der Topic-Destination des Standardproviders

Verwenden Sie diese Seite, um die Konfigurationseigenschaften der ausgewählten JMS-Topic-Destination für den internen JMS-Provider (Java Message Service) des Produkts, der mit WebSphere Application Server installiert wird, anzuzeigen oder zu ändern.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Messaging-Provider > Standardprovider. Klicken Sie mit der rechten Maustaste auf Topic-Destinations, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Der Name des Topic-Destination-Eintrags.

Datentyp String
Beschreibung

Eine Beschreibung des Eintrags.

Datentyp String
JNDI-Name

Der JNDI-Name, der verwendet wird, um die Definition dem Namen des Elements resource-env-ref eines Implementierungsdeskriptors zuzuordnen.

Datentyp String
Topic-Bereich

Der Name des Topic-Bereichs. Dieses Feld ist erforderlich.

Datentyp String
Standardeinstellung DEFAULT_TOPIC_SPACE
Topic-Name

Der Name des Topics. Dieses Feld ist erforderlich.

Datentyp String
Übermittlungsmodus

Der Standardmodus für Nachrichten, die an diese Destination gesendet werden.

Datentyp String
Einstellmöglichkeiten Anwendung, Persistent und Nicht persistent
Standardeinstellung Anwendung
Lebensdauer

Diese Einstellung gibt an, wie lange (gerechnet aber der Übermittlungszeit) eine Nachricht, die an diese Destination gesendet wird, vom System aufbewahrt werden soll. Der Wert 0 gibt an, dass keine Verfallszeit festgelegt wird. Wenn Sie dieses Feld nicht ausfüllen, wird der Wert des Erzeugers verwendet.

Datentyp Long
Einheiten Millisekunden
Priorität

Die Priorität für Nachrichten, die an diese Destination gesendet werden. Wenn Sie dieses Feld nicht ausfüllen, wird der Wert des Erzeugers verwendet.

Datentyp Integer
Einstellmöglichkeiten Die gültigen Werte sind 0 bis 9, wobei 0 für die niedrigste und 9 für die höchste Priorität steht.
Vorauslesen

Mit dieser Einstellung wird die Optimierung für Vorauslesen während der Nachrichtenübermittlung gesteuert.

Datentyp String
Einstellmöglichkeiten AsConnection, AlwaysOn und AlwaysOff
Standardeinstellung AsConnection
