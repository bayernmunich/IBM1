Einstellungen der JMS-Provider für Anwendungsclients

Verwenden Sie diese Seite, um die Eigenschaften des JMS-Providers (Java Message Service) zu konfigurieren, wenn Sie einen anderen JMS-Provider als den Standard-Messaging-Provider oder WebSphere MQ als JMS-Provider verwenden möchten.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Blenden Sie die Struktur der ausgewählten JAR-Datei ein. Klicken Sie mit der rechten Maustaste auf Messaging-Provider, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Der Name, unter dem der JMS-Provider für Verwaltungszweck bekannt ist.

Datentyp String
Beschreibung

Eine Beschreibung des JMS-Providers für Verwaltungszwecke.

Datentyp String
Klassenpfad

Eine Liste von Pfaden bzw. Namen von .jar-Dateien, die zusammen die Speicherposition der Ressourcenproviderklassen angeben.

Klasse der Verbindungs-Factory

Der Java-Klassenname der Ausgangskontext-Factory für den JMS-Provider.

Für einen LDAP-Service-Provider hat der Wert folgendes Format: com.sun.jndi.ldap.LdapCtxFactory.

Datentyp String
Provider-URL

Der URL des JMS-Providers für externe JNDI-Lookup-Operationen.

Beispielsweise hat ein LDAP-URL für einen JMS-Provider das Format: ldap://Hostname.Unternehmen.com/Kontextname.

Datentyp String
Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
