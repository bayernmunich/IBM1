Einstellungen der Verbindungs-Factory für Topics des Standardproviders

Verwenden Sie diese Seite, um die Konfigurationseigenschaften der ausgewählten Verbindungs-Factory für JMS-Topics für den internen JMS-Provider (Java Message Service) des Produkts, der mit WebSphere Application Server installiert wird, anzuzeigen oder zu ändern. Diese Konfigurationseigenschaften steuern, wie Verbindungen zwischen dem JMS-Provider und dem verwendeten Service Integration Bus erstellt werden.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Messaging-Provider > Standardprovider. Klicken Sie mit der rechten Maustaste auf Verbindungs-Factory für Topics, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Für die Einstellungen, die einen Standardwert haben, wird der entsprechende Wert angezeigt. Alle Einstellungen mit festen Werten haben ein Dropdown-Menü.

Name

Der Name der Topic-Verbindungs-Factory.

Datentyp String
Beschreibung

Eine Beschreibung der Topic-Verbindungs-Factory für Verwaltungszwecke in IBM WebSphere Application Server.

Datentyp String
JNDI-Name

Der JNDI-Name, der verwendet wird, um diese Topic-Verbindungs-Factory dem Implementierungsdeskriptor zuzuordnen. Dieser Eintrag ist ein resource-ref-Name.

Datentyp String
Benutzername

Der Benutzername wird zusammen mit der Eigenschaft Kennwort für die Authentifizierung verwendet, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt. Wenn Sie dieses Feld verwenden, wird das Eigenschaftsfeld Benutzername ignoriert.

Wenn Sie einen Wert für die Eigenschaft Benutzername angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Die Verbindungs-Factory-Eigenschaften Benutzername und Kennwort werden verwendet, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und Kennwort angibt. Wenn ein Benutzername und ein Kennwort angegeben sind, wird ein Authentifizierungsalias für die Factory erstellt, wobei das Kennwort verschlüsselt wird.

Datentyp String
Kennwort

Das Kennwort wird verwendet, um eine verschlüsselte Verbindung zu erstellen. Wenn Sie dieses Feld ausfüllen, wird das Kennwortfeld im Fenster "Eigenschaften" ignoriert.

Wenn Sie einen Wert für die Eigenschaft Benutzername angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Datentyp String
Kennwort bestätigen

Bestätigt das Kennwort.

Busname

Der Name des Busses, zu dem die Topic-Verbindungs-Factory eine Verbindung herstellt.

Datentyp String
Client-ID

Der Name des Clients. Dieses Feld muss für dauerhafte Topic-Subskriptionen ausgefüllt werden.

Datentyp String
Zuverlässigkeit nicht persistenter Nachrichten

Die Zuverlässigkeit, die auf nicht persistente JMS-Nachrichten angewendet wird, die mit dieser Verbindungs-Factory gesendet werden.

Wenn Sie unterschiedliche Zuverlässigkeitsoptionen für die Zustellung für einzelne JMS-Destinations festlegen möchten, können Sie diese Eigenschaft auf Wie Bus-Destination einstellen. Damit wird die Zuverlässigkeit durch die Eigenschaft "Zuverlässigkeit" der Bus-Destination bestimmt, der die JMS-Destination zugeordnet ist.

Standardeinstellung Zuverlässig, persistent
Einstellmöglichkeiten
Ohne
Für nicht persistente Nachrichten wird keine Nachrichtenzuverlässigkeit verwendet. Wenn eine nicht persistente Nachricht nicht zugestellt werden kann, wird sie verworfen.
Bestmöglich, nicht persistent
Nachrichten werden nicht auf die Platte geschrieben und verworfen, wenn der Speichercache überläuft.
Express, nicht persistent
Nachrichten werden asynchron in den persistenten Speicher geschrieben, wenn der Speichercache überläuft, aber bei einem Serverneustart nicht gespeichert.
Zuverlässig, nicht persistent
Nachrichten können beim Ausfall einer Messaging-Steuerkomponente und unter normalen Betriebsbedingungen verloren gehen.
Zuverlässig, persistent
Nachrichten können beim Ausfall einer Messaging-Steuerkomponente, aber unter normalen Betriebsbedingungen nicht verloren gehen.
Garantiert, persistent
Die höchste Stufe der Zuverlässigkeit, die eine garantierte Nachrichtenübertragung unterstützt.
Wie Bus-Destination
Die Zustelloption, die für die Bus-Destination konfiguriert ist.

Home-Schnittstelle für permanente Subskriptionen

Der Name der Home-Schnittstelle für permanente Subskriptionen.

Datentyp String
Permanente Subskriptionen freigeben

Steuert, ob permanente Subskriptionen von den Membern eines Serverclusters in Verbindungen gemeinsam genutzt werden.

Normalerweise kann zu einer Zeit nur eine Sitzung einen TopicSubscriber für eine bestimmte dauerhafte Subskription haben. Mit dieser Eigenschaft können Sie dieses Verhalten außer Kraft setzen, wenn Sie möchten, dass eine dauerhafte Subskription mehrere Konsumenten parallel haben darf.

Datentyp Auswahlliste
Standardeinstellung Im Cluster
Einstellmöglichkeiten
Im Cluster
Lässt die gemeinsame Nutzung dauerhafter Subskriptionen zu, wenn Verbindungen innerhalb eines Servercluster hergestellt werden.
Immer freigeben
Dauerhafte Subskriptionen können von mehreren Verbindungen gemeinsam genutzt werden.
Nie freigeben
Dauerhafte Subskriptionen können nicht von mehreren Verbindungen gemeinsam genutzt werden.
Vorauslesen

Mit dieser Einstellung wird die Optimierung für Vorauslesen während der Nachrichtenübermittlung gesteuert.

Standardeinstellung Default
Einstellmöglichkeiten Default, AlwaysOn und AlwaysOff
Ziel

Der Name der Workload-Manager-Zielgruppe, die die Messaging-Steuerkomponente enthält.

Datentyp String
Zieltyp

Der Typ der Workload-Manager-Zielgruppe, die die Messaging-Steuerkomponente enthält.

Standardeinstellung BusMember
Einstellmöglichkeiten BusMember, Custom, ME
Bedeutung des Ziels

Die Bedeutung (Priorität) des angegebenen Ziels.

Standardeinstellung Bevorzugt
Einstellmöglichkeiten Bevorzugt, Erforderlich
Zieltransportkette für eingehende Anforderungen

Der Name des Protokolls, das in eine Gruppe von Messaging-Steuerkomponenten aufgelöst wird.

Datentyp String
Providerendpunkte

Die durch Kommas getrennte Liste mit Endpunkten, die für die Herstellung einer Verbindung zu einem Bootstrap-Server verwendet werden.

Geben Sie eine Liste mit Endpunkttriplets der Form :Host:Port:Kette" ein, in der Kommas als Trennzeichen verwendet werden.

Beispiel 
localhost:7777:BootstrapBasicMessaging
Für diese Angaben gilt Folgendes:

BootstrapBasicMessaging entspricht dem fernen Protokoll InboundBasicMessaging (JFAP-TCP/IP).
Standardeinstellung
  • Wenn der Hostname nicht angegeben ist, wird localhost als Standardwert verwendet.
  • Wenn keine Portnummer angegeben ist, wird 7276 als Standardwert verwendet.
  • Wenn kein Kettenname angegeben ist, wird eine vordefinierte Kette wie BootstrapBasicMessaging als Standardwert verwendet.
Proximität für Verbindung

Die Proximität, die die Messaging-Steuerkomponente zum Requester haben muss.

Standardeinstellung Bus
Einstellmöglichkeiten Bus, Host, Cluster, Server
Namenspräfix für temporäre Topics

Das Präfix, das für die Namen temporärer Topics verwendet wird. Dieser Name darf maximal 12 Zeichen lang sein.

Datentyp String

Dateiname: ucli_dptopicconnect.html
