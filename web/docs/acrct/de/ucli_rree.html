Einstellungen der Ressourcenumgebungseinträge für Anwendungsclients

Auf dieser Seite können Sie die Eigenschaften von Einträgen der Ressourcenumgebung angeben.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Ressourcenumgebungsprovider > Ressourcenumgebungsinstanz. Klicken Sie mit der rechten Maustaste auf Ressourcenumgebungseinträge, und klicken Sie anschließend auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Gibt den Administrationsnamen für den Ressourcenumgebungseintrag an.

Beschreibung

Optionale Beschreibung des URL-Providers für die Administration.

JNDI-Name

Gibt den JNDI-Namen (Java Naming and Directory Interface) für die Ressource einschließlich aller Subkontexte an.

Mit diesem Namen können Sie die Bindungsinformationen der Plattformen verknüpfen. Die Bindung ordnet die im Implementierungsdeskriptor des Moduls zugeordneten Ressourcen den echten (oder physischen) Ressourcen zu, die von der Plattform in JNDI gebunden sind.

Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
