Einstellungen der Mail-Sitzungen für Anwendungsclients

Verwenden Sie diese Seite, um die Eigenschaften von Mail-Sitzungen zu konfigurieren.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Mail-Provider > Mail-Povider-Instanz. Klicken Sie mit der rechten Maustaste auf Mail-Sitzungen, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Gibt den Verwaltungsnamen des JavaMail-Sitzungsobjekts an.

Beschreibung

Gibt eine optionale Beschreibung für Ihre Verwaltungsdatensätze an.

JNDI-Name

Die Laufzeitumgebung des Anwendungsclients verwendet dieses Feld zum Abrufen von Konfigurationsdaten.

Host für Mail-Transport

Gibt den Server an, zu dem zum Senden von Mail eine Verbindung hergestellt werden soll.

Protokoll für Mail-Transport

Gibt das für das Senden von Mail zu verwendende Transportprotokoll an.

Benutzer-ID für Mail-Transport

Gibt die Benutzer-ID an, die verwendet werden soll, wenn der Mail-Transporthost eine Authentifizierung verlangt.

Kennwort für Mail-Transporte

Gibt das Kennwort an, das verwendet werden soll, wenn der Mail-Transporthost eine Authentifizierung verlangt.

Strenge Syntaxanalyse von Internet-Adressen aktivieren

Gibt an, ob die Empfängeradressen streng nach RFC 822 syntaktisch analysiert werden müssen. RFC 822 ist ein vom Internet Architecture Board herausgegebenes Spezifikationsdokument.

Diese Einstellung wird im Allgemeinen für die meisten Mail-Anwendungen nicht verwendet. Die in RFC 822 beschriebene Syntax für die Überprüfung von Adressen legt eine strikte Definition für eine gültige E-Mail-Adresse fest. Wenn Sie diese Einstellung auswählen, richtige sich JavaMail an die in RFC 822 beschriebene Syntax und weist Empfängeradressen zurück, die der Definition (gemäß Spezifikation) für gültige E-Mail-Adressen nicht entsprechen. Falls Sie die Einstellung nicht auswählen, richtet sich JavaMail nicht nach der in RFC 822 definierten Syntax und akzeptiert auch Empfängeradressen, die der Spezifikation nicht entsprechen. Standardmäßig ist diese Einstellung abgewählt. Sie finden die Spezifikation RFC 822 unter dem folgenden URL des World Wide Web Consortium (W3C): http://www.w3.org/Protocols/rfc822/.

Kennwort bestätigen

Bestätigt das Kennwort.

Mail von

Gibt den Absender der Mail an.

Host für Mail-Speicher

Der Hostname (oder Domänenname) des Mail-Account.

Benutzer für Mail-Speicher

Gibt die Benutzer-ID für den Mail-Account an.

Kennwort für Mail-Speicher

Gibt das Kennwort für den Mail-Account an.

Kennwort bestätigen

Bestätigt das Kennwort.

Protokoll für Mail-Speicher

Gibt das Protokoll an, das beim Empfangen von Mails verwendet wird.

Mail-Debug

Wenn diese Eigenschaft auf true gesetzt ist, werden JavaMail-Interaktionen mit Mail-Servern zusammen mit diesen Mail-Sitzungseigenschaften in den Standardausgabedatenstrom ausgegeben.

Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
