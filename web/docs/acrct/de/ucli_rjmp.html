Einstellungen der Mail-Provider für Anwendungsclients

Auf dieser Seite können Sie die JavaMail-API implementieren und Mail-Sitzungen erstellen.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Blenden Sie die Struktur der ausgewählten JAR-Datei ein. Klicken Sie mit der rechten Maustaste auf Mail-Provider, und klicken Sie anschließend auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Der Name des JavaMail-Ressourcenproviders.

Beschreibung

Eine optionale Beschreibung des Ressourcenproviders.

Klassenpfad

Eine Liste von Pfaden bzw. Namen von JAR-Dateien, die zusammen die Speicherposition der Ressourcen-Provider-Klassen angeben.

Protokoll

Gibt den Namen des Protokolls an.

Klassenname

Gibt den Namen der Klasse an, die das Protokoll implementiert. Lassen Sie dieses Feld leer, wenn Sie die Standardimplementierung verwenden möchten.

Typ

Dieses Menü enthält die folgenden zwei Werte: TRANSPORT bzw. STORE.

Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
