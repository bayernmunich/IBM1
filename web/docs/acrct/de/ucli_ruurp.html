Einstellungen für den URL-Provider für Anwendungsclients

Auf dieser Seite können Sie neue URL-Provider erstellen.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Blenden Sie die Struktur der ausgewählten JAR-Datei ein. Klicken Sie mit der rechten Maustaste auf URL-Provider, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Ein URL-Provider implementiert die Funktion für ein bestimmtes URL-Protokoll wie beispielsweise Hyper Text Transfer Protocol (HTTP). Dieser Provider, der aus Klassen besteht, erweitert die Klassen java.net.URLStreamHandler und java.net.URLConnection.

Name

Verwaltungsname für den URL.

Beschreibung

Optionale Beschreibung des URL für Verwaltungszwecke.

Klassenpfad

Eine Liste mit Pfaden oder Namen von JAR-Dateien, die die Adresse der Ressourcenproviderklassen bilden.

Protokoll

Von diesem Datenstrom-Handler unterstütztes Protokoll, z. B. nntp, smtp, ftp usw.

Wenn Sie das Standardprotokoll verwenden möchten, lassen Sie dieses Feld leer.

Klasse für Datenstrom-Handler

Vollständig qualifizierter Name einer angepassten Java-Klasse, die java.net.URLStreamHandler für ein bestimmtes URL-Protokoll, wie z. B. FTP, erweitert.

Wenn Sie das Standarddatenstrom-Handler verwenden möchten, lassen Sie dieses Feld leer.

Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.



Mit (online) gekennzeichnete Links setzen einen Internet-Zugang voraus.
