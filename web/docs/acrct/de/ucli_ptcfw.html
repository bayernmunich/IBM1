Einstellungen der Topic-Verbindungs-Factory des Standardproviders in Version 5 für Anwendungsclients

Verwenden Sie diese Anzeige, um die Konfigurationseigenschaften der ausgewählten Topic-Verbindungs-Factory für den internen JMS-Provider des Produkts anzuzeigen oder zu ändern. Diese Konfigurationseigenschaften steuern, wie Verbindungen zwischen dem JMS-Provider und dem verwendeten Messaging-System erstellt werden.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Klicken Sie, nachdem Sie eine EAR-Datei lokalisiert haben, auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Messaging-Provider > Standardprovider der Version 5. Klicken Sie mit der rechten Maustaste auf Verbindungs-Factorys für WebSphere-Topics, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Eine Topic-Verbindungs-Factory des Standardproviders in Version 5 hat die folgenden Eigenschaften.

Name

Der Name, unter dem diese Topic-Verbindungs-Factory zu Verwaltungszwecken in IBM WebSphere Application Server bekannt ist. Der Name muss unter den JMS-Verbindungs-Factorys innerhalb der Verwaltungsdomäne von WebSphere Application Server eindeutig sein.

Datentyp String
Beschreibung

Eine Beschreibung der Verbindungs-Factory für Verwaltungszwecke in WebSphere Application Server.

Datentyp String
JNDI-Name

Die Laufzeitumgebung des Anwendungsclients verwendet dieses Feld zum Abrufen von Konfigurationsdaten.

Benutzer-ID

Die Benutzer-ID, die in Kombination mit der Eigenschaft Kennwort zur Authentifizierung verwendet wird, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt.

Wenn Sie einen Wert für die Eigenschaft Benutzer-ID angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Die Eigenschaften Benutzer-ID und Kennwort werden verwendet, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt, z. B. bei Verwendung der Methode "createTopicConnection()". Der JMS-Client übergibt die Benutzer-ID und das Kennwort an den JMS-Server.

Datentyp String
Kennwort

Das Kennwort, das in Kombination mit der Eigenschaft Benutzer-ID zur Authentifizierung verwendet wird, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt.

Wenn Sie einen Wert für die Eigenschaft Benutzer-ID angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Datentyp String
Kennwort bestätigen

Bestätigt das Kennwort.

Knoten

Der Knotenname des Verwaltungsknotens von WebSphere Application Server, auf dem der JMS-Server für diese Verbindungs-Factory ausgeführt wird. Ziel der Verbindungen, die von dieser Factory erstellt werden, ist der JMS-Server.

Datentyp Enum
Einstellmöglichkeiten Pulldown-Liste der Knoten in der Verwaltungsdomäne von WebSphere Application Server.
Anwendungsserver

Geben Sie den Namen des Anwendungsservers ein. Dieser Name ist nicht der Hostname der Maschine, sondern der Name des konfigurierten Anwendungsservers.

Port

Gibt an, welcher der beiden Ports für Verbindungen zum JMS-Server verwendet wird. Der Porttyp "Aus der Warteschlange" ist für den mit allen Funktionen ausgestatteten JMS-Publish/Subscribe-Support, der Porttyp "Direkt" dagegen ausschließlich für nicht persistente, nicht transaktionsbasierte und nicht dauerhafte Subskriptionen vorgesehen.

Anmerkung: MDBs können den Listener-Porttyp "Direkt" nicht für die Publish/Subscribe-Unterstützung verwenden. Deshalb können Verbindungs-Factory für Topics, bei denen Port auf Direkt gesetzt ist, nicht mit MDBs verwendet werden.
Datentyp Enum
Standardeinstellung WARTESCHLANGE
Einstellmöglichkeiten
WARTESCHLANGE
Der Listener-Port, der für die vollständige, JMS-kompatible Publish/Subscribe-Unterstützung verwendet wird.
DIREKT
Der Listener-Port, der für die direkte TCP/IP-Übertragung (nur nicht transaktionsbasierte, nicht persistente und nicht dauerhafte Subskriptionen) für die Publish/Subscribe-Unterstützung verwendet wird.

Die TCP/IP-Portnummern für diese Ports sind im produktinternen JMS-Server definiert.

Client-ID

Die JMS-Client-ID, die für Verbindungen zum MQSeries-Warteschlangenmanager verwendet wird.

Datentyp String
Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
