Java-Messaging-Clientressourcen konfigurieren

Zum Konfigurieren von Java-Messaging-Clientressourcen erstellen Sie neue JMS-Providerkonfigurationen für Ihren Anwendungsclient. Der Anwendungsclient kann über die JMS-APIs (Java Message Service) einen Messaging-Service verwenden. Ein JMS-Provider stellt zwei Arten von J2EE-Factorys bereit: eine JMS-Verbindungs-Factory und eine JMS-Ziel-Factory.

Vorbereitungen

Installieren Sie in einer separaten Verwaltungs-Task den JMS-Client (Java Message Service) auf der Clientmaschine, auf der sich der Anwendungsclient befindet. Der Hersteller des Messaging-Produkts muss eine Implementierung des JMS-Clients zur Verfügung stellen. Weitere Informationen enthält die Dokumentation des Messaging-Produkts.

Anmerkung: Sie können einen neuen Messaging-Provider erstellen oder einen vorhandenen verwenden.

Prozedur

  1. Starten Sie das Application Client Resource Configuration Tool (ACRCT).
  2. Öffnen Sie die EAR-Datei, für die Sie den neuen JMS-Provider konfigurieren möchten. Der Inhalt der EAR-Datei wird in einer Baumstruktursicht angezeigt.
  3. Wählen Sie in der Baumstruktur die JAR-Datei aus, in der Sie den neuen JMS-Provider konfigurieren möchten.
  4. Erweitern Sie die JAR-Datei, um den Dateiinhalt anzuzeigen.
  5. Klicken Sie optional mit der rechten Maustaste auf Messaging-Provider, und wählen Sie Neu aus, wenn Sie einen neuen Messaging-Provider erstellen und verwenden möchten.
  6. Konfigurieren Sie in dem darauf angezeigten Eigenschaftendialog die Eigenschaften für den JMS-Provider.
  7. Klicken Sie auf OK.
  8. Klicken Sie auf Datei > Speichern.


Mit (online) gekennzeichnete Links setzen einen Internet-Zugang voraus.
