Generische Einstellungen von JMS-Destinations für Anwendungsclients

Verwenden Sie diese Anzeige, um die Konfigurationseigenschaften der ausgewählten JMS-Destination, die mit dem zugeordneten JMS-Provider verwendet werden soll, anzuzeigen oder zu ändern.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Messaging-Provider > neue_JMS-Providerinstanz. Klicken Sie mit der rechten Maustaste auf Destinations, und klicken Sie dann auf Neu. Auf der Registerkarte Allgemein werden die folgenden Felder angezeigt.

Eine JMS-Destination wird verwendet, um die Eigenschaften einer JMS-Destination für den zugeordneten generischen JMS-Provider zu konfigurieren. Die Verbindungen zur JMS-Destination werden von der zugehörigen JMS-Verbindungs-Factory erstellt. Eine JMS-Destination für einen generischen JMS-Provider (nicht den Standard-Messaging-Provider oder WebSphere MQ als JMS-Provider) hat die folgenden Eigenschaften.

Name

Der Name, unter dem die Warteschlange für Verwaltungszwecke in WebSphere Application Server bekannt wird.

Datentyp String
Beschreibung

Eine Beschreibung der Warteschlange für Verwaltungszwecke.

JNDI-Name

Der JNDI-Name des tatsächlichen (physischen) Namens der an JNDI gebundenen JMS-Destination.

Externer JNDI-Name

Der JNDI-Name, der verwendet wird, um die Warteschlange an den Namespace des Anwendungsservers zu binden.

Verwenden Sie als Konvention den vollständig qualifizierten JNDI-Namen, z. B. im Format "jms/Name". Dabei bezeichnet Name den logischen Namen der Ressource.

Dieser Name wird verwendet, um die Bindungsinformationen für die Plattform zu verlinken. Die Bindung ordnet die im Implementierungsdeskriptor des Moduls definierten Ressourcen den "echten" (physischen) Ressourcen zu, die über die Plattform an JNDI gebunden sind.

Datentyp String
Destination-Typ

Gibt an, ob diese JMS-Destination eine Warteschlange (Punkt-zu-Punkt) oder ein Topic (Publish/Subscribe) ist.

Wählen Sie eine der folgenden Optionen aus:
Warteschlange
Eine JMS-Warteschlangen-Destination für Punkt-zu-Punkt-Messaging.
Topic
Ein JMS-Topic für Publish/Subscribe-Messaging.
Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
