Einstellungen für Ressourcenadapter

Verwenden Sie diese Anzeige, um die Konfigurationseigenschaften des Ressourcenadapters anzuzeigen oder zu ändern. Diese Konfigurationseigenschaften steuern, wie Ressourcenadapter erstellt werden.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und dann auf Ressourcenadapter. Klicken Sie mit der rechten Maustaste auf Ressourcenadapter, und klicken Sie dann auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Der Name, unter dem dieser Ressourcenadapter zu Verwaltungszwecken in IBM WebSphere Application Server bekannt ist. Der Name des Ressourcenadapters muss innerhalb der Verwaltungsdomäne des Produkts eindeutig sein.

Datentyp String
Beschreibung

Eine Beschreibung des Ressourcenadapters für Verwaltungszwecke in IBM WebSphere Application Server.

Datentyp String
Klassenpfad

Ein zusätzlicher Klassenpfad. Der Pfad zum Ressourcenadapterverzeichnis wird automatisch hinzugefügt.

Datentyp String
Standardeinstellung Der Pfad zum Verzeichnis des Ressourcenadapters.
Nativer Pfad

Der native Pfad, in dem sich der Ressourcenadapter befindet. Geben Sie in diesem Feld alle zusätzlichen nativen Klassenpfade ein.

Datentyp String
Name des Ressourcenadapters

Ein Musseingabefeld, das auf das Unterverzeichnis eines installierten Ressourcenadapters zeigt. Der Eintrag ist nicht der vollständige Verzeichnisname für den Ressourcenadapter. Der vollständige Verzeichnisname umfasst den Pfad des installierten Ressourcenadapters und den Namen des Ressourcenadapters.

Datentyp String
Pfad des installierten Ressourcenadapters

Das Verzeichnis, in dem die Ressourcenadapter installiert sind. Wenn Sie dieses Feld nicht ausfüllen, werden die Standardwerte verwendet.

Wenn Sie den Wert ${CONNECTOR_INSTALL_ROOT} angeben, ersetzt dieser Wert auf der Maschine, auf der die Clientanwendung ausgeführt wird, den Wert der Variablen CLIENT_CONNECTOR_INSTALL_ROOT. Mit dieser Aktion kann die Anwendung bequem auf unterschiedlichen Maschinen ausgeführt werden, auf denen der Client in unterschiedlichen Verzeichnissen installiert sein kann.

Datentyp String
Standardeinstellung ${CONNECTOR_INSTALL_ROOT}
