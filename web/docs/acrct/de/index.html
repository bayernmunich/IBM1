  • URL-Provider für das Application Client Resource Configuration Tool
    Ein URL-Provider implementiert die Funktion für ein bestimmtes URL-Protokoll wie beispielsweise HyperText Transfer Protocol (HTTP). Dieser Provider, der aus einem Klassenpaar besteht, erweitert die Klassen java.net.URLStreamHandler und java.net.URLConnection.
  • URLs für Anwendungsclients
    Ein URL (Uniform Resource Locator) ist eine Kennung, die auf eine elektronisch verfügbare Ressource, wie z. B. eine Verzeichnisdatei auf einer Maschine in einem Netz oder ein in einer Datenbank gespeichertes Dokument zeigt.
  • Asynchrones Messaging in WebSphere Application Server mit JMS
    WebSphere Application Server unterstützt Asynchronous Messaging als eine Methode der Kommunikation, die auf der JMS-Programmierschnittstelle (Java Message Service) basiert. Die JMS-Schnittstelle bietet Java-Programmen (Clients und Java-EE-Anwendungen) eine Möglichkeit zum Erzeugen, Senden, Empfangen und Lesen asynchroner Anforderungen als JMS-Nachrichten.
  • JMS-Provider für Clients
    In diesem Artikel werden die verschiedenen Verwendungsmöglichkeiten von JMS-Providern (Java Message Service) für WebSphere Application Server in Clientanwendungen beschrieben.
  • Ressourcenadapter für den Client
    Ein Ressourcenadapter ist ein Softwaretreiber auf Systemebene, den eine Java-Anwendung für die Verbindung zu einem unternehmensweiten Informationssystem (EIS, Enterprise Information System, EIS) nutzt. Ein Ressourcenadapter integriert sich in einen Anwendungsclient und unterstützt die Konnektivität zwischen dem EIS und der Unternehmensanwendung.
  • Ressourcenumgebungsprovider und Ressourcenumgebungseinträge
    Eine Referenz auf eine Ressourcenumgebung ordnet einen von der Clientanwendung verwendeten logischen Namen dem physischen Namen eines Objekts zu.
  • Java-Messaging-Clientressourcen konfigurieren
    Zum Konfigurieren von Java-Messaging-Clientressourcen erstellen Sie neue JMS-Providerkonfigurationen für Ihren Anwendungsclient. Der Anwendungsclient kann über die JMS-APIs (Java Message Service) einen Messaging-Service verwenden. Ein JMS-Provider stellt zwei Arten von J2EE-Factorys bereit: eine JMS-Verbindungs-Factory und eine JMS-Ziel-Factory.
