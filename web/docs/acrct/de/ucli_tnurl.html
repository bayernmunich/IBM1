Neue URL-Provider für Anwendungsclients konfigurieren

Sie können URL-Provider und URLs für Ihre Clientanwendung mit Application Client Resource Configuration Tool (ACRCT) erstellen.

Vorbereitungen

Bei dieser Task erstellen Sie URL-Provider und URLs für Ihre Clientanwendung. Sie müssen in einer separaten Verwaltungs-Task den Java-Code für den erforderlichen URL-Provider auf der Clientmaschine installieren, auf der die Clientanwendung sich befindet.

Informationen zu dieser Task

Prozedur

  1. Starten Sie das Application Client Resource Configuration Tool (ACRCT).
  2. Öffnen Sie die EAR-Datei, für die Sie den neuen URL-Provider konfigurieren möchten. Der Inhalt der EAR-Datei wird in einer Baumstruktursicht angezeigt.
  3. Wählen Sie in der Baumstruktur die JAR-Datei aus, in der Sie den neuen URL-Provider konfigurieren möchten.
  4. Erweitern Sie die JAR-Datei, um den Dateiinhalt anzuzeigen.
  5. Klicken Sie auf den Ordner URL-Provider. Führen Sie eine der folgenden Aktionen aus:
    • Klicken Sie mit der rechten Maustaste auf den Ordner, und wählen Sie Neu aus.
    • Klicken Sie in der Menüleiste auf Bearbeiten > Neu.
  6. Konfigurieren Sie in dem darauf angezeigten Eigenschaftendialog die Eigenschaften für den URL-Provider.
  7. Klicken Sie auf OK.
  8. Klicken Sie in der Menüleiste auf Datei > Speichern, um Ihre Änderungen zu speichern.


Mit (online) gekennzeichnete Links setzen einen Internet-Zugang voraus.
