JMS-Provider für Clients

In diesem Artikel werden die verschiedenen Verwendungsmöglichkeiten von JMS-Providern (Java Message Service) für WebSphere Application Server in Clientanwendungen beschrieben.

IBM WebSphere Application Server unterstützt asynchrones Messaging über die Verwendung eines JMS-Providers und des zugehörigen Messaging-Systems. JMS-Provider müssen der JMS-Spezifikation Version 1.1 entsprechen. Zur Verwendung von MDBs muss der JMS-Provider die optionale Funktion Application Server Facility (ASF), die in dieser Spezifikation definiert ist, oder einen eingehenden Ressourcenadapter unterstützen, der in der JCA-Spezifikation Version 1.5 definiert ist.

Die Serviceintegrationstechnologien von IBM WebSphere Application Server können als Messaging-System eingesetzt werden, sofern Sie einen Service Integration Bus konfiguriert haben, der über den Standard-Messaging-Provider aufgerufen wird. Diese Unterstützung wird mit WebSphere Application Server installiert, in der Administrationskonsole verwaltet und ist vollständig in die Laufzeitumgebung von WebSphere Application Server integriert.

WebSphere Application Server unterstützt außerdem die folgenden JMS-Provider:
WebSphere MQ
Für die unterstützten Versionen von WebSphere MQ.
Generisch
Für alle Messaging-Systeme von Fremdanbietern, die ASF unterstützen.

Für die Abwärtskompatibilität mit früheren Releases unterstützt WebSphere Application Server auch den Standard-Messaging-Provider der Version 5, mit dem Sie Ressourcen für Embedded Messaging von WebSphere Application Server Version 5 konfigurieren können. Der Standard-Messaging-Provider der Version 5 kann auch für einen Service Integration Bus verwendet werden.

WebSphere-Anwendungen können Messaging-Ressourcen von jedem dieser JMS-Provider verwenden. Die Auswahl des Providers richtet sich jedoch häufig nach den Anforderungen eines bereits vorhandenen Messaging-Systems. Wenn Sie beispielsweise bereits eine Messaging-Infrastruktur haben, die auf WebSphere MQ basiert, können Sie über die integrierte Unterstützung für WebSphere MQ als JMS-Provider eine direkte Verbindung herstellen oder einen Service Integration Bus mit Verbindungen zu einem WebSphere-MQ-Netz konfigurieren und dann über den Standard-Messaging-Provider auf den Bus zugreifen.

Der Service Integration Bus unterstützt auch den Zugriff auf einen Standard-Messaging-Provider. Es handelt sich dabei um einen mit J2EE 1.4 kompatiblen JMS-Messaging-Provider, der vollständig in WebSphere Application Server integriert ist. Sie können diesen Messaging-Provider in mehreren Serverkonfigurationen für Messaging-Interaktionen mit einem WebSphere-MQ-Netz verwenden.
