WebSphere MQ als JMS-Provider und die zugehörigen JMS-Ressourcenkonfigurationen für Anwendungsclients aktualisieren

Sie können eine vorhandene Konfiguration von WebSphere MQ als JMS-Provider und die Konfiguration der WebSphere-MQ-Verbindungs-Factorys bzw. WebSphere-MQ-Destinations aktualisieren.

Informationen zu dieser Task

Verwenden Sie diese Task, um eine vorhandene Konfiguration von WebSphere MQ als JMS-Provider und die Konfiguration von WebSphere-MQ-Verbindungs-Factorys oder WebSphere-MQ-Destinations zu aktualisieren.

Prozedur

  1. Starten Sie das Application Client Resource Configuration Tool (ACRCT).
  2. Öffnen Sie die EAR-Datei mit dem WebSphere-MQ-JMS-Provider, der WebSphere-MQ-Verbindungs-Factory bzw. der WebSphere-MQ-Destination. Der Inhalt der EAR-Datei wird in einer der Navigationsstruktur angezeigt.
  3. Wählen Sie die JAR-Datei mit dem JMS-Provider, der Verbindungs-Factory bzw. der Destination aus, den bzw. die Sie aktualisieren möchten.
  4. Erweitern Sie die Ansicht mit dem Inhalt der JAR-Datei so weit, bis Sie den gewünschten JMS-Provider, die gewünschte Verbindungs-Factory oder Destination sehen. Führen Sie eine der folgenden Aktionen aus:
    • Klicken Sie mit der rechten Maustaste auf das Objekt, und klicken Sie dann auf Eigenschaften.
    • Klicken Sie in der Menüleiste auf Bearbeiten > Eigenschaften.
  5. Aktualisieren Sie die Eigenschaften in den angezeigten Feldern. Ausführliche Hilfe zu den Feldern finden Sie unter:
    • Eigenschaften des JMS-Providers
    • Eigenschaften der MQ-Warteschlangenverbindungs-Factory
    • Eigenschaften der MQ-Topic-Verbindungs-Factory
    • Eigenschaften der MQ-Warteschlangen-Destination
    • Eigenschaften der MQ-Topic-Destination
  6. Klicken Sie auf OK.
  7. Klicken Sie auf Datei > Speichern, um Ihre Änderungen zu speichern.
