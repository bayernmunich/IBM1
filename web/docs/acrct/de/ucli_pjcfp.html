Generische Einstellungen der JMS-Verbindungs-Factorys für Anwendungsclients

Verwenden Sie diese Anzeige, um die Konfigurationseigenschaften der ausgewählten JMS-Verbindungs-Factory, die mit dem zugeordneten JMS-Provider verwendet werden soll, anzuzeigen oder zu ändern. Diese Konfigurationseigenschaften steuern, wie Verbindungen zwischen dem JMS-Provider und dem verwendeten Messaging-System erstellt werden.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Wenn Sie die EAR-Datei gefunden haben, klicken Sie auf Öffnen. Klicken Sie auf die ausgewählte JAR-Datei und anschließend auf Messaging-Provider > neue_JMS-Providerinstanz. Klicken Sie mit der rechten Maustaste auf Verbindungs-Factorys, und klicken Sie dann auf Neu. Auf der Registerkarte Allgemein werden die folgenden Felder angezeigt.

Eine JMS-Verbindungs-Factory erstellt Verbindungen zu JMS-Destinations. Die JMS-Verbindungs-Factory wird vom zugeordneten JMS-Provider erstellt. Eine JMS-Verbindungs-Factory für einen generischen JMS-Provider (nicht der interne Standard-Messaging-Provider oder WebSphere MQ als JMS-Provider) hat die folgenden Eigenschaften:

Name

Der Name, unter dem diese JMS-Verbindungs-Factory zu Verwaltungszwecken in IBM WebSphere Application Server bekannt ist. Der Name muss innerhalb des zugeordneten JMS-Providers eindeutig sein.

Datentyp String
Beschreibung

Eine Beschreibung der Verbindungs-Factory für Verwaltungszwecke in IBM WebSphere Application Server.

Datentyp String
Standardeinstellung Ohne
JNDI-Name

Die Laufzeitumgebung des Anwendungsclients verwendet dieses Feld zum Abrufen von Konfigurationsdaten.

Benutzer-ID

Die Benutzer-ID, die in Kombination mit der Eigenschaft Kennwort zur Authentifizierung verwendet wird, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt.

Wenn Sie einen Wert für die Eigenschaft Benutzer-ID angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Die Eigenschaften Benutzer-ID und Kennwort werden verwendet, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt, z. B. bei Verwendung der Methode "createQueueConnection()". Der JMS-Client übergibt die Benutzer-ID und das Kennwort an den JMS-Server.

Datentyp String
Kennwort

Das Kennwort, das in Kombination mit der Eigenschaft Benutzer-ID zur Authentifizierung verwendet wird, wenn die aufrufende Anwendung nicht explizit eine Benutzer-ID und ein Kennwort angibt.

Wenn Sie einen Wert für die Eigenschaft Benutzer-ID angeben, müssen Sie auch einen Wert für die Eigenschaft Kennwort angeben.

Datentyp String
Standardeinstellung Ohne
Kennwort bestätigen

Bestätigt das Kennwort, das im Feld Kennwort eingegeben wurde.

Externer JNDI-Name

Der JNDI-Name, der verwendet wird, um die Warteschlange an den Namespace des Anwendungsservers zu binden.

Verwenden Sie als Konvention den vollständig qualifizierten JNDI-Namen, z. B. jms/Name. Dabei bezeichnet Name den logischen Namen der Ressource.

Dieser Name wird verwendet, um die Bindungsinformationen für die Plattform zu verlinken. Die Bindung ordnet die im Implementierungsdeskriptor des Moduls definierten Ressourcen den "echten" (physischen) Ressourcen zu, die über die Plattform an die JNDI-API gebunden sind.

Datentyp String
Verbindungstyp

Gibt an, ob diese JMS-Destination eine Warteschlange (Punkt-zu-Punkt) oder ein Topic (Publish/Subscribe) ist.

Wählen Sie eine der folgenden Optionen aus:
Warteschlange
Eine JMS-Warteschlangen-Destination für Punkt-zu-Punkt-Messaging.
Topic
Eine JMS-Topic-Destination für Publish/Subscribe-Messaging.
Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
