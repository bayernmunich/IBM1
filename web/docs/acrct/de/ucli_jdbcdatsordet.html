Einstellungen für den Datenquellenprovider von Anwendungsclients

Auf dieser Seite können Sie eine Datenquelle unter einem JDBC-Provider erstellen, der die Implementierungsklasse des spezifischen JDBC-Treibers bereitstellt.

Klicken Sie zum Anzeigen dieser ACRCT-Seite (Application Client Resource Configuration Tool) auf Datei > Öffnen. Klicken Sie, nachdem Sie eine EAR-Datei lokalisiert haben, auf Öffnen. Blenden Sie die Struktur der ausgewählten JAR-Datei ein. Klicken Sie mit der rechten Maustaste auf Datenquellenprovider, und klicken Sie anschließend auf Neu. Die folgenden Felder erscheinen unter dem Register Allgemein.

Name

Gibt den Anzeigenamen für die Datenquelle an.

Beispielsweise können Sie dieses Feld auf Datenquelle testen einstellen.

Datentyp String
Beschreibung

Gibt eine Textbeschreibung der Ressource an.

Datentyp String
Klassenpfad

Eine Liste von Pfaden bzw. Namen von .jar-Dateien, die zusammen die Speicherposition der Ressourcenproviderklassen angeben.

Implementierungsklasse

Mit dieser Einstellung können Sie datenbankspezifische Funktionen ausführen.

Datentyp String
Standardeinstellung Ist abhängig von der Implementierungsklasse des JDBC-Treibers.
Angepasste Eigenschaften

Gibt Name/Wert-Paare für die Definition zusätzlicher Eigenschaften des Objekts an, das zur Laufzeit für diese Ressource erstellt wird.

Sie müssen einen Namen eingeben, der eine öffentliche Eigenschaft des Objekts ist. Außerdem müssen Sie einen Wert eingeben, der von einer Zeichenfolge in den für die Setter-Methode der Eigenschaft erforderlichen Typ konvertiert werden kann. Die gültigen Eigenschaften und Werte sind von dem zu erstellenden Objekt abhängig. Eine Liste der gültigen Eigenschaften und Eigenschaftswerte finden Sie in der Objektdokumentation.
