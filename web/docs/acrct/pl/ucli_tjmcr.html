Konfigurowanie zasobów klienta usługi przesyłania komunikatów Java

Aby skonfigurować zasoby klienta usługi przesyłania komunikatów Java, należy utworzyć nowe konfiguracje dostawcy JMS dla klienta aplikacji. Klient aplikacji może używać usługi przesyłania komunikatów za pośrednictwem interfejsu API JMS (Java Message Service). Dostawca JMS udostępnia dwa rodzaje fabryk J2EE. Pierwszy to Fabryka połączeń JMS, a drugi Fabryka miejsc docelowych JMS.

Czynności wstępne

W osobnym zadaniu administracyjnym należy zainstalować klienta JMS (Java Message Service) na komputerze, na którym rezyduje klient aplikacji. Dostawca produktu przesyłania komunikatów musi dostarczyć implementację klienta JMS. Dodatkowe informacje zawiera dokumentacja produktu przesyłania komunikatów.

Uwaga: Wykonując to zadanie, można utworzyć nowego dostawcę przesyłania komunikatów lub używać istniejącego.

Procedura

  1. Uruchom narzędzie ACRCT (Application Client Resource Configuration Tool).
  2. Otwórz plik EAR, dla którego zostanie utworzony nowy dostawca JMS. Zawartość pliku EAR zostanie wyświetlona w widoku drzewa.
  3. Wybierz z drzewa plik JAR, w którym zostanie skonfigurowany nowy dostawca JMS.
  4. Rozwiń plik JAR, aby wyświetlić jego zawartość.
  5. Ewentualnie, kliknij prawym przyciskiem myszy opcję Dostawcy przesyłania komunikatów i wybierz opcję Nowy, aby utworzyć nowego dostawcę przesyłania komunikatów.
  6. Skonfiguruj właściwości dostawcy JMS w wyświetlonym oknie dialogowym właściwości.
  7. Kliknij przycisk OK.
  8. Kliknij opcję Plik > Zapisz.
