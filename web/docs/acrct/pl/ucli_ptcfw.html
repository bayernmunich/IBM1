Ustawienia fabryki połączeń tematów dostawcy domyślnego w wersji 5 dla klientów aplikacji

Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranej fabryki połączeń tematów dla wewnętrznego dostawcy JMS (Java Message Service) produktu. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS oraz używanym przez niego systemem przesyłania komunikatów.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Następnie należy rozwinąć wybrany plik JAR > Dostawcy przesyłania komunikatów > Dostawca domyślny wersji 5. Kliknij prawym przyciskiem myszy pozycję Fabryki połączeń tematu i wybierz opcję Nowa. Na karcie Ogólne zostaną wyświetlone następujące pola.

Fabryka połączeń tematu dostawcy domyślnego w wersji 5 ma następujące właściwości.

Nazwa

Nazwa służąca do identyfikowania tej fabryki połączeń kolejek w celach administracyjnych na serwerze WebSphere Application Server. Nazwa ta musi być unikalna w obrębie fabryk połączeń JMS dla domeny administracyjnej produktu WebSphere Application Server.

Typ danych String
Opis

Opis fabryki połączeń tematów używany w celach administracyjnych na serwerze WebSphere Application Server.

Typ danych String
Nazwa JNDI

Środowisko wykonawcze klienta aplikacji używa tego pola w celu pobrania informacji konfiguracyjnych.

Identyfikator użytkownika

Identyfikator użytkownika używany z właściwością Hasło w celu uwierzytelnienia, jeśli aplikacja wywołująca nie udostępni identyfikatora użytkownika oraz hasław sposób jawny.

Jeśli zostanie podana wartość właściwości Identyfikator użytkownika, należy podać także wartość właściwości Hasło.

Właściwości Identyfikator użytkownika oraz Hasło fabryki połączeń są używane, jeśli aplikacja wywołująca nie udostępnia identyfikatora użytkownika oraz hasła w sposób jawny, na przykład gdy aplikacja wywołująca korzysta z metody createTopicConnection(). Klient JMS przesyła identyfikator użytkownika oraz hasło do serwera JMS.

Typ danych String
Hasło

Hasło używane z właściwością Identyfikator użytkownika do uwierzytelnienia, jeśli aplikacja wywołująca nie udostępnia identyfikatora użytkownika oraz hasła w sposób jawny.

Jeśli zostanie podana wartość właściwości Identyfikator użytkownika, należy podać także wartość właściwości Hasło.

Typ danych String
Wprowadź ponownie hasło

Potwierdzenie hasła.

Węzeł

Nazwa węzła administracyjnego produktu WebSphere Application Server, w którym działa serwer JMS dla danej fabryki połączeń. Połączenia utworzone przez daną fabrykę łączą się z tym serwerem JMS.

Typ danych Wyliczeniowy
Zakres Lista rozwijana węzłów w domenie administracyjnej produktu WebSphere Application Server.
Serwer aplikacji

Nazwa serwera aplikacji. Nie jest to nazwa hosta komputera, lecz nazwa konfigurowanego serwera aplikacji.

Port

Określa, który z dwóch portów używanych przez połączenia do nawiązywania połączenia z serwerem JMS, ma zostać użyty. Port W KOLEJCE jest przeznaczony do obsługi wszystkich funkcji publikowania i subskrypcji JMS. Port BEZPOŚREDNI jest przeznaczony tylko dla nietrwałych, nietransakcyjnych i niestałych subskrypcji.

Uwaga: Komponenty bean sterowane komunikatami nie mogą używać bezpośredniego portu nasłuchiwania do obsługi funkcji publikowania lub subskrypcji. Dlatego fabryki połączeń tematów, których porty są skonfigurowane jako Bezpośredni, nie mogą być używane z komponentami bean sterowanymi komunikatami.
Typ danych Wyliczeniowy
Wartość domyślna QUEUED (W KOLEJCE)
Zakres
QUEUED (W KOLEJCE)
Port nasłuchiwania używany do obsługi wszystkich funkcji publikowania i subskrypcji kompatybilnych z JMS.
DIRECT (BEZPOŚREDNI)
Port nasłuchiwania używany do obsługi funkcji publikowania i subskrypcji bezpośredniego połączenia TCP/IP (tylko dla nietransakcyjnych, nietrwałych i niestałych subskrypcji).

Numery portów TCP/IP dla tych portów są definiowane na wewnętrznym serwerze JMS produktu.
Identyfikator klienta

Identyfikator klienta JMS używany w przypadku połączeń z menedżerem kolejek produktu MQSeries.

Typ danych String
Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
