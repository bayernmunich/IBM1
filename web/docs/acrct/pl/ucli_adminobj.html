Ustawienia obiektów administrowanych

Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranych obiektów administrowanych.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Adaptery zasobów > instancja_adaptera_zasobów. Kliknij prawym przyciskiem myszy pozycję Obiekty administrowane, a następnie kliknij polecenie Nowy. Na karcie Ogólne zostaną wyświetlone następujące pola.

Ustawienia obiektów administracyjnych są obsługiwane w podobny sposób, jak fabryki połączeń. Przy aktualizowaniu obiektów administracyjnych należy używać tych samych paneli, co przy ich tworzeniu.

Nazwa

Nazwa identyfikująca obiekt administrowany w celach administracyjnych na serwerze IBM WebSphere Application Server. Ta nazwa musi być unikalna w obrębie obiektów administrowanych adaptera zasobów w domenie administracyjnej produktu.

Typ danych String
Opis

Opcjonalny opis fabryki połączeń używany w celach administracyjnych na serwerze IBM WebSphere Application Server.

Typ danych String
Nazwa JNDI

Jest to nazwa resource-env-ref, nazwa message-destination-ref (jeśli nazwa message-destination-ref nie posiada odsyłacza) lub odsyłacz message-destination.

Typ danych String
Typ

Lista rozwijana wszystkich par klasa obiektu administrowanego-interfejs zdefiniowanych dla obiektów administrowanych w pliku RAR (Resource Adapter Archive).

Każdy Typ posiada zestaw właściwości określonych w polu Właściwości. Taki zestaw właściwości jest konstruowany przez wydobycie właściwości z definicji poszczególnych obiektów administrowanych. W przypadku istniejących obiektów administrowanych wyświetlanych do aktualizacji taka lista właściwości zostaje przesłonięta właściwościami określonymi dla poszczególnych obiektów. Po zmianie wartości w polu Typ właściwości również ulegają zmianie na odpowiednie dla danego typu.
