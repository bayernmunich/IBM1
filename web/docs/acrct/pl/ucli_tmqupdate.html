Aktualizowanie produktu WebSphere MQ jako dostawcy JMS (Java Message Service) oraz konfiguracji jego zasobów JMS dla klientów aplikacji

Można zaktualizować istniejącą konfigurację produktu WebSphere MQ jako dostawcy JMS (Java Message Service) oraz zaktualizować konfigurację fabryk połączeń produktu WebSphere MQ lub miejsc docelowych produktu WebSphere MQ.

Informacje dotyczące tego tematu

To zadanie służy do aktualizowania istniejącej konfiguracji produktu WebSphere MQ jako dostawcy JMS (Java Message Service) oraz aktualizowania konfiguracji fabryk połączeń produktu WebSphere MQ lub miejsc docelowych produktu WebSphere MQ.

Procedura

  1. Uruchom narzędzie ACRCT (Application Client Resource Configuration Tool).
  2. Otwórz plik EAR (Enterprise Archive) zawierający dostawcę JMS produktu WebSphere MQ, fabrykę połączeń produktu WebSphere MQ lub miejsce docelowe produktu WebSphere MQ. Zawartość pliku EAR zostanie wyświetlona w widoku drzewa.
  3. Wybierz plik JAR (Java Archive) zawierający dostawcę JMS, fabrykę połączeń lub miejsce docelowe do zaktualizowania.
  4. Rozwiń plik JAR, aby wyświetlić jego zawartość i odnaleźć danego dostawcę JMS, fabrykę połączeń lub miejsce docelowe przeznaczone do aktualizacji. Wykonaj jedną z następujących czynności:
    • Kliknij prawym przyciskiem myszy odpowiedni obiekt, a następnie wybierz opcję Właściwości.
    • Kliknij opcję Edytuj > Właściwości na pasku menu.
  5. Zaktualizuj właściwości w wyświetlonych polach. Szczegółowe informacje dotyczące pól zawiera temat:
    • Właściwości dostawcy JMS
    • Właściwości fabryki połączeń kolejki MQ
    • Właściwości fabryki połączeń tematów MQ
    • Właściwości miejsca docelowego kolejki MQ
    • Właściwości miejsca docelowego tematu MQ
  6. Kliknij przycisk OK.
  7. Kliknij opcję Plik > Zapisz, aby zapisać zmiany.
