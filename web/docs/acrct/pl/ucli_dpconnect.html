Ustawienia fabryki połączeń domyślnego dostawcy

Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server. Takie właściwości konfiguracyjne określają sposób tworzenia połączeń między dostawcą JMS a używaną przez niego magistralą integracji usług.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Dostawcy przesyłania komunikatów > Domyślny dostawca. Kliknij prawym przyciskiem myszy folder Fabryki połączeń, a następnie kliknij polecenie Nowa. Na karcie Ogólne zostaną wyświetlone następujące pola.

Dla ustawień posiadających wartość domyślną wyświetlane są odpowiednie wartości. Dla ustawień posiadających stałe wartości wyświetlane jest menu rozwijane.

Nazwa

Nazwa fabryki połączeń.

Typ danych String
Opis

Opis fabryki połączeń używany w celach administracyjnych na serwerze IBM WebSphere Application Server.

Typ danych String
Nazwa JNDI

Nazwa JNDI umożliwiająca dopasowanie tej definicji fabryki połączeń adaptera zasobów do deskryptora wdrażania. Wpisem tym jest nazwa resource-ref.

Typ danych String
Nazwa użytkownika

Nazwa użytkownika używana wraz z właściwością Hasło do połączenia się z aplikacją.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Właściwości fabryki połączeń ID użytkownika i Hasło są używane, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła. Jeśli nazwa użytkownika i hasło są określone, dla fabryki utworzony zostaje alias uwierzytelniania, w którym hasło jest zaszyfrowane.

Typ danych String
Hasło

Hasło używane do uwierzytelnienia połączenia z aplikacją.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Typ danych String

Potwierdzenie hasła.

Typ danych String

Nazwa klienta. Wymagana dla trwałych subskrypcji tematu.

Typ danych String
Niezawodność komunikatów nietrwałych

Niezawodność stosowana względem nietrwałych komunikatów JMS wysłanych przy użyciu tej fabryki połączeń.

Aby mieć do dyspozycji inne opcje niezawodności dostarczania komunikatów do poszczególnych miejsc docelowych JMS, można dla tej właściwości ustawić wartość Jako miejsce docelowe magistrali. Niezawodność jest wówczas definiowana przez właściwość Niezawodność miejsca docelowego magistrali, do którego przypisane jest miejsce docelowe JMS.

Wartość domyślna Niezawodny trwały
Zakres

Możliwie optymalny nietrwały
Komunikaty nie są zapisywane na dysku i w razie przepełnienia pamięci podręcznej zostają odrzucone.
Ekspresowy nietrwały

Niezawodny nietrwały
Komunikaty mogą zostać utracone w razie awarii mechanizmu przesyłania komunikatów oraz w normalnych warunkach pracy.
Niezawodny trwały

Typ danych String
Współużytkuj trwałe subskrypcje

Określa, czy trwałe subskrypcje są współużytkowane w ramach połączeń z elementami klastra serwerów.

W normalnej sytuacji tylko jedna sesja na raz może mieć subskrybenta tematów (TopicSubscriber) dla danej subskrypcji trwałej. Ta właściwość umożliwia przesłonięcie takiego zachowania, aby dana subskrypcja trwała mogła mieć wiele symultanicznych konsumentów.

Nazwa protokołu, która identyfikuje grupę mechanizmów przesyłania komunikatów.

Typ danych String
Punkty końcowe dostawcy

Rozdzielana przecinkami lista punktów końcowych służących do łączenia się z serwerem startowym.

Wprowadź listę rozdzielonych przecinkami punktów końcowych w postaci trójek o składni host:port:protokół.

Przykład 
merlin:7276:BootstrapBasicMessaging,Gandalf:5557:BootstrapSecureMessaging
gdzie

BootstrapBasicMessaging odpowiada zdalnemu protokołowi InboundBasicMessaging (JFAP-TCP/IP).
Wartość domyślna
  • Jeśli nazwa hosta nie zostanie określona, przyjmowana jest wartość domyślna localhost.
  • Jeśli numer portu nie zostanie określony, przyjmowana jest wartość domyślna 7276.
  • Jeśli nazwa łańcucha nie zostanie określona, jako wartość domyślna przyjmowany jest predefiniowany łańcuch, taki jak BootstrapBasicMessaging.
