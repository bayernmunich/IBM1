Właściwości źródeł danych dla klientów aplikacji

Ta strona umożliwia tworzenie i modyfikowanie źródeł danych.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Dostawcy źródeł danych > instancja_dostawcy_źródła_danych. Kliknij prawym przyciskiem myszy folder Źródła danych, a następnie kliknij polecenie Nowe. Na karcie Ogólne zostaną wyświetlone następujące pola:

Nazwa

Określa wyświetlaną nazwę tego źródła danych.

Typ danych String
Opis

Opis tekstowy źródła danych.

Typ danych String
Nazwa JNDI

Środowisko wykonawcze klienta aplikacji używa tego pola w celu pobrania informacji konfiguracyjnych.

Nazwa bazy danych

Nazwa bazy danych, z którą ma zostać nawiązane połączenie.

Użytkownik

Na potrzeby uwierzytelniania należy użyć właściwości ID użytkownika i Hasło, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła.

Jeśli określona zostanie wartość właściwości ID użytkownika, należy również określić wartość właściwości Hasło. Właściwości fabryki połączeń ID użytkownika i Hasło są używane, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła.

Hasło

Na potrzeby uwierzytelniania należy użyć właściwości ID użytkownika i Hasło, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła.

Jeśli określona zostanie wartość właściwości Hasło, należy również określić wartość właściwości ID użytkownika.

Wprowadź ponownie hasło

Potwierdzenie hasła.

Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
