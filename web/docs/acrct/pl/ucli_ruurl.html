Ustawienia adresu URL dla klientów aplikacji

Ta strona służy do implementowania funkcji dla danego protokołu URL, jak np. HTTP (Hyper Text Transfer Protocol).

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Następnie należy rozwinąć wybrany plik JAR > Dostawcy URL > Instancja dostawcy URL. Należy kliknąć prawym przyciskiem myszy opcję Adresy URL, a następnie kliknij opcję Nowy. Na karcie Ogólne zostaną wyświetlone następujące pola.

Ten dostawca, składający się z klas jest rozszerzeniem klas java.net.URLStreamHandler i java.net.URLConnection .

Nazwa

Nazwa administracyjna dla adresu URL.

Opis

Jest to opcjonalny opis adresu URL dla rekordów administracyjnych.

Nazwa JNDI

Środowisko wykonawcze klienta aplikacji używa tego pola w celu pobrania informacji konfiguracyjnych.

Adres URL

Nazwa URL (Uniform Resource Locator) wskazująca na zasób Internetu lub intranetu. Na przykład: http://www.ibm.com.

Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
