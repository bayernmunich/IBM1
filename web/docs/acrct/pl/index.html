Narzędzie konfigurowania zasobów klienta aplikacji

Przejdź do tematów pomocy znajdujących się w tej sekcji.

  • Aktualizowanie produktu WebSphere MQ jako dostawcy JMS (Java Message Service) oraz konfiguracji jego zasobów JMS dla klientów aplikacji
    Można zaktualizować istniejącą konfigurację produktu WebSphere MQ jako dostawcy JMS (Java Message Service) oraz zaktualizować konfigurację fabryk połączeń produktu WebSphere MQ lub miejsc docelowych produktu WebSphere MQ.
  • Konfigurowanie dostawców poczty i sesji dla klientów aplikacji
    Za pomocą narzędzia ACRCT (Application Client Resource Configuration Tool) można edytować konfiguracje sesji poczty i dostawców dla klientów aplikacji.
  • Konfigurowanie dostawców źródeł danych (dostawców JDBC) dla klientów aplikacji
    Za pomocą narzędzia ACRCT (Application Client Resource Configuration Tool) można tworzyć nowych dostawców źródła danych, nazywanych też dostawcami JDBC, dla klienta aplikacji.
  • Konfigurowanie nowych dostawców środowiska zasobów dla klientów aplikacji
    Za pomocą narzędzia ACRCT (Application Client Resource Configuration Tool) można tworzyć nowe konfiguracje dostawy środowiska zasobów dla klienta aplikacji.
  • Konfigurowanie nowych dostawców URL dla klientów aplikacji
    Za pomocą narzędzia ACRCT (Application Client Resource Configuration Tool) można tworzyć dostawców URL i adresy URL dla aplikacji klienckiej.
  • Konfigurowanie nowych pozycji środowiska zasobów dla klientów aplikacji
    Za pomocą narzędzia ACRCT (Application Client Resource Configuration Tool) można utworzyć nowe pozycje środowiska zasobów dla aplikacji klienckiej.
  • Konfigurowanie nowych sesji poczty dla klientów aplikacji
    Narzędzie ACRCT (Application Client Resource Configuration Tool) służy do konfigurowania nowych sesji poczty dla klienta aplikacji.
  • Konfigurowanie obiektów administrowanych
    Ta sekcja ułatwia konfigurowanie nowych obiektów administrowanych.
  • Konfigurowanie zasobów klienta usługi przesyłania komunikatów Java
    Aby skonfigurować zasoby klienta usługi przesyłania komunikatów Java, należy utworzyć nowe konfiguracje dostawcy JMS dla klienta aplikacji. Klient aplikacji może używać usługi przesyłania komunikatów za pośrednictwem interfejsu API JMS (Java Message Service). Dostawca JMS udostępnia dwa rodzaje fabryk J2EE. Pierwszy to Fabryka połączeń JMS, a drugi Fabryka miejsc docelowych JMS.
  • Adaptery zasobów dla klienta
    Adapter zasobu jest to sterownik programowy na poziomie systemu, używany przez aplikację Java do łączenia z globalnym systemem informacyjnym (enterprise information system - EIS). Adapter zasobów łączy się z klientem aplikacji i umożliwia połączenie między systemem EIS a aplikacją korporacyjną.
  • Asynchroniczne przesyłanie komunikatów na serwerze WebSphere Application Server za pomocą usługi JMS
    Serwer WebSphere Application Server obsługuje funkcję asynchronicznego przesyłania komunikatów jako metodę komunikacji w oparciu o interfejs programistyczny JMS (Java Message Service). Interfejs JMS udostępnia programom Java (klientom i aplikacjom J2EE Java Platform, Enterprise Edition) wspólny sposób tworzenia, wysyłania, otrzymywania i odczytywania żądań asynchronicznych jako komunikatów JMS.
  • Dostawcy i wpisy środowiska zasobów
    Odwołanie do środowiska zasobów odwzorowuje nazwę logiczną używaną przez aplikację klienta na nazwę fizyczną obiektu.
  • Dostawcy JMS (Java Message Service) dla klientów
    W tym temacie opisano różne sposoby wykorzystania dostawców JMS (Java Message Service) przez aplikacje klienckie na serwerze WebSphere Application Server.
  • Dostawcy adresów URL dla narzędzia Application Client Resource Configuration Tool
    Dostawca adresu URL implementuje funkcję dla danego protokołu URL, jak np. HTTP (HyperText Transfer Protocol). Dostawca taki, składający się z pary klas, stanowi rozszerzenie klas java.net.URLStreamHandler i java.net.URLConnection.
  • Konfigurowanie nowych fabryk połączeń dla adapterów zasobów
    Do konfigurowania nowych fabryk połączeń dla adapterów zasobów służy narzędzie Application Client Resource Configuration Tool (ACRCT).
  • Ogólne ustawienia fabryki połączeń JMS dla klientów aplikacji
    Ten panel służy do przeglądania i zmiany właściwości konfiguracyjnych wybranej fabryki połączeń JMS (Java Message Service) na potrzeby używania z powiązanym dostawcą JMS. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS oraz używanym przez niego systemem przesyłania komunikatów.
  • Ogólne ustawienia miejsc docelowych JMS dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranego miejsca docelowego JMS, wykorzystywanych przez przypisanego do tego miejsca dostawcę JMS.
  • URLs for application clients
    A Uniform Resource Locator (URL) is an identifier that points to an electronically accessible resource, such as a directory file on a machine in a network, or a document stored in a database.
  • Ustawienia adaptera zasobów
    W tym panelu można wyświetlić lub zmienić właściwości konfiguracyjne adaptera zasobów. Właściwości te kontrolują sposób, w jaki tworzone są adaptery zasobów.
  • Ustawienia adresu URL dla klientów aplikacji
    Ta strona służy do implementowania funkcji dla danego protokołu URL, jak np. HTTP (Hyper Text Transfer Protocol).
  • Ustawienia dostawcy JMS dla klientów aplikacji
    Ta strona służy do konfigurowania właściwości dostawcy JMS (Java Message Service), jeśli ma być używany dostawca JMS inny niż domyślny dostawca funkcji przesyłania komunikatów lub jako dostawca JMS ma być używany produkt WebSphere MQ.
  • Ustawienia dostawcy poczty dla klientów aplikacji
    Ta strona służy do implementowania interfejsu JavaMail API i tworzenia sesji poczty.
  • Ustawienia dostawcy środowiska zasobów dla klientów aplikacji
    Ta strona służy do ustawienia właściwości pozycji środowiska zasobów.
  • Ustawienia dostawcy URL dla klientów aplikacji
    Ta strona służy do utworzenia nowych dostawców URL.
  • Ustawienia dostawcy źródeł danych dla klientów aplikacji
    Ta strona umożliwia utworzenie źródła danych w ramach dostawcy JDBC, który dostarcza określoną klasę implementacji sterownika JDBC.
  • Ustawienia fabryki połączeń adaptera zasobów
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń adaptera zasobów.
  • Ustawienia fabryki połączeń domyślnego dostawcy
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server. Takie właściwości konfiguracyjne określają sposób tworzenia połączeń między dostawcą JMS a używaną przez niego magistralą integracji usług.
  • Ustawienia fabryki połączeń kolejki domyślnego dostawcy
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń kolejki JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server. Takie właściwości konfiguracyjne określają sposób tworzenia połączeń między dostawcą JMS a używaną przez niego magistralą integracji usług.
  • Ustawienia fabryki połączeń kolejki dostawcy domyślnego w wersji 5 dla klientów aplikacji
    Ten panel służy do przeglądania lub zmiany właściwości konfiguracji wybranej fabryki połączeń kolejki JMS na potrzeby przesyłania komunikatów w trybie punkt z punktem przez aplikacje serwera WebSphere Application Server 5. Poniższe właściwości konfiguracji określają sposób tworzenia połączeń między danym dostawcą JMS i używanym przez niego domyślnym systemem przesyłania komunikatów.
  • Ustawienia fabryki połączeń kolejki dostawcy WebSphere MQ dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranej fabryki połączeń kolejki dla dostawcy JMS (Java Message Service) produktu MQSeries. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS a produktem WebSphere MQ.
  • Ustawienia fabryki połączeń tematów domyślnego dostawcy
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń tematów JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server. Takie właściwości konfiguracyjne określają sposób tworzenia połączeń między dostawcą JMS a używaną przez niego magistralą integracji usług.
  • Ustawienia fabryki połączeń tematów dostawcy domyślnego w wersji 5 dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranej fabryki połączeń tematów dla wewnętrznego dostawcy JMS (Java Message Service) produktu. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS oraz używanym przez niego systemem przesyłania komunikatów.
  • Ustawienia fabryki połączeń tematu dostawcy WebSphere MQ dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranej fabryki połączeń tematu dla dostawcy JMS (Java Message Service) produktu WebSphere MQ. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS a produktem WebSphere MQ.
  • Ustawienia miejsca docelowego Domyślnego dostawcy przesyłania komunikatów (wersja 5) dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranego miejsca docelowego tematu dla wewnętrznego dostawcy JMS (Java Message Service) produktu.
  • Ustawienia miejsca docelowego kolejki domyślnego dostawcy
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranego miejsca docelowego kolejki JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server.
  • Ustawienia miejsca docelowego kolejki dostawcy domyślnego w wersji 5 dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranego miejsca docelowego kolejki dla dostawcy JMS (Java Message Service) produktu.
  • Ustawienia miejsca docelowego kolejki dostawcy produktu WebSphere MQ dla klientów aplikacji
    Ten panel służy do wyświetlania lub modyfikowania właściwości konfiguracyjnych wybranego miejsca docelowego kolejki dla dostawcy JMS (Java Message Service) produktu WebSphere MQ.
  • Ustawienia miejsca docelowego tematu domyślnego dostawcy
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranego miejsca docelowego tematu JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server.
  • Ustawienia miejsca docelowego tematu dostawcy produktu WebSphere MQ dla klientów aplikacji
    Ten panel służy do wyświetlania lub zmiany właściwości konfiguracyjnych wybranego miejsca docelowego tematu dla dostawcy JMS (Java Message Service) produktu WebSphere MQ.
  • Ustawienia obiektów administrowanych
    Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranych obiektów administrowanych.
  • Ustawienia pozycji środowiska zasobów dla klientów aplikacji
    Ta strona służy do ustawienia właściwości pozycji środowiska zasobów.
  • Ustawienia sesji poczty dla klientów aplikacji
    Ta strona służy do konfigurowania właściwości sesji poczty.
  • Właściwości źródeł danych dla klientów aplikacji
    Ta strona umożliwia tworzenie i modyfikowanie źródeł danych.
