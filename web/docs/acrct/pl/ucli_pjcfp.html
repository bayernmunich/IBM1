Ogólne ustawienia fabryki połączeń JMS dla klientów aplikacji

Ten panel służy do przeglądania i zmiany właściwości konfiguracyjnych wybranej fabryki połączeń JMS (Java Message Service) na potrzeby używania z powiązanym dostawcą JMS. Te właściwości konfiguracyjne sterują sposobem tworzenia połączeń między dostawcą JMS oraz używanym przez niego systemem przesyłania komunikatów.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Następnie należy rozwinąć wybrany plik JAR > Dostawcy przesyłania komunikatów > nowa_instancja_dostawcy_JMS. Kliknij prawym przyciskiem myszy pozycję Fabryki połączeń i wybierz opcję Nowa. Na karcie Ogólne zostaną wyświetlone następujące pola.

Fabryka połączeń JMS (Java Message Service) tworzy połączenia z miejscami docelowymi JMS. Fabryka połączeń JMS jest tworzona przez przypisanego do niej dostawcę JMS. Fabryka połączeń JMS ogólnego dostawcy JMS (innego, niż wewnętrzny domyślny dostawca przesyłania komunikatów lub produkt WebSphere MQ) ma następujące właściwości:

Nazwa

Nazwa służąca do identyfikowania tej fabryki połączeń JMS w celach administracyjnych na serwerze IBM WebSphere Application Server. Nazwa musi być unikalna w środowisku przypisanego do niej dostawcy JMS.

Typ danych String
Opis

Opis fabryki połączeń używany w celach administracyjnych na serwerze IBM WebSphere Application Server.

Typ danych String
Wartość domyślna NULL
Nazwa JNDI

Środowisko wykonawcze klienta aplikacji używa tego pola w celu pobrania informacji konfiguracyjnych.

Identyfikator użytkownika

Identyfikator użytkownika używany z właściwością Hasło w celu uwierzytelnienia, jeśli aplikacja wywołująca nie udostępni identyfikatora użytkownika oraz hasław sposób jawny.

Jeśli zostanie podana wartość właściwości Identyfikator użytkownika, należy podać także wartość właściwości Hasło.

Właściwości ID użytkownika oraz Hasło fabryki połączeń są używane, jeśli aplikacja wywołująca nie udostępnia identyfikatora użytkownika i hasła w sposób jawny; na przykład wtedy, gdy aplikacja wywołująca korzysta z metody createQueueConnection(). Klient JMS przesyła identyfikator użytkownika oraz hasło do serwera JMS.

Typ danych String
Hasło

Hasło używane z właściwością Identyfikator użytkownika do uwierzytelnienia, jeśli aplikacja wywołująca nie udostępnia identyfikatora użytkownika oraz hasła w sposób jawny.

Jeśli zostanie podana wartość właściwości Identyfikator użytkownika, należy podać także wartość właściwości Hasło.

Typ danych String
Wartość domyślna NULL
Wprowadź ponownie hasło

Potwierdzenie hasła wpisanego w polu Hasło.

Zewnętrzna nazwa JNDI

Nazwa JNDI używana do wiązania kolejki z przestrzenią nazw serwera aplikacji.

Przyjęto zasadę używania pełnej nazwy JNDI, na przykład jms/Nazwa, gdzie Nazwa jest nazwą logiczną zasobu.

Nazwa ta jest używana do łączenia informacji o powiązaniach platformy. Powiązania te służą do tworzenia powiązań zasobów zdefiniowanych w deskryptorze wdrażania modułu z rzeczywistym (fizycznym) zasobem przypisanym do interfejsu API JNDI przez platformę.

Typ danych String
Typ połączenia

Wskazuje, czy dane miejsce docelowe JMS to kolejka (przesyłanie komunikatów w trybie punkt z punktem) czy temat (przesyłanie komunikatów w trybie publikowania i subskrypcji).

Wybierz jedną z następujących opcji:
Kolejka
Miejsce docelowe kolejki JMS na potrzeby przesyłania komunikatów między punktami.
Temat
Miejsce docelowe kolejki JMS na potrzeby przesyłania komunikatów w trybie publikowania i subskrypcji.
Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
