Ustawienia sesji poczty dla klientów aplikacji

Ta strona służy do konfigurowania właściwości sesji poczty.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Następnie należy rozwinąć wybrany plik JAR > Dostawcy poczty > Instancja dostawcy poczty. Kliknij prawym przyciskiem myszy pozycję Sesje poczty i wybierz Nowa. Na karcie Ogólne pojawią się poniższe pola:

Nazwa

Nazwa administracyjna obiektu sesji poczty JavaMail.

Opis

Opcjonalny opis rekordów administracyjnych.

Nazwa JNDI

Środowisko wykonawcze klienta aplikacji używa tego pola w celu pobrania informacji konfiguracyjnych.

Host transportu poczty

Serwer używany do połączenia podczas wysyłania poczty.

Protokół transportu poczty

Protokół transportowy poczty używany podczas wysyłania poczty.

Użytkownik transportu poczty

Określa identyfikator użytkownika, gdy host transportu poczty wymaga uwierzytelnienia.

Hasło transportu poczty

Hasło używane, gdy host transportu poczty wymaga uwierzytelnienia.

Włącz ścisłe analizowanie adresów internetowych

Określa, czy adresy odbiorców powinny być analizowane ściśle zgodnie z dokumentem RFC 822, który zawiera specyfikacje opublikowane przez komisję IAB (Internet Architecture Board).

Ustawienie to stosuje większość serwerów poczty. Zastosowanie składni RFC 822 do analizowania adresów umożliwia skuteczną selekcję ważnych adresów e-mail, ściśle zgodnych z definicją. Jeśli zostanie wybrane to ustawienie, składnia używana przez program JavaMail będzie zgodna ze składnią opisaną w dokumencie RFC 822, a adresy odbiorców, które nie zostaną zanalizowane jako poprawne adresy e-mail (tak jak to określono w specyfikacji), będą odrzucane. Jeśli to ustawienie nie zostanie wybrane, składnia używana przez program JavaMail nie będzie zgodna ze składnią RFC 822, a adresy odbiorców, które nie będą zgodne ze specyfikacją, będą akceptowane. Domyślnie to ustawienie jest wyłączone. Specyfikację RFC 822 można przejrzeć pod następującym adresem URL należącym do World Wide Web Consortium (W3C): http://www.w3.org/Protocols/rfc822/.

Wprowadź ponownie hasło

Potwierdzenie hasła.

Poczta od

Określa nadawcę poczty.

Host składnicy poczty

Nazwa (lub "domena") hosta konta poczty.

Użytkownik składnicy poczty

Identyfikator użytkownika konta poczty.

Hasło składnicy poczty

Hasło konta poczty.

Wprowadź ponownie hasło

Potwierdzenie hasła.

Protokół składnicy poczty

Protokół używany podczas odbierania poczty.

Debugowanie poczty

Jeśli ma wartość true, interakcje JavaMail z serwerami poczty oraz właściwości sesji poczty są umieszczane na wyjściu standardowym (w pliku stdout).

Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
