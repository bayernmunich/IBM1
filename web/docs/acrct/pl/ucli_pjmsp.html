Ustawienia dostawcy JMS dla klientów aplikacji

Ta strona służy do konfigurowania właściwości dostawcy JMS (Java Message Service), jeśli ma być używany dostawca JMS inny niż domyślny dostawca funkcji przesyłania komunikatów lub jako dostawca JMS ma być używany produkt WebSphere MQ.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń plik JAR. Kliknij prawym przyciskiem myszy pozycję Dostawcy przesyłania komunikatów i wybierz opcję Nowy. Na karcie Ogólne zostaną wyświetlone następujące pola.

Nazwa

Nazwa służąca do identyfikowania dostawcy JMS w celach administracyjnych.

Typ danych String
Opis

Opis dostawcy JMS używany do celów administracyjnych.

Typ danych String
Ścieżka klasy

Lista ścieżek lub nazw plików .jar, które łącznie tworzą położenie klas dostawcy zasobów.

Klasa fabryki kontekstów

Nazwa klasy Java fabryki początkowych kontekstów dla dostawcy JMS.

Na przykład dla dostawcy usług LDAP wartość przyjmuje formę: com.sun.jndi.ldap.LdapCtxFactory.

Typ danych String
Adres URL dostawcy

Adres URL dostawcy JMS na potrzeby zewnętrznego wyszukiwania JNDI.

Na przykład adres URL LDAP używany przez dostawcę JMS ma postać: ldap://hostname.company.com/contextName.

Typ danych String
Właściwości niestandardowe

Pary nazwa-wartość do ustawiania dodatkowych właściwości obiektu, który jest tworzony w czasie wykonywania dla tego zasobu.

Należy wprowadzić nazwę będącą właściwością publiczną obiektu oraz wartość, która może być przekształcona z łańcucha tekstowego w typ wymagany przez metodę ustawioną dla tej właściwości. Dopuszczalne właściwości oraz wartości zależą od obiektu, który jest tworzony. Wykaz poprawnych wartości można znaleźć w dokumentacji obiektu.
