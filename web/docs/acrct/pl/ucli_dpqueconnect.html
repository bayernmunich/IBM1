Ustawienia fabryki połączeń kolejki domyślnego dostawcy

Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń kolejki JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server. Takie właściwości konfiguracyjne określają sposób tworzenia połączeń między dostawcą JMS a używaną przez niego magistralą integracji usług.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Dostawcy przesyłania komunikatów > Domyślny dostawca. Kliknij prawym przyciskiem myszy folder Fabryki połączeń kolejki, a następnie kliknij polecenie Nowa. Na karcie Ogólne zostaną wyświetlone następujące pola.

Dla ustawień posiadających wartość domyślną wyświetlane są odpowiednie wartości. Dla ustawień posiadających stałe wartości wyświetlane jest menu rozwijane.

Nazwa

Nazwa fabryki połączeń kolejki.

Typ danych String
Opis

Opis fabryki połączeń kolejki używany w celach administracyjnych na serwerze IBM WebSphere Application Server.

Typ danych String
Nazwa JNDI

Nazwa JNDI umożliwiająca dopasowanie tej definicji fabryki połączeń kolejki do deskryptora wdrażania. Wpisem tym jest nazwa resource-ref.

Typ danych String
Nazwa użytkownika

Nazwa użytkownika, wraz z właściwością Hasło używana na potrzeby uwierzytelniania, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła. W przypadku użycia tego pola pole Nazwa użytkownika w obszarze Właściwości jest ignorowane.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Właściwości fabryki połączeń Nazwa użytkownika i Hasło są używane, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła. Jeśli nazwa użytkownika i hasło są określone, dla fabryki utworzony zostaje alias uwierzytelniania, w którym hasło jest zaszyfrowane.

Typ danych String
Hasło

Hasło używane do utworzenia zaszyfrowanego W przypadku wypełnienia tego pola pole Hasło w obszarze Właściwości jest ignorowane.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Typ danych String
Wprowadź ponownie hasło

Potwierdzenie hasła.

Nazwa magistrali

Nazwa magistrali, z którą łączy się fabryka połączeń kolejki.

Typ danych String
Identyfikator klienta

Identyfikator klienta. Wymagana dla trwałych subskrypcji tematu.

Typ danych String
Niezawodność komunikatów nietrwałych

Niezawodność stosowana względem nietrwałych komunikatów JMS wysłanych przy użyciu tej fabryki połączeń.

Aby mieć do dyspozycji inne opcje niezawodności dostarczania komunikatów do poszczególnych miejsc docelowych JMS, można dla tej właściwości ustawić wartość Jako miejsce docelowe magistrali. Niezawodność jest wówczas definiowana przez właściwość Niezawodność miejsca docelowego magistrali, do którego przypisane jest miejsce docelowe JMS.

Wartość domyślna Niezawodny trwały
Zakres
Brak

Możliwie optymalny nietrwały
Komunikaty nie są zapisywane na dysku i w razie przepełnienia pamięci podręcznej zostają odrzucone.
Ekspresowy nietrwały
W razie przepełnienia pamięci podręcznej komunikaty są zapisywane asynchronicznie w trwałej składnicy, lecz nie są zachowywane przy restartowaniu serwera.
Niezawodny nietrwały
Komunikaty mogą zostać utracone w razie awarii mechanizmu przesyłania komunikatów oraz w normalnych warunkach pracy.
Niezawodny trwały
Komunikaty mogą zostać utracone w razie awarii mechanizmu przesyłania komunikatów, ale nie w normalnych warunkach pracy.

Nazwa protokołu, która identyfikuje grupę mechanizmów przesyłania komunikatów.

Typ danych String
Punkty końcowe dostawcy

Rozdzielana przecinkami lista punktów końcowych służących do łączenia się z serwerem startowym.

Wprowadź listę rozdzielonych przecinkami punktów końcowych w postaci trójek o składni host:port:protokół.

Przykład 
localhost:7777:BootstrapBasicMessaging
gdzie

BootstrapBasicMessaging odpowiada zdalnemu protokołowi InboundBasicMessaging (JFAP-TCP/IP).
Wartość domyślna
  • Jeśli nazwa hosta nie zostanie określona, przyjmowana jest wartość domyślna localhost.
  • Jeśli numer portu nie zostanie określony, przyjmowana jest wartość domyślna 7276.
  • Jeśli nazwa łańcucha nie zostanie określona, jako wartość domyślna przyjmowany jest predefiniowany łańcuch, taki jak BootstrapBasicMessaging.
Bliskość połączeń

Bliskość mechanizmu przesyłania komunikatów w stosunku do requestera.

Wartość domyślna Magistrala, Klaster, Serwer
Zakres Magistrala, Host
