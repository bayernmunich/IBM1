Ustawienia fabryki połączeń adaptera zasobów

Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranej fabryki połączeń adaptera zasobów.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Adaptery zasobów. Kliknij prawym przyciskiem myszy folder Fabryki połączeń, a następnie kliknij polecenie Nowa. Na karcie Ogólne zostaną wyświetlone następujące pola.

Nazwa

Nazwa służąca do identyfikowania tej fabryki połączeń w celach administracyjnych na serwerze WebSphere Application Server. Ta nazwa musi być unikalna w obrębie fabryk połączeń adaptera zasobów w domenie administracyjnej produktu.

Typ danych String
Opis

Opcjonalny opis fabryki połączeń używany w celach administracyjnych na serwerze IBM WebSphere Application Server.

Typ danych String
Nazwa JNDI

Nazwa JNDI umożliwiająca dopasowanie tej definicji fabryki połączeń adaptera zasobów do deskryptora wdrażania. Wpisem tym powinna być nazwa resource-ref.

Typ danych String
Nazwa użytkownika

Nazwa użytkownika, wraz z właściwością Hasło używana na potrzeby uwierzytelniania, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła przy uzyskiwaniu połączenia. W przypadku użycia tego pola pole Nazwa użytkownika w obszarze Właściwości jest ignorowane.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Właściwości Nazwa użytkownika i Hasło są używane, jeśli aplikacja wywołująca nie przekazuje jawnie identyfikatora użytkownika i hasła przy uzyskiwaniu połączenia.

Typ danych String
Hasło

Określa zaszyfrowane hasło. W przypadku wypełnienia tego pola pole Hasło w obszarze Właściwości jest ignorowane.

Jeśli określona zostanie wartość właściwości Nazwa użytkownika, należy również określić wartość właściwości Hasło.

Typ danych String
Wprowadź ponownie hasło

Potwierdzenie hasła.

Typ

Lista rozwijana wszystkich interfejsów connectionFactoryInterfaces zdefiniowanych dla fabryk w pliku RAR (Resource Adapter Archive).

Każdy Typ posiada zestaw właściwości określonych w polu Właściwości. Taki zestaw właściwości jest konstruowany przez wydobycie właściwości z poszczególnych obiektów definicji połączeń. W przypadku istniejących fabryk połączeń wyświetlanych do aktualizacji taka lista właściwości zostaje przesłonięta właściwościami określonymi dla poszczególnych obiektów. Po zmianie wartości w polu Typ właściwości również ulegają zmianie na odpowiednie dla danego typu.
