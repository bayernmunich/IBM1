Dostawcy JMS (Java Message Service) dla klientów

W tym temacie opisano różne sposoby wykorzystania dostawców JMS (Java Message Service) przez aplikacje klienckie na serwerze WebSphere Application Server.

Serwer IBM WebSphere Application Server obsługuje funkcję asynchronicznego przesyłania komunikatów z wykorzystaniem dostawcy JMS i odpowiadającego mu systemu przesyłania komunikatów. Dostawcy usług JMS musza być zgodni ze specyfikacją JMS w wersji 1.1. W celu wykorzystania komponentów bean sterowanych komunikatami dostawca JMS musi obsługiwać funkcję ASF (Application Server Facility), zdefiniowaną w tej specyfikacji, lub obsługę adaptera zasobów przychodzących, zdefiniowanego w specyfikacji JCA w wersji 1.5.

Technologie integracji usług serwera IBM WebSphere Application Server mogą działać jako system przesyłania komunikatów przez skonfigurowanie magistrali integracji usługi, do której dostęp odbywa się za pomocą domyślnego dostawcy usługi przesyłania komunikatów. Obsługa ta jest instalowana jako składnik serwera WebSphere Application Server, administrowana za pomocą Konsoli administracyjnej i jest w pełni zintegrowana ze środowiskiem wykonawczym serwera WebSphere Application Server.

Serwer WebSphere Application Server zawiera także obsługę następujących dostawców JMS:
WebSphere MQ
Produkt udostępniony do stosowania z obsługiwanymi wersjami produktu WebSphere MQ.
Ogólny
Jest ona udostępniona do wykorzystania z dowolnymi systemami przesyłania komunikatów innych dostawców obsługujących ASF.

W celu zapewnienia kompatybilności wstecznej z poprzednimi wersjami produktu serwer WebSphere Application Server zawiera również obsługę domyślnego dostawcy przesyłania komunikatów w wersji 5. Dzięki temu możliwe jest skonfigurowanie zasobów do korzystania z systemu Embedded Messaging produktu WebSphere Application Server 5. Domyślny dostawca przesyłania komunikatów w wersji 5 produktu może być używany również z magistralą integracji usługi.

Aplikacje WebSphere mogą korzystać z zasobów usługi przesyłania komunikatów udostępnianych przez dowolnego z powyższych dostawców JMS. Wybór dostawcy jest jednak najczęściej dyktowany przez wymagania związane z użyciem lub integracją z istniejącym systemem przesyłania komunikatów. Na przykład użytkownik może już mieć infrastrukturę usługi przesyłania komunikatów opartą na produkcie WebSphere MQ. W takim wypadku użytkownik może albo połączyć się bezpośrednio jako dostawca JMS - korzystając z zawartej w produkcie obsługi produktu WebSphere MQ - lub skonfigurować magistralę integracji usługi z łączami do sieci WebSphere MQ, a następnie uzyskać dostęp do tej magistrali za pomocą domyślnego dostawcy usługi przesyłania komunikatów.

Magistrala integracji usługi zapewnia także dostęp do domyślnego dostawcy usługi komunikatów. Jest to dostawca JMS usługi przesyłania komunikatów zgodny ze standardem J2EE 1.4 i w pełni zintegrowany z serwerem WebSphere Application Server. Może być używany w wielu konfiguracjach serwera do obsługi interakcji usługi przesyłania komunikatów w sieci produktu WebSphere MQ.
