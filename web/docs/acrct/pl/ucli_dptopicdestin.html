Ustawienia miejsca docelowego tematu domyślnego dostawcy

Ten panel umożliwia wyświetlanie i zmianę właściwości konfiguracyjnych wybranego miejsca docelowego tematu JMS, które ma być używane z wewnętrznym dostawcą JMS (Java Message Service) produktu, instalowanym wraz z serwerem WebSphere Application Server.

Aby wyświetlić tę stronę narzędzia ACRCT (Application Client Resource Configuration Tool), należy kliknąć opcję Plik > Otwórz. Po wyszukaniu pliku EAR należy kliknąć przycisk Otwórz. Rozwiń wybrany plik JAR > Dostawcy przesyłania komunikatów > Domyślny dostawca. Kliknij prawym przyciskiem myszy folder Miejsca docelowe tematu, a następnie kliknij polecenie Nowe. Na karcie Ogólne zostaną wyświetlone następujące pola.

Nazwa

Nazwa wpisu miejsca docelowego tematu.

Typ danych String
Opis

Opis wpisu.

Typ danych String
Nazwa JNDI

Nazwa JNDI umożliwiająca dopasowanie tej definicji do nazwy resource-env-ref deskryptora wdrażania.

Typ danych String
Obszar tematu

Nazwa obszaru tematu. To pole jest wymagane.

Typ danych String
Wartość domyślna DEFAULT_TOPIC_SPACE
Nazwa tematu

Nazwa tematu. To pole jest wymagane.

Typ danych String
Tryb dostarczania

Domyślny tryb wysyłania komunikatów wysłanych do tego miejsca docelowego.

Typ danych String
Zakres Application, Trwałe lub Nietrwałe
Wartość domyślna Aplikacja
Czas życia

Domyślny czas przechowywania przez system komunikatu wysłanego do tego miejsca docelowego (liczony od momentu jego wysłania), gdzie wartość 0 oznacza nieskończony czas życia. Jeśli pole to nie jest wypełnione, użyta zostaje wartość producenta.

Typ danych Całkowitoliczbowy długi
Jednostki Milisekundy
Priorytet

Priorytet wysyłania komunikatów do tego miejsca docelowego. Jeśli pole to nie jest wypełnione, użyta zostaje wartość producenta.

Typ danych Integer
Zakres Od 0 do 9, gdzie 0 oznacza najniższy priorytet, a 9 najwyższy
Odczyt z wyprzedzeniem

Służy do sterowania optymalizacją odczytu z wyprzedzeniem podczas dostarczania komunikatu.

Typ danych String

Wartość domyślna Jako połączenie
