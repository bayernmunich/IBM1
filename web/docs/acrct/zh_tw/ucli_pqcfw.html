應用程式用戶端的第 5 版預設提供者 Queue Connection Factory 設定

請利用這個畫面來瀏覽或變更所選取 JMS Queue Connection Factory 的配置內容，供 WebSphere® Application Server 第 5 版應用程式進行點對點傳訊。 這些配置內容控制在 JMS 提供者與其所使用的預設傳訊系統之間建立連線的方式。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 第 5 版預設提供者。 用滑鼠右鍵按一下 Queue Connection Factory，再按一下新建。這時一般標籤會出現下列欄位。

Queue Connection Factory 用來建立通往 Queue Destination 的 JMS 連線。Queue Connection Factory 是內部 WebSphere Application Server 產品 JMS 提供者所建立的。 第 5 版預設提供者 Queue Connection Factory 有下列內容：

名稱

用來識別這個 Queue Connection Factory 的名稱，供 IBM® WebSphere Application Server 內管理作業使用。 在整個 WebSphere 管理網域的各個 JMS Connection Factory 內，這個名稱必須是唯一的。

資料類型 字串
說明

這個 Connection Factory 的說明，供 IBM WebSphere Application Server 內管理作業使用。

資料類型 字串
預設值 空值
JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

使用者 ID

在呼叫端應用程式沒有明確提供使用者 ID 和密碼時，要搭配密碼內容供鑑別使用的使用者 ID。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

當呼叫端應用程式沒有明確提供使用者 ID 和密碼時，比方說，當呼叫端應用程式使用 createQueueConnection() 方法時，會使用 Connection Factory 使用者 ID 和密碼內容。 JMS 用戶端會將使用者 ID 和密碼傳送給 JMS 伺服器。

資料類型 字串
密碼

當呼叫端應用程式沒有明確提供使用者 ID 和密碼時，要搭配使用者 ID 內容供鑑別使用的密碼。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

重新輸入密碼

確認密碼。

節點

這個 Connection Factory 的 JMS 伺服器執行所在之管理節點的 WebSphere 節點名稱。 這個 Factory 所建立的連線會連接至該 JMS 伺服器。

資料類型 字串
應用程式伺服器

請輸入應用程式伺服器的名稱。這個名稱不是機器的主機名稱，而是所配置應用程式伺服器的名稱。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
