應用程式用戶端的「URL 提供者」設定

請利用這個頁面來建立新的 URL 提供者。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔。用滑鼠右鍵按一下 URL 提供者，再按一下新建。 這時一般標籤會出現下列欄位。

URL 提供者負責實作特定 URL 通訊協定的功能，如超文字傳送通訊協定 (HTTP)。這個由類別組成的提供者繼承 java.net.URLStreamHandler 和 java.net.URLConnection 類別。

名稱

URL 的管理名稱。

說明

管理記錄的 URL 的選用說明。

類別路徑

這是共同形成資源提供者類別位置的路徑或 JAR 檔名稱的清單。

通訊協定

這個串流處理常式所支援的通訊協定。例如，nntp、smtp、ftp 等等。

若要使用預設通訊協定，請保留這個欄位空白。

串流處理常式類別

使用者定義的 Java™ 類別的完整名稱，繼承特定 URL 通訊協定（如 FTP）的 java.net.URLStreamHandler。

若要使用預設串流處理常式，請保留這個欄位空白。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
