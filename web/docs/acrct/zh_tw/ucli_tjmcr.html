配置 Java 傳訊用戶端資源

若要配置 Java™ 傳訊用戶端資源，請建立應用程式用戶端的新 JMS 提供者配置。 應用程式用戶端可以透過 Java 訊息服務 API 來使用傳訊服務。JMS 提供者提供兩類 J2EE Factory。 一類是 JMS Connection Factory，另一類是 JMS 目的地 Factory。

開始之前

請在個別管理作業中，將 Java 訊息服務 (JMS) 用戶端安裝在應用程式用戶端所在的用戶端機器中。 傳訊產品供應商必須提供 JMS 用戶端的實作。如需相關資訊，請參閱您的傳訊產品文件。

註: 完成這項作業之後，您可以建立一個新的傳訊提供者，也可以使用現有的傳訊提供者。

程序

  1. 啟動應用程式用戶端資源配置工具 (ACRCT)。
  2. 開啟要配置新 JMS 提供者的 EAR 檔。 EAR 檔內容在所顯示的樹狀視圖中。
  3. 從樹狀結構中，選取您要配置新 JMS 提供者的 JAR 檔。
  4. 展開 JAR 檔，以檢視其內容。
  5. 如果您要建立和使用新的傳訊提供者，請選擇性地用滑鼠右鍵按一下傳訊提供者，再選取新建。
  6. 在出現的內容對話框中，配置 JMS 提供者內容。
  7. 按一下確定。
  8. 按一下檔案 > 儲存。


標示（線上）的鏈結表示需要存取網際網路。
