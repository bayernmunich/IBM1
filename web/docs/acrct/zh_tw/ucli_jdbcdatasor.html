應用程式用戶端的資料來源內容

請利用這個頁面來建立或修改資料來源。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 資料來源提供者 > 資料來源提供者實例。 用滑鼠右鍵按一下資料來源，再按一下新建。 這時一般標籤會出現下列欄位：

名稱

請指定這個資料來源的顯示名稱。

資料類型 字串
說明

請指定資料來源的文字說明。

資料類型 字串
JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

資料庫名稱

要連接的資料庫名稱。

使用者

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼，請搭配密碼內容來利用使用者 ID 進行鑑別。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。 如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼，就會使用 Connection Factory 使用者 ID 和密碼內容。

密碼

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼，請搭配使用者 ID 內容來使用密碼進行鑑別。

如果您指定了「密碼」內容的值，您也必須指定「使用者 ID」內容的值。

重新輸入密碼

確認密碼。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。



標示（線上）的鏈結表示需要存取網際網路。
