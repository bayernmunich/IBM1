應用程式用戶端的 JMS 提供者設定

如果您要使用預設傳訊提供者或作為 JMS 提供者之 WebSphere® MQ 以外的 JMS 提供者，請利用這個頁面來配置 Java™ 訊息服務 (JMS) 提供者的內容。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔。用滑鼠右鍵按一下傳訊提供者，然後按一下新建。這時一般標籤會出現下列欄位。

名稱

用來識別 JMS 提供者供管理使用的名稱。

資料類型 字串
說明

JMS 提供者的說明，供管理使用。

資料類型 字串
類別路徑

共同形成資源提供者類別位置的路徑或 .jar 檔名稱的清單。

環境定義 Factory 類別

JMS 提供者的起始環境定義 Factory 的 Java 類別名稱。

比方說，如果是 LDAP 服務供應商，值的格式如下：com.sun.jndi.ldap.LdapCtxFactory。

資料類型 字串
提供者 URL

外部 JNDI 查閱的 JMS 提供者 URL。

例如，JMS 提供者的 LDAP URL 格式如下：ldap://hostname.company.com/contextName。

資料類型 字串
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
