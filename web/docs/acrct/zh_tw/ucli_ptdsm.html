應用程式用戶端的 WebSphere MQ Provider 主題目的地設定

請利用這個畫面來檢視或變更所選主題目的地的配置內容，以便搭配使用 WebSphere® MQ 產品「Java™ 訊息服務 (JMS)」提供者。

如果要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。 展開選取的 JAR 檔 > 傳訊提供者 > WebSphere MQ 提供者。 用滑鼠右鍵按一下 Topic Destination，再按一下新建。 這時一般標籤會出現下列欄位。

附註：
  • 您指定的內容值必須符合配置 WebSphere MQ 的 JMS 資源時所指定的值。 如需配置 WebSphere MQ 之 JMS 資源的相關資訊，請參閱 WebSphere MQ 資訊中心的《使用 Java》。
  • 在 WebSphere MQ 中，名稱最多可有 48 個字元。

Topic Destination 用來配置相關聯的 JMS 提供者之 JMS 主題的內容。 搭配 WebSphere MQ 產品 JMS 提供者來使用的主題，其內容如下。

名稱

用來識別主題的名稱，供 WebSphere Application Server 內管理作業使用。

資料類型 String
說明

主題的說明，供 WebSphere Application Server 內管理作業使用。

資料類型 String
JNDI 名稱

應用程式用戶端執行時期環境利用這個欄位來擷取配置資訊。

持續性

傳送至目的地的所有訊息是全部具有持續性、不具持續性，或由應用程式來定義其持續性。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在目的地的訊息，其持續性由將它們放入佇列的應用程式來定義。
已定義的佇列
[只適用於 WebSphere MQ 目的地] 在目的地的訊息，其持續性由 WebSphere MQ 佇列定義內容來定義。
持續性
在目的地的訊息具有持續性。
非持續性
在目的地的訊息沒有持續性。
優先順序

這個目的地的訊息優先順序是由應用程式或指定的優先順序內容來定義的。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在這個目的地之訊息，其優先順序由將它們放入目的地的應用程式來定義。
已定義的佇列
[只適用於 WebSphere MQ 目的地] 在目的地的訊息，其持續性由 WebSphere MQ 佇列定義內容來定義。
指定的
這個目的地的訊息優先順序由指定的優先順序內容來定義。 如果您選取這個選項，您必須定義一個優先順序給指定的優先順序內容。
指定的優先順序

如果優先順序內容設定為指定的，請指定這個佇列的訊息優先順序，範圍是 0（最低）至 9（最高）。

如果優先順序內容設定為指定的，送往這個佇列的訊息，其優先順序值由這個內容來指定。

資料類型 Integer
單位 訊息優先順序層次
範圍 0（最低優先順序）至 9（最高優先順序）
期限

這個佇列的期限逾時值是由應用程式來定義，或由指定的期限內容來定義，或是佇列中的訊息永遠不會到期（期限逾時值無限）。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在這個佇列之訊息，其期限逾時值由將它們放入佇列的應用程式來定義。
指定的
這個佇列中的訊息，其期限逾時值由指定的期限內容來定義。如果您選取這個選項，您必須定義一個逾時值給指定的期間內容。
無限
這個佇列中的訊息沒有期限逾時值，因此，這些訊息永遠不會到期。
指定的期限

如果期限逾時值內容設為指定，請輸入一個毫秒數（大於 0），指定這個佇列中的訊息過了多久就到期。

資料類型 Integer
單位 毫秒
範圍 大於或等於 0
  • 0 表示訊息永遠不會逾時。
  • 其他值是整數的毫秒數。
基本主題名稱

訊息要送往的主題名稱。

資料類型 String
CCSID

搭配 WebSphere MQ 佇列管理程式使用的編碼字集 ID。

這個編碼字集 ID (CCSID) 必須是 WebSphere MQ 支援的其中一個 CCSID。請參閱「WebSphere MQ 傳訊提供者佇列和主題進階內容設定」，以取得詳細資料。

資料類型 String
單位 Integer
範圍 1 至 65535 個字元
整數編碼

如果沒有啟用原生編碼，請選取整數編碼是正常或反向。

資料類型 列舉
預設值 NORMAL
範圍
NORMAL
使用正常整數編碼。
REVERSED
使用反向整數編碼。

如需編碼內容的相關資訊，請參閱 WebSphere MQ 資訊中心的《使用 Java》。
十進位編碼

如果沒有啟用原生編碼，請選取十進位編碼是正常或反向。

資料類型 列舉
預設值 NORMAL
範圍
NORMAL
使用正常十進位編碼。
REVERSED
使用反向十進位編碼。

如需編碼內容的相關資訊，請參閱 WebSphere MQ 資訊中心的《使用 Java》。
浮點數編碼

如果沒有啟用原生編碼，請選取浮點數編碼的類型。

資料類型 列舉
預設值 IEEENORMAL
範圍
IEEENORMAL
使用 IEEE 正常浮點數編碼。
IEEEREVERSED
使用 IEEE 反向浮點數編碼。
S390
使用 S/390® 浮點數編碼。

如需編碼內容的相關資訊，請參閱 WebSphere MQ 資訊中心的《使用 Java》。
原生編碼

指示在您選取這個勾選框時，Queue Destination 使用原生編碼（Java 平台所適用的編碼值）。

資料類型 列舉
預設值 已清除
範圍
已清除
沒有使用原生編碼，因此，請指定整數、十進位和浮點數編碼的上述內容。
已選取
使用原生編碼（以提供 Java 平台所適用的編碼值）。

如需編碼內容的相關資訊，請參閱 WebSphere MQ 資訊中心的《使用 Java》。
BrokerDurSubQueue

從中擷取延續性訂閱訊息的分配管理系統佇列之名稱。

訂閱者在登錄訂閱時指定佇列的名稱。

資料類型 String
單位 En_US ASCII 字元
範圍 1 至 48 個 ASCII 字元
BrokerCCDurSubQueue

從中擷取 ConnectionConsumer 的延續性訂閱訊息的分配管理系統佇列之名稱。這個內容只適用於 Web 儲存器。

資料類型 String
單位 En_US ASCII 字元
範圍 1 至 48 個 ASCII 字元
目標用戶端

接收端應用程式是 JMS 標準應用程式或傳統 WebSphere MQ 應用程式。

資料類型 列舉
預設值 WebSphere MQ
範圍
WebSphere MQ
目標是不支援 JMS 的傳統 WebSphere MQ 應用程式。
JMS
目標應用程式支援 JMS。
訊息內文

指定應用程式是否將 WebSphere MQ 訊息的 RFH 第 2 版標頭，當作 JMS 訊息內文的一部分來處理。

資料類型 下拉清單
預設值 UNSPECIFIED
範圍
UNSPECIFIED
當傳送訊息時，依「將 RFH 第 2 版標頭附加於傳送到此目的地的訊息」內容值而定，WebSphere MQ 傳訊提供者有可能會（或不會）產生及併入 RFH 第 2 版標頭。 當收到訊息時，WebSphere MQ 傳訊提供者會依照設定為 JMS 值的方式來運作。
JMS
當傳送訊息時，WebSphere MQ 傳訊提供者會自動產生 RFH 第 2 版標頭，並將它併入 WebSphere MQ 訊息中。 當收到訊息時，WebSphere MQ 傳訊提供者會根據 RFH 第 2 版標頭中的值（如果這些值存在的話）來設定 JMS 訊息內容；它不會將 RFH 第 2 版標頭呈現為 JMS 訊息內文的一部分。
MQ
當傳送訊息時，WebSphere MQ 傳訊提供者不會產生 RFH 第 2 版標頭。 當收到訊息時，WebSphere MQ 傳訊提供者會將 RFH 第 2 版標頭呈現為 JMS 訊息內文的一部分。
ReplyTo 目的地樣式

指定 JMSReplyTo 欄位格式。

資料類型 下拉清單
預設值 DEFAULT
範圍
DEFAULT
預設值相當於 RFH 第 2 版標頭中的資訊。
MQMD
使用 MQMD 所提供的值。 這會將 MQMD 的值移入回覆目的地佇列管理程式欄位，相當於 WebSphere MQ 6.0.2.4 版和 6.0.2.5 版的預設行為。
RFH2
使用 RFH 第 2 版標頭所提供的值。 如果傳送端應用程式設定了 JMSReplyTo 值，就使用這個值。
多重播送

這個 Connection Factory 是否使用多重播送傳輸。

資料類型 列舉
預設值 AS_CF
範圍
AS_CF
這個 Connection Factory 使用多重播送傳輸。
DISABLED
這個 Connection Factory 不使用多重播送傳輸。
NOT_RELIABLE
這個 Connection Factory 一律使用多重播送傳輸。
RELIABLE
當主題目的地不可靠時，這個 Connection Factory 會使用多重播送傳輸。
ENABLED
這個 Connection Factory 使用可靠的多重播送傳輸。
已啟用 MQMD 讀取

指定應用程式是否能夠讀取利用 WebSphere MQ 傳訊提供者來傳送或接收之 JMS 訊息中的 MQMD 欄位值。

資料類型 勾選框
預設值 已清除
範圍
已清除
應用程式無法讀取 MQMD 欄位值。
已選取
應用程式可以讀取 MQMD 欄位值。
已啟用 MQMD 寫入

指定應用程式是否能夠寫入利用 WebSphere MQ 傳訊提供者來傳送或接收之 JMS 訊息中的 MQMD 欄位值。

資料類型 勾選框
預設值 已清除
範圍
已清除
應用程式無法寫入 MQMD 欄位值。
已選取
應用程式可以寫入 MQMD 欄位值。
MQMD 訊息環境定義

定義將訊息傳送到目的地時，所指定的訊息環境定義選項。

資料類型 下拉清單
預設值 DEFAULT
範圍
DEFAULT
MQOPEN API 呼叫和 MQPMO 結構不指定明確的訊息環境定義選項。
SET_IDENTITY_CONTEXT
MQOPEN API 呼叫指定訊息環境定義選項 MQOO_SET_IDENTITY_CONTEXT，MQPMO 結構指定 MQPMO_SET_IDENTITY_CONTEXT。
SET_ALL_CONTEXT
MQOPEN API 呼叫指定訊息環境定義選項 MQOO_SET_ALL_CONTEXT，MQPMO 結構指定 MQPMO_SET_ALL_CONTEXT。
