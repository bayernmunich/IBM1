應用程式用戶端的通用 JMS Connection Factory 設定

請利用這個畫面來檢視或變更所選取 Java™ 訊息服務 (JMS) Connection Factory 的配置內容，搭配使用相關的 JMS 提供者。這些配置內容用來控制如何建立 JMS 提供者與 JMS 提供者所使用的傳訊系統之間的連線。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > new_JMS_Provider_instance。 用滑鼠右鍵按一下 Connection Factory，再按一下新建。這時一般標籤會出現下列欄位。

Java 訊息服務 (JMS) Connection Factory 用來建立與 JMS 目的地的連線。 JMS Connection Factory 是相關聯的 JMS 提供者所建立的。一般 JMS 提供者（不是內部預設傳訊提供者或作為 JMS 提供者的 WebSphere® MQ）的 JMS Connection Factory 具有下列內容：

名稱

用來識別這個 JMS Connection Factory 的名稱，供 IBM® WebSphere Application Server 內管理作業使用。在相關聯的 JMS 提供者內，名稱必須是唯一的。

資料類型 字串
說明

這個 Connection Factory 的說明，供 IBM WebSphere Application Server 內管理作業使用。

資料類型 字串
預設值 空值
JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

使用者 ID

指示如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼時，就要搭配密碼內容來進行鑑別的使用者 ID。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

當呼叫端應用程式沒有明確提供使用者 ID 和密碼時，比方說，當呼叫端應用程式使用 createQueueConnection() 方法時，會使用 Connection Factory 使用者 ID 和密碼內容。 JMS 用戶端會將使用者 ID 和密碼傳送給 JMS 伺服器。

資料類型 字串
密碼

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼時，就要搭配使用者 ID 內容來進行鑑別的密碼。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

資料類型 字串
預設值 空值
重新輸入密碼

確認在密碼欄位中輸入的密碼。

外部 JNDI 名稱

用來將佇列連結至應用程式伺服器名稱空間的 JNDI 名稱。

請依照慣例使用完整的 JNDI 名稱，比方說，jms/Name，其中 Name 是資源的邏輯名稱。

這個名稱用來鏈結平台的連結資訊。連結會將模組部署描述子所定義的資源關聯到由平台連結至 JNDI API 內的實際（實體）資源。

資料類型 字串
連線類型

這個 JMS 目的地是 Queue（用於點對點）或 Topic（用於發佈/訂閱）。

請選取下列其中一個選項：
Queue
點對點傳訊所用的 JMS Queue Destination。
Topic
發佈訂閱傳訊所用的 JMS Topic Destination。
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
