預設提供者 Queue Connection Factory 設定

請利用這個畫面來檢視或變更所選取 JMS Queue Connection Factory 的配置內容，以與隨著 WebSphere® Application Server 一起安裝的內部產品 Java™ 訊息服務 (JMS) 提供者搭配使用。 這些配置內容用來控制 JMS 提供者和它所用的服務整合匯流排之間的連線建立方式。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 預設提供者。 用滑鼠右鍵按一下 Queue Connection Factory，再按一下新建。這時一般標籤會出現下列欄位。

具有預設值的設定會顯示適當的值。具有固定值的任何設定則會有下拉功能表。

名稱

Queue Connection Factory 的名稱。

資料類型 字串
說明

這個 Queue Connection Factory 的說明，供 IBM® WebSphere Application Server 內管理作業使用。

資料類型 字串
JNDI 名稱

用來在部署描述子比對這個 Queue Connection Factory 定義的 JNDI 名稱。這個項目是 resource-ref 名稱。

資料類型 字串
使用者名稱

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼時，就要搭配密碼內容供鑑別使用的使用者名稱。如果使用這個欄位，則會忽略 UserName 內容欄位。

如果您指定了使用者名稱內容的值，您也必須指定密碼內容的值。

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼，就會使用 Connection Factory 使用者名稱和密碼內容。如果有指定使用者名稱和密碼，則會為加密密碼的 Factory 建立鑑別別名。

資料類型 字串
密碼

如果您指定了使用者名稱內容的值，您也必須指定密碼內容的值。

資料類型 字串

用戶端 ID

用戶端 ID。可延續的主題訂閱需要這個項目。

資料類型 字串
非持續性傳訊可靠性

套用在使用這個 Connection Factory 所傳送之非持續性 JMS 訊息的可靠性。

如果個別 JMS 目的地要有不同的可靠性遞送選項，您可以將這個內容設定為作為匯流排目的地。 之後，可靠性就由被指派 JMS 目的地的匯流排目的地之「可靠性」內容來定義。

預設值 ReliablePersistent
範圍
無
沒有非持續性訊息的訊息可靠性。如果無法遞送非持續性訊息，就會捨棄它。
最大努力非持續性
訊息絕不寫入磁碟中，如果記憶體快取超限運轉，就會將它們捨棄。
快速非持續性
如果記憶體快取超限運轉，訊息會以非同步方式寫入持續性儲存庫，但是伺服器重新啟動後不會保存。
可靠非持續性
如果傳訊引擎失敗，訊息可能會遺失，在正常作業狀況下，也可能會遺失。
可靠持續性
如果傳訊引擎失敗，訊息可能會遺失，但在正常作業狀況下，不會遺失。
確定持續性
當支援擔保訊息遞送時，最高程度的可靠性。
作為匯流排目的地
使用配置給匯流排目的地的遞送選項。

目標入埠傳輸鏈

資料類型 字串
提供者端點

用來連接引導伺服器的端點清單，以逗點區隔各個端點。

請輸入端點 Triplet 的逗點分隔清單，語法如下：host:port:protocol。

範例 
localhost:7777:BootstrapBasicMessaging
其中

BootstrapBasicMessaging 對應於遠端通訊協定 InboundBasicMessaging (JFAP-TCP/IP)。
預設值
  • 如果未指定主機名稱，就會使用預設的 localhost 作為預設值。
  • 如果未指定埠號，預設值就是 7276。
  • 如果未指定鏈名稱，就會使用預先定義的鏈（如 BootstrapBasicMessaging）作為預設值。
連線近似性

傳訊引擎對要求端應該具有的近似性。

預設值 匯流排、叢集、伺服器
範圍 匯流排、主機
暫時佇列名稱字首

套用暫時佇列名稱的字首。這個名稱最長為 12 個字元。
