用戶端的 Java 訊息服務 (JMS) 提供者

這個主題將說明用戶端應用程式將 Java™ 訊息服務 (JMS) 提供者搭配 WebSphere® Application Server 使用的幾種方式。

IBM® WebSphere Application Server 利用 JMS 提供者及相關的傳訊系統來支援非同步傳訊。 JMS 提供者必須符合 JMS 規格 1.1 版。若要使用訊息驅動 Bean，JMS 提供者必須支援這個規格所定義的選用應用程式伺服器機能 (ASF) 功能，或支援 JCA 規格 1.5 版所定義的入埠資源配接器。

當您已配置透過預設傳訊提供者來存取的服務整合匯流排，IBM WebSphere Application Server 的服務整合技術可以作為傳訊系統。 這項支援是隨著 WebSphere Application Server 而一起安裝，利用管理主控台來管理，與 WebSphere Application Server 執行時期完全整合。

WebSphere Application Server 也支援下列 JMS 提供者：
WebSphere MQ
專用來搭配支援的 WebSphere MQ 版本。
通用
專用來搭配任何支援 ASF 的協力廠商傳訊系統。

為了與舊版相容，WebSphere Application Server 也支援第 5 版預設傳訊提供者，可讓您配置搭配 WebSphere Application Server 第 5 版內嵌傳訊系統使用的資源。 第 5 版的預設傳訊提供者也可以搭配服務整合匯流排來使用。

WebSphere 應用程式可以使用上述任何 JMS 提供者所提供的傳訊資源。 不過，選擇哪一個提供者通常是取決於與現有傳訊系統搭配或整合的需求。 比方說，您可能已有一個以 WebSphere MQ 為基礎的傳訊基礎架構。 這時您可以將內含的 WebSphere MQ 支援當作 JMS 提供者而直接連接，或利用通往 WebSphere MQ 網路的鏈結來配置服務整合匯流排，再利用預設的傳訊提供者來存取匯流排。

另外，服務整合匯流排也可用來存取預設傳訊提供者。 這是完全整合在 WebSphere Application Server 中的 J2EE 1.4 標準 JMS 傳訊提供者。 您可以在多個伺服器配置中，利用它來與 WebSphere MQ 網路進行傳訊互動。
