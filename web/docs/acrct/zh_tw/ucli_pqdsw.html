應用程式用戶端的第 5 版預設提供者 Queue Destination 設定

請利用這個畫面來檢視或變更所選取 Queue Destination 的配置內容，以與產品 Java™ 訊息服務 (JMS) 提供者搭配使用。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 第 5 版預設提供者。 用滑鼠右鍵按一下 Queue Destination，再按一下新建。 這時一般標籤會出現下列欄位。

Queue Destination 用來配置 JMS 佇列的內容。 第 5 版預設提供者 Queue Destination 有下列內容。

名稱

用來識別佇列的名稱，供 WebSphere® Application Server 內管理作業使用。

資料類型 字串
說明

佇列的說明，用來進行管理。

資料類型 字串
JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

持續性

傳送至目的地的所有訊息是全部具有持續性、不具持續性，或由應用程式來定義其持續性。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在目的地的訊息，其持續性由將它們放入佇列的應用程式來定義。
已定義的佇列
[只適用於 WebSphere MQ 目的地] 在目的地的訊息，其持續性由 WebSphere MQ 佇列定義內容來定義。
持續性
在目的地的訊息具有持續性。
非持續性
在目的地的訊息沒有持續性。
優先順序

這個目的地的訊息優先順序是由應用程式或指定的優先順序內容來定義的。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
在這個目的地之訊息，其優先順序由將它們放入目的地的應用程式來定義。
已定義的佇列
[只適用於 WebSphere MQ 目的地] 在目的地的訊息，其持續性由 WebSphere MQ 佇列定義內容來定義。
指定的
這個目的地的訊息優先順序由指定的優先順序內容來定義。 如果您選取這個選項，您必須在指定的優先順序內容中定義優先順序。
指定的優先順序

如果優先順序內容設定為指定的，請在這裡輸入這個佇列的訊息優先順序，範圍是 0（最低）至 9（最高）。

如果優先順序內容設定為指定的，送往這個佇列的訊息，其優先順序值由這個內容來指定。

資料類型 整數
單位 訊息優先順序層次
範圍 0（最低優先順序）至 9（最高優先順序）
期限

這個佇列的期限逾時值是由應用程式來定義，或由指定的期限內容來定義，或佇列中的訊息是否永遠不會到期（期限逾時值無限）。

資料類型 列舉
預設值 APPLICATION_DEFINED
範圍
定義的應用程式
這個佇列中的訊息，其期限逾時值由將它們放入佇列的應用程式來定義。
指定的
這個佇列中的訊息，其期限逾時值由指定的期限內容來定義。如果您選取這個選項，您必須在指定的期限內容中定義一個逾時值。
無限
這個佇列中的訊息沒有期限逾時值，因此，訊息永遠不會到期。
指定的期限

如果期限逾時值內容設定為指定的，請指定這個佇列中的訊息到期之前的毫秒數（大於 0）。

資料類型 整數
單位 毫秒
範圍 大於或等於 0
  • 0 表示訊息永遠不會逾時。
  • 其他值是整數的毫秒數。
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
