用戶端的資源配接器

資源配接器是一個系統層次的軟體驅動程式，Java™ 應用程式利用它來連接企業資訊系統 (EIS)。 資源配接器會插入應用程式用戶端中，並且在 EIS 和企業應用程式之間提供連線。

J2EE 用戶端應用程式的資源配接器支援是伺服器支援的一部分。 就拿利用 clientRAR 工具安裝的資源配接器來說，用戶端資源配接器是用在非管理的環境中，它必須符合 J2EE 連接器架構規格 1.5 版或更新的版本。只有 EIS 的出埠連線，才能透過 ManagedConnectionFactory 介面加以支援。入埠訊息支援 (EIS)、生命週期管理以及規格的工作管理層面，用戶端都不支援。

如果是使用資源配接器的用戶端應用程式，它必須安裝在執行 setupCmdLine Script 時定義的 CLIENT_CONNECTOR_INSTALL_ROOT 環境變數所指定的目錄之下。 launchClient 工具、應用程式用戶端資源配置工具 (ACRCT) 和 clientRAR 工具，都是利用這個變數，來尋找所有安裝之資源配接器的預設位置。若要把資源配接器安裝在用戶端，請使用 clientRAR 工具。 資源配接器一安裝之後，就必須用 ACRCT 加以配置。用戶端配置工具會把資源配接器配置加到 EAR 檔中。這樣，就定義了 Connection Factory 和受管理的物件。

在執行 J2EE 應用程式用戶端時，launchClient Script 會指定一個系統內容 com.ibm.ws.client.installedConnector，這個內容與 CLIENT_CONNECTOR_INSTALL_ROOT 變數同值。 這是已安裝之資源配接器的預設位置，您可以在每一個 launchClient 呼叫指定 -CCD 參數來置換它。 當用戶端儲存器啟動時，所有在資源配接器目錄指定預設位置下的資源配接器子目錄，都會加到類別路徑中。這個動作可讓用戶端應用程式使用資源配接器，而不必以 ACRCT 指定任何用戶端資源。

使用資源配接器是一種新的機制，可以讓您輕鬆的延伸用戶端應用程式。
