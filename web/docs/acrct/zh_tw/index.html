應用程式用戶端資源配置工具

導覽至這個區段中的說明主題。

  • 配置 Java 傳訊用戶端資源
    若要配置 Java 傳訊用戶端資源，請建立應用程式用戶端的新 JMS 提供者配置。 應用程式用戶端可以透過 Java 訊息服務 API 來使用傳訊服務。JMS 提供者提供兩類 J2EE Factory。 一類是 JMS Connection Factory，另一類是 JMS 目的地 Factory。
  • 配置應用程式用戶端的新 URL 提供者
    您可以利用應用程式用戶端資源配置工具 (ACRCT)，來建立用戶端應用程式的 URL 提供者和 URL。
  • 配置應用程式用戶端的新資料來源提供者（JDBC 提供者）
    您可以利用應用程式用戶端資源配置工具 (ACRCT)，來建立應用程式用戶端的新資料來源提供者，也稱為 JDBC 提供者。
  • 配置應用程式用戶端的新資源環境提供者
    您可以利用應用程式用戶端資源配置工具 (ACRCT)，來建立用戶端應用程式的新資源環境提供者配置。
  • 配置應用程式用戶端的新資源環境項目
    您可以使用應用程式用戶端資源配置工具 (ACRCT) 來建立用戶端應用程式的新資源環境項目。
  • 配置應用程式用戶端的新郵件階段作業
    您可以利用應用程式用戶端資源配置工具 (ACRCT)，來配置應用程式用戶端的新郵件階段作業。
  • 配置應用程式用戶端的郵件提供者和階段作業
    您可以利用應用程式用戶端資源配置工具 (ACRCT)，來編輯應用程式用戶端的郵件階段作業和提供者的配置。
  • 配置管理物件
    這一節協助您配置新的受管理物件。
  • 針對應用程式用戶端，更新作為 Java 訊息服務提供者的 WebSphere MQ 及其 JMS 資源配置
    您可以更新作為 Java 訊息服務 (JMS) 提供者之 WebSphere MQ 的現有配置，以及更新 WebSphere MQ Connection Factory 或 WebSphere MQ 目的地的配置。
  • 在 WebSphere Application Server 中使用 JMS 進行非同步傳訊
    WebSphere Application Server 支援將非同步傳訊當作以 Java 訊息服務 (JMS) 程式設計介面為基礎的通訊方法。 JMS 介面提供一種共用的方法，讓 Java 程式（用戶端及 Java Platform Enterprise Edition (Java EE) 應用程式）能夠將非同步要求當作 JMS 訊息來建立、傳送、接收和讀取。
  • 用戶端的 Java 訊息服務 (JMS) 提供者
    這個主題將說明用戶端應用程式將 Java 訊息服務 (JMS) 提供者搭配 WebSphere Application Server 使用的幾種方式。
  • 用戶端的資源配接器
    資源配接器是一個系統層次的軟體驅動程式，Java 應用程式利用它來連接企業資訊系統 (EIS)。 資源配接器會插入應用程式用戶端中，並且在 EIS 和企業應用程式之間提供連線。
  • 資源環境提供者和資源環境項目
    資源環境參照會將用戶端應用程式所用的邏輯名稱，對映至物件的實體名稱。
  • URLs for application clients
    A Uniform Resource Locator (URL) is an identifier that points to an electronically accessible resource, such as a directory file on a machine in a network, or a document stored in a database.
