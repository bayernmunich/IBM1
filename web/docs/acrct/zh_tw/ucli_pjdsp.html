應用程式用戶端的通用 JMS 目的地設定

請利用這個畫面來檢視或變更選取要搭配相關 JMS 提供者使用之 JMS 目的地的配置內容。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 新建 JMS 提供者實例。 用滑鼠右鍵按一下目的地，然後按一下新建。這時一般標籤會出現下列欄位。

JMS 目的地用來配置相關聯的通用 JMS 提供者之 JMS 目的地的內容。 通往 JMS 目的地的連線是由相關聯的 JMS Connection Factory 來建立的。用來搭配一般 JMS 提供者（不是預設傳訊提供者或作為 JMS 提供者的 WebSphere® MQ）的 JMS 目的地有下列內容。

名稱

用來識別佇列的名稱，供 WebSphere Application Server 內管理作業使用。

資料類型 字串
說明

佇列的說明，用來進行管理。

JNDI 名稱

連結至 JNDI 之 JMS 目的地的實際（實體）名稱之 JNDI 名稱。

外部 JNDI 名稱

用來將佇列連結至應用程式伺服器名稱空間的 JNDI 名稱。

請依照慣例使用完整的 JNDI 名稱；比方說，在 jms/Name 格式中，Name 是資源的邏輯名稱。

這個名稱用來鏈結平台的連結資訊。連結會將模組部署描述子所定義的資源關聯到由平台連結至 JNDI 內的實際（實體）資源。

資料類型 字串
目的地類型

這個 JMS 目的地是 Queue（用於點對點）或 Topic（用於發佈/訂閱）。

請選取下列其中一個選項：
Queue
點對點傳訊所用的 JMS Queue Destination。
Topic
發佈/訂閱傳訊所用的 JMS Topic Destination。
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
