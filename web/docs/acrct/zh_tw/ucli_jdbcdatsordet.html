應用程式用戶端的資料來源提供者設定

請利用這個頁面，在 JDBC 提供者下建立一個資料來源，來提供特定 JDBC 驅動程式實作類別。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔。用滑鼠右鍵按一下資料來源提供者 >，再按一下新建。這時一般標籤會出現下列欄位：

名稱

請指定資料來源的顯示名稱。

例如，您可以將這個欄位設定為測試資料來源。

資料類型 字串
說明

請指定資源的文字說明。

資料類型 字串
類別路徑

共同形成資源提供者類別位置的路徑或 .jar 檔名稱的清單。

實作類別

請利用這個頁面來執行資料庫特定的功能。

資料類型 字串
預設值 相依於 JDBC 驅動程式實作類別
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
