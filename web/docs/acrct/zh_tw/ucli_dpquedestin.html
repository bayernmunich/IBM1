預設提供者 Queue Destination 設定

請利用這個畫面來檢視或變更所選取 JMS Queue Destination 的配置內容，以與隨著 WebSphere® Application Server 一起安裝的內部產品 Java™ 訊息服務 (JMS) 提供者搭配使用。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 預設提供者。 用滑鼠右鍵按一下 Queue Destinations。然後再按一下新建。這時一般標籤會出現下列欄位。

名稱

Queue Destination Factory 的名稱。您必須完成這個欄位。

資料類型 字串
說明

這個 Queue Destination 的說明，供 WebSphere Application Server 內管理作業使用。

資料類型 字串
JNDI 名稱

用來將這個定義與部署描述子 resource-env-ref 名稱比對的 JNDI 名稱。

資料類型 字串
佇列名稱

佇列的名稱。

資料類型 字串
遞送模式

訊息傳送到這個目的地的遞送模式。

資料類型 字串
範圍 應用程式、持續或非持續
預設值 應用程式
存活時間

從訊息傳送到這個目的地的分派時間開始，系統應予保留的預設時間長度；其中 0 表示存活時間值不會到期。如果未指定「存活時間」欄位，則會使用生產者的值。

資料類型 整數
單位 毫秒
優先順序

訊息傳送到這個目的地的優先順序。如果未指定，則會使用生產者的值。

資料類型 整數
範圍 0 到 9，0 表示最低優先順序，9 表示最高優先順序。
先讀

用來在訊息遞送期間控制先讀最佳化。

資料類型 字串
範圍 AsConnection、AlwaysOn 和 AlwaysOff
預設值 AsConnection
