資源配接器設定

請利用這個畫面來檢視或變更資源配接器的配置內容。這些配置內容控制如何建立資源配接器。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 資源配接器。用滑鼠右鍵按一下資源配接器，再按一下新建。這時一般標籤會出現下列欄位。

名稱

用來識別這個資源配接器的名稱，供 IBM® WebSphere® Application Server 內管理作業使用。 在跨產品管理網域的資源配接器內，這個名稱必須是唯一的。

資料類型 字串
說明

這個資源配接器的說明，供 IBM WebSphere Application Server 內管理作業使用。

資料類型 字串
類別路徑

任何其他類別路徑。要自動新增的資源配接器目錄路徑。

資料類型 字串
預設值 您的資源配接器目錄的路徑。
原生路徑

資源配接器所在的原生路徑。在此輸入任何其他的原始類別路徑。

資料類型 字串
資源配接器名稱

指向安裝的資源配接器子目錄的必要輸入欄位。這個項目不代表資源配接器的完整目錄名稱。完整的目錄名稱是安裝的資源配接器路徑加上資源配接器名稱。

資料類型 字串
安裝的資源配接器路徑

安裝資源配接器的目錄。如果您未完成這個欄位，則預設值會生效。

如果您指定值 ${CONNECTOR_INSTALL_ROOT}，則這個值會取代用戶端應用程式執行機器上的 CLIENT_CONNECTOR_INSTALL_ROOT 變數的值。這個動作讓應用程式很容易地就能在不同的機器上（用戶端可能安裝在不同的位置）執行。

資料類型 字串
預設值 ${CONNECTOR_INSTALL_ROOT}
