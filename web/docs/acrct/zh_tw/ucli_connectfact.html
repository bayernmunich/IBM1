資源配接器 Connection Factory 設定

請利用這個畫面來檢視或變更選定之資源配接器 Connection Factory 的配置內容。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 資源配接器。用滑鼠右鍵按一下 Connection Factory 資料夾，然後按一下新建。這時一般標籤會出現下列欄位。

名稱

用來識別這個 Connection Factory 的名稱，供 WebSphere® Application Server 內管理作業使用。 在整個產品管理網域中的各資源配接器 Connection Factory 內，這個名稱必須是唯一的。

資料類型 字串
說明

這個 Connection Factory 的選用說明，供 IBM® WebSphere Application Server 內管理作業使用。

資料類型 字串
JNDI 名稱

用來在部署描述子比對這個資源配接器 Connection Factory 定義的 JNDI 名稱。這個項目應該是 resource-ref 名稱。

資料類型 字串
使用者名稱

在取得連線時，如果呼叫的應用程式未明確提供使用者 ID 和密碼，則這是搭配密碼內容來用於鑑別的使用者名稱。如果使用這個欄位，則會忽略內容欄位使用者名稱。

如果您指定了使用者名稱內容的值，您也必須指定密碼內容的值。

在取得連線時，如果呼叫的應用程式未明確提供使用者 ID 和密碼，則會使用 Connection Factory 的使用者名稱和密碼內容。

資料類型 字串
密碼

指定加密過的密碼。如果您完成這個欄位，則會忽略「內容」框中的密碼欄位。

如果您指定了使用者名稱內容的值，您也必須指定密碼內容的值。

資料類型 字串
重新輸入密碼

確認密碼。

類型

針對「資源配接器保存檔」中的 Factory，所定義之所有 connectionFactoryInterfaces 的下拉清單。

每一種類型在「內容」框中都指定有一組內容。這組內容係透過從每一個連線定義物件擷取內容所建構而成。 對於顯示供更新的所有現有的 Connection Factory 而言，這個內容清單會被物件所指定的內容覆蓋。當類型欄位變更時，這些內容也會變更，以反映該類型正確的內容。
