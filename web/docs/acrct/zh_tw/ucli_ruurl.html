應用程式用戶端的 URL 設定

請利用這個頁面來實作特定 URL 通訊協定的功能，如超文字轉送通訊協定 (HTTP)。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > URL 提供者 > URL 提供者實例。 用滑鼠右鍵按一下 URL，再按一下新建。 這時一般標籤會出現下列欄位。

這個由類別組成的提供者繼承 java.net.URLStreamHandler 和 java.net.URLConnection 類別。

名稱

URL 的管理名稱。

說明

這是您管理記錄的 URL 選用說明。

JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

URL

指向網際網路或企業內部網路資源的統一資源定位器 (URL) 名稱。比方說，http://www.ibm.com。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。



標示（線上）的鏈結表示需要存取網際網路。
