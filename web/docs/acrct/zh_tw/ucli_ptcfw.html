應用程式用戶端的第 5 版預設提供者 Topic Connection Factory 設定

請利用這個畫面來檢視或變更所選取 Topic Connection Factory 的配置內容，以與內部產品 Java™ 訊息服務 (JMS) 提供者搭配使用。 這些配置內容用來控制如何建立 JMS 提供者與 JMS 提供者所使用的傳訊系統之間的連線。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 傳訊提供者 > 第 5 版預設提供者。 用滑鼠右鍵按一下 Topic Connection Factory，再按一下新建。 這時一般標籤會出現下列欄位。

第 5 版預設提供者 Topic Connection Factory 有下列內容。

名稱

用來識別這個 Queue Connection Factory 的名稱，供 WebSphere® Application Server 內管理作業使用。 在整個 WebSphere Application Server 管理網域的各個 JMS Connection Factory 內，這個名稱必須是唯一的。

資料類型 字串
說明

這個 Topic Connection Factory 的說明，供 WebSphere Application Server 內管理作業使用。

資料類型 字串
JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

使用者 ID

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼時，就要搭配密碼內容來進行鑑別的使用者 ID。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

如果發出呼叫的應用程式沒有明確提供使用者 ID 和密碼，比方說，發出呼叫的應用程式使用 createTopicConnection() 方法，就會使用 Connection Factory 使用者 ID 和密碼內容。 JMS 用戶端會將使用者 ID 和密碼傳送給 JMS 伺服器。

資料類型 字串
密碼

當呼叫端應用程式沒有明確提供使用者 ID 和密碼時，要搭配使用者 ID 內容供鑑別使用的密碼。

如果您指定了使用者 ID 內容值，您也必須指定密碼內容值。

資料類型 字串
重新輸入密碼

確認密碼。

節點

這個 Connection Factory 的 JMS 伺服器執行所在之管理節點的 WebSphere Application Server 節點名稱。 這個 Factory 所建立的連線會連接至該 JMS 伺服器。

資料類型 列舉
範圍 WebSphere Application Server 管理網域中的下拉節點清單。
應用程式伺服器

請輸入應用程式伺服器的名稱。這個名稱不是機器的主機名稱，而是所配置應用程式伺服器的名稱。

埠

連線將使用哪一個埠來連接到「JMS 伺服器」。 QUEUED 埠是全功能的 JMS 發佈/訂閱支援，DIRECT 埠只適用於非持續性、非交易式、不可延續的訂閱。

註: 訊息驅動 Bean 無法對發佈或訂閱支援使用直接的接聽器埠。 因此，任何將「埠」設成直接來配置的 Topic Connection Factory 都無法與訊息驅動 Bean 一起使用。
資料類型 列舉
預設值 QUEUED
範圍
QUEUED
全功能 JMS 標準發佈或訂閱支援所用的接聽器埠。
DIRECT
發佈或訂閱支援的直接 TCP/IP 連線（只適用於非交易式、非持續性及不可延續的訂閱）所用的接聽器埠。

這些埠的 TCP/IP 埠號定義在產品內部 JMS 伺服器中。
用戶端 ID

通往 MQSeries® 佇列管理程式的連線所用的 JMS 用戶端 ID。

資料類型 字串
自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
