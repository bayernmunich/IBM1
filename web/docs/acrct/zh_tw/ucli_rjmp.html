應用程式用戶端的郵件提供者設定

請利用這個頁面來實作 JavaMail API 和建立郵件階段作業。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔。用滑鼠右鍵按一下郵件提供者 >，再按一下新建。 這時一般標籤會出現下列欄位：

名稱

JavaMail 資源提供者的名稱。

說明

資源提供者的選用說明。

類別路徑

指定共同形成資源提供者類別位置的路徑或 JAR 檔名稱的清單。

通訊協定

指定通訊協定的名稱。

類別名稱

指定實作通訊協定的類別名稱。如果您要使用預設實作，請將這個欄位留白。

類型

這個功能表包含下面這兩個值：TRANSPORT 或 STORE。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
