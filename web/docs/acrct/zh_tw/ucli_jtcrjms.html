應用程式用戶端的郵件階段作業設定

請利用這個頁面來配置郵件階段作業內容。

若要檢視這個「應用程式用戶端資源配置工具」(ACRCT) 頁面，請按一下檔案 > 開啟。瀏覽 EAR 檔之後，請按一下開啟。展開選取的 JAR 檔 > 郵件提供者 > 郵件提供者實例。 用滑鼠右鍵按一下郵件階段作業，再按一下新建。 這時一般標籤會出現下列欄位：

名稱

代表 JavaMail 階段作業物件的管理名稱。

說明

提供管理記錄的選用說明。

JNDI 名稱

應用程式用戶端執行時期利用這個欄位來擷取配置資訊。

郵件傳輸主機

指定傳送郵件時要連接的伺服器。

郵件傳輸通訊協定

指定傳送郵件時要用的傳輸通訊協定。

郵件傳輸使用者

指定郵件傳輸主機需要鑑別時所用的使用者 ID。

郵件傳輸密碼

指定郵件傳輸主機需要鑑別時所用的密碼。

啟用嚴格網際網路位址剖析

指定是否必須嚴格遵守 Internet Architecture Board 發出的規格文件 RFC 822 來剖析接收者位址。

大部分的郵件應用程式很少使用這項設定。 用來剖析位址的 RFC 822 語法會實際採用有效電子郵件位址的嚴格定義。 如果您選取這個設定，JavaMail 將遵循 RFC 822 語法，且會拒絕未剖析成有效電子郵件位址（依照規格所定義）的接收者位址。 如果您沒有選取這個設定，JavaMail 就不會遵循 RFC 822 語法，且會接受不符合規格的接收者位址。 依預設，會取消選取這個設定。 您可以檢視全球資訊網協會 (W3C) 下列 URL 所提供的 RFC 822 規格：http://www.w3.org/Protocols/rfc822/。

重新輸入密碼

確認密碼。

郵件來源

指定郵件發送端。

郵件儲存庫主機

指定郵件帳戶主機（或「網域」）名稱。

郵件儲存庫使用者

指定郵件帳戶的使用者 ID。

郵件儲存庫密碼

指定郵件帳戶的密碼。

重新輸入密碼

確認密碼。

郵件儲存庫通訊協定

指定接收郵件時要用的通訊協定。

郵件除錯

當設成 true 時，JavaMail 會與郵件伺服器互動，且這些郵件階段作業內容都會列印在 stdout 檔中。

自訂內容

請指定名稱/值配對，用來設定在這項資源的執行時期建立的物件的其他內容。

您必須輸入名稱和值，名稱必須是物件的公開內容，值必須能夠從字串轉換成內容的 set 方法所需要的類型。可接受的內容和值會隨著所建立的物件而不同。請參閱物件文件，以取得有效內容和值的清單。
