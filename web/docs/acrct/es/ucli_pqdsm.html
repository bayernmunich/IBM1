Valores de destino de cola del proveedor de WebSphere MQ para clientes de aplicaciones

Utilice este panel para ver o cambiar las propiedades de configuración del destino de cola seleccionado para su uso con el proveedor JMS (Java™ Message Service) del producto WebSphere® MQ.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > Proveedor WebSphere MQ. Pulse el botón derecho del ratón en Destinos de cola y pulse Nuevo. Los campos siguientes se muestran en la pestaña General.

Nota:
  • Los valores de propiedad que especifique deben coincidir con los valores que ha especificado al configurar los recursos JMS para WebSphere MQ. Para obtener más información sobre la configuración de recursos JMS para WebSphere MQ, consulte Utilización de Java en el centro de información de WebSphere MQ.
  • En WebSphere MQ, los nombres pueden tener un máximo de 48 caracteres.

Una cola que se vaya a utilizar con el proveedor de JMS de WebSphere MQ tiene las propiedades siguientes.

Nombre

Nombre por el que se conoce la cola para fines administrativos dentro de WebSphere Application Server.

Tipo de datos Serie
Descripción

Descripción de la cola, para fines administrativos en WebSphere Application Server.

Tipo de datos Serie
Nombre JNDI

El entorno de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

Persistencia

Especifica si todos los mensajes enviados al destino son persistentes, no persistentes o la persistencia está definida por la aplicación.

Tipo de datos Enum
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por la aplicación
La persistencia de los mensajes en el destino está definida por la aplicación que los ha colocado en la cola.
Definida por la cola
[Sólo para destino de WebSphere MQ] La persistencia de los mensajes en el destino está definida por las propiedades de definición de cola de WebSphere MQ.
Persistente
Los mensajes en el destino son persistentes.
No persistente
Los mensajes en el destino no son persistentes.
Prioridad

Especifica si la prioridad del mensaje para este destino está definida por la aplicación o por la propiedad Prioridad especificada.

Tipo de datos Enum
Unidades No se aplica
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por la aplicación
La prioridad de los mensajes en este destino está definida por la aplicación que los ha colocado en el destino.
Definida por la cola
[Sólo para destino de WebSphere MQ] La persistencia de los mensajes en el destino está definida por las propiedades de definición de cola de WebSphere MQ.
Especificado
La prioridad de los mensajes de este destino está definida por la propiedad Prioridad especificada. Si selecciona esta opción, debe definir una prioridad en la propiedad Prioridad especificada.
Prioridad especificada

Si la propiedad Prioridad está establecida en Especificada, especifique la prioridad de mensajes de esta cola, en el rango de 0 (menor) a 9 (mayor).

Tipo de datos Entero
Unidades Nivel de prioridad de los mensajes
Rango 0 (prioridad menor) a 9 (prioridad mayor)
Caducidad

Especifica si el valor de tiempo de espera de caducidad para esta cola está definida por la aplicación o por la propiedad Caducidad especificada o si, de lo contrario, los mensajes de la cola nunca caducan (tienen un tiempo espera de caducidad ilimitado).

Tipo de datos Enum
Unidades No se aplica
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por la aplicación
El tiempo de espera de caducidad de los mensajes de esta cola está definido por la aplicación que los ha colocado en la cola.
Especificado
El tiempo de espera de caducidad de los mensajes de esta cola está definido por la propiedad Caducidad especificada. Si selecciona esta opción, debe definir un tiempo de espera en la propiedad Caducidad especificada.
Ilimitado
Los mensajes de esta cola no tienen tiempo de espera de caducidad y no caducan nunca.
Caducidad especificada

Si la propiedad Tiempo de espera de caducidad está establecida en Especificada, escriba aquí el número de milisegundos (mayor que 0) transcurridos los cuales los mensajes de esta cola caducan.

Tipo de datos Entero
Unidades Milisegundos
Rango Mayor que o igual a 0
  • El valor 0 indica que los mensajes nunca superan el tiempo de espera
  • Los demás valores son un número entero de milisegundos

Cliente de destino

Especifica si la aplicación receptora es compatible con JMS o es una aplicación de WebSphere MQ tradicional.

Tipo de datos Enum
Valor predeterminado WebSphere MQ
Rango
WebSphere MQ

Cuerpo del mensaje

Especifica si una aplicación procesa la cabecera RFH versión 2 de un mensaje WebSphere MQ como parte del cuerpo del mensaje JMS.

Tipo de datos Lista desplegable
Valor predeterminado SIN ESPECIFICAR
Rango
SIN ESPECIFICAR
Cuando se envían mensajes, el proveedor de mensajería de WebSphere MQ genera o no genera e incluye una cabecera RFH versión 2, dependiendo del valor de la propiedad Adjuntar cabeceras RFH versión 2 a los mensajes enviados a este destino. Al recibir mensajes, el proveedor de mensajería de WebSphere MQ actúa como si el valor estuviera establecido en JMS.
JMS
Cuando se envían mensajes, el proveedor de mensajería de WebSphere MQ genera automáticamente una cabecera RFH versión 2 y la incluye en el mensaje de WebSphere MQ. Al recibir mensajes, el proveedor de mensajería de WebSphere MQ establece las propiedades de mensaje JMS de acuerdo con los valores de la cabecera RFH versión 2 (si existen estos valores); no presenta la cabecera RFH versión 2 como parte del cuerpo del mensaje JMS.
MQ
Cuando se envían mensajes, el proveedor de mensajería de WebSphere MQ no genera una cabecera RFH versión 2. Al recibir mensajes, el proveedor de mensajería de WebSphere MQ presenta la cabecera RFH versión 2 como parte del cuerpo del mensaje JMS.
Estilo de destino ReplyTo

Especifica el formato del campo JMSReplyTo.

Tipo de datos Lista desplegable
Valor predeterminado DEFAULT
Rango
DEFAULT
El valor predeterminado es equivalente a la información de la cabecera RFH versión 2.
MQMD
Utilice el valor proporcionado en MQMD. Esto llena el campo de respuesta al gestor de colas con el valor de MQMD, equivalente al comportamiento predeterminado de WebSphere MQ Versión 6.0.2.4 y 6.0.2.5.
RFH2
Utilice el valor proporcionado en la cabecera RFH versión 2. Si la aplicación remitente ha establecido un valor JMSReplyTo, se utiliza dicho valor.

Contexto de mensaje MQMD

Define las opciones de contexto de mensaje especificadas al enviar mensajes a un destino.

Tipo de datos Lista desplegable
Valor predeterminado DEFAULT
Rango
DEFAULT
La llamada de API MQOPEN y la estructura MQPMO no especifican ninguna opción de contexto de mensaje explícita.
SET_IDENTITY_CONTEXT
La llamada de API MQOPEN especifica la opción de contexto de mensaje MQOO_SET_IDENTITY_CONTEXT y la estructura MQPMO especifica MQPMO_SET_IDENTITY_CONTEXT.
SET_ALL_CONTEXT
La llamada de API MQOPEN especifica la opción de contexto de mensaje MQOO_SET_ALL_CONTEXT y la estructura MQPMO especifica MQPMO_SET_ALL_CONTEXT.
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Debe entrar un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
