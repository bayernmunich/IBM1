Valores de la fábrica de conexiones JMS genéricas para clientes de aplicaciones

Utilice este panel para ver o modificar las propiedades de configuración de la fábrica de conexiones JMS (Java Message Service) seleccionada para su uso con el proveedor de JMS asociado. Estas propiedades de configuración controlan cómo se crean las conexiones entre el proveedor de JMS y el sistema de mensajería que utiliza.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > nueva_instancia_proveedor_JMS. Pulse con el botón derecho del ratón en Fábrica de conexiones y pulse Nueva. Los campos siguientes se muestran en el separador General.

Una fábrica de conexiones JMS (Java Message Service) crea conexiones con destinos de JMS. El proveedor de JMS asociado crea la fábrica de conexiones JMS. Una fábrica de conexiones JMS para un proveedor de JMS genérico (un proveedor de JMS que no sea el proveedor de mensajería interno predeterminado ni WebSphere MQ como proveedor de JMS) presenta las propiedades siguientes:

Nombre

Nombre por el que se conoce esta fábrica de conexiones JMS para fines administrativos dentro de IBM WebSphere Application Server. El nombre debe ser exclusivo dentro del proveedor de JMS asociado.

Tipo de datos String
Descripción

Descripción de esta fábrica de conexiones con fines administrativos dentro de IBM WebSphere Application Server.

Tipo de datos String
Valor predeterminado Null
Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

ID de usuario

indica el ID de usuario utilizado con la propiedad Contraseña en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Las propiedades ID de usuario y Contraseña de la fábrica de conexiones se utilizan si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña; por ejemplo, si la aplicación que llama utiliza el método createQueueConnection(). El cliente de JMS dirige el ID de usuario y la contraseña al servidor JMS.

Tipo de datos String
Contraseña

Contraseña utilizada con la propiedad ID de usuario en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Tipo de datos String
Valor predeterminado Null
Vuelva a entrar la contraseña

Confirma la contraseña entrada en el campo Contraseña.

Nombre externo de JNDI

Nombre JNDI utilizado para enlazar la cola con el espacio de nombres del servidor de aplicaciones.

Como convenio, utilice el nombre JNDI totalmente cualificado, por ejemplo, jms/Nombre, donde Nombre es el nombre lógico del recurso.

Este nombre se utiliza para enlazar la información de enlace de la plataforma. El enlace asocia los recursos definidos en el descriptor de despliegue del módulo con los recursos reales (físicos) enlazados con la API JNDI por la plataforma.

Tipo de datos String
Tipo de conexión

Especifica si este destino de JMS es una cola (para punto a punto) o un tema (para publicación o suscripción).

Seleccione una de las opciones siguientes:
Cola
Destino de cola JMS para mensajería punto a punto.
Tema
Destino de tema JMS para mensajería de publicación/suscripción.
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
