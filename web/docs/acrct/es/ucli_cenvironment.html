Entradas de entorno de recursos y proveedores de entorno de recursos

Las referencias de entorno de recursos establecen una correlación entre los nombres lógicos que utiliza la aplicación cliente con los nombres físicos de los objetos.

No todos los objetos enlazados al espacio de nombres JNDI del servidor se han diseñado para que los utilicen los clientes de aplicaciones. Por ejemplo, el tiempo de ejecución de cliente de WebSphere Application Server no da soporte al uso de objetos Java 2 Connector (J2C) en el cliente. Es necesario que el objeto pueda ser remoto y las implementaciones en el cliente deben estar disponibles en la classpath del programa de tiempo de ejecución del cliente de aplicaciones.

Las referencias de entorno de recursos son distintas de las referencias de recursos. Las referencias de entorno de recursos permiten al cliente de aplicaciones utilizar nombres lógicos para buscar enlaces de recursos en el espacio de nombres JNDI del servidor. Las referencias de recursos permiten a la aplicación utilizar nombres lógicos para buscar recursos J2EE locales. La especificación J2EE no especifica una implementación particular de un recurso.



Los enlaces marcados (en línea) requieren acceso a Internet.
