Valores de destino de cola del proveedor de mensajería predeterminado V5 para los clientes de aplicaciones

Utilice este panel para ver o modificar las propiedades de configuración del destino de cola seleccionado que se va a utilizar con el proveedor de JMS (Java Message Service) del producto.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > Proveedor predeterminado versión 5. Pulse con el botón derecho del ratón en Destinos de cola y pulse Nuevo. Los campos siguientes se muestran en el separador General.

Un destino de cola se utiliza para configurar las propiedades de una cola JMS. Un destino de cola del proveedor de mensajería predeterminado V5 presenta las propiedades siguientes.

Nombre

Nombre por el que se conoce la cola con fines administrativos dentro de WebSphere Application Server.

Tipo de datos String
Descripción

Descriptor de la cola, a efectos administrativos.

Tipo de datos String
Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

Persistencia

Especifica si todos los mensajes enviados al destino son persistentes o no persistentes, o si la persistencia está definida por la aplicación.

Tipo de datos Enum
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por aplicación
La persistencia de los mensajes en el destino está definida por la aplicación que los transfiere a la cola.
Cola definida
[Sólo para destino de WebSphere MQ] La persistencia de los mensajes en el destino está definida por las propiedades de definición de cola de WebSphere MQ.
Persistente
Los mensajes en el destino son persistentes.
No persistentes
Los mensajes en el destino no son persistentes.
Prioridad

Especifica si la prioridad del mensaje para este destino está definida por la aplicación o por la propiedad Prioridad especificada.

Tipo de datos Enum
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por aplicación
La prioridad de los mensajes en este destino está definida por la aplicación que los transfiere al destino.
Cola definida
[Sólo para destino de WebSphere MQ] La persistencia de los mensajes en el destino está definida por las propiedades de definición de cola de WebSphere MQ.
Especificada
La prioridad de los mensajes de este destino está definida por la propiedad Prioridad especificada. Si selecciona esta opción, debe definir una prioridad en la propiedad Prioridad especificada.
Prioridad especificada

Si la propiedad Prioridad está establecida en Especificada, escriba la prioridad de mensajes de esta cola aquí, en el rango de 0 (menor) a 9 (mayor).

Si la propiedad Prioridad está establecida en Especificada, los mensajes enviados a esta cola tienen el valor de prioridad especificado por esta propiedad.

Tipo de datos Integer
Unidades Nivel de prioridad de los mensajes
Rango 0 (prioridad menor) a 9 (prioridad mayor)
Caducidad

Especifica si el tiempo de espera de caducidad para esta cola está definida por la aplicación o por la propiedad Caducidad especificada o si los mensajes de la cola caducan (tienen un tiempo espera de caducidad ilimitado).

Tipo de datos Enum
Valor predeterminado DEFINIDA_POR_APLICACIÓN
Rango
Definida por aplicación
El tiempo de espera de caducidad de los mensajes de esta cola está definido por la aplicación que los transfiere a la cola.
Especificada
El tiempo de espera de caducidad de los mensajes de esta cola está definida por la propiedad Caducidad especificada.Si selecciona esta opción, debe definir un tiempo de espera en la propiedad Caducidad especificada.
Ilimitado
Los mensajes de esta cola no tienen tiempo de espera de caducidad y no caducan nunca.
Caducidad especificada

Si la propiedad Caducidad especificada está establecida en Especificado, especifique el número de milisegundos (mayor que 0) después de los cuales caducan los mensajes de esta cola.

Tipo de datos Integer
Unidades Milisegundos
Rango Mayor o igual a 0
  • El valor 0 indica que los mensajes nunca superan el tiempo de espera.
  • Los demás valores son números enteros en milisegundos.
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
