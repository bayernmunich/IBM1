Configuración de recursos del cliente de mensajería Java

Para configurar recursos del cliente de mensajería Java, debe crear nuevas configuraciones de proveedor JMS para el cliente de aplicaciones. El cliente de aplicaciones puede utilizar un servicio de mensajería a través de las API de JMS (Java Message Service). Un proveedor de JMS proporciona dos tipos de fábricas de J2EE. Una es una fábrica de conexiones JMS y la otra es una fábrica de destinos de JMS.

Antes de empezar

En una tarea administrativa independiente, instale el cliente JMS (Java Message Service) en la máquina cliente donde reside el cliente de aplicaciones. El proveedor del producto de mensajería debe proporcionar una implementación del cliente JMS. Para obtener más información, consulte la documentación del producto de mensajería.

Nota: Cuando haya completado esta tarea, puede crear un nuevo proveedor de mensajería o bien utilizar uno existente.

Procedimiento

  1. Inicie ACRCT (Application Client Resource Configuration Tool).
  2. Abra el archivo EAR para el que desea configurar el nuevo proveedor de JMS. El contenido del archivo EAR se muestra en una vista de árbol.
  3. En esta vista de árbol, seleccione el archivo JAR donde desea configurar el nuevo proveedor de JMS.
  4. Expanda el archivo JAR para ver su contenido.
  5. Opcionalmente, pulse con el botón derecho Proveedores de mensajería y seleccione Nuevo, si desea crear y utilizar un proveedor de mensajería nuevo.
  6. Configure las propiedades del proveedor de JMS en el diálogo de propiedades resultante.
  7. Pulse Aceptar.
  8. Pulse Archivo > Guardar.
