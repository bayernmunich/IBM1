Valores de la fábrica de conexiones de tema de mensajería predeterminada V5 para clientes de aplicaciones.

Utilice este panel para ver o modificar las propiedades de configuración de la fábrica de conexiones de tema seleccionada para su uso con el proveedor JMS (Java Message Service) del producto interno. Estas propiedades de configuración controlan cómo se crean las conexiones entre el proveedor de JMS y el sistema de mensajería que utiliza.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > Proveedor predeterminado versión 5. Pulse con el botón derecho del ratón en Fábrica de conexiones de tema y pulse Nueva. Los campos siguientes aparecen en el separador General.

Una fábrica de conexiones del proveedor de mensajería predeterminada V5 presenta las propiedades siguientes.

Nombre

Nombre por el que se conoce esta fábrica de conexiones con fines administrativos dentro de WebSphere Application Server. El nombre debe ser exclusivo dentro de las fábricas de conexiones JMS a lo largo del dominio administrativo de WebSphere Application Server.

Tipo de datos String
Descripción

Descripción de esta fábrica de conexiones de tema con fines administrativos dentro de WebSphere Application Server.

Tipo de datos String
Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

ID de usuario

ID de usuario utilizado, junto con la propiedad Contraseña, en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Las propiedades ID de Usuario y Contraseña de la fábrica de conexiones se utilizan si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña, por ejemplo, si la aplicación que llama utiliza el método createTopicConnection(). El cliente de JMS dirige el ID de usuario y la contraseña al servidor JMS.

Tipo de datos String
Contraseña

Contraseña utilizada, junto con la propiedad ID de usuario, en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Tipo de datos String
Vuelva a entrar la contraseña

Confirma la contraseña.

Nodo

Nombre de nodo de WebSphere Application Server del nodo administrativo donde se ejecuta el servidor JMS para esta fábrica de conexiones. Las conexiones creadas por esta fábrica se conectan al servidor JMS.

Tipo de datos Enum
Rango Lista desplegable de nodos en el dominio administrativo de WebSphere Application Server.
Servidor de aplicaciones

Entre el nombre del servidor de aplicaciones. Este nombre no es el nombre de host de la máquina, sino el nombre del servidor de aplicaciones configurado.

Puerto

Especifica cuál de los dos puertos van a utilizar las conexiones para conectarse al servidor JMS. El puerto EN COLA se utiliza para dar soporte de publicación/suscripción de JMS de funciones completas, mientras que los puertos DIRECTOS sólo se utilizan para suscripciones no persistentes, no transaccionales y no duraderas.

Nota: Los beans controlados por mensajes no pueden utilizar el puerto de escucha directo para soporte de publicación o suscripción. Por lo tanto, no se puede utilizar con los beans controlados por mensajes ninguna fábrica de conexiones de tema cuya configuración tenga establecido Puerto en Directo.
Tipo de datos Enum
Valor predeterminado EN COLA
Rango
EN COLA
Puerto de escucha utilizado para soporte de publicación o suscripción compatible con JMS de funciones completas.
DIRECTO
Puerto de escucha utilizado para conexiones TCP/IP directas (sólo suscripciones no transaccionales, no persistentes y no duraderas) en soporte de publicación o suscripción.

Los números de puerto TCP/IP para estos puertos se definen en el servidor JMS interno del producto.
ID de cliente

Identificador de cliente de JMS que se utiliza para las conexiones con el gestor de colas de MQSeries.

Tipo de datos String
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
