Valores de la fábrica de conexiones de cola del proveedor de WebSphere MQ para clientes de aplicaciones

Utilice este panel para ver o cambiar las propiedades de configuración de la fábrica de conexiones de cola seleccionada para su uso con el proveedor de JMS (Java™ Message Service) del producto MQSeries®. Estas propiedades de configuración controlan cómo se crean las conexiones entre el proveedor de JMS y WebSphere® MQ.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > Proveedor WebSphere MQ. Pulse con el botón derecho del ratón en Fábricas de conexiones de cola y pulse Nueva. Los campos siguientes se muestran en la pestaña General.

Nota:
  • Los valores de propiedad que especifique deben coincidir con los valores que ha especificado al configurar los recursos JMS para WebSphere MQ. Para obtener más información sobre la configuración de recursos JMS para WebSphere MQ, consulte el apartado Utilización de Java del centro de información de WebSphere MQ.
  • En WebSphere MQ, los nombres pueden tener un máximo de 48 caracteres, a excepción de los canales, que tienen un máximo de 20 caracteres.

Una fábrica de conexiones de cola del proveedor de JMS tiene las propiedades siguientes.

Nombre

Nombre por el que se conoce esta fábrica de conexiones de cola para fines administrativos dentro de WebSphere Application Server. El nombre debe ser exclusivo dentro de las fábricas de conexiones de JMS en el dominio administrativo de WebSphere.

Tipo de datos Serie
Descripción

Descripción de esta fábrica de conexiones para fines administrativos en WebSphere Application Server.

Tipo de datos Serie
Valor predeterminado Null (Nulo)
Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

ID de usuario

ID de usuario utilizado, con la propiedad de contraseña, para la autenticación si la aplicación que realiza la llamada no proporciona explícitamente un ID de usuario y una contraseña.

Si especifica un valor para la propiedad de ID de usuario, también debe especificar un valor para la propiedad de contraseña.

Las propiedades de ID de usuario y contraseña de la fábrica de conexiones se utilizan si la aplicación que realiza la llamada no proporciona explícitamente un ID de usuario y una contraseña; por ejemplo si la aplicación que realiza la llamada utiliza el método createQueueConnection(). El cliente JMS pasa el ID de usuario y la contraseña al servidor JMS.

Tipo de datos Serie
Contraseña

Contraseña utilizada, con la propiedad de ID de usuario, para la autenticación si la aplicación que realiza la llamada no proporciona explícitamente un ID de usuario y una contraseña.

Si especifica un valor para la propiedad de ID de usuario, también debe especificar un valor para la propiedad de contraseña.

Tipo de datos Serie
Valor predeterminado Null (Nulo)
Vuelva a entrar la contraseña

Confirma la contraseña.

Gestor de colas

Nombre del gestor de colas de WebSphere MQ de esta fábrica de conexiones.

Las conexiones creadas por dicha fábrica se conectan a este gestor de colas.

Tipo de datos Serie
Entrar información de nombre de host y puerto

Este botón de selección estará seleccionado de forma predeterminada y, si se selecciona, habilita las propiedades de host y puerto e inhabilita la propiedad de lista de nombres de conexión.

Tipo de datos Botón de selección
Valor predeterminado Seleccionado
Host

Nombre del host en el que se ejecuta WebSphere MQ Queue Manager sólo para la conexión del cliente.

Tipo de datos Serie
Valor predeterminado Null (Nulo)
Rango Un nombre de host TCP/IP válido
Puerto

Número de puerto TCP/IP utilizado para la conexión con WebSphere MQ Queue Manager, sólo para la conexión del cliente.

Este puerto debe configurarse en WebSphere MQ Queue Manager.

Tipo de datos Entero
Valor predeterminado Null (Nulo)
Rango Un número de puerto TCP/IP válido, configurado en WebSphere MQ Queue Manager.
Entrar información de lista de nombres de conexión

Si se selecciona, este botón de selección habilita la propiedad de lista de nombres de conexión e inhabilita las propiedades de nombre de host y puerto. Seleccione este botón de selección si desea conectarse a un gestor de colas de varias instancias.

Tipo de datos Botón de selección
Valor predeterminado Deseleccionado
Lista de nombres de conexión

Lista separada por comas de información de host y puerto que se puede utilizar para conectarse a un gestor de colas de varias instancias.

El formato de la lista es: 

host[(puerto)],[host[(puerto)]]
donde puerto es opcional y toma de forma predeterminada 1414 si no se ha establecido. Por ejemplo:
hostname1,hostname2(1415)

Para obtener más información sobre gestores de colas de varias instancias, consulte el centro de información de WebSphere MQ.

Esta propiedad sólo se debe utilizar para conectarse a un gestor de colas de varias instancias. No se debe utilizar para conectarse a una lista de gestores de colas diferentes porque esto puede generar problemas de integridad de transacción.

Canal

Nombre del canal utilizado para la conexión con WebSphere MQ Queue Manager, sólo para la conexión del cliente.

Tipo de datos Serie
Valor predeterminado Null (Nulo)
Rango De 1 a 20 caracteres ASCII
Tipo de transporte

Especifica si se utilizan la conexión del cliente de WebSphere MQ o enlaces JNDI para la conexión con WebSphere MQ Queue Manager. El proveedor de JMS externo controla los protocolos de comunicación entre clientes y servidores JMS. Ajuste el tipo de transporte cuando utilice un servicio de mensajes que no sea ASF, no persistente, no perdurable y no transaccional o cuando desea satisfacer asuntos de seguridad y el cliente es local al nodo del gestor de colas.

Tipo de datos Enum
Unidades No se aplica
Valor predeterminado ENLACES
Rango
BINDINGS
Los enlaces JNDI se utilizan para conectar con el gestor de colas. BINDINGS es un protocolo de memoria compartida y sólo se puede utilizar cuando el gestor de colas se encuentra en el mismo nodo que el cliente de JMS y provoca algunos riesgos de seguridad que se deben tratar mediante el uso de roles de EJB.
CLIENT
La conexión del cliente de WebSphere MQ se utiliza para conectar con el gestor de colas. CLIENT es un protocolo TCP típico.
DIRECT
Para WebSphere MQ Event Broker mediante la modalidad DIRECT. DIRECT es un protocolo Lightweight Sockets que se utiliza en servicios de mensajes de publicación/suscripción no transaccionales, no perdurables y no persistentes. DIRECT sólo funciona en clientes y beans controlados por mensajes mediante el protocolo que no es ASF.
QUEUED
QUEUED es un protocolo TCP sin estado.
Recomendado
Tipo de transporte de fábrica de conexiones de cola
BINDINGS es un 30% o más rápido, pero carece de seguridad. Cuando le atañe la seguridad, es preferible BINDINGS que CLIENT.
Tipo de transporte de fábrica de conexiones de tema
DIRECT es el tipo más rápido y debe utilizarse cuando sea posible. Utilice BINDINGS cuando desee satisfacer tareas de seguridad adicionales y el gestor de colas sea local para el cliente JMS. QUEUED es lo que se utiliza para todos los demás casos. WebSphere MQ 5.3 antes de CSD2 con el valor DIRECT puede perder mensajes cuando se utiliza con beans controlados por mensajes y bajo carga. Esta pérdida también ocurre con las aplicaciones en el extremo del cliente, a menos que maxClientQueueSize del intermediario esté establecido en 0. Puede establecer este valor en 0 con el mandato: 
#wempschangeproperties WAS_nodeName_server1 -e default -o DynamicSubscriptionEngine -n maxClientQueueSize -v 0 -x executionGroupUUID
donde executionGroupUUID se puede encontrar iniciando el intermediario y buscando en el archivo de anotaciones cronológicas o aplicaciones de sucesos del suceso 2201. Este valor suele ser ffffffff-0000-0000-000000000000.
Nota: El JMS de WebSphere MQ 5.3 no se puede utilizar en WAS 6.1 porque WAS 6.1 tiene un tiempo de ejecución Java 5. Por lo tanto, no se pueden establecer conexiones entre memoria con los gestores de colas WebSphere MQ 5.3. Esto puede producir una disminución del rendimiento si anteriormente estaba utilizando WebSphere MQ 5.3 y BINDINGS para las conexiones y pasa a las conexiones de red CLIENT al migrar a WebSphere Application Server Versión 6.1. Si está utilizando WebSphere MQ 5.3 para z/OS®, es posible que tenga que comprar un paquete de características adicional.

Cuando se ejecuta en z/OS de 64 bits, el tipo de Transporte se debe establecer en CLIENT porque WebSphere MQ z/OS de 64 bits no está disponible actualmente y no se puede utilizar la modalidad BINDINGS para conectarse a WebSphere MQ z/OS de 31 bits. Es posible que también tenga que adquirir un paquete de características de WebSphere MQ adicional para obtener este soporte.
ID de cliente

Identificador de cliente de JMS utilizado para conexiones con MQSeries Queue Manager.

Tipo de datos Serie
CCSID

Identificador del juego de caracteres codificado para utilizarlo con WebSphere MQ Queue Manager.

Este CCSID (identificador del juego de caracteres codificado) debe ser uno de los CCSID a los que WebSphere MQ da soporte.

Tipo de datos Serie

Para obtener más información sobre los CCSID soportados y sobre cómo convertir entre los datos de mensajes de un juego de caracteres codificado a otro, consulte las secciones sobre Administración del sistema y Consulta de programación de aplicaciones del centro de información de WebSphere MQ.

Seleccione este recuadro de selección para especificar que desea conservar en la cola los mensajes no deseados. De lo contrario, dichos mensajes se manejan de acuerdo con sus opciones de configuración.

Tipo de datos Enum
Unidades No se aplica
Valor predeterminado Deseleccionado
Rango
Seleccionado
Los mensajes no deseados permanecen en la cola.
Deseleccionado
Los mensajes no deseados se manejan de acuerdo con sus opciones de configuración.

Tipo de datos Serie
Rango De 1 a 48 caracteres ASCII

Prefijo utilizado para la denominación de colas dinámicas.

Tipo de datos Serie
Fallo si se desactiva temporalmente

Especifica si las aplicaciones realizan la devolución de una llamada a método si el gestor de colas ha entrado en una anomalía controlada.

Tipo de datos Recuadro de selección
Valor predeterminado Seleccionado
Dirección de servidor local

Especifica la dirección local del servidor.

Intervalo de sondeo

Especifica el intervalo, en milisegundos, entre exploraciones de todos los receptores durante la entrega de mensajes asíncronos.

Tipo de datos Entero

Este intervalo controla la exploración de mensajes que se han añadido a un tema fuera de servicio con respecto al cursor para examinar de WebSphere MQ.

Tipo de datos Entero

Valor predeterminado 5000
Suite de cifrado SSL

Especifica el suite de cifrado que se utilizará para la conexión SSL con WebSphere MQ.

Establezca esta propiedad en un suite de cifrado válido proporcionado por el proveedor de JSSE. El valor debe coincidir con la CipherSpec especificada en el canal SVRCONN como la propiedad Canal..

Debe establecer esta propiedad, si establece la propiedad Nombre de igual SSL .

Almacén de certificados SSL

Especifica una lista de cero o más servidores CRL (Lista de revocación de certificados) utilizada para comprobar la revocación de certificados SSL. Si especifica un valor para la propiedad ID de usuario, debe utilizar la JVM de WebSphere MQ en Java 2 versión 1.4.

El valor es una lista de entradas delimitada por espacios con el siguiente aspecto:
ldap://nombresistprpal:[puerto]

A continuación de este valor va una sola barra inclinada (/). Si se omite puerto, se asume que el puerto LDAP predeterminado es de 389. Durante la conexión, el certificado SSL presentado por el servidor se comprueba contra los servidores CRL especificados. Para obtener más información sobre la seguridad de CRL, consulte la información sobre la "Utilización de listas de revocación de certificados" en la sección de Seguridad del centro de información de WebSphere MQ.

Nombre de igual SSL

En SSL, un archivo skeleton de nombre distinguido que debe coincidir con el nombre proporcionado por WebSphere MQ Queue Manager. El nombre distinguido se utiliza para comprobar el certificado identificador presentado por el servidor durante el tiempo de conexión.

Si no se ha establecido esta propiedad, se lleva a cabo esta comprobación de certificado.

La propiedad de nombre de igual SSL se ignora si no se especifica la propiedadSuite de cifrado SSL.

Esta propiedad es una lista de pares de nombre y valor de atributo separados por comas o punto y coma. Por ejemplo: 
CN=QMGR.*, OU=IBM, OU=WEBSPHERE

El ejemplo proporcionado comprueba el certificado identificador presentado por el servidor durante el tiempo de conexión. Para que la conexión sea satisfactoria, el certificado debe tener un Nombre común que empiece por QMGR. y debe tener como mínimo dos nombres de unidad organizativa, siendo el primero IBM y el segundo WEBSPHERE. La comprobación no es sensible a las mayúsculas y minúsculas.

Para obtener más detalles sobre los nombres distinguidos y su uso con WebSphere MQ, consulte la sección "Nombres distinguidos" en el centro de información de WebSphere MQ.

Agrupación de conexiones

Especifica un conjunto opcional de valores de agrupación de conexiones.

Las propiedades de agrupación de conexiones son comunes a todos los conectores J2C.

El servidor de aplicaciones agrupa conexiones y sesiones con el proveedor de JMS para mejorar el rendimiento. Esta acción es independiente de cualquier agrupación de conexiones de WebSphere MQ. Es necesario que configure las propiedades de agrupación de sesiones y conexiones para las aplicaciones adecuadamente; de lo contrario, es posible que no obtenga el comportamiento de sesión o conexión deseado.

Cambie el tamaño de la agrupación de conexiones si el acceso simultáneo en el servidor al recurso JMS supera el valor predeterminado. El tamaño de la agrupación de conexiones se establece por cola o por tema.

Tipo de datos Recuadro de selección
Valor predeterminado Seleccionado
Opciones de reconexión de cliente

Especifica si una conexión de modalidad de cliente se vuelve a conectar automáticamente, o no, en el caso de una anomalía de comunicaciones o del gestor de colas. Esta propiedad se ignora a menos que la fábrica de conexiones se esté utilizando en un entorno de cliente ligero o gestionado.

Tipo de datos Lista desplegable
Valor predeterminado INHABILITADO
Rango
DISABLED
La reconexión de cliente no se produce automáticamente.
ASDEF
Se utiliza el valor del atributo DefRecon de la stanza channels del archivo de configuración de cliente. Si no hay ningún valor DefRecon especificado, este valor tiene el mismo efecto que un valor de DISABLED.
RECONNECT
La reconexión se produce con cualquier gestor de colas coherente con el valor del atributo de gestor de colas, que puede ser un gestor de colas distinto de aquél con el cual se había establecido originalmente la conexión.
QMGR
La reconexión sólo se produce con el gestor de colas con el que se ha establecido originalmente la conexión.

Para obtener más información sobre la reconexión de cliente automática, consulte el centro de información de WebSphere MQ.

Tiempo de espera de reconexión de cliente

Periodo máximo de tiempo, en segundos, que una conexión de modalidad de cliente empleará intentando reconectarse automáticamente a un gestor de colas tras una anomalía de comunicaciones o del gestor de colas. Este parámetro se ignora a menos que la fábrica de conexiones se utilice en un entorno de cliente ligero o gestionado. El hecho de que este parámetro se utilice o no depende del valor del parámetro de opciones de reconexión de cliente.

Tipo de datos Entero
Unidades Segundos
Valor predeterminado 1800
Rango Un valor mayor que cero y hasta un máximo de 2147483647

Para obtener más información sobre la reconexión de cliente automática, consulte el centro de información de WebSphere MQ.
