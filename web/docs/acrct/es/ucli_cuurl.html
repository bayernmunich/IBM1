URL para clientes de aplicaciones

Un URL (Uniform Resource Locator) es un identificador que apunta a un recurso accesible electrónicamente, como un archivo de directorio en una máquina de una red, o un documento almacenado en una base de datos.

Los URL tienen el formato esquema:información_esquema.

Puede representar un esquema como http, ftp, file u otro término que identifique el tipo de recurso y el mecanismo por el que puede acceder al recurso.

En una ubicación de navegador de Internet o recuadro de dirección, el URL de un archivo disponible mediante HTTP (HyperText Transfer Protocol) empieza por http:. Por ejemplo: http://www.ibm.com. Los archivos disponibles mediante FTP (File Transfer Protocol) empiezan por ftp:. Los archivos disponibles localmente empiezan por file:.

La información_esquema identifica comúnmente la máquina de Internet que pone a disposición un recurso, la vía de acceso a dicho recurso y el nombre del mismo. La información_esquema para HTTP, FTP y File suele empezar por dos barras inclinadas (//) seguida de la dirección de Internet, separada del nombre de la vía de acceso del recurso por una barra inclinada (/). Por ejemplo:

http://www.ibm.com/software/webservers/appserv/library.html.

En HTTP y FTP, el nombre de vía de acceso finaliza con una barra inclinada cuando el URL apunta a un directorio. En tal caso, el servidor devuelve generalmente el índice predeterminado del directorio.



Los enlaces marcados (en línea) requieren acceso a Internet.
