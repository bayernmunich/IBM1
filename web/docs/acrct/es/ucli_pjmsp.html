Valores del proveedor de JMS para clientes de aplicaciones

Utilice esta página para configurar las propiedades del proveedor de JMS (Java Message Service) si desea utilizar un proveedor de JMS que no sea el proveedor de mensajería predeterminado ni WebSphere MQ como proveedor de JMS.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado. Pulse con el botón derecho del ratón en Proveedores de mensajes y pulse Nuevo. Los campos siguientes aparecen en el separador General.

Nombre

Nombre por el que se conoce el proveedor de JMS por razones administrativas.

Tipo de datos String
Descripción

Descriptor del proveedor de JMS, a efectos administrativos.

Tipo de datos String
Classpath

Lista de vías de acceso o nombres de archivos .jar que forman conjuntamente la ubicación de las clases de proveedores de recursos.

Clase de fábrica de contexto

El nombre de clase Java de la fábrica de contexto inicial para el proveedor de JMS.

Por ejemplo, para un proveedor de servicio LDAP, el valor tiene el formato siguiente: com.sun.jndi.ldap.LdapCtxFactory.

Tipo de datos String
URL de proveedor

URL de proveedor de JMS para búsquedas JNDI externas.

Por ejemplo, el URL LDAP para un proveedor de JMS tiene el formato siguiente: ldap://nombrehost.empresa.com/contextName.

Tipo de datos String
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
