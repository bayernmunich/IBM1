Herramienta de configuración de recursos del cliente de aplicaciones

Vaya a los temas de ayuda dentro de esta sección.

  • Adaptadores de recursos para el cliente
    Un adaptador de recursos es un controlador de software de nivel de sistema que una aplicación Java utiliza para conectarse a un sistema de información de empresa (EIS). El adaptador de recursos se puede conectar al cliente de aplicaciones y proporciona conectividad entre el EIS y la aplicación empresarial.
  • Entradas de entorno de recursos y proveedores de entorno de recursos
    Las referencias de entorno de recursos establecen una correlación entre los nombres lógicos que utiliza la aplicación cliente con los nombres físicos de los objetos.
  • Mensajería asíncrona en WebSphere Application Server utilizando JMS
    WebSphere Application Server da soporte a la mensajería asíncrona como método de comunicación basado en la interfaz de programación JMS (Java Message Service). La interfaz JMS proporciona un método común para que los programas Java (clientes y aplicaciones Java Platform, Enterprise Edition (Java EE) creen, envíen, reciban y lean solicitudes asíncronas como mensajes JMS.
  • Proveedores de Java Message Service para clientes
    Las aplicaciones cliente pueden utilizar recursos de mensajería de tres tipos principales de proveedores de JMS (Java Message Service en WebSphere Application Server: El proveedor de mensajería predeterminado de WebSphere Application Server (que utiliza la integración de servicios como proveedor), el proveedor de mensajería de WebSphere MQ (que utiliza el sistema WebSphere MQ como proveedor) y los proveedores de mensajería de terceros (que utilizan el producto de otra empresa como proveedor).
  • Actualización de WebSphere MQ como proveedor de Java Message Service y las configuraciones de recursos JMS, para clientes de aplicaciones
    Puede actualizar una configuración existente de WebSphere MQ como proveedor de JMS (Java Message Service) y actualizar la configuración de fábricas de conexiones de WebSphere MQ o destinos de WebSphere MQ.
  • Configuración de nuevas entradas de entorno de recursos para clientes de aplicaciones
    Puede crear nuevas entradas de recursos para la aplicación de cliente utilizando la herramienta ACRCT (Application Client Resource Configuration Tool).
  • Configuración de nuevas sesiones de correo para clientes de aplicaciones
    Puede utilizar la herramienta ACRCT (Application Client Resource Configuration Tool) para configurar nuevas sesiones de correo para el cliente de aplicaciones.
  • Configuración de nuevos proveedores de entorno de recursos para clientes de aplicaciones
    Puede crear nuevas configuraciones de proveedores de entorno de recursos para el cliente de aplicación utilizando la herramienta ACRCT (Application Client Resource Configuration Tool).
  • Configuración de nuevos proveedores de orígenes de datos (proveedores de JDBC) para clientes de aplicaciones
    Puede crear nuevos proveedores de datos, también conocidos como proveedores de JDBC, para el cliente de aplicaciones que utiliza la herramienta ACRCT (Application Client Resource Configuration Tool).
  • Configuración de nuevos proveedores de URL para clientes de aplicaciones
    Puede crear URL y proveedores de URL para la aplicación de cliente utilizando la herramienta ACRCT (Application Client Resource Configuration Tool).
  • Configuración de objetos administrados para adaptadores de recursos para el cliente
    Esta sección le ayuda a configurar nuevos objetos administrados para el cliente.
  • Configuración de recursos del cliente de mensajería Java
    Para configurar recursos del cliente de mensajería Java, debe crear nuevas configuraciones de proveedor JMS para el cliente de aplicaciones. El cliente de aplicaciones puede utilizar un servicio de mensajería a través de las API de JMS (Java Message Service). Un proveedor de JMS proporciona dos tipos de fábricas de J2EE. Una es una fábrica de conexiones JMS y la otra es una fábrica de destinos de JMS.
  • Configuración de sesiones y proveedores de correo para clientes de aplicaciones
    Puede editar las configuraciones de todas las sesiones y proveedores de correo para la herramienta ACRT (Application Client Resource Configuration Tool).
  • Configuración de nuevas fábricas de conexiones para adaptadores de recursos para el cliente
    Utilice ACRC (Application Client Resource Configuration Tool) para configurar nuevas fábricas de conexiones para adaptadores de recursos para el cliente.
  • Propiedades del origen de datos para clientes de aplicaciones

  • Proveedores de URL para Application Client Resource Configuration Tool
    Un proveedor URL implementa la función de un protocolo URL determinado, por ejemplo HTTP (HyperText Transfer Protocol). Este proveedor, que consta de un par de clases, amplía las clases java.net.URLStreamHandler y java.net.URLConnection.
  • URL para clientes de aplicaciones
    Un URL (Uniform Resource Locator) es un identificador que apunta a un recurso accesible electrónicamente, como un archivo de directorio en una máquina de una red, o un documento almacenado en una base de datos.
