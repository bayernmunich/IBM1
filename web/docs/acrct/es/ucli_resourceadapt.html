Adaptadores de recursos para el cliente

Un adaptador de recursos es un controlador de software de nivel de sistema que una aplicación Java utiliza para conectarse a un sistema de información de empresa (EIS). El adaptador de recursos se puede conectar al cliente de aplicaciones y proporciona conectividad entre el EIS y la aplicación empresarial.

Importante: Este tema no es relevante para el adaptador de recursos de WebSphere MQ. Las clases de WebSphere MQ las selecciona automáticamente el contenedor del cliente (tanto para una instalación autónoma como para una instalación de WebSphere Application Server).

El soporte de adaptador de recursos para aplicaciones cliente de Java EE es un subconjunto del soporte para el servidor. Para cualquier adaptador de recursos instalado con la herramienta ClientRAR, el adaptador de recursos del cliente se utiliza en un entorno no gestionado y debe cumplir la especificación Java EE Connector Architecture Specification versión 1.5 o superior. Sólo las conexiones de salida con EIS se admiten en las interfaces ManagedConnectionFactory. El soporte de mensajería de entrada (desde EIS), la gestión del ciclo de vida y los aspectos de gestión de trabajo de la especificación no se admiten en el cliente.

Para que una aplicación cliente utilice un adaptador de recursos, debe estar instalado en el directorio especificado por la variable de entorno, CLIENT_CONNECTOR_INSTALL_ROOT, definida cuando se ejecuta el script setupCmdLine. Las herramientas launchClient, ACRCT (Application Client Resource Configuration Tool) y clientRAR utilizan esta variable para buscar la ubicación predeterminada de todos los recursos instalados. Para instalar un adaptador de recursos en el cliente, utilice la herramienta clientRAR. Una vez que se haya instalado el adaptador de recursos, debe configurarse mediante la herramienta ACRCT. La herramienta de configuración del cliente añade la configuración de adaptador de recursos al archivo EAR. A continuación, se definen las fábricas de conexiones y los objetos administrados.

Al ejecutar clientes de aplicaciones Java EE, el script launchClient especifica una propiedad del sistema llamada com.ibm.ws.client.installedConnector, que está establecida en el mismo valor que la variable CLIENT_CONNECTOR_INSTALL_ROOT. Esta es la ubicación predeterminada para los adaptadores de recursos instalados y se puede sobrescribir para todas las llamadas launchClient especificando el parámetro -CCD. Cuando el contenedor de cliente está activado, todos los subdirectorios de adaptador de recursos bajo la ubicación predeterminada especificada para el directorio de adaptadores de recursos se añaden a la classpath. Esta acción permite que la aplicación cliente utilice los adaptadores de recursos sin utilizar la herramienta ACRCT para especificar cualquiera de los recursos de cliente.

La utilización de adaptadores de recursos es un nuevo mecanismo para ampliar fácilmente las aplicaciones cliente.
