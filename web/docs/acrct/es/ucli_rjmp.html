Valores del proveedor de correo para clientes de aplicaciones

Utilice esta página para implementar la API de JavaMail y crear sesiones de correo.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado. Pulse con el botón derecho del ratón en Proveedores de correo > y pulse Nuevo. Los campos siguientes aparecen en el separador General:

Nombre

Nombre del proveedor de recursos de JavaMail.

Descripción

Descripción opcional del proveedor de recursos.

Classpath

Especifica una lista de vías de acceso o nombres de archivos JAR que forman conjuntamente la ubicación de las clases de proveedores de recursos.

Protocolo

Especifica el nombre del protocolo.

Nombre de clase

Especifica el nombre de la clase que implementa el protocolo. Deje este campo en blanco si desea utilizar la implementación predeterminada.

Tipo

Este menú contiene los dos valores siguientes: TRANSPORT o STORE.

Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
