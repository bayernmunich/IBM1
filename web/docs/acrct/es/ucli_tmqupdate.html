Actualización de WebSphere MQ como proveedor de Java Message Service y las configuraciones de recursos JMS, para clientes de aplicaciones

Puede actualizar una configuración existente de WebSphere MQ como proveedor de JMS (Java Message Service) y actualizar la configuración de fábricas de conexiones de WebSphere MQ o destinos de WebSphere MQ.

Acerca de esta tarea

Utilice esta tarea para actualizar una configuración existente de WebSphere MQ como proveedor de JMS (Java Message Service) y para actualizar la configuración de fábricas de conexiones de WebSphere MQ o destinos de WebSphere MQ.

Procedimiento

  1. Inicie ACRCT (Application Client Resource Configuration Tool).
  2. Abra el archivo EAR (Enterprise Archive) que contiene el proveedor de JMS de WebSphere MQ, la fábrica de conexiones de WebSphere MQ o el destino de WebSphere MQ. El contenido del archivo EAR se muestra en una vista de árbol de navegación.
  3. Seleccione el archivo JAR (Java Archive) que contiene el proveedor de JMS, la fábrica de conexiones o el destino que desea actualizar.
  4. Expanda el archivo JAR para ver su contenido hasta que encuentre el proveedor de JMS, la fábrica de conexiones o el destino que desea actualizar. Realice una de las acciones siguientes:
    • Pulse con el botón derecho del ratón en el objeto y pulse Propiedades.
    • Pulse Editar > Propiedades en la barra de menús.
  5. Actualice las propiedades en los campos visualizados. Para obtener ayuda de campos detallada, consulte:
    • Propiedades del proveedor de JMS
    • Propiedades de fábrica de conexiones de cola MQ
    • Propiedades de fábrica de conexiones de tema MQ
    • Propiedades de destino de cola MQ
    • Propiedades de destino de tema MQ
  6. Pulse Aceptar.
  7. Pulse Archivo > Guardar para guardar los cambios.


Los enlaces marcados (en línea) requieren acceso a Internet.
