Valores de la fábrica de conexiones de cola del proveedor de mensajería predeterminado V5 para clientes de aplicaciones

Utilice este panel para examinar o modificar las propiedades de configuración de la fábrica de conexiones de cola JMS seleccionada para la mensajería punto a punto que se va a utilizar con las aplicaciones de WebSphere Application Server de la versión 5. Estas propiedades de configuración controlan cómo se crean las conexiones entre el proveedor de JMS y el sistema de mensajería predeterminado que utiliza.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedor de mensajería > Proveedor predeterminado versión 5. Pulse con el botón derecho del ratón en Fábrica de conexiones de cola y pulse Nueva. Los campos siguientes aparecen en el separador General.

Una fábrica de conexiones de cola sirve para crear conexiones JMS con destinos de cola. La fábrica de conexiones de cola la crea el proveedor de JMS del producto WebSphere Application Server interno. Una fábrica de conexiones del proveedor de mensajería predeterminado V5 presenta las propiedades siguientes:

Nombre

Nombre por el que se conoce esta fábrica de conexiones con fines administrativos dentro de IBM WebSphere Application Server. El nombre debe ser exclusivo dentro de las fábricas de conexiones JMS a lo largo del dominio administrativo de WebSphere.

Tipo de datos String
Descripción

Descripción de esta fábrica de conexiones con fines administrativos dentro de IBM WebSphere Application Server.

Tipo de datos String
Valor predeterminado Null
Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

ID de usuario

ID de usuario utilizado, junto con la propiedad Contraseña, en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Las propiedades ID de usuario y Contraseña de la fábrica de conexiones se utilizan si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña, por ejemplo, si la aplicación que llama utiliza el método createQueueConnection(). El cliente de JMS dirige el ID de usuario y la contraseña al servidor JMS.

Tipo de datos String
Contraseña

Contraseña utilizada, junto con la propiedad ID de usuario, en la operación de autenticación si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña.

Si especifica un valor para la propiedad ID de usuario, también debe especificar un valor para la propiedad Contraseña.

Vuelva a entrar la contraseña

Confirma la contraseña.

Nodo

Nombre de nodo de WebSphere del nodo administrativo donde se ejecuta el servidor de JMS para esta fábrica de conexión. Las conexiones creadas por esta fábrica se conectan al servidor JMS.

Tipo de datos String
Servidor de aplicaciones

Entre el nombre del servidor de aplicaciones. Este nombre no es el nombre de host de la máquina, sino el nombre del servidor de aplicaciones configurado.

Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
