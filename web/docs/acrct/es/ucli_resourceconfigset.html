Valores del adaptador de recursos

Utilice este panel para ver o modificar las propiedades de configuración del adaptador de recursos. Estas propiedades de configuración controlan cómo se crean los adaptadores de recursos.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Adaptador de recursos. Pulse con el botón derecho del ratón en Adaptador de recursos y pulse Nuevo. Los campos siguientes aparecen en el separador General.

Nombre

Nombre por el que se conoce este Adaptador de recursos con fines administrativos dentro de IBM WebSphere Application Server. El nombre debe ser exclusivo dentro de los adaptadores de recursos en el dominio administrativo del producto.

Tipo de datos String
Descripción

Descripción de este adaptador de recursos con fines administrativos dentro de IBM WebSphere Application Server.

Tipo de datos String
Classpath

Cualquier classpath adicional. La vía de acceso al directorio del adaptador de recursos se añade automáticamente.

Tipo de datos String
Valor predeterminado Vía de acceso al directorio del adaptador de recursos.
Vía de acceso nativa

Vía de acceso donde está ubicado el adaptador de recursos. Especifique aquí cualquier classpath nativa adicional.

Tipo de datos String
Nombre del adaptador de recursos

Campo obligatorio que señala a un subdirectorio del adaptador de recursos instalado. La entrada no representa el nombre del directorio completo del adaptador de recursos. El nombre del directorio completo es la vía de acceso del adaptador de recursos instalado, junto con el nombre del adaptador de recursos.

Tipo de datos String
Vía de acceso del adaptador de recursos instalado

Directorio donde están instalados los adaptadores de recursos. Si no rellena este campo, entrará en vigor el valor predeterminado.

Si especifica el valor, ${CONNECTOR_INSTALL_ROOT}, éste sustituye al de la variable CLIENT_CONNECTOR_INSTALL_ROOT en la máquina en la que se ejecuta la aplicación cliente. Esta acción permite que la aplicación se ejecute fácilmente en máquinas distintas, donde la instalación del cliente podría estar en ubicaciones distintas.

Tipo de datos String
Valor predeterminado ${CONNECTOR_INSTALL_ROOT}
