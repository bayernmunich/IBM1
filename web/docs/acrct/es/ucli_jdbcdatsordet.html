Valores del proveedor de orígenes de datos para clientes de aplicaciones

Utilice esta página para crear un origen de datos bajo un proveedor de JDBC que proporcione la clase de implementación del controlador JDBC específica.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado. Pulse con el botón derecho del ratón Proveedores de orígenes de datos > y pulse Nuevo. Los campos siguientes aparecen en el separador General:

Nombre

Especifica el nombre de visualización del origen de datos.

Por ejemplo, puede establecer este campo en Probar origen de datos.

Tipo de datos String
Descripción

Especifica una descripción de texto del recurso.

Tipo de datos String
Classpath

Lista de vías de acceso o nombres de archivos .jar que forman conjuntamente la ubicación de las clases de proveedores de recursos.

Clase de implementación

Utilice este valor para realizar funciones específicas de la base de datos.

Tipo de datos String
Valor predeterminado Depende de la clase de implementación del controlador JDBC
Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
