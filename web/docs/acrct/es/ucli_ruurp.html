Valores del proveedor de URL para clientes de aplicaciones

Utilice esta página para crear nuevos proveedores de URL.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado. Pulse con el botón derecho del ratón en Proveedores de URL y pulse Nuevo. Los campos siguientes aparecen en el separador General.

Un proveedor de URL implementa la función de un protocolo de URL determinado, como por ejemplo HTTP (Hyper Text Transfer Protocol). Este proveedor, formado por clases, amplía las clases java.net.URLStreamHandler y java.net.URLConnection.

Nombre

Nombre administrativo del URL.

Descripción

Descripción opcional del URL, para los registros de administración.

Classpath

Lista de vías de acceso o nombres de archivos JAR que forman conjuntamente la ubicación de las clases de proveedores de recursos.

Protocolo

Protocolo soportado por este manejador de corrientes de datos. Por ejemplo, nntp, smtp, ftp, etc.

Para utilizar el protocolo predeterminado, deje este campo en blanco.

Clase de manejador de corriente de datos

Nombre completo de una clase Java definida por el usuario que amplía java.net.URLStreamHandler para un protocolo URL determinado, como, por ejemplo, FTP.

Para utilizar el manejador de corrientes de datos predeterminadas, deje este campo en blanco.

Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.



Los enlaces marcados (en línea) requieren acceso a Internet.
