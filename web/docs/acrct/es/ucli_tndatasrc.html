Configuración de nuevos proveedores de orígenes de datos (proveedores de JDBC) para clientes de aplicaciones

Puede crear nuevos proveedores de datos, también conocidos como proveedores de JDBC, para el cliente de aplicaciones que utiliza la herramienta ACRCT (Application Client Resource Configuration Tool).

Antes de empezar

Durante esta tarea, podrá crear nuevos proveedores de orígenes de datos, también conocidos como proveedores de JDBC, para el cliente de aplicaciones. En una tarea administrativa independiente, instale el código Java para el proveedor de orígenes de datos necesario en la máquina cliente en la que reside el cliente de aplicaciones.

Acerca de esta tarea

Utilice esta tarea para conectar clientes de aplicaciones a bases de datos relacionales.

Procedimiento

  1. Inicie ACRCT (Application Client Resource Configuration Tool) y abra el archivo EAR para el que desea configurar el nuevo proveedor de orígenes de datos. El contenido del archivo EAR se muestra en una vista de árbol.
  2. En esta vista de árbol, seleccione el archivo JAR donde desea configurar el nuevo proveedor de orígenes de datos.
  3. Expanda el archivo JAR para ver su contenido.
  4. Pulse la carpeta Proveedores de orígenes de datos. Realice una de las operaciones siguientes:
    • Pulse con el botón derecho del ratón en la carpeta y pulse Nuevo proveedor.
    • Pulse Editar > Nuevo en la barra de menús.
  5. Configure las propiedades del proveedor de orígenes de datos en el diálogo de propiedades resultante.
  6. Pulse Aceptar cuando haya terminado.
  7. Pulse Archivo > Guardar en la barra de menús para guardar los cambios.


Los enlaces marcados (en línea) requieren acceso a Internet.
