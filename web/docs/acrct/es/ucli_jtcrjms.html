Valores de la sesión de correo para clientes de aplicaciones

Utilice esta página para configurar las propiedades de la sesión de correo.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de correo > instancia de proveedor de correo. Pulse con el botón derecho del ratón en Sesiones de correo y pulse Nueva. Los campos siguientes aparecen en el separador General:

Nombre

Representa el nombre administrativo del objeto de sesión de JavaMail.

Descripción

Proporciona una descripción opcional de los registros de administración.

Nombre JNDI

El programa de tiempo de ejecución del cliente de aplicaciones utiliza este campo para recuperar información de configuración.

Host de transporte de correo

Especifica el servidor al que conectarse al enviar correo.

Protocolo de transporte de correo

Especifica el protocolo de transporte que se debe utilizar al enviar correo.

Usuario de transporte de correo

Especifica el ID de usuario que se debe utilizar cuando el host de transporte de correo requiere autenticación.

Contraseña de transporte de correo

Especifica la contraseña que se debe utilizar cuando el host de transporte de correo requiere autenticación.

Habilitar análisis estricto de direcciones de Internet

Especifica si las direcciones del destinatario deben analizarse estrictamente en cumplimiento con RFC 822, que es un documento de especificaciones emitido por el comité IAB (Internet Architecture Board).

Este valor no se suele utilizar con la mayoría de las aplicaciones de correo. La sintaxis de RFC 822 para analizar direcciones de forma eficaz obliga a una definición estricta de direcciones válidas de correo electrónico. Si selecciona este valor, JavaMail se adherirá a la sintaxis de RFC 822 y rechazará las direcciones de destinatarios cuyo análisis no sea una dirección válida de correo electrónico (tal como se define en la especificación). Si no selecciona este valor, JavaMail no se adherirá a la sintaxis de RFC 822 y aceptará las direcciones de destinatarios que no cumplen con la especificación. De forma predeterminada, este valor no está seleccionado. Puede consultar la especificación RFC 822 en la dirección URL siguiente de World Wide Web Consortium (W3C): http://www.w3.org/Protocols/rfc822/.

Vuelva a entrar la contraseña

Confirma la contraseña.

Correo de

Especifica el emisor de correo.

Host de almacenamiento de correo

Especifica el nombre del host (o "dominio") de la cuenta de correo.

Usuario de almacenamiento de correo

Especifica el ID de usuario de la cuenta de correo.

Contraseña de almacenamiento de correo

Especifica la contraseña de la cuenta de correo.

Vuelva a entrar la contraseña

Confirma la contraseña.

Protocolo de almacenamiento de correo

Especifica el protocolo que se debe utilizar al recibir correo.

Depurar correo

Cuando este valor está establecido en true, la interacción de JavaMail con los servidores de correo, junto con estas propiedades de la sesión de correo, se imprimen en el archivo stdout.

Propiedades personalizadas

Especifica los pares nombre-valor para establecer propiedades adicionales en el objeto creado durante el tiempo de ejecución de este recurso.

Especifique un nombre que sea una propiedad pública en el objeto y un valor que se pueda convertir desde una serie en el tipo necesario para el método set de la propiedad. Las propiedades y valores aceptables dependen del objeto creado. Consulte la documentación del objeto para obtener una lista de propiedades válidas.
