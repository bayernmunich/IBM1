Valores de la fábrica de conexiones del proveedor predeterminado

Utilice este panel para ver o modificar las propiedades de configuración de la fábrica de conexiones JMS seleccionada para su uso con el proveedor de JMS (Java Message Service) del producto interno que está instalado con WebSphere Application Server. Estas propiedades de configuración controlan cómo se crean las conexiones entre el proveedor de JMS y el bus de integración de servicios que utiliza.

Para ver esta página de la herramienta ACRCT (Application Client Resource Configuration Tool), pulse Archivo > Abrir. Después de buscar un archivo EAR, pulse Abrir. Expanda el archivo JAR seleccionado > Proveedores de mensajería > Proveedor predeterminado. Pulse con el botón derecho del ratón en Fábrica de conexiones y pulse Nueva. Los campos siguientes aparecen en el separador General.

Los valores que tienen un valor predeterminado, muestran el valor adecuado. Todos los valores que tengan un valor fijo, tienen un menú desplegable.

Nombre

Nombre de la fábrica de conexiones.

Tipo de datos String
Descripción

Descripción de esta fábrica de conexiones con fines administrativos dentro de IBM WebSphere Application Server.

Tipo de datos String
Nombre JNDI

Nombre JNDI que se utiliza para hacer coincidir esta definición de la fábrica de conexiones de adaptador de recursos con el descriptor de despliegue. Esta entrada es un nombre resource-ref.

Tipo de datos String
Nombre de usuario

Nombre de usuario que se utiliza con la propiedad Contraseña para conectar con aplicaciones.

Si especifica un valor para la propiedad Nombre de usuario, también debe especificar un valor para la propiedad Contraseña.

Las propiedades ID de usuario y Contraseña de la fábrica de conexiones se utilizan si la aplicación que llama no proporciona explícitamente un ID de usuario ni una contraseña. Si se especifican un nombre de usuario y una contraseña, se crea un alias de autenticación para la fábrica donde está cifrada la contraseña.

Tipo de datos String
Contraseña

Contraseña que se utiliza para autenticar la conexión con aplicaciones.

Si especifica un valor para la propiedad Nombre de usuario, también debe especificar un valor para la propiedad Contraseña.

Tipo de datos String
Vuelva a entrar la contraseña

Confirma la contraseña.

Nombre de bus

Nombre de bus al que se conecta la fábrica de conexiones.

Tipo de datos String
Identificador de cliente

Nombre del cliente. Necesario para subscripciones de tema duraderas.

Tipo de datos String
Fiabilidad de mensajes no persistentes

Fiabilidad aplicada a los mensajes JMS no persistentes enviados utilizando esta fábrica de conexiones.

Si desea opciones de entrega de fiabilidad diferentes para destinos de JMS individuales, puede establecer esta propiedad en Como destino de bus. De esta forma la fiabilidad está definida por la propiedad Fiabilidad del destino del bus al que se asigna este destino de JMS.

Valor predeterminado ReliablePersistent
Rango
Ninguno
No hay fiabilidad del mensaje para mensajes no persistente. Si no se puede entregar un mensaje no persistente, se descarta.
Mejor esfuerzo no persistente
Los mensajes no se escriben nunca en disco y se pierden si la memoria caché se desborda.
Express no persistente
Si se produjera un desbordamiento de memoria en memoria caché, se grabarán los mensajes de modo asíncrono en el almacén persistente, pero no se conservarán al reiniciar el servidor.
Fiable no persistente
Los mensajes se pueden perder si se produce una anomalía en el motor de mensajería, pero pueden perderse en condiciones de funcionamiento normal.
Fiable persistente
Los mensajes se pueden perder si se produce una anomalía en el motor de mensajería, pero no se pierden en condiciones de funcionamiento normal.
Seguro persistente
El mayor grado de fiabilidad donde se admite la entrega de mensajes segura.
Como destino de bus
Utilice la opción de entrega configurada para el destino del bus.

Inicio de suscripciones duraderas

Nombre de la ubicación inicial de suscripciones duraderas.

Tipo de datos String
Compartir suscripciones duraderas

Controla si las suscripciones duraderas se comparten entre conexiones con miembros de un clúster de servidores.

Normalmente, sólo una sesión a la vez puede tener un TopicSubscriber para una suscripción duradera concreta. Esta propiedad le permite alterar este comportamiento para que una suscripción duradera pueda tener varios consumidores simultáneos,

Tipo de datos Lista de selección
Valor predeterminado En clúster
Rango
En clúster
Permite compartir suscripciones duraderas cuando se realizan conexiones desde un clúster de servidores.
Siempre compartidas
Las suscripciones duraderas pueden compartirse entre conexiones.
Nunca compartidas
Las suscripciones duraderas nunca se comparten entre conexiones.
Lectura hacia adelante

Controla la optimización de lectura hacia adelante durante el envió de mensajes.

Valor predeterminado Valor predeterminado
Rango De forma predeterminada, Siempre activo y Siempre inactivo

Cadena de transporte de entrada de destino

Nombre del protocolo que define un grupo de motores de mensajería.

Tipo de datos String
Puntos finales del proveedor

Lista de puntos finales separados por comas utilizada para conectar con un servidor de rutina de carga.

Escriba una lista separada por comas de tripletes de puntos finales con la sintaxis: host:puerto:protocolo.

Ejemplo 
merlin:7276:BootstrapBasicMessaging,Gandalf:
5557:BootstrapSecureMessaging
donde

BootstrapBasicMessaging se corresponde con el protocolo remoto InboundBasicMessaging (JFAP-TCP/IP).
Valor predeterminado
  • Si no se especifica el nombre de host, se utiliza el valor predeterminado localhost.
  • Si no se especifica el número de puerto, se utiliza el valor predeterminado 7276.
  • Si no se especifica el nombre de cadena, se utiliza como valor predeterminado una cadena predefinida, como BootstrapBasicMessaging.
Proximidad de conexión

La proximidad que el motor de mensajería debe tener con el solicitante.

Valor predeterminado Bus
Rango Bus, Host, Cluster, Server
