SIBusMember

This type is a class for model objects.

A bus has a number of bus members. Each bus member is a server or a cluster. A server will have one messaging engine on the bus, while a cluster will have one or more messaging engine on a bus. Buses and bus members are held in a separate document to messaging engines - buses and bus members are held in a file called sib-bus.xml at bus scope while messaging engines are held in files called sib-engine.xml at server and/or cluster scope as appropriate. To avoid cross-document links, a bus member simply indicates the scope (server or cluster) at wich the bus member's engines can be found.
