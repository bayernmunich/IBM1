Opis

Narzędzie IBM Support Assistant Data Collector udostępnia funkcję automatycznego gromadzenie danych dla produktów IBM. Konfiguracja fabryczna tego narzędzia umożliwia odnajdywanie istotnych danych diagnostycznych w używanym systemie komputerowym i kopiowanie ich do pliku modułu gromadzącego dane. Przykładem danych diagnostycznych jest plik dziennika wygenerowany przez produkt IBM, zawierający szczegółową historię zdarzeń, które miały miejsce w trakcie jego działania. Plik taki może być bardzo pomocny przy określaniu rodzaju i przyczyny błędu oprogramowania. Inne przykłady danych diagnostycznych to: pliki inicjujące, pliki konfiguracyjne, wersja systemu operacyjnego, miejsce na dysku i połączenia sieciowe. Narzędzie można uruchamiać w trybie interfejsu użytkownika WWW lub w trybie wiersza komend konsoli. Tryb konsoli umożliwia sterowanie skryptami gromadzenia danych programu IBM Support Assistant Data Collector z poziomu wiersza komend. Narzędzie zawiera wiele funkcji wspomagających interakcje z nim w trybie konsoli, między innymi funkcję umożliwiającą rejestrowanie w pliku odpowiedzi użytkownika udzielanych podczas sesji w trybie konsoli, a następnie możliwość kolejnego uruchomienia tego samego skryptu gromadzenia danych za pomocą tego pliku.

Ułatwienia dostępu

IBM® stara się dostarczać produkty dostępne dla wszystkich użytkowników, niezależnie od ich wieku czy kondycji. Interfejs konsoli programu IBM ISA Data Collector obsługuje operacje wykonywane jedynie za pomocą klawiatury i może być również używany przy użyciu aplikacji lektora ekranowego.

Instalacja narzędzia i składnia

W większości przypadków zainstalowanie i skonfigurowanie narzędzia wymaga wykonania przedstawionych poniżej kroków. W przypadku wystąpienia problemów lub aby uzyskać więcej informacji na temat któregoś z tych kroków, można odwołać się do poniższych sekcji.


  1. Rozpakuj pliki narzędzia w dowolnie wybranym katalogu. W poniższych sekcjach dokładniej przedstawiono sposób przeprowadzenia ekstrakcji.
  2. Uruchom narzędzie w trybie interfejsu użytkownika WWW lub w trybie wiersza komend konsoli. Wykonaj opisaną poniżej procedurę ustawiania zmiennej środowiskowej JAVA_HOME. Następnie można wykonać skrypt uruchamiania. Uruchomione narzędzie rozpocznie gromadzenie danych. Wykonaj poniższe instrukcje w celu interakcji z narzędziem. Instalowanie narzędzia

We wszystkich przypadkach instalacja narzędzia IBM Support Assistant Data Collector sprowadza się do rozpakowania plików z zarchiwizowanego pliku .zip lub .tar.gz. Pliki można wyodrębnić w dowolnym miejscu systemu plików, w którym narzędzie będzie uruchamiane. Spowoduje to utworzenie podkatalogu ISALite w katalogu docelowym.

Uruchamianie narzędzia w trybie interfejsu użytkownika WWW

W trybie interfejsu użytkownika WWW należy otworzyć plik index.html znajdujący się w podkatalogu isadc.

Tryb wiersza komend konsoli - ustawianie zmiennej środowiskowej JAVA_HOME

W trybie wiersza komend konsoli odpowiedni skrypt uruchamiania wywoływany jest z poziomu wiersza komend. W systemie Windows skrypty uruchomieniowe są plikami wsadowymi. W innych środowiskach są one skryptami powłoki. Ponieważ narzędzie wiersza komend jest zaimplementowane jako aplikacja języka Java, przed jego uruchomieniem należy znaleźć interpreter tego języka. Jeśli interpreter języka Java jest niedostępny w ścieżce określonej przez zmienną środowiskową PATH, należy ustawić zmienną środowiskową JAVA_HOME ręcznie. Narzędzie IBM Support Assistant Data Collector wymaga środowiska JRE na poziomie 1.5 lub nowszego, dlatego należy najpierw upewnić się, że odpowiednie środowisko JRE jest zainstalowane w systemie, w którym narzędzie będzie działać. Jeśli odpowiednie środowisko JRE jest zainstalowane, należy wykonać komendę systemu operacyjnego tak, aby zmienna JAVA_HOME wskazywała na to środowisko JRE. Środowiska Microsoft JVM/JDK i gij (biblioteka libgcj GNU) nie są obsługiwane.

Na przykład, jeśli na platformie Windows środowisko jre 1.5 jest zainstalowane w katalogu c:\jre1.5, należy ustawić zmienną JAVA_HOME przy użyciu następującej komendy:

SET JAVA_HOME=c:\jre1.5
UWAGA: w argumencie komendy SET nie należy stosować znaków cudzysłowu, nawet jeśli argument ten zawiera białe znaki.

Na platformach Linux, AIX lub Solaris, jeśli środowisko JRE jest zainstalowane w katalogu /opt/jre15, należy ustawić zmienną JAVA_HOME przy użyciu następującej komendy:

export JAVA_HOME=/opt/jre15

Uruchamianie narzędzia w trybie wiersza komend konsoli

Jeśli nie ma możliwości uruchomienia interfejsu użytkownika WWW, narzędzie powinno zostać uruchomione w trybie wiersza komend. Wystarczy uruchomić skrypt z poziomu wiersza komend (isadc/isadc.bat lub isadc/isadc.sh).

Pliki są zapisywane do katalogu instalacyjnego

Domyślnie katalog instalacyjny produktu isadc jest używany do przechowywania plików utworzonych podczas wykonywania. W niektórych systemach katalog instalacyjny produktu isadc będzie dostępny tylko do odczytu. W takim przypadku należy użyć parametru -useHome (dostępny tylko w trybie konsoli). Ten parametr spowoduje zapisanie plików tymczasowych w katalogach tymczasowych systemów i plików trwałych w katalogu domowym użytkownika. UWAGA: po wyodrębnieniu pliki .zip lub .tar.gz instalacji użytkownik, który wyodrębnił plik, musi uruchomić operację gromadzenia danych, zanim jakikolwiek innych użytkownik podejmie próbę uruchomienia operacji gromadzenia danych przy użyciu parametru -useHome. Zapewni to odpowiednią konfigurację przed wykonaniem przez innego użytkownika nie będącego administratorem.

Interakcja z narzędziem

Po uruchomieniu narzędzia IBM Support Assistant Data Collector w przeglądarce WWW zostanie wyświetlony interfejs graficzny. Zanim będzie można użyć narzędzia do gromadzenia danych i analizy, należy najpierw wybrać typ problemu w oknie głównym. Po wybraniu typu problemu następnym krokiem jest kliknięcie przycisku Uruchom. W trakcie działania skrypt gromadzenia danych wyświetli zapytanie o dodatkowe informacje potrzebne do zakończenia działania gromadzenia. Skrypt może również wyświetlić prośbę o podanie informacji dotyczących konfiguracji, kolejności zdarzeń prowadzącej do wystąpienia danego problemu lub preferencji użytkownika odnoszących się do sposobu zakończenia gromadzenia. Po uzyskaniu wszystkich wymaganych informacji skrypt przechodzi do ostatniego etapu gromadzenia danych.

Po zakończeniu gromadzenia danych przez narzędzie wyniki można wysłać do działu wsparcia IBM. Do przesyłania plików można wybrać niezaszyfrowany protokół FTP lub zaszyfrowany protokół HTTPS. Narzędzie prześle do działu wsparcia IBM plik ZIP o nazwie widocznej w obszarze statusu gromadzenia.

Jeśli program IBM Support Assistant Data Collector działa w trybie tekstowym, użytkownik nie ma do dyspozycji list wyboru ani pól wprowadzania. Zamiast nich dostępne opcję są reprezentowane przez numerowane listy. Użytkownik musi wprowadzić numer wybranej opcji, a następnie nacisnąć klawisz Enter. Zamiast pól wejściowych stosowane są podpowiedzi. Użytkownik musi wpisać odpowiedź i nacisnąć klawisz Enter. Po zakończeniu gromadzenia danych plikiem wyjściowym będzie kolejny plik ZIP. Wyjściowy plik ZIP może zostać wysłany do działu wsparcia IBM lub sprawdzony lokalnie.

Aby zatrzymać narzędzie modułu gromadzącego dane, w trybie tekstowym wpisz opcję quit lub zamknij okno przeglądarki.

Narzędzie wyświetli pytanie o nazwy plików. Na platformach Unix użycie znaku "~" jako oznaczenia katalogu osobistego (HOME) użytkownika nie jest obsługiwane. Jeśli zostanie użyty znak "~" przywołany zostanie podkatalog w obecnym katalogu roboczym o nazwie "~".

Określanie informacji o wersji

Domyślnie wersja narzędzia (i jego różnych komponentów podrzędnych) jest wyświetlana w konsoli, z poziomu której zostało ono uruchomione. Jeśli narzędzie działa w trybie interfejsu użytkownika WWW, informacje o wersji można znaleźć również przy użyciu opcji Pomoc->Informacje. Jeśli nie jest dostępny język Java lub nie można uruchomić aplikacji Java, informacje o wersji narzędzia można również uzyskać, wywołując skrypty uruchamiania z opcją "-version".

*UWAGA: skrypty uruchamiania wywołane z użyciem opcji "-version" wyświetlą informacje o wersji bez wywołania narzędzia.

Wyświetlanie pliku zasobów

Narzędzie może gromadzić informacje o zasobach bieżącego systemu. Informacje o zasobach są automatycznie umieszczane w pliku ZIP gromadzenia, jeśli jest to możliwe.
Dane wyjściowe stanowi plik zasobów (ISALite/inventory/inventory_rcf.xml), który najlepiej przeglądać przy użyciu przeglądarki WWW.

Aktualizacje, ograniczenia i znane problemy

Poniżej wymieniono aktualizacje dostępne w momencie publikacji tej wersji oraz znane ograniczenia i problemy. Spisywanie zasobów sprzętowych dostępne jedynie w środowisku JRE firmy IBM

Jeśli podczas gromadzenia danych ma miejsce spisywanie zasobów sprzętowych lub oprogramowania dostępnych na komputerze i używane jest środowisko JRE inne niż firmy IBM, informacje dotyczące sprzętu mogą nie zostać zwrócone. Wynika to z ograniczenia mechanizmu gromadzenia danych. Obejściem jest użycie środowiska JRE firmy IBM.

Linux on IBM System z (zLinux)

Gromadzenie danych w systemie zLinux jest obsługiwane tylko w trybie konsoli. Aby uruchomić gromadzenie danych, należy wywołać skrypt isadc.sh z poziomu powłoki.

Problemy z instalacją

Konfiguracja wtyczki Java

Przeglądarka, w której ma zostać uruchomiony program IBM Support Assistant Data Collector, musi zostać skonfigurowana za pomocą wtyczki Java 1.6. Aby sprawdzić, czy przeglądarka została poprawnie skonfigurowana, można użyć następującego odsyłacza: http://www.java.com/en/download/testjava.jsp. Jeśli przeglądarka działa na platformie Linux lub Unix, konieczne może być ręczne zainstalowanie i skonfigurowanie wtyczki Java.
