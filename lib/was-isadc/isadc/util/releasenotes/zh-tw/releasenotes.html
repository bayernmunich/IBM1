說明

IBM Support Assistant Data Collector 工具為 IBM 產品提供自動資料收集。此工具會預先配置成尋找您電腦系統中的重要診斷資料，並複製到「收集器」檔案中。舉例來說，診斷資料可以是您 IBM 產品產生的日誌檔，其中含有產品作業期間發生的詳細事件歷程。如此的檔案對於判斷軟體問題的本質和原因可能非常有用。其他像是起始設定檔、配置檔、作業系統版本、磁碟空間，以及網路連線等，都可算是診斷資料。這套工具可以在 Web UI 模式或指令行主控台模式中執行。主控台模式可讓您利用指令行來控制 IBM Support Assistant Data Collector 收集 Script。此工具隨附數個功能，可協助您在主控台模式下與此工具進行互動；其中一個功能可讓您將主控台模式階段作業的回應記錄在檔案中，這樣，以後就可以使用該檔案來驅動執行相同的收集 Script。

入門

工具安裝與用法

在大多數情況下，執行下列步驟序列可讓您啟動並執行此工具。如果您遇到問題，或需要其中任何步驟的相關資訊，則可以參閱此步驟下面的區段。


  1. 將此工具解壓縮到您所選擇的任何目錄。請參閱下列章節，以取得有關如何執行解壓縮作業的詳細資料。
  2. 在 Web UI 模式或指令行主控台模式下執行此工具。遵循下面說明的程序來設定 JAVA_HOME 環境變數。在執行此作業後，您可以執行啟動 Script。 在啟動此工具以執行收集程序後，請遵循下面的指示，與此工具進行互動。安裝工具

無論如何，只需解壓縮保存的 .zip 檔或 .tar.gz 檔中的檔案，即可完成 IBM Support Assistant Data Collector 工具的安裝作業。可以將這些檔案解壓縮到預期執行此工具的檔案系統位置 (您可以在系統上選擇任何檔案系統位置來執行此工具)。這會在您的目標目錄下建立 ISALite 子目錄。

在 Web UI 模式下啟動工具

若為 Web UI 模式，請開啟位於 isadc 子目錄的 index.html 檔。

指令行主控台模式 - 設定 JAVA_HOME 環境變數

若為指令行模式，您可以從指令行呼叫適當的啟動 Script。若為 Windows 系統，則這些啟動 Script 是批次檔。若為其他環境，則它們是 Shell Script。由於此工具是實作為 Java 應用程式，因此必須先找到 Java，才能啟動此指令行工具。如果 PATH 上沒有 Java，則您將必須手動設定 JAVA_HOME 環境變數。IBM Support Assistant Data Collector 工具需要層次 JRE 1.5 或更高版本，因此您必須先確定預期執行此工具的系統上有安裝適當的 JRE。如果有安裝適當的 JRE，則您將需要發出作業系統特定指令，以便將 JAVA_HOME 變數設為指向此 JRE。不支援 Microsoft JVM/JDK 及 gij (GNU libgcj)。

例如，在 Windows 平台上，如果您將 jre1.5 安裝在 c:\jre1.5 中，請使用下列指令來設定 JAVA_HOME：

SET JAVA_HOME=c:\jre1.5
附註：請勿在 SET 指令的值中使用引號（即使您的值中有空格字元）。

在 Linux、AIX 或 Solaris 上，如果您將 JRE 安裝在 /opt/jre15 中，請使用下列指令來設定 JAVA_HOME：

export JAVA_HOME=/opt/jre15

在指令行主控台模式下啟動工具

如果無法選擇在 Web UI 中執行，則應在指令行模式下啟動工具。只要從指令行執行啟動 Script 即可（isadc/isadc.bat 或 isadc/isadc.sh）。

檔案會寫入安裝目錄

依預設，isadc 安裝目錄會用來儲存在執行期間所建立的檔案。在某些系統上，isadc 安裝目錄是唯讀目錄。在此實例中，請使用 -useHome 參數（僅限主控台模式）。 此參數會使暫存檔寫入系統暫存目錄，而持續檔案則會寫入使用者起始目錄。附註：在解壓縮安裝 .zip/.tar.gz 檔之後， 在任何其他的使用者試圖使用 -useHome 參數執行收集之前，解壓縮該檔案的使用者必須先執行收集。這麼做將可確保在任何其他的非 admin/root 使用者執行之前，已完成適當的設定。

與工具互動

在 Web 瀏覽器中啟動 IBM Support Assistant Data Collector 工具之後，即會顯示圖形介面。 您必須先從主視窗中選取問題類型，才能使用此工具來執行資料收集及分析。 選取問題類型之後，下一步是按「啟動」按鈕。作業繼續執行時，收集 Script 會要求您提供完成其收集活動所需的任何其他資訊。Script 可能也會要求您提供配置資訊、會導致您目前所處理問題的事件順序相關資訊，或者是您希望如何完成收集的相關喜好設定。一旦它具備所有的必要資訊後，Script 會繼續處理後段的收集程序。

當此工具完成資料收集時，您可以將結果傳送給「IBM 支援中心」。您可以選擇使用未加密的 FTP 或加密的 HTTPS 來傳送檔案。收集狀態區中將會顯示此工具傳送給「IBM 支援中心」的 ZIP 檔名稱。

當 IBM Support Assistant Data Collector 在文字模式下執行時，不會提供選項清單或輸入欄位供使用者輸入。 可選的選項採編號清單呈現，請輸入您的選項編號，然後按 Enter 鍵。輸入欄位則轉變成提示，請輸入您的回應，然後按 Enter 鍵。當資料收集完成時，輸出為另一個 ZIP 檔。可在此機器上將輸出 ZIP 檔案傳送至「IBM 支援中心」，或在本端機器上檢查。

若要停止收集器工具，請在文字模式下鍵入 quit 選項，或是關閉瀏覽器視窗。

此工具會提示您輸入檔名。在 Unix 平台上，不支援使用 "~" 來指定使用者的 HOME 目錄。如果使用 "~"，則會參照目前工作目錄下名稱為 "~" 的子目錄。

判斷版本資訊

依預設，此工具（及其各種子元件）的版本會列印到在其中啟動此工具的主控台中。在 Web UI 下執行時，使用「說明 -> 關於」功能表選項，也可以找到版本資訊。 如果 Java 無法使用，或者 Java 應用程式無法啟動，則也可以藉由執行含有 "-version" 選項的啟動 Script，來取得此工具的版本。

*附註：使用 "-version" 選項時，啟動 Script 會印出版本資訊，但不會實際呼叫此工具。

檢視庫存檔

此工具可收集現行系統的庫存資訊。庫存資訊會自動置於收集 ZIP 檔中（如果有的話）。
輸出是庫存檔 (ISALite/inventory/inventory_rcf.xml)，最好的檢視方式是使用 Web 瀏覽器。

更新資料、限制及已知問題

此版本發佈時具有下列已知更新資料、限制及問題。IBM Support Assistant Support 網站上記載了 IBM Support Assistant Data Collector 的其他相關更新資料：http://www.ibm.com/software/support/isa。

只有 IBM JRE 隨附的硬體資產管理

當執行試圖對機器執行軟體/硬體資產管理的收集， 並使用非 IBM JRE 時，可能不會傳回硬體資訊。這是收集引擎的限制，可藉由使用 IBM JRE 來防止發生。

Linux on IBM System z (zLinux)

在 zLinux 上進行資料收集僅在主控台模式中受支援。若要啟動收集，應該在 Shell 內呼叫 isadc.sh Script。

安裝問題

Java 外掛程式配置

您想要在其內執行 IBM Support Assistant Data Collector 的瀏覽器必須配置 Java 1.6 外掛程式。您可以開啟下列鏈結 http://www.java.com/en/download/testjava.jsp 來驗證瀏覽器的配置是否適當。如果您是在 Linux 或 Unix 平台上執行瀏覽器，您可能必須手動安裝及配置 Java 外掛程式。
