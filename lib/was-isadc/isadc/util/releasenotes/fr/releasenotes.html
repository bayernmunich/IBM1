Description

L'outil IBM Support Assistant Data Collector collecte automatiquement les données des produits IBM. Il est préconfiguré pour localiser les données de diagnostic importantes sur votre ordinateur et les copier dans un fichier de collecteur. Le fichier journal généré par votre produit IBM et contenant l'historique détaillé des événements survenus durant le fonctionnement du produit est un exemple de donnée de diagnostic. Ce type de fichier peut s'avérer utile pour déterminer la nature et la cause d'un problème logiciel. D'autres exemples de données de diagnostic incluent les fichiers d'initialisation, les fichiers de configuration, la version du système d'exploitation, l'espace disque et les connexions réseau. L'outil peut être exécuté dans mode Interface graphique Web ou Console de ligne de commande. Le mode Console fournit un contrôle de ligne de commande des scripts de collecte BM Support Assistant Data Collector. L'outil inclut plusieurs fonctions qui vous assistent lorsque vous interagissez en mode console, notamment une fonction qui vous permet d'enregistrer vos réponses dans un fichier à partir d'une session en mode console, puis d'utiliser le fichier pour lancer des exécutions ultérieures du même script de collecte.

Accessibilité

IBM® s'efforce de fournir des produits accessibles à tous, quel que soit l'âge ou les capacités. L'interface IBM ISA Data Collector Console permet d'exécuter des opérations à partir du clavier uniquement et elle peut être également utilisée avec des applications de lecteur d'écran.

Guide d'initiation

Installation et utilisation de l'outil

Dans la plupart des cas, la séquence d'étapes suivante va vous permettre d'optimiser votre utilisation de l'outil. Si vous rencontrez des problèmes ou si vous avez besoin d'informations supplémentaires sur l'une des étapes, vous pouvez consulter les sections qui suivent la présente section.


  1. Extrayez l'outil dans le répertoire de votre choix. Reportez-vous aux sections ci-dessous pour obtenir des détails sur les modalités d'exécution des extractions.
  2. Exécutez l'outil en mode Interface graphique Web ou Console de ligne de commande. Suivez la procédure décrite ci-dessous pour définir la variable d'environnement JAVA_HOME. Après quoi, vous pouvez exécuter le script de lancement. Une fois que vous avez lancé l'outil, suivez les instructions ci-dessous pour interagir avec ce dernier au moment d'une collecte. Installation de l'outil

Dans tous les cas, l'installation de l'outil IBM Support Assistant Data Collector consiste simplement à extraire les fichiers du fichier .zip ou .tar.gz. Il est possible d'extraire les fichiers vers n'importe quel emplacement de système de fichiers que vous choisissez, sur le système où vous exécuterez l'outil. Un sous-répertoire ISALite est ainsi créé sous votre répertoire cible.

Démarrage de l'outil en mode Interface graphique Web

Pour le mode Interface graphique Web, ouvrez e fichier index.html dans le sous-répertoire d'installation isadc.

Mode Console de ligne de commande - Définition de la variable d'environnement JAVA_HOME

Pour le mode Ligne de commande, vous appelez le script de lancement approprié depuis une ligne de commande. Dans le cas d'un système Windows, ces scripts de lancement sont des fichiers de commandes. Pour les autres environnements, il s'agit de scripts de shell. L'outil étant mis en oeuvre en tant qu'application Java, il est nécessaire de localiser Java pour que l'outil de ligne de commande puisse démarrer. Si Java n'est pas disponible dans la variable PATH, vous devez définir la variable d'environnement JAVA_HOME manuellement. L'outil IBM Support Assistant Data Collector requiert un environnement JRE de niveau 1.5 ou supérieur ; vous devez donc vous assurer, dans un premier temps, qu'un environnement JRE adéquat est installé sur le système où l'outil sera exécuté. Si tel est le cas, vous devez émettre une commande propre au système d'exploitation pour définir la variable JAVA_HOME de façon à ce qu'elle pointe vers cet environnement JRE. Microsoft JVM/JDK et gij (GNU libgcj) ne sont pas pris en charge.

Par exemple, s'il s'agit d'une plateforme Windows avec jre1.5 installé dans c:\jre1.5, vous devez configurer la variable JAVA_HOME avec la commande suivante :

SET JAVA_HOME=c:\jre1.5
REMARQUE : n'utilisez pas de guillemets dans la valeur de la commande SET, même si votre valeur comporte des espaces vides.

Sur une plateforme Linux, AIX ou Solaris, si JRE est installé dans l'emplacement /opt/jre15, vous devez configurer la variable JAVA_HOME avec la commande suivante :

export JAVA_HOME=/opt/jre15

Démarrage de l'outil en mode console de ligne de commande

Si vous ne pouvez pas exécuter l'outil en mode Interface graphique Web, l'outil doit être démarré en mode Ligne de commande. Exécutez simplement le script de lancement depuis la ligne de commande (isadc/isadc.bat ou isadc/isadc.sh).

Les fichiers sont écrits dans le répertoire d'installation.

Par défaut, le répertoire d'installation isadc est utilisé pour enregistrer les fichiers créés lors de l'exécution. ur certains systèmes, ce répertoire est en lecture seule. Dans ce cas, utilisez le paramètre -useHome (mode Console uniquement). Ce paramètre permet d'écrire les fichiers temporaires dans le répertoire temporaire du système et les fichiers permanents dans le répertoire de base de l'utilisateur. REMARQUE : après avoir extrait le fichier d'installation .zip/.tar.gz, l'utilisateur qui a extrait le fichier doit exécuter une collecte avant qu'un autre utilisateur en exécute une en utilisant le paramètre -useHome. Ainsi, vous pouvez vérifier que l'installation est correcte avant toute exécution par un utilisateur non-administrateur ou non root.

Interaction avec l'outil

Une fois que l'outil IBM Support Assistant Data Collector a été démarré dans un navigateur Web, l'interface graphique s'affiche. Pour pouvoir utiliser l'outil pour collecter et analyser des données, vous devez sélectionner un type de problème dans la fenêtre principale. Une fois le type de problème sélectionné, cliquez sur le bouton Démarrer. Ensuite, le script de collecte va vous demander les autres informations dont il a besoin pour achever ses activités de collecte. Un script peut également vous demander des informations de configuration, des informations sur la séquence d'événements à l'origine de l'incident que vous gérez ou vos préférences concernant la façon de terminer la collecte. Une fois toutes les informations réunies, le script poursuit la collecte.

Lorsque l'outil a terminé la collecte des données, vous pouvez envoyer les résultats au support IBM. Pour le transfert de fichiers, vous pouvez choisir entre FTP, qui n'est pas chiffré, et HTTPS, qui est chiffré. Le nom du fichier compressé que l'outil envoie au support IBM s'affiche dans la zone d'état de la collecte.

Lorsque l'outil IBM Support Assistant Data Collector s'exécute en mode Texte, il ne comporte aucune liste de sélection ou zone de saisie destinée à l'utilisateur. Les choix disponibles sont présentés sous la forme de listes numérotées ; vous entrez le numéro de votre sélection et vous appuyez sur la touche Entrée. Les zones de saisie sont transformées en invites, dans lesquelles vous entrez votre réponse avant d'appuyer sur la touche Entrée. Une fois la collecte des données terminée, la sortie correspond à un autre fichier ZIP. Ce fichier peut être envoyé au support IBM ou examiné localement.

Pour arrêter l'outil de collecte, entrez quit en mode texte ou fermez la fenêtre du navigateur.

L'outil demande des noms de fichier. Sur les plateformes Unix, l'utilisation de "~" pour désigner le répertoire HOME de l'utilisateur n'est pas prise en charge. Si "~" est utilisé, un sous-répertoire dans le répertoire de travail actuel avec le nom "~" sera référencé.

Détermination des informations de version

Par défaut, la version de l'outil (et ses différents sous-composants) est affichée sur la console à partir de laquelle il a été lancé. Lors d'une exécution en mode Interface graphique Web, les informations de version sont également disponibles en utilisant l'option de menu Aide->A propos de. Si Java n'est pas disponible ou que l'application Java ne peut pas démarrer, vous pouvez également obtenir la version de l'outil en exécutant les scripts de lancement avec l'option "-version".

*REMARQUE : lorsque vous utilisez l'option "-version", les scripts de lancement afficheront les informations de version sans appeler l'outil.

Affichage du fichier d'inventaire

L'outil peut collecter les informations d'inventaire du système en cours. Ces informations sont placées automatiquement dans le fichier ZIP de collecte, s'il est disponible
La sortie est un fichier d'inventaire (ISALite/inventory/inventory_rcf.xml) dont l'affichage est optimisé dans un navigateur Web.

Mises à jour, limitations et incidents recensés

Voici les mises à jour, limitations et incidents recensés au moment de la publication. Des mises à jours supplémentaires d'IBM Support Assistant Data Collector sont décrites sur le site de support Web IBM Support Assistant : http://www.ibm.com/software/support/isa.

Inventaire matériel disponible avec IBM JRE uniquement

Lorsque vous exécutez une collecte pour établir un inventaire logiciel/matériel sur un ordinateur et qu'un JRE non-IBM est utilisé, les informations sur le matériel peuvent ne pas être retournées. Il s'agit d'une limitation du moteur de collecte qui peut être résolue en utilisant un JRE IBM.

Linux sur IBM System z (zLinux)

La collecte des données sur zLinux fonctionne uniquement en mode Console. Pour lancer une collecte, appelez le script isadc.sh dans un shell.

Incidents liés à l'installation

Configuration de plug-in Java

Le navigateur dans lequel vous voulez exécuter IBM Support Assistant Data Collector doit être configuré avec un plug-in Java 1.6. Vous pouvez vérifier que le navigateur est correctement configuré en ouvrant le lien http://www.java.com/en/download/testjava.jsp. Si vous exécutez le navigateur sur une plateforme Linux ou Unix, il peut être nécessaire d'installer et de configurer manuellement le plug-in Java.
