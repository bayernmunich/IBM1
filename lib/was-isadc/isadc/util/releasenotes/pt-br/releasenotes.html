Descrição

O IBM Support Assistant Data Collector Tool fornece uma coleção de dados automática para produtos IBM. A ferramenta é pré-configurada para localizar dados diagnósticos importantes no seu sistema de computador e copiá-lo em um arquivo do Coletor. Um exemplo de dados de diagnóstico é um arquivo de log gerado por seu produto IBM que contém um histórico detalhado de eventos que ocorrem durante o funcionamento do produto. Esse arquivo pode ser útil na determinação da natureza e causa de um problema de software. Outros exemplos dos dados de diagnósticos incluem arquivos de inicialização, arquivos de configuração, versão do sistema operacional, espaço em disco e conexões de rede. A ferramenta pode ser executada no modo de UI da Web ou em um modo de console da linha de comandos. O modo do console fornece controle de linha de comandos dos scripts de coleção IBM Support Assistant Data Collector. A ferramenta inclui vários recursos que irão ajudá-lo ao interagir com ela no modo de console, incluindo um que permite a gravação das respostas de uma sessão no modo de console em um arquivo e, em seguida, o uso do arquivo para direcionar execuções subsequentes do mesmo script de coleta.

Acessibilidade

A IBM® se esforça em fornecer produtos com acesso utilizável para todos, independente da idade ou capacidade. A interface IBM ISA Data Collector Console suporta operação apenas de teclado e também pode ser usada com aplicativos de leitura de tela.

Introdução

Instalação da Ferramenta e Uso

Na maioria dos casos, a sequência de etapas a seguir o ajudará a instalar e executar a ferramenta. Se encontrar problemas, ou se quiser mais informações sobre alguma das etapas, consulte as seções subsequentes a esta.


  1. Extraia a ferramenta em qualquer diretório escolhido. Consulte as seções a seguir para obter detalhes de como executar as extrações.
  2. Execute a ferramenta no modo de UI da Web ou modo do console da linha de comandos. Siga o procedimento descrito abaixo para configurar a variável de ambiente JAVA_HOME. Depois de fazer isso, é possível executar o script de ativação. Depois de iniciar a ferramenta, siga as instruções abaixo para interagir com ela durante a execução de uma coleta. Instalando a Ferramenta

Em todos os casos, a instalação da ferramenta IBM Support Assistant Data Collector é simplesmente uma questão de extrair os arquivos do arquivo .zip ou arquivo .tar.gz arquivados. Os arquivos podem ser extraídos em qualquer local do sistema de arquivos escolhido no sistema em que a ferramenta será executada. Isso criará um subdiretório ISALite sob seu diretório de destino.

Iniciando a Ferramenta no Modo de UI da Web

Para o modo de UI da Web, abra o arquivo index.html localizado no subdiretório isadc.

Modo do Console da Linha de Comandos - Configurando a Variável de Ambiente JAVA_HOME

Para o modo de linha de comando, chame o script de ativação apropriado a partir da linha de comandos. No caso de sistemas Windows, esses scripts de ativação serão arquivos em lote. Para outros ambientes, eles serão shell scripts. Como a ferramenta é implementada como um aplicativo Java, é necessário que o Java possa ser localizado antes que a ferramenta da linha de comandos possa ser iniciada. Se Java não estiver disponível no PATH, a variável de ambiente JAVA_HOME precisará ser configurada manualmente. A ferramenta IBM Support Assistant Data Collector requer um JRE no nível 1.5 ou superior; portanto, primeiro você deve assegurar-se de que um JRE adequado esteja instalado no sistema no qual a ferramenta estará sendo executado. Se houver, será necessário emitir um comando específico do sistema operacional para configurar a variável JAVA_HOME para que aponte para esse JRE. O Microsoft JVM/JDK e gij (GNU libgcj) não são suportados.

Por exemplo, se em uma plataforma Windows você tiver o jre1.5 instalado em c:\jre1.5, você configuraria JAVA_HOME usando o comando a seguir:

SET JAVA_HOME=c:\jre1.5
NOTA: Não utilize aspas no valor do comando SET, mesmo que seu valor contenha espaços em branco.

Em um Linux, AIX ou Solaris, se você tiver o JRE instalado no /opt/jre15, você configuraria JAVA_HOME usando o comando a seguir:

export JAVA_HOME=/opt/jre15

Iniciando a Ferramenta no Modo de Console da Linha de Comandos

Se executar na UI da Web não for uma opção, a ferramenta deve ser iniciada no modo de linha de comandos. Simplesmente execute o script de ativação a partir da linha de comandos (isadc/isadc.bat ou isadc/isadc.sh).

Os arquivos são gravados no diretório de instalação

Por padrão, o diretório de instalação isadc é usado para armazenar os arquivos criados durante a execução. Em alguns sistemas, o diretório de instalação isadc será somente leitura. Nessa instância, use o parâmetro -useHome (apenas modo do console). Esse parâmetro fará com que os arquivos temporários sejam gravados no diretório temporário systems e os arquivos persistentes gravados no diretório inicial do usuário. NOTA: Depois de extrair o arquivo .zip/.tar.gz de instalação, o usuário que extraiu o arquivo deve executar uma coleção antes de qualquer outra tentativa de o usuário executar uma coleção usando o parâmetro -useHome. Fazer isso assegurará que a configuração adequada tenha sido feita antes da execução por qualquer outro usuário não admin/raiz.

Interagindo com a Ferramenta

Depois que a ferramenta IBM Support Assistant Data Collector tiver sido iniciada em um navegador da Web, a interface gráfica será exibida. Antes que a ferramenta possa ser usada para executar a coleção de dados e a análise, primeiro você deve selecionar um tipo de problema a partir da janela principal. Com o tipo de problema selecionado, a próxima etapa é pressionar o botão Iniciar. Na sequência, o script de coleta solicitará informações adicionais necessárias para concluir suas atividades de coleta. Um script também pode solicitar informações de configuração, informações sobre a sequência de eventos que levaram ao problema com o qual você está lidando ou preferências relacionadas a como ele deve concluir a coleta. Quando tiver todas as informações de que precisa, o script prosseguirá com o restante da coleta.

Quando a ferramenta concluir a coleta de dados, os resultados poderão ser enviados para o Suporte IBM. É possível escolher entre FTP, que não é criptografado, e HTTPS, que é criptografado, para a transferência de arquivos. O nome do arquivo zip que a ferramenta envia para o Suporte IBM será exibido na área de status de coleção.

Quando o IBM Support Assistant Data Collector é executado no modo de texto, não há listas de seleção ou campos de entrada para entrada do usuário. Em vez disso, as opções disponíveis são apresentadas como listas numeradas e você digita o número de sua seleção seguido pela tecla Enter. Os campos de entrada são transformados em prompts, nos quais você digita sua resposta e pressiona Enter. Quando a coleção de dados é concluída, a saída é outro arquivo ZIP. O arquivo ZIP de saída pode ser enviado para Suporte IBM ou examinado localmente.

Para parar a ferramenta coletora, digite a opção de saída no modo de texto ou feche a janela do navegador.

A ferramenta solicitará nomes de arquivos. Nas plataformas Unix, usar "~" como uma designação do diretório HOME do usuário não é suportado. Se um "~" for usado, um subdiretório sob o diretório ativo presente pelo nome de "~" será referenciado.

Determinando Informações de Versão

Por padrão, a versão da ferramenta (e seus vários subcomponentes) é impressa no console a partir do qual ela foi ativada. Ao executar na UI da web, as informações de versão também podem ser localizadas usando a opção de menu Ajuda->Sobre. Se Java não estiver disponível, ou o aplicativo Java não puder ser iniciado, a versão da ferramenta também poderá ser obtida executando os scripts de ativação com a opção "-version".

*NOTA: Ao executar a opção "-version", os scripts de ativação imprimirão as informações de versão sem realmente chamar a ferramenta.

Visualizando o Arquivo de Inventário

A ferramenta pode coletar informações sobre o inventário do sistema atual. As informações sobre o inventário são automaticamente colocadas no arquivo ZIP de coleta, se disponíveis.
A saída é um arquivo de inventário (ISALite/inventory/inventory_rcf.xml), que é melhor visualizado com um navegador da web.

Atualizações, Limitações e Problemas Conhecidos

As seguintes atualizações, limitações e problemas foram conhecidos no momento deste release. As atualizações adicionais sobre o IBM Support Assistant Data Collector são documentadas no website do IBM Support Assistant Support em: http://www.ibm.com/software/support/isa.

Inventário de Hardware Disponível com IBM JRE Apenas

Ao executar uma coleta que tenta executar o inventário de software/hardware em uma máquina e um JRE não IBM é usado, as informações de hardware podem não ser retornadas. Essa é uma limitação do mecanismo de coleta e pode ser contornada usando um IBM JRE.

Linux no IBM System z (zLinux)

A coleção de dados no zLinux é suportada apenas no modo do console. Para iniciar uma coleção, o script isadc.sh deve ser chamado em um shell.

Problemas de Instalação

Configuração de Plug-in Java

O navegador no qual você deseja executar o IBM Support Assistant Data Collector deve estar configurado com um Plug-in Java 1.6. Você pode validar que seu navegador está corretamente configurado abrindo o link a seguir http://www.java.com/en/download/testjava.jsp. Se você estiver executando o navegador em uma plataforma Linux ou Unix, pode ser necessário instalar e configurar manualmente o Plug-in Java.
