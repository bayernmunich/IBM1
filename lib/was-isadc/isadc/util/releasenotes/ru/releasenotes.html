Описание

Программа IBM Support Assistant Data Collector обеспечивает автоматический сбор данных о продуктах IBM. Эта программа предназначена для поиска важных диагностических данных в системе, которые копируются в файл инструмента сбора данных. В качестве примера диагностической информации можно привести созданный продуктом IBM файл протокола, содержащий подробную хронологию событий, возникающих во время работы продукта. С помощью этого файла можно определить характер и причину неполадки программы. Другие примеры диагностической информации: файлы инициализации, файлы конфигурации, версия операционной системы, объем дисковой памяти и сетевые соединения. В режиме консоли можно управлять сценариями сбора данных IBM Support Assistant Data Collector с помощью командной строки. Инструмент содержит несколько средств, помогающих взаимодействовать с ним в режиме консоли, включая функцию, которая позволяет регистрировать ответы из сеанса режима консоли в файле и затем использовать этот файл для управления последующими действиями того же сценария сбора данных.

Специальные возможности

Компания IBM® стремится предоставлять продукты, которые может использовать кто угодно вне зависимости от возраста и физических возможностей. Интерфейс консоли IBM ISA Data Collector поддерживает только ввод с клавиатуры и совместим с программами чтения с экрана.

Установка и использование программы

В большинстве случаев следующая последовательность шагов позволит вам приступить к работе с программой. В случае затруднений или для получения дополнительной информации о любом из этих шагов обратитесь к последующим разделам.


  1. Извлеките инструмент в любой каталог. Подробная информация об извлечении содержится в следующих разделах.
  2. Запустите программу в режиме веб-интерфейса или режиме консоли с командной строкой. Для установки переменной среды JAVA_HOME следуйте описанной процедуре. После этого можно выполнить сценарий запуска. После запуска инструмента следуйте приведенным инструкциям для взаимодействия с инструментом в процессе сбора данных. Установка программы

Для установки программы IBM Support Assistant Data Collector достаточно извлечь файлы из архивного файла .zip или .tar.gz. Файлы могут быть извлечены в любое место файловой системы, выбранное вами для запуска инструмента. В указанном каталоге создается подкаталог ISALite.

Запуск программы в режиме веб-интерфейса

Для применения графического веб-интерфейса откройте файл index.html, расположенный в подкаталоге isadc.

Режим консоли с командной строкой - настройка переменной среды JAVA_HOME

В режиме командной строки необходимо вызвать сценарий запуска из командной строки. В случае системы Windows эти сценарии запуска представляют собой пакетные файлы. В других средах они являются сценариями командного процессора. Поскольку программа реализована как приложение на Java, для ее запуска в режиме командной строки должно быть известно расположение Java. Если Java в PATH отсутствует, то придется настроить переменную среды JAVA_HOME вручную. Для программы IBM Support Assistant Data Collector необходима среда JRE версии 1.5 или более поздней, поэтому перед запуском программы убедитесь в наличии совместимой среды JRE. Если она установлена, необходимо с помощью команды операционной системы настроить переменную JAVA_HOME так, чтобы она указывала на эту JRE. Microsoft JVM/JDK и gij (GNU libgcj) не поддерживаются.

Например, если на платформе Windows среда jre1.5 установлена в каталоге c:\jre1.5, то для настройки JAVA_HOME применяется следующая команда:

SET JAVA_HOME=c:\jre1.5
Примечание: Значение команды SET не следует заключать в кавычки даже в том случае, если оно содержит пробелы.

Если в Linux, AIX или Solaris среда JRE установлена в каталоге /opt/jre15, то для настройки JAVA_HOME применяется следующая команда:

export JAVA_HOME=/opt/jre15

Запуск программы в режиме командной строки

Если запуск в режиме веб-интерфейса невозможен, программу можно запустить в режиме командной строки. Для этого вызовите сценарий запуска из командной строки (isadc/isadc.bat или isadc/isadc.sh).

Запись файлов в каталог установки

По умолчанию каталог установки isadc используется для хранения файлов, созданных во время выполнения. В ряде систем каталог установки isadc доступен только для чтения. В этом случае следует использовать параметр -useHome (в режиме консоли). При этом временные файлы будут записываться в системный каталог временных файлов, а остальные - в домашний каталог пользователя. ПРИМЕЧАНИЕ: первую операцию сбора данных после распаковки установочного файла .zip или .tar.gz должен выполнить тот пользователь, который распаковывал архив, прежде чем другой пользователь попытается выполнить сбор данных с помощью параметра -useHome. Это обеспечит выполнение правильной настройки перед запуском сбора данных пользователем, не имеющим прав администратора или root.

Взаимодействие с программой

После запуска программы IBM Support Assistant Data Collector в веб-браузере будет показан графический интерфейс программы. Перед тем как приступать к сбору и анализу данных с помощью программы, необходимо выбрать тип неполадки в главном окне. После выбора типа неполадки необходимо нажать кнопку Запустить. При выполнении сценария сбора данных может быть запрошена дополнительная информация. Например, сценарий может запросить имя установочного каталога продукта, дополнительные сведения о конфигурации, информацию о последовательности событий, вызывающих неполадку, или ваши предпочтения относительно сбора данных. После сбора всей нужной информации сценарий выполняет заключительную часть операции.

После завершения сбора данных можно отправить результаты в службу IBM Support. Можно выбрать передачу данных по FTP (незашифрованное соединение), либо по HTTPS (зашифрованное соединение). Имя файла zip, отправляемого в IBM Support, показано в области состояния сбора данных.

Когда программа IBM Support Assistant Data Collector запущена в текстовом режиме, списки вариантов и поля для ввода данных пользователем отсутствуют. Доступные опции представлены в виде пронумерованных списков, в которых требуется ввести номер нужной опции и нажать клавишу Enter. Вместо полей ввода применяются приглашения, в которых требуется ввести ответ и нажать клавишу Enter. После завершения сбора данных вывод сохраняется в другом файле zip. Файл zip с данными вывода можно отправить в службу IBM Support или просмотреть в локальной системе.

Для того чтобы завершить работу инструмента сбора данных, введите quit в текстовом режиме или закройте окно браузера.

Инструмент запросит имена файлов. На платформах Unix не поддерживается использование символа "~" в качестве обозначения домашнего каталога пользователя. Символ "~" обозначает подкаталог текущего рабочего каталога с именем "~".

Определение версии

По умолчанию версия инструмента и его компонентов выводится на консоль, из которой был запущен данный инструмент. В веб-интерфейсе версию можно просмотреть, выбрав в меню пункт Справка->О программе. Если Java недоступна, либо не удается запустить приложение на Java, то версию программы можно узнать, вызвав сценарий запуска с параметром "-version".

*ПРИМЕЧАНИЕ: при использовании параметра "-version" сценарий запуска показывает информацию о версии, не запуская программу.

Просмотр файла реестра

Эта программа может собрать информацию о реестре текущей системы. Информация о реестре автоматически добавляется в файл zip собранных данных, если он доступен.
На выходе формируется файл реестра (ISALite/inventory/inventory_rcf.xml), который удобнее всего просматривать в веб-браузере.

Обновления, ограничения и известные неполадки

Ниже перечислены обновления, неполадки и ограничения, известные на момент выхода этого выпуска. Новая информация о программе IBM Support Assistant Data Collector публикуется на веб-сайте поддержки IBM Support Assistant: http://www.ibm.com/software/support/isa.

Реестр аппаратного обеспечения доступен только при использовании IBM JRE

Если во время сбора данных делается попытка создать реестр программного и аппаратного обеспечения системы и применяется среда JRE, предоставленная не IBM, а другим поставщиком, то информация об аппаратном обеспечении может быть не собрана. Это ограничение инструмента сбора данных, которое можно обойти, установив IBM JRE.

Linux в IBM System z (zLinux)

Сбор данных в zLinux возможен только в режиме консоли. Для запуска сбора данных необходимо вызвать сценарий isadc.sh в оболочке.

Неполадки, связанные с установкой

Конфигурация встраиваемого модуля Java

Браузер, применяемый для запуска программы IBM Support Assistant Data Collector, должен содержать встраиваемый модуль Java 1.6. Для проверки правильности конфигурации браузера откройте в нем следующую ссылку: http://www.java.com/en/download/testjava.jsp. Если браузер установлен на платформе Linux или Unix, вам может потребоваться установить и настроить модуль Java вручную.
