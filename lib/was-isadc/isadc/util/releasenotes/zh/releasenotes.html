描述

IBM Support Assistant Data Collector 工具提供针对 IBM 产品的自动数据收集。该工具已预先配置为在您的计算机系统上查找重要诊断数据并将其复制到收集器文件。由 IBM 产品生成的日志文件就是诊断数据的一个示例，该日志文件中包含在执行产品操作期间发生的事件的详细历史记录。此类文件在确定软件问题的性质和原因时可能有用。诊断数据的其他示例包括初始化文件、配置文件、操作系统版本、磁盘空间和网络连接。该工具可在 Web 用户界面方式或命令行控制台方式下运行。控制台方式为您提供了通过命令行来控制 IBM Support Assistant Data Collector 收集脚本的方式。该工具包括若干功能部件，可协助您在控制台方式下与其交互，其中一个功能部件允许您将来自控制台方式会话的响应记录到文件中，然后使用该文件来驱动同一收集脚本的后续执行。

入门

工具安装和使用

在大多数情况下，执行以下一系列步骤将启动并运行该工具。如果遇到问题，或者您希望了解关于任何这些步骤的更多信息，那么可以参阅此章节后面的以下部分。


  1. 将该工具解压缩到您选择的任何目录。要了解如何执行解压缩的详细信息，请参阅以下章节。
  2. 以 Web 用户界面方式或命令行控制台方式运行该工具。请按照以下描述的过程设置 JAVA_HOME 环境变量。完成此过程之后，就可以开始执行启动脚本。 一旦启动该工具之后，遵循以下指示信息以在其执行收集时与其交互。安装该工具

在任何情况下，安装 IBM Support Assistant Data Collector 工具都只需要将文件从归档的 .zip 文件或 .tar.gz 文件中解压缩出来即可。可以将这些文件解压缩到您在将运行此工具的系统上选择的任何文件系统位置。 这将在目标目录下创建 ISALite 子目录。

以 Web 用户界面方式启动该工具

对于 Web 用户界面方式，打开位于 isadc 子目录的 index.html 文件。

命令行控制台方式 - 设置 JAVA_HOME 环境变量

对于命令行方式，您从命令行调用相应的启动脚本。在 Windows 系统中，这些启动脚本是批处理文件。 而在其他环境中，这些启动脚本是 shell 脚本。因为该工具作为 Java 应用程序来实现，所以此工具必须在在启动命令行工具之前可以找到 Java。如果 PATH 中未提供 Java，那么必须手动设置 JAVA_HOME 环境变量。 IBM Support Assistant Data Collector 工具需要 1.5 或更高级别的 JRE，因此必须确保在将运行该工具的系统上已安装适合的 JRE。如果已满足此条件，那么您将需要发出一个特定于操作系统的命令将 JAVA_HOME 变量设置为指向此 JRE。 Microsoft JVM/JDK 和 gij (GNU libgcj) 不受支持。

例如，如果您在 Windows 平台的 c:\jre1.5 位置安装了 jre1.5，那么应使用以下命令来设置 JAVA_HOME：

SET JAVA_HOME=c:\jre1.5
注意：请不要在 SET 命令的值中使用引号，即使您的值具有空格字符。

在 Linux、AIX 或 Solaris 上，如果您已在 /opt/jre15 位置安装了 JRE，那么应使用以下命令来设置 JAVA_HOME：

export JAVA_HOME=/opt/jre15

以命令行控制台方式启动该工具

如果未提供在 Web 用户界面中运行的选项，那么应以命令行方式启动该工具。只需从命令行执行启动脚本（isadc/isadc.bat 或 isadc/isadc.sh）。

文件将被写入安装目录

缺省情况下，isadc 安装目录用来存储在执行期间创建的文件。在某些系统上，isadc 安装目录将为只读。在这种情况下，请使用 -useHome 参数（仅适用于控制台方式）。该参数会将临时文件写入系统临时目录，而将持久文件写入用户主目录。注意：在解压缩安装 .zip/.tar.gz 文件操作之后，在任何其他用户尝试使用 -useHome 参数运行收集之前，解压缩该文件的用户必须运行收集。这将确保在任何其他非管理员/root 用户执行之前，已进行过适当设置。

与该工具交互

一旦在 Web 浏览器中启动 IBM Support Assistant Data Collector 之后，将显示图形界面。必须先从主窗口中选择问题类型，然后才可将该工具用于执行数据收集和分析。选择问题类型之后，下一个步骤是按“启动”按钮。随着收集过程的进行，收集脚本将要求您提供它完成收集活动所需的更多信息。 某个脚本可能还会要求您提供配置信息、有关导致您正在处理的问题的事件序列的信息，或者提供与其如何完成收集有关的首选项。一旦它获得了它需要的所有信息，脚本就会继续完成收集活动的其余任务。

当该工具完成数据收集时，可将结果发送给 IBM 支持机构。您可以选择使用 FTP（不加密），也可以选择 HTTPS（加密）进行文件传输。 该工具发送给 IBM 支持机构的 zip 文件的名称将显示在收集状态区域中。

当 IBM Support Assistant Data Collector 以文本方式运行时，没有用于用户输入的选择列表或输入字段。可用的选项是以编号列表形式提供的，在输入您所作选择的编号之后按 Enter 键。 输入字段将转换为提示，此时您需要在输入字段中输入回答并按 Enter 键。 数据收集完成时，输出为另一个 ZIP 文件。可将输出 ZIP 文件发送给 IBM 支持机构或在本地进行检查。

要停止收集器工具，请在控制台方式下输入退出选项或关闭浏览器窗口。

该工具会提示输入文件名。在 Unix 平台上，不支持将 "~" 指定为用户的主目录。如果使用了 "~"，那么将引用当前工作目录下名为 "~" 的子目录。

确定版本信息

缺省情况下，该工具的版本（及其各种子组件）会在启动它的控制台上显示。当在 Web 用户界面中运行时，也可通过使用“帮助->关于”菜单选项找到版本信息。如果 Java 不可用，或者 Java 应用程序无法启动，那么也可以通过使用 "-version" 选项来运行启动脚本以获取该工具的版本。

*注意：使用“-version”选项时，启动脚本将显示版本信息，但不实际调用该工具。

查看库存文件

该工具可以收集当前系统的库存信息。库存信息（如可用）将自动放置在收集 ZIP 文件中。
输出是一个库存文件 (ISALite/inventory/inventory_rcf.xml)，最适合在 Web 浏览器中查看。

更新、局限性和已知问题

发布此发行版时，存在下列已知更新、局限性和问题。在 IBM Support Assistant 支持 Web 站点上提供了有关 IBM Support Assistant Data Collector 的其他更新，网址为：http://www.ibm.com/software/support/isa。

硬件库存仅在使用 IBM JRE 时可用

当在使用非 IBM JRE 的机器上运行尝试执行软件/硬件库存的收集操作时，可能不会返回硬件信息。这是收集机器的一项限制，可通过使用 IBM JRE 来避免。

Linux on IBM System z (zLinux)

仅支持以控制台方式在 zLinux 上进行数据收集。要启动收集，应在 shell 内部调用 isadc.sh 脚本。
