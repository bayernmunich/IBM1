Beschreibung

Das Tool IBM Support Assistant Data Collector unterstützt die automatische Datenerfassung für IBM Produkte. Das Tool ist so vorkonfiguriert, dass es wichtige Diagnosedaten auf Ihrem Computersystem aufspürt und in eine Collectordatei kopiert. Ein Beispiel von Diagnosedaten ist eine Protokolldatei, die Ihr IBM Produkt generiert hat und ein detailliertes Verlaufsprotokoll von Ereignissen enthält, die während des Betriebs der Software eintreten. Eine solche Datei kann hilfreich sein, um den Charakter und die Ursache eines Softwareproblems zu bestimmen. Initialisierungsdateien, Konfigurationsdateien, die Betriebssystemversion, der Plattenspeicherplatz und die Netzverbindungen sind weitere Beispiele für derartige Diagnosedaten. Das Tool kann im Webbenutzerschnittstellenmodus oder im Befehlszeilenkonsolmodus ausgeführt werden. Der Konsolmodus ermöglicht Ihnen die Steuerung der Erfassungsscripts von IBM Support Assistant Data Collector über die Befehlszeile. Das Tool umfasst verschiedene Funktionen, die Sie unterstützen, wenn Sie mit dem Tool im Konsolmodus interagieren. Dazu gehört auch eine Funktion zur Aufzeichnung Ihrer Antworten aus einer Konsolmodussitzung in einer Datei, die Sie anschließend zur Steuerung nachfolgender Ausführungen desselben Erfassungsscripts verwenden können.

Erste Schritte

Installation und Verwendung des Tools

In den meisten Fällen können Sie das Tool in den folgenden Schritten installieren und einsatzbereit machen. Wenn Probleme auftreten oder weitere Informationen zu einem dieser Schritte gewünscht werden, können Sie die Abschnitte lesen, die auf diesen Abschnitt folgen.


  1. Extrahieren Sie das Tool in ein beliebiges Verzeichnis, das Sie auswählen. Die nachfolgenden Abschnitte enthalten detaillierte Informationen dazu, wie die Extraktion ausgeführt wird.
  2. Führen Sie das Tool entweder im Webbenutzerschnittstellenmodus oder im Befehlszeilenkonsolmodus aus. Führen Sie die unten beschriebene Prozedur aus, um die Umgebungsvariable JAVA_HOME festzulegen. Anschließend können Sie das Startscript ausführen. Folgen Sie, nachdem das Tool gestartet worden ist, den unten beschriebenen Anweisungen zur Interaktion mit dem Tool während der Ausführung einer Erfassung. Installieren Sie das Tool wie folgt:

In allen Fällen besteht die Installation des Tools IBM Support Assistant Data Collector einfach darin, die Dateien aus der archivierten Datei mit der Erweiterung .zip oder .tar.gz zu extrahieren. Die Dateien können an eine beliebige Dateisystemposition extrahiert werden, die Sie auf dem System auswählen, auf dem das Tool ausgeführt werden soll. Durch diesen Vorgang wird ein Unterverzeichnis 'ISALite' unter Ihrem Zielverzeichnis erstellt.

Tool im Webbenutzerschnittstellenmodus starten

Im Webbenutzerschnittstellenmodus öffnen Sie die Datei index.html im Unterverzeichnis isadc.

Befehlszeilenkonsolmodus - Umgebungsvariable JAVA_HOME festlegen

Im Befehlszeilenmodus rufen Sie das entsprechende Startscript in einer Befehlszeile auf. Bei einem Windows-System sind diese Startscripts Batchdateien. In anderen Umgebungen sind sie Shell-Scripts. Da das Tool als Java-Anwendung implementiert ist, muss Java auffindbar sein, bevor das Befehlszeilentool gestartet werden kann. Wenn Java in der Variablen PATH nicht verfügbar ist, müssen Sie die Umgebungsvariable JAVA_HOME manuell festlegen. Das Tool IBM Support Assistant Data Collector erfordert Java Runtime Environment (JRE) 1.5 oder eine höhere Version. Daher müssen Sie zunächst sicherstellen, dass eine entsprechende JRE auf dem System installiert ist, auf dem das Tool ausgeführt werden soll. Ist eine solche JRE vorhanden, müssen Sie mithilfe eines betriebssystemspezifischen Befehls die Variable JAVA_HOME so definieren, dass sie auf diese JRE verweist. Die JVM/JDK von Microsoft sowie gij ('GNU libgcj') werden nicht unterstützt.

Wenn Sie auf einer Windows-Plattform jre1.5 beispielsweise in c:\jre1.5 installiert haben, müssen Sie JAVA_HOME mit dem folgenden Befehl setzen:

SET JAVA_HOME=c:\jre1.5
HINWEIS: Geben Sie keine Anführungszeichen im Wert für den Befehl SET an, auch wenn der Wert Leerzeichen enthält.

Wenn Sie die JRE unter Linux, AIX oder Solaris in /opt/jre15 installiert haben, müssen Sie JAVA_HOME mit dem folgenden Befehl setzen:

export JAVA_HOME=/opt/jre15

Tool im Befehlszeilenkonsolmodus starten

Wenn die Ausführung in der Webbenutzerschnittstelle keine Option ist, muss das Tool im Befehlszeilenmodus gestartet werden. Führen Sie einfach das Startscript über die Befehlszeile aus (isadc/isadc.bat bzw. isadc/isadc.sh).

Dateien werden in das Installationsverzeichnis geschrieben

Standardmäßig wird das Installationsverzeichnis isadc zum Speichern von Dateien verwendet, die während der Ausführung erstellt wurden. Auf manchen Systemen ist das Installationsverzeichnis isadc schreibgeschützt. In diesem Fall müssen Sie den Parameter '-useHome' verwenden (nur im Konsolmodus). Dieser Parameter bewirkt, dass temporäre Dateien in das temporäre Verzeichnis des Systems und als persistent definierte Dateien in das Ausgangsverzeichnis des Benutzers geschrieben werden. Anmerkung: Nachdem Sie die Installationsdatei mit der Erweiterung .zip/.tar.gz entpackt haben, muss der Benutzer, der die entpackt hat, eine Erfassung mit dem Parameter -useHome durchführen, bevor andere Benutzer versuchen, eine Erfassung durchzuführen. Auf diese Weise wird sichergestellt, dass die Umgebung ordnungsgemäß konfiguriert ist, bevor Erfassungen von anderen Benutzern ohne Administrator- bzw. Rootrechte durchgeführt werden.

Interaktion mit dem Tool

Nachdem Sie das Tool IBM Support Assistant Data Collector in einem Web-Browser gestartet haben, wird die grafische Schnittstelle angezeigt. Damit das Tool zum Ausführen einer Datenerfassung und -analyse verwendet werden kann, müssen Sie zunächst im Hauptfenster einen Problemtyp auswählen. Nach der Auswahl des Problemtyps klicken Sie auf die Schaltfläche Starten. Bei der Verarbeitung werden Sie vom Erfassungsscript zur Angabe aller zusätzlichen Informationen aufgefordert, die für die vollständige Ausführung der Erfassungsaktivitäten erforderlich sind. Ein Script kann Sie auch auffordern, Konfigurationsinformationen bereitzustellen, Informationen zu der Folge von Ereignissen anzugeben, die dem gegenwärtig vorliegenden Problem vorangegangen sind, oder Einstellungen zur Durchführung und Beendigung der Erfassung festzulegen. Nachdem alle benötigten Informationen bereitgestellt wurden, fährt das Script mit dem Rest der Datenerfassung fort.

Nachdem das Tool die Datenerfassung abgeschlossen hat, können Sie die Ergebnisse an die IBM Unterstützungsfunktion senden. Bei der Dateiübertragung können Sie zwischen der unverschlüsselten FTP-Übertragung und der verschlüsselten HTTPS-Übertragung wählen. Der Name der ZIP-Datei, die das Tool an den IBM Support sendet, wird im Statusbereich für die Erfassung angezeigt.

Wenn IBM Support Assistant Data Collector im Textmodus ausgeführt wird, sind keine Auswahllisten oder Eingabefelder für Benutzereingaben verfügbar. Stattdessen werden die verfügbaren Auswahlmöglichkeiten in Form von nummerierten Listen angezeigt. Um eine Option auszuwählen, geben Sie die Nummer für die entsprechende Auswahlmöglichkeit ein und drücken anschließend die Eingabetaste. Eingabefelder werden in Eingabeaufforderungen umgesetzt, in die Sie Ihre Antwort eingeben und anschließend die Eingabetaste drücken. Nach Abschluss der Datenerfassung wird die Ausgabe in eine andere ZIP-Datei geschrieben. Die ZIP-Ausgabedatei kann an den IBM Support gesendet oder lokal untersucht werden.

Zum Stoppen des Collector-Tools geben Sie im Textmodus die Option quit ein, oder schließen Sie das Browserfenster.

Das Tool fordert zur Angabe von Dateinamen auf. Auf UNIX-Plattformen wird die Verwendung von "~" als Bezeichnung für das Ausgangsverzeichnis (HOME) des Benutzers nicht unterstützt. Wird "~" verwendet, so wird ein Unterverzeichnis unter dem gegenwärtigen Arbeitsverzeichnis namens "~" referenziert.

Versionsinformationen ermitteln

Standardmäßig wird die Version des Tools (und seiner diversen Unterkomponenten) auf der Konsole ausgegeben, auf der es gestartet wurde. Wenn Sie in der Webbenutzerschnittstelle arbeiten, finden Sie die Versionsinformationen auch unter dem Menüpunkt "Hilfe - > Produktinfo". Ist Java nicht verfügbar oder sollte die Java-Anwendung nicht gestartet werden können, können Sie die Version des Tools auch erfahren, indem Sie die Startscripts mit der Option "-version" ausführen.

*HINWEIS: Bei Verwendung der Option "-version" geben die Startscripts die Versionsinformationen aus, ohne dass jedoch das Tool tatsächlich aufgerufen wird.

Inventardatei anzeigen

Das Tool kann Inventarinformationen des aktuellen Systems erfassen. Inventarinformationen werden automatisch in die ZIP-Datei der Erfassung kopiert, sofern verfügbar.
Die Ausgabe erfolgt in Form einer Inventardatei (ISALite/inventory/inventory_rcf.xml), die am besten mit einem Web-Browser angezeigt wird.

Aktualisierungen, Einschränkungen und bekannte Probleme

Die folgenden Aktualisierungen, Einschränkungen und Probleme sind zum Zeitpunkt dieses Release bekannt. Weitere Aktualisierungen zu IBM Support Assistant Data Collector sind auf der Unterstützungswebsite zu IBM Support Assistant unter folgender Adresse dokumentiert: http://www.ibm.com/software/support/isa.

Hardwareinventarinformationen nur mit IBM JRE verfügbar

Wenn Sie eine Erfassung ausführen, die versucht, das Software/Hardware-Inventar auf einer Maschine zu erfassen, und eine JRE eines anderen Anbieters verwendet wird, werden unter Umständen keine Hardwareinformationen zurückgegeben. Dies ist eine Einschränkung der Erfassungsengine und kann durch Verwendung einer IBM JRE umgangen werden.

Linux on IBM System z (zLinux)

Die Datenerfassung unter zLinux wird nur im Konsolmodus unterstützt. Zum Starten einer Erfassung muss das Script isadc.sh in einer Shell aufgerufen werden.

Installationsprobleme

Java-Plug-in-Konfiguration

Der Browser, den Sie für die Ausführung von IBM Support Assistant Data Collector verwenden möchten, muss mit einem Java-Plug-in der Version 1.6 konfiguriert sein. Sie können prüfen, ob Ihr Browser ordnungsgemäß konfiguriert ist, indem Sie den folgenden Link aufrufen: http://www.java.com/en/download/testjava.jsp. Wenn Sie den Browser auf einer Linux- oder UNIX-Plattform ausführen, müssen Sie möglicherweise das Java-Plug-in manuell installieren und konfigurieren.
