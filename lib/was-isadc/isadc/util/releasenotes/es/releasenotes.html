Descripción

La herramienta IBM Support Assistant Data Collector proporciona una recogida de datos automática para los productos de IBM. La herramienta está preconfigurada para localizar datos de diagnóstico importantes en el sistema y copiarlos en un archivo de Colector. Un ejemplo de datos de diagnóstico es un archivo de anotaciones generado por el producto IBM que contenga un historial detallado de los eventos ocurridos durante la operación del producto. Este tipo de archivo puede resultar útil para determinar la naturaleza y causa de un problema de software. Otros ejemplos de datos de diagnóstico son archivos de inicialización, archivos de configuración, versión del sistema operativo, espacio en disco y conexiones de redes. La herramienta se puede ejecutar en modalidad UI de web o en modalidad de consola de línea de mandatos. La modalidad de consola le proporciona un control de línea de mandatos de los scripts de recopilación de IBM Support Assistant Data Collector. La herramienta incluye varias características para ayudarle cuando interactúe con ella en la modalidad de consola, que incluye una que le permite registrar las respuestas de una sesión de modalidad de consola en un archivo y, a continuación, utilizar el archivo para realizar ejecuciones posteriores del mismo script de recogida.

Iniciación

Instalación de herramienta y uso

En la mayoría de los casos, la siguiente secuencia de pasos le guiará para iniciar y ejecutar la herramienta. Si se experimenta problemas, o si desea más información sobre cualquiera de estos pasos, puede hacer referencia a las secciones siguientes que siguen a ésta.


  1. Extraiga la herramienta en cualquier directorio de su elección. Consulte las secciones siguientes para obtener detalles sobre cómo realizar las extracciones.
  2. Ejecute la herramienta en la modalidad UI de web o bien en la modalidad de consola de línea de mandatos. Siga el procedimiento descrito a continuación para definir la variable de entorno JAVA_HOME. Una vez haya hecho esto, podrá ejecutar el script de lanzamiento. Una vez que haya iniciado la herramienta, siga las instrucciones siguientes para interactuar con la misma cuando realiza una recogida de datos. Instalación de la herramienta

En todos los casos, la instalación de la herramienta IBM Support Assistant Data Collector es una simple cuestión de extraer los archivos desde el archivo .zip archivado o archivo .tar.gz. Los archivos se pueden extraer en cualquier ubicación del sistema que elija en el sistema donde ejecutará la herramienta. Esto creará un subdirectorio ISALite debajo del directorio de destino.

Inicio de la herramienta en modalidad UI de web

En modalidad UI de web, abra el archivo index.html ubicado en el subdirectorio isadc.

Modalidad de consola de línea de mandatos - Definición de la variable de entorno JAVA_HOME

En modalidad de línea de mandatos, invoque el script de lanzamiento desde una línea de mandatos. En el caso de un sistema Windows, estos scripts de lanzamiento son archivos de proceso por lotes. Para los demás entornos, se trata de scripts de shell. Puesto que la herramienta no se implementa como una aplicación Java, es necesario localizar Java para poder iniciar la herramienta de línea de mandatos. Si no está disponible Java en la PATH, tendrá que definir la variable de entorno JAVA_HOME manualmente. La herramienta IBM Support Assistant Data Collector requiere un JRE en el nivel 1.5 o superior, por lo que, en primer lugar, debe asegurarse de que está instalado un JRE apropiado en el sistema donde se ejecutará la herramienta. En caso afirmativo, deberá emitir un mandato específico del sistema operativo para definir la variable JAVA_HOME para indicar este JRE. El Microsoft JVM/JDK y gij (GNU libgcj) no están soportados.

Por ejemplo, si en una plataforma Windows tiene instalado jre1.5 en c:\jre1.5, defina JAVA_HOME mediante el mandato siguiente:

SET JAVA_HOME=c:\jre1.5
NOTA: no utilice comillas en el valor del mandato SET, aunque el valor contenga espacios en blanco.

En Linux, AIX o Solaris, si tiene JRE instalado en /opt/jre15, defina JAVA_HOME mediante el mandato siguiente:

export JAVA_HOME=/opt/jre15

Inicio de la herramienta en la modalidad de consola de línea de mandatos

Si no dispone de la interfaz de usuario de web, inicie la herramienta en modalidad de línea de mandatos. Ejecute el script de lanzamiento desde la línea de mandatos (isadc/isadc.bat o isadc/isadc.sh).

Los archivos se escriben en el directorio de instalación.

De forma predeterminada, el directorio de instalación isadc se utiliza para almacenar los archivos creados durante la ejecución. En algunos sistemas, el directorio de instalación isadc será de sólo lectura. En este caso, utilice el parámetro -useHome (sólo en modalidad de consola). El parámetro provocará que los archivos temporales se escriban en el directorio temporal de los sistemas y que los archivos persistentes se escriban en el directorio de inicio del usuario. NOTA: después de extraer el archivo de instalación .zip/.tar.gz, el usuario que ha extraído el archivo debe ejecutar una recopilación antes de que otro usuario intente ejecutar una recopilación mediante el parámetro -useHome. De esta manera, se garantiza una instalación adecuada antes de que otro usuario no admin/root realice una ejecución.

Interacción con la herramienta

Una vez que se ha iniciado la herramienta IBM Support Assistant Data Collector en un navegador web, se mostrará la interfaz gráfica. Antes de que se pueda utilizar la herramienta para realizar una recopilación y análisis de datos, primero debe seleccionar un tipo de problema en la ventana principal. Una vez seleccionado el tipo de problema, el siguiente paso es pulsar el botón Iniciar. Durante el proceso, el script de recopilación de datos le pedirá cualquier información adicional que necesite para completar sus actividades de recopilación de datos. Un script también le podría solicitar información de configuración, información sobre la secuencia de sucesos que genera el problema con el que está trabajando, o las preferencias respecto a cómo se debe completar la recogida de datos. Una vez que disponga de toda la información necesaria, el script continuará con el resto de la recogida.

Cuando la herramienta completa la recopilación de datos, puede enviar los resultados al servicio de soporte de IBM. Puede elegir entre FTP, que está sin cifrar, y HTTPS, que está cifrado, para la transferencia de archivos. El nombre del archivo zip que la herramienta envía al servicio de soporte de IBM se mostrará en el área de estado de la recopilación.

Cuando IBM Support Assistant Data Collector se ejecuta en la modalidad de texto, no hay listas de selección ni campos de entrada para que especifique información el usuario. En lugar de ello, se presentan opciones disponibles en forma de listas numeradas, y el usuario especifica el número de la selección seguido de la tecla Intro. Los campos de entrada se transforman en solicitudes, en las que se especifica la respuesta y se pulsa Intro. Cuando la recopilación de datos ha finalizado, la salida es otro archivo comprimido. El archivo comprimido de salida se puede enviar al servicio de soporte de IBM o se puede examinar localmente.

Para detener la herramienta de recopilador, escriba la opción quit en modalidad de texto o cierre la ventana del navegador.

La herramienta le solicitará los nombres de archivos. En plataformas Unix, el uso de "~" como designación del directorio HOME del usuario no está admitido. Si se utiliza "~", se hará referencia a un subdirectorio debajo del directorio de trabajo actual por el nombre de "~".

Determinación de la información de versión

De forma predeterminada, la versión de la herramienta (y sus distintos subcomponentes) se imprime en la consola desde la que se ha iniciado. Si se ejecuta en la modalidad UI de web, encontrará información de versión si utiliza la la opción del menú Ayuda->Acerca de. Si Java no está disponible, o la aplicación Java no se puede iniciar, también puede obtener la versión de la herramienta ejecutando los scripts de lanzamiento con la opción"-version".

*NOTA: al utilizar la opción "-version", los scripts de lanzamiento imprimirán la información de versión sin invocar realmente la herramienta.

Visualización del archivo de inventario

La herramienta puede recopilar información de inventario del sistema actual. La información de inventario se coloca automáticamente en el archivo ZIP de la recopilación si está disponible.
La salida es un archivo de inventario (ISALite/inventory/inventory_rcf.xml), que se visualiza mejor con un navegador web.

Actualizaciones, limitaciones y problemas conocidos

En el momento de este release ya se conocen las actualizaciones, limitaciones y los problemas siguientes. Las actualizaciones adicionales sobre IBM Support Assistant Data Collector se documentan en el sitio web de IBM Support Assistant Support en: http://www.ibm.com/software/support/isa.

Inventario de hardware disponible con IBM JRE solamente

Al ejecutar una recopilación que intenta realizar un inventario de software/hardware en una máquina, si se utiliza JRE que no es de IBM, puede que no se proporcione información de hardware. Se trata de una limitación del motor de recopilación, que puede solventar utilizando un JRE de IBM.

Linux en IBM System z (zLinux)

En zLinux la recopilación de datos sólo se admite en modalidad de consola. Para iniciar una recopilación, el script isadc.sh debe invocarse dentro de un shell.

Problemas de instalación

Configuración del plug-in de Java

El navegador en el que desea ejecutar IBM Support Assistant Data Collector debe configurarse con un plug-in Java 1.6. Para validar si su navegador se ha configurado correctamente, abra el enlace siguiente http://www.java.com/en/download/testjava.jsp. Si su navegador se ejecuta en una plataforma Linux o Unix, puede que necesite instalar manualmente y configurar el plug-in de Java.
