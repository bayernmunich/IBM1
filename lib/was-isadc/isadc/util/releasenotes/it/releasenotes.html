Descrizione

Lo strumento IBM Support Assistant Data Collector offre la funzione di raccolta automatica dei dati per i prodotti IBM. Lo strumento è preconfigurato in modo da individuare i dati diagnostici importanti sul sistema di computer e copiarli in un file del Programma di raccolta. Un esempio di dati diagnostici è un file di log generato dal proprio prodotto IBM, che contiene una cronologia dettagliata degli eventi che si verificano durante il funzionamento del prodotto. Tale file può essere utile per la determinazione della natura e della causa di un problema software. Altri esempi di dati diagnostici includono i file di inizializzazione, i file di configurazione, la versione del sistema operativo, lo spazio su disco e le connessioni di rete. Lo strumento può essere eseguito in modalità UI Web o in una modalità console della riga comandi. La modalità console offre all'utente il controllo da riga comandi degli script di raccolta di IBM Support Assistant Data Collector. Sono incluse diverse funzioni che assistono l'utente quando utilizza lo strumento in modalità console, ad esempio una funzione che consente di registrare le risposte dell'utente da una sessione in modalità console in un file e di riutilizzare tale file per le esecuzioni successive dello stesso script di raccolta.

Introduzione

Installazione e utilizzo dello strumento

Nella maggior parte dei casi, la seguente sequenza di operazioni consentirà all'utente di iniziare ad utilizzare lo strumento. In caso di problemi o se si desidera acquisire ulteriori informazioni su tali operazioni, è possibile consultare le sezioni riportate di seguito.


  1. Estrarre lo strumento in una directory a propria scelta. Consultare le sezioni riportate di seguito per i dettagli su come eseguire le estrazioni.
  2. Eseguire lo strumento in modalità UI Web o in modalità console della riga comandi. Attenersi alla procedura descritta di seguito per impostare la variabile di ambiente JAVA_HOME. Dopo avere effettuato tale operazione, è possibile eseguire lo script di avvio. Dopo avere avviato lo strumento, attenersi alle istruzioni riportate di seguito per interagire con esso mentre esegue una raccolta. Installazione dello strumento

In tutti i casi, l'installazione dello strumento IBM Support Assistant Data Collector consiste nell'estrarre i file dal file .zip archiviato o dal file .tar.gz. I file possono essere estratti in un'ubicazione qualsiasi del file system scelta dall'utente, sul sistema su cui verrà eseguito lo strumento. In tal modo, verrà creata una directory secondaria ISALite nella directory di destinazione.

Avvio dello strumento in modalità UI Web

Per la modalità UI Web, aprire il file index.html contenuto nella directory secondaria isadc.

Modalità console della riga comandi - Impostazione della variabile di ambiente JAVA_HOME

Per la modalità riga comandi, l'utente richiama lo script di avvio appropriato da una riga comandi. Nel caso di un sistema Windows, questi script di avvio sono i file batch. Per gli altri ambienti, si tratta degli script della shell. Poiché lo strumento viene implementato come un'applicazione Java, per potere avviare lo strumento dalla riga comandi, è necessario individuare Java. Se Java non è disponibile sul percorso, sarà necessario impostare manualmente la variabile di ambiente JAVA_HOME. Lo strumento IBM Support Assistant Data Collector richiede JRE livello 1.5 o un livello successivo, quindi è necessario verificare che sul sistema su cui verrà eseguito lo strumento sia installato un livello JRE appropriato. In caso affermativo, sarà necessario eseguire un comando specifico del sistema operativo per impostare la variabile JAVA_HOME, in modo che punti a questo JRE. Microsoft JVM/JDK e gij (GNU libgcj) non sono supportati.

Ad esempio, se su una piattaforma Windows jre1.5 è installato in c:\jre1.5, impostare JAVA_HOME utilizzando il seguente comando:

SET JAVA_HOME=c:\jre1.5
NOTA: non utilizzare le virgolette nel valore del comando SET, anche se il valore contiene degli spazi.

Su Linux, AIX o Solaris, se JRE è installato in /opt/jre15, impostare JAVA_HOME utilizzando il seguente comando:

export JAVA_HOME=/opt/jre15

Avvio dello strumento in modalità console della riga comandi

Se l'esecuzione in modalità UI Web non è consentita, è necessario avviare lo strumento in modalità riga comandi. È sufficiente eseguire lo script di avvio dalla riga comandi (isadc/isadc.bat o isadc/isadc.sh).

I file vengono scritti nella directory di installazione

Per impostazione predefinita, la directory di installazione isadc viene utilizzata per memorizzare i file creati durante l'esecuzione. Su alcuni sistemi la directory di installazione isadc sarà in sola lettura. In questo caso, utilizzare il parametro -useHome (solo modalità console). Utilizzando tale parametro, i file temporanei verranno scritti nella directory temporanea dei sistemi e i file persistenti verranno scritti nella directory home dell'utente. NOTA: dopo avere estratto il file di installazione .zip/.tar.gz, l'utente che ha estratto il file deve eseguire una raccolta, prima che altri utenti tentino di eseguire una raccolta mediante il parametro -useHome. In tal modo, si verifica che l'impostazione sia stata eseguita correttamente prima che altri utenti non admin/root procedano con l'esecuzione.

Interazione con lo strumento

Dopo avere avviato lo strumento IBM Support Assistant Data Collector in un browser Web, verrà visualizzata l'interfaccia grafica. Prima di utilizzare lo strumento per eseguire la raccolta e l'analisi dei dati, è necessario selezionare un tipo di problema dalla finestra principale. Una volta selezionato il tipo di problema, premere il pulsante Avvia. Durante l'esecuzione, lo script di raccolta richiederà all'utente altre informazioni, necessarie per completare le attività di raccolta. Uno script può anche richiedere all'utente le informazioni di configurazione, le informazioni sulla sequenza di eventi da cui è scaturito il problema in questione oppure come preferisce che venga completata la raccolta. Dopo avere acquisito tutte le informazioni necessarie, lo script procederà con la parte rimanente della raccolta.

Quando lo strumento ha terminato la raccolta dei dati, è possibile inviare i risultati al supporto IBM. Per il trasferimento file è possibile scegliere tra FTP, senza codifica, ed HTTPS, con codifica. Il nome del file zip che lo strumento invia al supporto IBM verrà visualizzato nell'area di stato della raccolta.

Quando IBM Support Assistant Data Collector viene eseguito in modalità testo, non sono disponibili elenchi di selezione o campi di immissione per l'input da parte dell'utente. Invece, le opzioni disponibili vengono presentate come elenchi numerati e l'utente immette il numero della selezione, quindi preme il tasto Invio. I campi di immissione vengono trasformati in prompt, in cui l'utente immette la risposta e preme Invio. Una volta completata la raccolta dei dati, l'output prodotto è un altro file ZIP. Il file ZIP di output può essere inviato al supporto IBM o esaminato in loco.

Per arrestare lo strumento Programma di raccolta, digitare l'opzione di uscita (quit) in modalità testo oppure chiudere la finestra del browser.

Lo strumento richiederà i nomi dei file. Sulle piattaforme Unix, l'utilizzo di "~" per designare la directory HOME dell'utente non è supportato. Se si utilizza "~", verrà indicata una directory secondaria nella directory di lavoro attuale denominata "~".

Determinazione delle informazioni sulla versione

Per impostazione predefinita, la versione dello strumento (e dei suoi componenti secondari) viene indicata sulla console da cui è stato avviato lo strumento. Quando si sceglie l'esecuzione in modalità UI Web, le informazioni sulla versione possono essere visualizzate anche utilizzando l'opzione di menu Guida->Informazioni su. Se Java non è disponibile oppure se non è possibile avviare l'applicazione Java, la versione dello strumento può essere determinata anche eseguendo gli script di avvio con l'opzione "-version".

*NOTA: quando si utilizza l'opzione "-version", gli script di avvio mostreranno le informazioni sulla versione senza richiamare effettivamente lo strumento.

Visualizzazione del file dell'inventario

Lo strumento può raccogliere le informazioni sull'inventario del sistema corrente. Le informazioni sull'inventario vengono inserite automaticamente nel file ZIP della raccolta, se sono disponibili.
L'output è un file di inventario (ISALite/inventory/inventory_rcf.xml) e il modo migliore per visualizzarlo è tramite un browser Web.

Aggiornamenti, limitazioni e problemi noti

Quelli che seguono sono gli aggiornamenti, le limitazioni e i problemi notati al momento in cui è stata resa disponibile questa release. Ulteriori aggiornamenti relativi ad IBM Support Assistant Data Collector sono documentati sul sito Web del supporto di IBM Support Assistant all'indirizzo: http://www.ibm.com/software/support/isa.

Inventario hardware disponibile solo con JRE IBM

Quando si esegue una raccolta che tenta di eseguire l'inventario software/hardware su una macchina e si utilizza un JRE non IBM, potrebbero non essere restituite le informazioni sull'hardware. Si tratta di una limitazione del motore di raccolta e il problema può essere risolto utilizzando un JRE IBM.

Linux su IBM System z (zLinux)

La raccolta dei dati su zLinux è supportata solo in modalità console. Per avviare una raccolta, è necessario richiamare lo script isadc.sh dall'interno di una shell.

Problemi di installazione

Configurazione del plug-in Java

Il browser in cui si desidera eseguire IBM Support Assistant Data Collector deve essere configurato con il plug-in Java 1.6. Per verificare che il proprio browser sia configurato correttamente, aprire il seguente link http://www.java.com/en/download/testjava.jsp. Se si sta eseguendo il browser su una piattaforma Linux o Unix, potrebbe essere necessario installare e configurare manualmente il plug-in Java.
