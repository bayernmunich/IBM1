설명

IBM Support Assistant Data Collector 도구는 IBM 제품에 대한 자동 데이터 수집 기능을 제공합니다. 이 도구는 사용자의 컴퓨터 시스템에서 중요한 진단 데이터를 찾고 이를 콜렉터 파일로 복사하도록 사전 구성되어 있습니다. 진단 데이터의 예로는, 제품 조작 중에 발생하는 이벤트에 대한 자세한 히스토리를 포함하는 IBM 제품에서 생성되는 로그 파일이 있습니다. 이러한 유형의 파일은 소프트웨어 문제점의 특성과 원인을 판별하는 데 유용합니다. 진단 데이터의 다른 예에는 초기화 파일, 구성 파일, 운영 체제 버전, 디스크 공간 및 네트워크 연결이 포함될 수 있습니다. 도구는 웹 UI 모드 또는 명령행 콘솔 모드에서 실행할 수 있습니다. 콘솔 모드에서는 IBM Support Assistant Data Collector 콜렉션 스크립트의 명령행 제어를 제공합니다. 이 도구에는 파일의 콘솔 모드 세션에서 응답을 기록하고 파일을 사용하여 동일한 수집 스크립트의 후속 실행을 추진하는 기능을 포함하여 콘솔 모드에서 상호 작용할 경우 사용자를 지원하는 여러 기능이 포함되어 있습니다.

도구 설치 및 사용법

대부분의 경우에는 다음과 같은 일련의 단계를 수행하여 도구를 시작하고 실행합니다. 문제가 발생하거나 이러한 단계에 대한 자세한 정보를 원하는 경우에는 아래 섹션을 참조하십시오.


  1. 선택한 디렉토리에 도구의 압축을 풉니다. 압축을 푸는 방법에 대한 세부사항은 아래 섹션을 참조하십시오.
  2. 웹 UI 모드 또는 명령행 콘솔 모드에서 도구를 실행하십시오. JAVA_HOME 환경 변수를 설정하는 방법은 아래에 설명된 프로시저를 따릅니다. 이 프로시저를 수행하면 실행 스크립트를 실행할 수 있습니다. 도구를 시작한 경우 콜렉션을 수행하는 경우와 마찬가지로 상호작용을 위해 아래 지시사항을 따르십시오. 도구 설치

IBM Support Assistant Data Collector 도구를 설치하려면 아카이브된 Zip 파일 또는 .tar .gz 파일의 압축을 풀기만 하면 됩니다. 이 파일은 도구를 실행할 시스템에서 선택한 파일 시스템 위치로 압축을 풀 수 있습니다. 그러면 대상 디렉토리 아래에 서브디렉토리 ISALite가 작성됩니다.

웹 UI 모드에서 도구 시작

웹 UI 모드의 경우 isadc 서브디렉토리에 있는 index.html 파일을 여십시오.

명령행 콘솔 모드 - JAVA_HOME 환경 변수 설정

명령행 모드의 경우 명령행에서 적절한 실행 스크립트를 호출하십시오. Windows 시스템의 경우 이러한 실행 스크립트는 일괄처리 파일입니다. 기타 환경의 경우에는 쉘 스크립트입니다. 도구는 Java 애플리케이션으로 구현되므로 명령행 도구를 시작하려면 먼저 Java를 찾아야 합니다. PATH에서 Java를 사용할 수 없는 경우 직접 JAVA_HOME 환경 변수를 설정해야 합니다. IBM Support Assistant Data Collector 도구를 사용하려면 JRE 레벨 1.5 이상이 필요하므로 먼저 도구를 실행할 시스템에 적절한 JRE가 설치되었는지 확인해야 합니다. 적절한 JRE가 설치된 경우 운영 체제에 특정한 명령을 실행하여 JAVA_HOME 변수가 이 JRE를 가리키도록 설정해야 합니다. Microsoft JVM/JDK 및 gij(GNU libgcj)는 지원되지 않습니다.

예를 들어, Windows 플랫폼에서 jre1.5를 c:\jre1.5에 설치한 경우, 다음 명령을 사용하여 JAVA_HOME을 설정합니다.

SET JAVA_HOME=c:\jre1.5
참고: 값에 공백 문자가 있어도 SET 명령의 값에 따옴표를 사용하지 마십시오.

Linux, AIX 또는 Solaris에서 /opt/jre15에 JRE를 설치한 경우 다음 명령을 사용하여 JAVA_HOME을 설정합니다.

export JAVA_HOME=/opt/jre15

명령행 콘솔 모드에서 도구 시작

웹 UI에서 실행하는 것이 옵션이 아닌 경우, 명령행 모드에서 도구를 시작해야 합니다. 명령행에서 실행 스크립트를 실행하십시오(isadc/isadc.bat 또는 isadc/isadc.sh).

파일은 설치 디렉토리에 작성됩니다.

기본적으로 isadc 설치 디렉토리는 실행 중에 작성된 파일을 저장하는 데 사용됩니다. 일부 시스템에서는 isadc 설치 디렉토리가 읽기 전용입니다. 이 경우에는 -useHome 매개변수를 사용합니다(콘솔 모드 전용). 이 매개변수를 사용하면 임시 파일은 시스템 임시 디렉토리에 쓰여지고 지속 파일은 사용자 홈 디렉토리에 쓰여집니다. 참고: 설치 .zip/.tar.gz 파일의 압출을 푼 후에 파일의 압축을 푼 사용자는 다른 사용자가 콜렉션 실행을 시도하기 전에 -useHome 매개변수를 사용하여 콜렉션을 실행해야 합니다. 이를 통해 다른 비관리/루트 사용자가 콜렉션을 실행하기 전에 필요한 설정을 수행할 수 있습니다.

도구와 상호작용

IBM Support Assistant Data Collector 도구가 웹 브라우저에서 시작되면 그래픽 인터페이스가 표시됩니다. 데이터 콜렉션 및 분석을 위해 도구를 사용하려면 먼저 기본 창에서 문제점 유형을 선택해야 합니다. 문제점 유형이 선택되면 다음 단계는 시작 단추를 누르는 것입니다. 이 단계가 진행되면 콜렉션 스크립트는 콜렉션 활동을 완료하기 위해 필요한 추가 정보를 사용자에게 요청합니다. 스크립트도 구성 정보, 처리하는 문제점의 발생 과정에 있는 일련의 이벤트에 대한 정보 또는 사용자가 선호하는 콜렉션 완료 방법을 요청할 수 있습니다. 필요한 정보를 모두 제공하면 스크립트에서 나머지 수집을 수행하게 됩니다.

도구가 데이터 콜렉션을 완료하면 결과를 IBM Support에 전송할 수 있습니다. 파일 전송을 위해 암호화되지 않은 FTP와 암호화된 HTTPS 사이에서 선택할 수 있습니다. 도구가 IBM 지원 센터에 전송하는 Zip 파일의 이름은 콜렉션 상태 영역에 표시됩니다.

IBM Support Assistant Data Collector가 명령행 텍스트 모드에서 실행되면 사용자 입력을 위한 입력 필드나 선택사항 목록이 없습니다. 대신, 번호가 있는 목록으로 사용 가능한 선택사항이 제시되므로 해당 번호를 입력한 후 Enter 키를 눌러 선택할 수 있습니다. 입력 필드는 프롬프트로 변환되고, 여기서 사용자 응답을 입력한 후 Enter 키를 누를 수 있습니다. 데이터 콜렉션이 완료되면 다른 Zip 파일로 출력이 작성됩니다. 출력 Zip 파일은 IBM 지원 센터로 전송하거나 로컬로 검사할 수도 있습니다.

콜렉터 도구를 중지하려면 텍스트 모드로 quit 옵션을 입력하거나 브라우저 창을 닫으십시오.

파일 이름을 입력하라는 메시지가 표시됩니다. Unix 플랫폼에서는 "~"를 사용자의 HOME 디렉토리로 지정할 수 없습니다. "~" 가 사용된 경우에는 이름이 "~"인 현재 작업 중인 디렉토리 아래의 서브디렉토리가 참조됩니다.

버전 정보 판별

기본적으로 도구의 버전(및 다양한 서브컴포넌트)은 실행된 콘솔에 인쇄됩니다. 웹 UI에서 실행하는 경우 도움말->정보 메뉴 옵션을 사용하여 버전 정보를 찾을 수도 있습니다. Java를 사용할 수 없거나 Java 애플리케이션이 시작될 수 없는 경우에는 "-version" 옵션을 사용하여 실행 스크립트를 실행하여 도구의 버전을 가져올 수도 있습니다.

*참고: "-version" 옵션을 사용하는 경우 실행 스크립트는 도구를 실제로 호출하지 않고 버전 정보를 인쇄합니다.

업데이트, 제한사항 및 알려진 문제점

다음의 업데이트, 제한사항 및 문제점은 이 릴리스가 제공되는 시점에 알려진 내용입니다. IBM Support Assistant Data Collector에 대한 추가 업데이트는 IBM Support Assistant 지원 웹 사이트(http://www.ibm.com/software/support/isa)에서 확인할 수 있습니다.

IBM JRE 전용 하드웨어 인벤토리

시스템에서 소프트웨어/하드웨어 인벤토리를 수행하는 콜렉션을 실행하면서 비IBM JRE를 사용하는 경우, 하드웨어 정보가 리턴되지 않을 수도 있습니다. 이는 콜렉션 엔진의 제한사항이며 IBM JRE를 사용하여 해제될 수 있습니다.

IBM System z의 Linux(zLinux)

zLinuxdml 데이터 콜렉션은 콘솔 모드에서만 지원됩니다. 콜렉션을 시작하려면 isadc.sh 스크립트를 쉘 내에서 호출해야 합니다.

설치 문제

Java 플러그인 구성

IBM Support Assistant Data Collector를 실행하려는 브라우저는 Java 1.6 플러그인으로 구성되어야 합니다. 링크(http://www.java.com/en/download/testjava.jsp)를 열어 브라우저가 올바르게 구성되었는지 확인할 수 있습니다. Linux 또는 Unix 플랫폼에서 브라우저를 실행하는 경우 Java 플러그인을 수동으로 설치 및 구성하십시오.
