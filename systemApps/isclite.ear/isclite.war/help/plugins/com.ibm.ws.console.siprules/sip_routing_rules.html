Routing policy SIP rules

Use this page to associate routing policy SIP rules with on demand routers

To view this administrative console page, click Servers > Server types > On demand routers > ODR > SIP On demand router settings > Routing policy SIP rules.

To modify rules with the rule builder, you must have administrator administrative privileges.

Add

Builds a new rule.

Delete

Deletes a rule.

Move up, Move down

Prioritizes the rules in the case of multiple rules. The order of rules is key in determining matches. If a match occurs on the initial rule, subsequent rules are not pursued.

Select action

Actions refer to the incoming SIP request.

Assigns the default routing action to one of the following options:
Permit
Permits the incoming SIP request.
Reject
Rejects any incoming SIP request with the specified SIP error code.

Specify by

Specifies cluster, server, module, expression.

Select cell name

Specifies cell.

Multi-cluster routing policy

The multi-cluster routing policy only applies if there are multiple target clusters. If the target clusters are dynamically computed, the number and order of target clusters is not known until runtime.

Assigns the multi-cluster routing policy to one of the following options:
Error
Disallow multi-cluster routing. Request will fail with a 500 error.
Failover
Find the initial cluster with an available server and load balance across that cluster. The order of a dynamically generated list of clusters is undefined.
Weighted round robin
Weighted round robin load balance. For UDP retransmission, maintain affinity.

Apply

Commits your new rule settings.
