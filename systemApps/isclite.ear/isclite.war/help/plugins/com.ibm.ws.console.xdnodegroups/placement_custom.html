Application placement controller custom properties

Use this page to view and manage a list of arbitrary name-value pairs of data, where the name is a property key and the value is a string value that can be used to set internal system configuration properties. Defining a new property enables you to configure a setting beyond that which is available in the administrative console.

To view this administrative console page, click Operational policies > Autonomic managers > Application placement controller > Custom properties > new.

Name

Specifies the name, or key, for the property.

Each property must be unique. If the same name is used for multiple properties, the value specified for the first property with that name is used. Do not start your property names with was. This prefix is reserved for properties that are predefined in WebSphere® Application Server.

Value

Specifies the value that is paired with the specified name.

The value is a string that can set internal system configuration properties.

Description

Provides descriptive information about the name-value pair.

Click Apply or OK to save your settings.
