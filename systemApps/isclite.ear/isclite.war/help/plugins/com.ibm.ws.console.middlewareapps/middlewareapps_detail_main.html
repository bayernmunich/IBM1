Middleware application settings

Use this page to view configurations of existing middleware applications.

To view this administrative console page, click Applications > All applications > middleware_application_name.

Application name

Specifies a name for the application. The name for a middleware application must be unique among all applications in the cell.

The name field cannot contain the following characters: # \ / , : ; " * ? < > | = + & % ' . The name cannot begin with a period (.) or a space. A space does not generate an error, but leading and trailing spaces are automatically deleted.

The name cannot be the same as any other defined application name in the cell. For example, an unmanaged Web application cannot have the same name as a Java Platform, Enterprise Edition (Java EE) application that is deployed in the same cell.

Application type

Specifies the type that is affiliated with the application.

Application edition

Specifies the application edition.

Edition description

Specifies the description of the application edition.

Deployment properties

Specifies the current modules, and where the modules are deployed in the configuration.

A list of the currently defined modules is displayed. To change an existing module, select the module and click Edit. To remove an existing module, select the module and click Remove. To add a module, click New. Define the properties for the current module and click Apply.

Module name

Specifies the name of the module that is deployed as part of the application.

Context root

Specifies the context root of the Web application (WAR).

Virtual host

Specifies the name of the virtual host to which the Web module is currently mapped.

Expanding the list displays a list of previously defined virtual hosts. To change a mapping, select a different virtual host from the list.

Deployment targets

Specifies where the web module is deployed. For WebSphere® Application Community Edition applications, this section precedes the Deployment Properties section on the panel, and the deployment targets are applicable to the application instead of being specific to individual modules.

You can filter the list of existing deployment targets by expanding the Filter list and choosing the way that you want to filter your deployment targets. To add a deployment target, click Add. The deployment target that you add displays in the added deployment targets. To remove a deployment target, select the deployment target from the added deployment targets, and click Remove.
