On demand router clusters collection

This page lists the on demand router (ODR) clusters in the cell. An ODR cluster consists of a group of ODRs. On this page, you can modify the configuration settings of an ODR cluster. If one of the members of an ODR cluster fails, requests are routed to the other members of the cluster.

To view this administrative console page, click Servers > Clusters > On demand router clusters.

Cluster

Specifies a logical name for the ODR cluster.

Protocol

Indicates the protocol or protocols that the ODR cluster is configured to handle.

Status

Indicates whether the ODR cluster is partially started, started, partially stopped, stopped, or unknown. If the status is unknown, the node agent is not running in that node and you must restart the node agent before you can start the server.

Icon Action Description
Partially started At least one of the cluster members is running.
Started All of the cluster members are running.
Partially stopped At least one of the cluster members has stopped running.
Stopped All of the cluster members have stopped running.
Unknown The status cannot be determined.
