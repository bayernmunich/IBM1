Service map settings

A service map defines how a request sent from a service client to a service provider is transformed or rerouted. Use this panel to view properties for a specific service map.

To view this panel in the administrative console, click Service integration > Service mapping > Service maps > service_map_definition_name.

This panel shows the following properties:

General Properties

Name
The unique name for this service map.
Description
The description of this service map.

Business-Level Application Properties

Name
The name of the business-level application used by this service map
Status
The state of the business-level application used by this service map. Possible values are: Started, Stopped, Partial start, or Unknown. For more information about business-level application states, see Business-level application collection (online).
Deployed to servers
Lists the names of the servers to which this service map is deployed.

Service Properties

Port type
The port type associated with the service request that is intercepted by the local mapping service. This field is optional.
Namespace
The namespace for the service and port.
Service name
The name of the web service request that is intercepted by the local mapping service.
Port name
The port name associated with the web service request that is intercepted by the local mapping service.
Note: The values for the fields on this panel must match the values of the local mapping service to which this service map is attached.

Attached Local Mapping Services

This list shows the local mapping services to which this service map is attached. If no local mapping services are attached to this service map, an information message is displayed.
