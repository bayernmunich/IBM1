Step 4: Select a server template

Select a server template on which to model the servers for the new dynamic cluster. You can select a default template, or map an existing server.

To access this administrative console page, click Servers > Clusters > Dynamic clusters > New.

Server template

Specifies the template that is a model for the servers in the dynamic cluster.

  • Select Create the cluster member using a server template to model the servers for the dynamic cluster after a defined server template. Choose a server template from the list.
    The following list describes the templates for application servers:
    • The default or defaultZOS templates are the WebSphere® Application Server Version 6 or higher templates.
    • The DeveloperServer template is a default template that you can use when you are developing your applications.

    You can also specify a template for middleware servers with complete lifecycle management, such as PHP and WebSphere Application Server Community Edition servers. Predefined templates exist that you can use in your dynamic cluster, or you can create your own server template. To create your own server template, click Servers > All servers > Templates... > New.

  • Select Create the cluster member using an existing server as a template to model the servers for the dynamic cluster after an existing application server. Choose a server from the list.
  • Select the core group to which the cluster member belongs. This option is available only when you select WebSphere Application Server as the dynamic cluster server type.

The templates that display correspond to the platform and product version of your deployment manager.
