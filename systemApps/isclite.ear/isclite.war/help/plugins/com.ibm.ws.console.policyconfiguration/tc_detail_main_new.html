Define transaction class general properties

Use this page to create new transaction classes, which are components of a service policy.

To view this administrative console page, click Operational policies > Service policies > New > Step 3: Define service policy memberships > New, or select Operational policies > Service policies > service_policy_name > Transaction classes: New.

To create, modify, or remove service policies and transaction classes, you must have administrator or configurator administrative privileges.

Name

Specifies the name of the transaction class. The name is a required, user-defined field. The name field cannot contain the following characters: # \ / , : ; " * ? < > | = + & % ' The name must be unique among transaction classes and cannot begin with a period (.) or a space. A space does not generate an error, but leading and trailing spaces are automatically deleted. The name must provide meaning, particularly in environments with multiple system administrators. For example, a user creating a transaction class for name field lookups for an online application might call the transaction class Name Lookups.

Description

Complements the name field and is user-defined. The description field is optional, but in environments with multiple system administrators, you must complete this field with useful information.
