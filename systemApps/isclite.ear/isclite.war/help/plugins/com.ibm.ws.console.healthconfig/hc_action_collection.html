Custom actions collection

By default, you can define a health policy to execute any of the predefined health actions, such as restart a server, when a health policy is breached. Configure a custom action when you want your environment to take an action other than these defaults.

To view this administrative console page, click Operational policies > Custom action.

A custom action is a process definition. You can then configure your custom actions as an ordered list of steps that occur when a health policy is breached. You can run custom actions in the following places in your environment:
  • The deployment manager
  • Any nodes that are hosting servers that breach health policies
  • The node that runs the health management monitor
  • A specified node

Name

Specifies a name for the custom action.

Supported operating systems

Specifies the operating system types that the action supports.

Action

Specifies the program or script that is run for the action, including any arguments that you are passing to the executable file.
