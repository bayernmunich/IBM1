Creating a health policy: Specify members to monitor

Use this page to assign membership properties to health policies.

To view this administrative console page, click Operational policies > Health policies > New.

If you are a user with either a monitor or an operator administrative role, you can only view health policy information. If you are a user with either a configurator or an administrator role, you have all configuration privileges for health policies.

Members of a policy can be clusters, dynamic clusters, application server instances, and other servers that are running on nodes. If you specify a health policy that applies to dynamic clusters, the minimum number of dynamic cluster instances remains up and running regardless of the health policy that you are using.

Member types

Select from this list to add instances as members to the health policy, which activates the defined health policy for the members. The potential members display in the Available for membership list. Membership is not a one-to-one relationship: you can associate members with multiple policies. The following member types apply:
  • Cells
  • Dynamic clusters
  • Clusters
  • Servers and nodes
  • On demand routers
  • On demand router (ODR) dynamic clusters

Available for membership

From the Available for membership field, select the appropriate members from those available in the list. Select multiple members by pressing the control key until all of your selections are highlighted. Click Add to create membership for the health policy. The resulting members display in the members of the health_policy_name list.
