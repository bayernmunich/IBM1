Middleware server template settings

Use this page to configure the settings for a newly created server template. Existing server types from which to create a template include application, generic, on demand router, proxy, or Web server. You can also create or add server templates for a middleware server that already exists in your environment such as PHP, Apache Tomcat, BEA WebLogic Server, JBoss, Custom HTTP server, Liberty profile server, WebSphere® Application Server Community Edition, or External WebSphere Application Server.

To view this administrative console page, click Servers > All servers > Templates > New, or select Servers, choose a specific middleware server type in Server types, and click Templates > New.

To edit a PHP server template, see External configuration for more information.

Step 1: Select an existing server

Select an existing server from which you can create a server template.

Enable the Select check box and click OK. If you click Servers > All servers > Templates > New, you must first select the server type from the list. For WebSphere Application Server Community Edition servers, you can optionally select the location of the server run time that you use to create the server.

The available server types to choose from are:
  • WebSphere application server
  • Liberty profile server
  • WebSphere proxy server
  • On demand router
  • PHP server
  • WebSphere Application Server Community Edition server
  • Generic server
  • WebSphere MQ server
  • Web server
  • Apache Tomcat server
  • BEA WebLogic server
  • JBoss server
  • External WebSphere Application Server
  • Apache server
  • Custom HTTP server

Click OK to continue defining the settings or click Cancel.

Step 2: Specify template specific properties

Specify the name and a description of the server template.

Click OK after you type the name and a description to return to the main page that displays the name, platform, version, and description of the server template that you just created.

Step 3: Save or discard the changes

You can then either click Save to save the changes directly to the mastery configuration, or click Review to view a summary of the changes before saving or discarding them.
