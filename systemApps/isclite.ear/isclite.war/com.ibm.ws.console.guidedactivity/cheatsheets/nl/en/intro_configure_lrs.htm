In addition to the transactional J2EE applications supported by WebSphere Application Server, WebSphere Extended Deployment Compute Grid supports transactional batch, compute-intensive, and native jobs. This support allows you to leverage your WebSphere Application Server infrastructure to host applications whose processing model does not fit the traditional J2EE request/response paradigm. WebSphere Extended Deployment allows you to deploy these applications alongside transactional J2EE applications and uses administratively-defined service policies to dynamically allocate resources among the entire set of applications.

Assumptions

  • The correct levels of WebSphere Extended Deployment are installed.
