Assumptions

It is assumed that the Web server and Application server are on different nodes. This environment is typically referred to as a "Remote" Web server because it is not local to the Application Server. For more information, see the following sources in the information center:
  • Configuring a Web server and an application server on separate machines (remote for Distributed and zOS)
  • Configuring a Web server and an application server on separate machines(remote for OS/400)
  • IBM Education Assistant

Before you begin

  • Verify that the Web server is supported by WebSphere Application Server. Check the WebSphere Application Server Support Web page for the latest information on supported software and hardware .
  • Verify that the Web server software installed successfully and that the server can accept requests.

When finished

After you have completed the cheat sheet, a Web server is defined in the WebSphere Application Server configuration that can route requests for dynamic content to the appropriate application server.
