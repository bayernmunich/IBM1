Assumptions

This cheat sheet outlines the basic settings required for WebSphere Application Server to authenticate users with an Lightweight Directory Access Protocol (LDAP) directory server. It is assumed that the LDAP server exists and has been populated with users and groups. For more information on this task, see the following information center articles:
  • Configuring Lightweight Directory Access Protocol user registries
  • Using specific directory servers as the LDAP server
  • Authenticate users with an LDAP user registry
  • Authenticate users with an LDAP user registry

Before you begin

  • Set up an LDAP-compliant directory server or verify that a directory server is available for storing users and group information for both your applications and for WebSphere Application Server administration.
  • Create any users and groups that are needed within the directory. Note: WebSphere Application Servers do not provide the direct management of user and group information within the directory itself.
