Assumptions

This cheat sheet outlines only the most basic set of steps for getting Global security enabled so that access to the administrative console is restricted to authorized users. For more information, see the following sources in the information center:
  • Planning to secure your environment
  • Securing the administrative environment

Before you begin

  • Verify that a supported user registry is configured or set up a new user registry. For more information, see the information center article entitled, User Registries for more information.
  • Define the users and groups in the user registry that need administrative access using the user management tools that are supplied by the vendor.

What you need

  • The type of user registry that you plan to use, including the information to connect to the user registry, if necessary.
  • The users that you plan to have administer the system.

When finished

After you have completed the cheat sheet, you are required to use a user ID and password to log into the administrative console because you have enabled security.
